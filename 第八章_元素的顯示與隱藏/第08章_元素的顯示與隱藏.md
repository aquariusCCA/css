遇到下面情境，可以回來查這章：

- 想讓元素完全從畫面和版面中消失。
- 想讓元素看不見，但仍保留原本位置。
- 想做淡入、淡出動畫。

- `overflow: auto` 沒有效果，不知道為什麼沒有出現捲軸。
- 切換顯示狀態時，不確定要用 Vue 的 `v-if` 還是 `v-show`。

## 這一章先抓結論

1. `overflow` 處理的是「盒子裡面超出去的內容」，不是讓元素本身消失。
2. `display: none` 會讓元素完全不顯示，也不佔版面位置。
3. `visibility: hidden` 會讓元素看不見，但原本佔用的版面位置仍保留。
4. `opacity: 0` 會讓元素完全透明，但元素通常仍佔位置，也可能仍可被點擊或聚焦。
5. 想做淡入淡出動畫，通常不要只靠 `display: none`，而是使用 `opacity` 搭配 `visibility`、`pointer-events` 或 class 切換。
6. 想控制溢出方向時，用 `overflow-x` 和 `overflow-y`，不要全部都用單一 `overflow` 解決。

## 30 秒複習入口

| 需求 | 優先考慮 |
| --- | --- |
| 內容超出盒子，要裁切或出現捲軸 | `overflow` |
| 只想控制水平或垂直方向的溢出 | `overflow-x` / `overflow-y` |
| 元素要完全消失，而且不佔位置 | `display: none` |
| 元素要看不見，但保留原本位置 | `visibility: hidden` |
| 元素要淡入淡出 | `opacity` + `transition` |
| 元素透明後不想被點擊 | `pointer-events: none` |
| 文字太長要省略號 | `overflow: hidden` + `white-space: nowrap` + `text-overflow: ellipsis` |

# 1. `overflow`：處理內容溢出

`overflow` 用來控制：當內容超出元素指定的寬高時，要怎麼顯示。

它處理的是「內容超出盒子」的問題，不是「元素本身是否存在」的問題。

```css
.box {
  width: 200px;
  height: 120px;
  overflow: auto;
}
```

如果 `.box` 裡面的內容高度超過 `120px`，就會出現垂直捲軸。

---

## 1.1 常見值

```css
.box {
  overflow: visible; /* 預設值，溢出內容仍然顯示 */
  overflow: hidden;  /* 超出的內容被裁切 */
  overflow: scroll;  /* 不管有沒有溢出，都保留捲軸區域 */
  overflow: auto;    /* 有溢出才出現捲軸 */
}
```

| 值 | 效果 | 常見使用情境 |
| --- | --- | --- |
| `visible` | 預設值，內容超出仍顯示 | 不想裁切內容 |
| `hidden` | 超出盒子的內容被裁掉 | 卡片裁切、圖片裁切、避免破版 |
| `scroll` | 強制顯示捲軸區域 | 需要固定捲軸空間時 |
| `auto` | 有需要才出現捲軸 | 最常用的捲動容器設定 |

### 重要觀念

`overflow` 要有明顯效果，通常需要盒子本身有尺寸限制，例如：

```css
.box {
  width: 300px;
  height: 200px;
  overflow: auto;
}
```

或：

```css
.panel {
  max-height: 400px;
  overflow-y: auto;
}
```

如果元素高度會被內容自然撐開，內容就不算「溢出」，因此你可能看不到捲軸效果。

---

## 1.2 `overflow-x` 和 `overflow-y`

`overflow` 是同時控制水平和垂直方向。

如果只想控制某一個方向，可以使用：

```css
.box {
  overflow-x: hidden;
  overflow-y: auto;
}
```

| 屬性 | 控制方向 |
| --- | --- |
| `overflow-x` | 水平方向，左右溢出 |
| `overflow-y` | 垂直方向，上下溢出 |

### 常見情境：固定高度清單

```css
.list {
  max-height: 300px;
  overflow-y: auto;
  overflow-x: hidden;
}
```

這樣可以讓清單最多只佔 `300px` 高，內容太多時用垂直捲軸瀏覽，同時避免水平破版。

---

## 1.3 實戰範例：內容區塊出現捲軸

```html
<div class="message-list">
  <p>訊息 1</p>
  <p>訊息 2</p>
  <p>訊息 3</p>
  <p>訊息 4</p>
  <p>訊息 5</p>
</div>
```

```css
.message-list {
  width: 320px;
  max-height: 160px;
  overflow-y: auto;
  border: 1px solid #ddd;
  padding: 12px;
}
```

這種寫法常見於：

- 訊息列表
- 通知列表
- 下拉選單
- Modal 內容區
- 側邊欄選單

---

## 1.4 實戰範例：單行文字省略號

單行文字太長時，常用以下組合：

```css
.title {
  width: 240px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

三個屬性分工如下：

| 屬性 | 作用 |
| --- | --- |
| `white-space: nowrap` | 文字不換行 |
| `overflow: hidden` | 超出範圍的文字裁掉 |
| `text-overflow: ellipsis` | 被裁掉的地方顯示省略號 |

只寫 `text-overflow: ellipsis` 通常不會有效，因為它需要和 `overflow: hidden`、`white-space: nowrap` 搭配使用。

---

## 1.5 實戰範例：圖片裁切與圓角

```html
<div class="avatar">
  <img src="avatar.png" alt="使用者頭像" />
</div>
```

```css
.avatar {
  width: 80px;
  height: 80px;
  border-radius: 50%;
  overflow: hidden;
}

.avatar img {
  width: 100%;
  height: 100%;
  object-fit: cover;
}
```

這裡的 `overflow: hidden` 不是用來隱藏整個 `.avatar`，而是把圖片超出圓形邊界的部分裁掉。

---

## 1.6 `overflow: hidden` 的常見坑

`overflow: hidden` 很常用，但也很容易誤傷版面。

### 坑 1：下拉選單被裁掉

```css
.card {
  overflow: hidden;
}
```

如果 `.card` 裡面有下拉選單、Tooltip、Popover，這些浮出的內容可能會被父層裁掉。

```html
<div class="card">
  <button>更多操作</button>
  <div class="dropdown">選單內容</div>
</div>
```

如果 `.dropdown` 超出 `.card` 邊界，就可能被看不見。

解法通常是：

- 不要在需要浮出內容的父層加 `overflow: hidden`。
- 把下拉選單移到外層容器。
- 使用 Portal / Teleport，例如 Vue 的 `<Teleport>`。
- 重新檢查定位層級和 stacking context。

### 坑 2：陰影被裁掉

```css
.card {
  overflow: hidden;
  box-shadow: 0 8px 24px rgba(0, 0, 0, 0.12);
}
```

如果陰影或子元素超出邊界，可能會被裁切。

這時要判斷你真正需要的是：

- 裁切內部圖片？
- 裁切整張卡片？
- 還是只是想讓圖片有圓角？

如果只是圖片需要裁切，可以只在圖片包裝層加 `overflow: hidden`，不要加在整個卡片外層。

### 坑 3：`overflow: auto` 沒有捲軸

常見原因：元素沒有高度限制。

錯誤示範：

```css
.panel {
  overflow-y: auto;
}
```

如果 `.panel` 沒有 `height`、`max-height`，它可能會被內容撐高，因此不會出現捲軸。

修正：

```css
.panel {
  max-height: 400px;
  overflow-y: auto;
}
```

---

## 1.7 `overflow` 使用重點

- `overflow` 控制的是內容溢出，不是元素本身的顯示或隱藏。
- 要讓 `overflow` 有效果，元素通常需要有固定尺寸或最大尺寸限制。
- `overflow: hidden` 可以裁切內容，但也可能裁掉下拉選單、Tooltip、陰影。
- `overflow: auto` 通常比 `overflow: scroll` 更常用，因為它只在需要時出現捲軸。
- 只想控制單一方向時，使用 `overflow-x` 或 `overflow-y`。

# 2. `display: none`：讓元素完全消失

`display` 用來決定元素如何參與版面配置。

在隱藏元素時，最常見的寫法是：

```css
.hidden {
  display: none;
}
```

`display: none` 的效果是：

- 元素不顯示。
- 元素不佔版面位置。
- 版面會像這個元素不存在一樣重新排列。
- 子元素也會一起不顯示。
- 不適合直接拿來做一般的淡入淡出動畫。

---

## 2.1 範例：元素不佔位置

```html
<div class="box box-a">A</div>
<div class="box box-b">B</div>
<div class="box box-c">C</div>
```

```css
.box {
  width: 100px;
  height: 100px;
  margin-bottom: 12px;
}

.box-b {
  display: none;
}
```

畫面上只會看到 A 和 C，B 不會顯示，也不會留下空白位置。

---

## 2.2 顯示回來時，不要一律寫 `display: block`

很多初學者會這樣寫：

```css
.target {
  display: none;
}

.target.active {
  display: block;
}
```

這在某些情況可以，但不是萬用寫法。

如果元素原本是 `flex`：

```css
.menu {
  display: flex;
}
```

隱藏後再用 `display: block` 顯示，就會破壞原本的 flex 版面。

比較好的做法是保留原本樣式，另外用狀態 class 控制隱藏：

```css
.menu {
  display: flex;
  gap: 12px;
}

.menu.is-hidden {
  display: none;
}
```

這樣移除 `.is-hidden` 時，`.menu` 會回到原本的 `display: flex`。

---

## 2.3 JavaScript 實戰寫法

不建議在 JS 裡到處硬寫：

```js
menu.style.display = 'block';
```

因為你不一定知道原本應該是 `block`、`flex`、`grid`、`inline-block` 還是其他值。

比較建議：

```js
menu.classList.add('is-hidden');
menu.classList.remove('is-hidden');
```

```css
.menu {
  display: flex;
}

.menu.is-hidden {
  display: none;
}
```

這樣樣式仍由 CSS 管理，JS 只負責切換狀態。

---

## 2.4 Vue 補充：`v-if` 和 `v-show`

在 Vue 中，顯示 / 隱藏常見有兩種：

```vue
<div v-if="isOpen">內容</div>
<div v-show="isOpen">內容</div>
```

| 寫法 | 大致概念 | 適合情境 |
| --- | --- | --- |
| `v-if` | 條件成立才建立 DOM，不成立就移除 DOM | 不常切換、內容較重、需要重新掛載 |
| `v-show` | DOM 一直存在，只是切換 `display` | 經常切換顯示 / 隱藏 |

可以把 `v-show` 理解成：元素還在 DOM 裡，只是透過 CSS 類似 `display: none` 的方式隱藏。

### 實戰判斷

- Modal 很少打開，而且內容很重：可以考慮 `v-if`。
- Tab、Dropdown、Tooltip 經常切換：可以考慮 `v-show`。
- 需要保留元件內部狀態：通常 `v-show` 比較直覺。
- 希望每次重新打開都重新初始化：可以考慮 `v-if`。

---

## 2.5 `display: none` 使用重點

- 想讓元素完全消失且不保留位置，用 `display: none`。
- 不要把 `display: block` 當成萬用還原方式。
- JS 中建議切 class，不要到處硬改 `style.display`。
- 它不適合直接處理淡入淡出，動畫通常改用 `opacity` 或 `transform`。

# 3. `visibility: hidden`：看不見，但位置還在

`visibility` 用來控制元素是否可見。

常見值：

```css
.box {
  visibility: visible;
  visibility: hidden;
}
```

| 值 | 效果 |
| --- | --- |
| `visible` | 元素正常顯示 |
| `hidden` | 元素不可見，但仍保留版面位置 |
| `collapse` | 主要用於表格列、欄等場景，一般版面較少使用 |

---

## 3.1 範例：保留空白位置

```html
<div class="box box-a">A</div>
<div class="box box-b">B</div>
<div class="box box-c">C</div>
```

```css
.box {
  width: 100px;
  height: 100px;
  margin-bottom: 12px;
}

.box-b {
  visibility: hidden;
}
```

畫面上會看到 A 和 C，中間 B 的位置仍然保留，只是 B 本身看不見。

---

## 3.2 `visibility: hidden` 和 `display: none` 的差異

```css
.a {
  display: none;
}

.b {
  visibility: hidden;
}
```

| 寫法 | 是否看得見 | 是否佔位置 | 版面是否重排 |
| --- | --- | --- | --- |
| `display: none` | 否 | 否 | 會，因為元素從版面流中移除 |
| `visibility: hidden` | 否 | 是 | 通常不會，因為位置仍保留 |

### 什麼時候用 `visibility: hidden`？

適合需要保留版面穩定性的場景，例如：

- 表格欄位暫時不顯示，但仍想維持對齊。
- 某個提示文字暫時隱藏，但希望版面高度不要跳動。
- 需要避免顯示 / 隱藏造成 layout shift。

---

## 3.3 `visibility` 會繼承

`visibility` 是一個會被子元素繼承的屬性。

```css
.parent {
  visibility: hidden;
}
```

如果父元素被設定為 `visibility: hidden`，裡面的子元素也會一起看不見。

不過子元素可以重新設定：

```css
.parent {
  visibility: hidden;
}

.parent .child {
  visibility: visible;
}
```

這是 `visibility` 比較特殊的地方。不過實務上不建議濫用這種寫法，因為容易讓顯示邏輯變得難追蹤。

---

## 3.4 `visibility: hidden` 使用重點

- 元素看不見，但空間仍然保留。
- 適合避免版面跳動。
- 不適合用來表示「這個元素完全不存在」。
- 如果你希望元素完全不影響版面，應該用 `display: none`。
- 如果你希望做淡入淡出動畫，可以搭配 `opacity` 使用。

---

# 4. `opacity: 0`：完全透明，但元素仍存在

`opacity` 控制元素透明度。

```css
.box {
  opacity: 1; /* 完全不透明 */
  opacity: 0; /* 完全透明 */
}
```

`opacity: 0` 的效果是：

- 元素完全透明，看不見。
- 元素仍然佔版面位置。
- 元素通常仍然可以被滑鼠點擊。
- 元素裡的文字、圖片、子元素也會一起變透明。
- 很適合搭配 `transition` 做淡入淡出動畫。

---

## 4.1 範例：元素看不見但仍佔位置

```html
<button class="ghost-button">看不見但還在</button>
```

```css
.ghost-button {
  opacity: 0;
}
```

這個按鈕在畫面上看不見，但它仍在原本位置，使用者仍可能點到它。

這是 `opacity: 0` 最常見的坑。

---

## 4.2 防止透明元素被點擊

如果元素透明後不應該被點擊，可以加上：

```css
.panel {
  opacity: 0;
  pointer-events: none;
}

.panel.is-open {
  opacity: 1;
  pointer-events: auto;
}
```

`pointer-events: none` 的意思是：滑鼠事件不要落在這個元素上。

常見於：

- 淡出的 Modal 背景
- Dropdown 關閉狀態
- Tooltip 關閉狀態
- Toast 動畫結束前

---

## 4.3 淡入淡出動畫常用組合

```css
.fade-panel {
  opacity: 0;
  visibility: hidden;
  pointer-events: none;
  transition: opacity 0.2s ease;
}

.fade-panel.is-open {
  opacity: 1;
  visibility: visible;
  pointer-events: auto;
}
```

這個組合的分工是：

| 屬性 | 分工 |
| --- | --- |
| `opacity` | 控制淡入淡出視覺效果 |
| `visibility` | 控制是否可見，避免關閉後仍被視為可見元素 |
| `pointer-events` | 控制能不能被滑鼠點擊 |
| `transition` | 控制動畫過渡 |

### 為什麼不只用 `display: none`？

因為 `display: none` 會讓元素不參與渲染，傳統上不適合直接做一般的 `opacity` 淡入淡出動畫。

實務上更常見的是：

1. 元素保留在 DOM 中。
2. 透過 class 切換 `opacity`。
3. 動畫結束後，必要時再處理 `display` 或 DOM 移除。

---

## 4.4 `opacity` 的額外提醒

- `opacity` 會套用到整個元素，包含所有子元素。
- 父元素 `opacity: 0.5` 時，子元素無法用 `opacity: 1` 變回完全不透明。
- 當 `opacity` 小於 `1` 時，元素會形成新的堆疊上下文，這可能影響 `z-index` 判斷。
- `opacity: 0` 只是看不見，不等於不存在。

# 5. 四者完整比較

| 屬性 / 寫法 | 控制對象 | 是否看得見 | 是否佔位置 | 是否可能被點擊 | 是否適合動畫 | 主要用途 |
| --- | --- | --- | --- | --- | --- | --- |
| `overflow` | 盒子內部超出的內容 | 元素本身仍可見 | 是 | 視內容而定 | 不是主要動畫工具 | 控制溢出內容 |
| `display: none` | 元素本身是否參與版面 | 否 | 否 | 否 | 不適合直接做一般淡入淡出 | 完全隱藏元素 |
| `visibility: hidden` | 元素可見性 | 否 | 是 | 通常否 | 可搭配 `opacity` | 隱藏但保留空間 |
| `opacity: 0` | 元素透明度 | 否 | 是 | 是，除非加 `pointer-events: none` | 適合 | 淡入淡出、透明效果 |

# 6. 實戰選型：到底該用哪一個？

## 6.1 想讓元素完全消失

使用：

```css
.is-hidden {
  display: none;
}
```

適合：

- 條件不成立時完全不顯示。
- 不想保留空白位置。
- 不需要動畫。

---

## 6.2 想讓元素隱藏，但版面不要跳動

使用：

```css
.is-invisible {
  visibility: hidden;
}
```

適合：

- 保留原本尺寸。
- 避免旁邊元素突然移動。
- 表格、表單、固定區塊需要對齊。

---

## 6.3 想做淡入淡出

使用：

```css
.fade {
  opacity: 0;
  visibility: hidden;
  pointer-events: none;
  transition: opacity 0.2s ease;
}

.fade.is-active {
  opacity: 1;
  visibility: visible;
  pointer-events: auto;
}
```

適合：

- Modal
- Dropdown
- Tooltip
- Popover
- Toast
- 自訂選單

---

## 6.4 想裁切超出的內容

使用：

```css
.crop {
  overflow: hidden;
}
```

適合：

- 圖片裁切
- 卡片圓角
- 文字省略
- 避免內容破版

---

## 6.5 想讓某個區塊內部可以捲動

使用：

```css
.scroll-area {
  max-height: 360px;
  overflow-y: auto;
}
```

適合：

- 側邊欄
- 表格外層
- 訊息列表
- Modal body
- 長表單區域

# 7. 常見錯誤整理

## 錯誤 1：把 `overflow` 當成隱藏元素本身

錯誤理解：

```css
.box {
  overflow: hidden;
}
```

以為這樣會讓 `.box` 不見。

正確理解：

`overflow: hidden` 只會裁掉 `.box` 內部超出邊界的內容，`.box` 本身仍會顯示，也仍然佔位置。

---

## 錯誤 2：用 `display: block` 還原所有元素

錯誤：

```css
.item.is-show {
  display: block;
}
```

如果 `.item` 原本應該是 `flex`、`grid`、`inline-block`，這樣會破壞原本版面。

比較好的方式：

```css
.item {
  display: flex;
}

.item.is-hidden {
  display: none;
}
```

---

## 錯誤 3：用 `opacity: 0` 隱藏按鈕，但忘記它還能被點擊

錯誤：

```css
.button {
  opacity: 0;
}
```

修正：

```css
.button {
  opacity: 0;
  pointer-events: none;
}
```

如果這個元素也不應該被鍵盤或輔助工具操作，還要從 HTML 結構、焦點管理和可及性語意一起檢查，不能只靠 `opacity`。

---

## 錯誤 4：`overflow: auto` 沒有效果

常見原因：沒有設定高度或最大高度。

```css
.content {
  overflow-y: auto;
}
```

修正：

```css
.content {
  max-height: 400px;
  overflow-y: auto;
}
```

---

## 錯誤 5：下拉選單被父層 `overflow: hidden` 裁掉

錯誤場景：

```css
.card {
  overflow: hidden;
}
```

`.card` 裡面的 dropdown 超出邊界後被裁掉。

可能解法：

- 取消父層 `overflow: hidden`。
- 只在需要裁切的內層元素上加 `overflow: hidden`。
- 把 dropdown 移出被裁切的容器。
- Vue 中可考慮 `<Teleport>` 到 `body`。

---

## 錯誤 6：以為 `visibility: hidden` 不佔位置

錯誤理解：

```css
.box {
  visibility: hidden;
}
```

以為 `.box` 會從版面消失。

正確理解：

`.box` 看不見，但原本的寬高和位置仍然保留。

# 8. Vue / 前端互動實戰整理

## 8.1 顯示 / 隱藏的常見策略

| 需求 | Vue 寫法 | CSS 概念 |
| --- | --- | --- |
| 條件成立才建立元素 | `v-if` | DOM 建立 / 移除 |
| 元素頻繁切換顯示 | `v-show` | 類似切換 `display` |
| 做淡入淡出 | `<Transition>` + `opacity` | CSS transition |
| 避免透明元素被點擊 | class + `pointer-events` | 滑鼠事件控制 |
| 下拉選單避免被裁切 | `<Teleport>` | 脫離被裁切父層 |

---

## 8.2 Dropdown 的基本 CSS 狀態

```css
.dropdown {
  opacity: 0;
  visibility: hidden;
  pointer-events: none;
  transform: translateY(-4px);
  transition:
    opacity 0.2s ease,
    transform 0.2s ease;
}

.dropdown.is-open {
  opacity: 1;
  visibility: visible;
  pointer-events: auto;
  transform: translateY(0);
}
```

這種寫法比單純 `display: none` 更適合做動畫。

---

## 8.3 Modal 的基本分層思路

Modal 通常至少有兩層：

```html
<div class="modal-mask">
  <div class="modal-panel">內容</div>
</div>
```

```css
.modal-mask {
  position: fixed;
  inset: 0;
  display: flex;
  align-items: center;
  justify-content: center;
  background: rgba(0, 0, 0, 0.4);
}

.modal-panel {
  width: 480px;
  max-height: 80vh;
  overflow-y: auto;
  background: #fff;
  border-radius: 12px;
  padding: 24px;
}
```

這裡同時用到：

- `display: flex`：讓 Modal 置中。
- `max-height: 80vh`：限制內容最大高度。
- `overflow-y: auto`：內容太多時，讓 Modal 內部捲動。

這是 `display` 和 `overflow` 一起工作的典型場景。

# 9. 可及性與互動提醒

這一節先抓方向即可，不需要一次背完整 ARIA 規範。

## 9.1 看不見不一定等於不能互動

尤其是：

```css
.opacity-hidden {
  opacity: 0;
}
```

這種元素看不見，但仍可能：

- 擋住下面的元素。
- 被滑鼠點擊。
- 被鍵盤焦點選到。

所以當元素關閉時，若不應該互動，常見會搭配：

```css
.opacity-hidden {
  opacity: 0;
  pointer-events: none;
}
```

如果是 Modal、Dropdown、Popover 這種互動元件，還要額外處理：

- 開啟後焦點要不要移入。
- 關閉後焦點要不要回到觸發按鈕。
- 背景內容能不能被鍵盤操作。
- 輔助工具是否應該讀到隱藏內容。

---

## 9.2 不要只用 CSS 假裝禁用互動

例如只寫：

```css
.button-disabled {
  opacity: 0.5;
}
```

這只是視覺上看起來像禁用，並不代表按鈕真的不能點。

如果是按鈕，應該搭配 HTML 狀態：

```html
<button disabled>送出</button>
```

CSS 負責外觀，HTML / JS 狀態負責真正的互動邏輯。

# 10. 決策流程圖

可以用下面順序判斷：

```text
你要處理的是內容超出盒子嗎？
├─ 是：用 overflow / overflow-x / overflow-y
│
└─ 否：你要讓元素本身不見嗎？
   ├─ 想完全不佔位置：display: none
   ├─ 想看不見但保留位置：visibility: hidden
   ├─ 想做淡入淡出：opacity + transition
   └─ 看不見後也不能被點擊：再加 pointer-events: none
```

# 12. 一句話抓核心

`overflow` 管「內容超出盒子」，`display: none` 管「元素整個不參與版面」，`visibility: hidden` 管「看不見但位置還在」，`opacity: 0` 管「透明但元素仍存在」。

---

# 13. 本章速查表

| 場景 | 建議寫法 | 注意事項 |
| --- | --- | --- |
| 完全隱藏元素 | `display: none` | 不佔位置，不適合直接淡入淡出 |
| 隱藏但保留空間 | `visibility: hidden` | 版面穩定，但仍保留尺寸 |
| 淡入淡出 | `opacity` + `transition` | 關閉狀態要注意點擊問題 |
| 透明後不可點 | `pointer-events: none` | 只處理滑鼠事件，不等於完整可及性處理 |
| 區塊內部捲動 | `max-height` + `overflow-y: auto` | 沒有限高通常不會出現捲軸 |
| 裁切圖片 | `overflow: hidden` | 常搭配 `border-radius`、`object-fit` |
| 單行省略號 | `overflow: hidden` + `white-space: nowrap` + `text-overflow: ellipsis` | 三者通常要一起用 |
| 下拉選單被裁切 | 檢查父層 `overflow: hidden` | 可考慮 Teleport / Portal |

# 14. 修正說明

本版相對原筆記做了以下強化：

1. 保留原本「`overflow` → `display` → `visibility` → 比較」的清楚結構。
2. 新增 `opacity: 0`，補齊實務中最常見的第四種隱藏方式。
3. 新增 `overflow-x` / `overflow-y`，補齊水平與垂直方向控制。
4. 新增文字省略號、圖片裁切、固定高度捲動區塊等常見實戰案例。
