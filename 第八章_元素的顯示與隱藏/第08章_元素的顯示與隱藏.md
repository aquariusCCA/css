# 第08章_元素的顯示與隱藏

> 本章在講三件事：`overflow` 怎麼處理溢出內容、`display` 怎麼直接讓元素消失、`visibility` 怎麼讓元素看不見但保留位置。
> 
> 這三個屬性都和「顯示 / 隱藏」有關，但作用層級不同，不能混用。
>
> 返回根目錄：[README.md](../README.md)

## 關鍵字

- `overflow`
- `display`
- `visibility`
- `hidden`
- `none`
- `scroll`
- `auto`

## 建議回查情境

- 內容超出固定寬高時，想決定要不要顯示捲軸
- 點擊切換顯示 / 隱藏功能時，想讓元素從版面消失
- 想隱藏元素，但又希望原本位置保留不變
- 想快速分辨 `display: none` 和 `visibility: hidden` 的差異

## 這一章先抓結論

1. `overflow` 是在處理「內容溢出盒子」時要怎麼顯示。
2. `display: none` 是把元素整個從版面流中移除。
3. `visibility: hidden` 是讓元素看不見，但原本位置還在。

## 30 秒複習入口

- 溢出內容要裁掉、顯示捲軸，先看 `overflow`
- 元素要整個消失且不占位置，用 `display: none`
- 元素要隱藏但保留位置，用 `visibility: hidden`

---

## 1. `overflow`：處理內容溢出

`overflow` 用來控制：當內容超出元素指定的寬高時，要怎麼顯示。

常見值如下：

```css
.box {
  overflow: visible; /* 預設值，溢出內容直接顯示出來 */
  overflow: hidden;   /* 超出的內容直接裁掉 */
  overflow: scroll;   /* 不管有沒有溢出，都顯示捲軸 */
  overflow: auto;     /* 只有真的溢出時，才顯示捲軸 */
}
```

### 什麼時候用

- `visible`
  - 預設行為
  - 內容超出去也照樣顯示
- `hidden`
  - 超出盒子的內容會被裁切
  - 適合不想讓內容破壞版面時使用
- `scroll`
  - 會強制顯示捲軸
  - 即使內容沒有溢出，也可能看到捲軸
- `auto`
  - 只有在內容真的超出時，才顯示捲軸
  - 一般比 `scroll` 更常用

### 範例

```css
.peppa {
  width: 200px;
  height: 200px;
  border: 3px solid pink;
  margin: 100px auto;
  overflow: auto;
}
```

```html
<div class="peppa">
  內容很多很多很多很多很多很多很多很多很多很多很多很多很多很多很多
</div>
```

### 使用提醒

- `overflow: hidden` 常用來裁切超出框線的內容。
- 如果盒子裡有需要顯示在外面的內容，或是定位後會超出邊界的元素，要小心使用。
- 這個屬性不是在「隱藏元素本身」，而是在處理「盒子裡超出去的部分」。

---

## 2. `display`：讓元素直接消失

`display` 可以決定元素怎麼參與版面配置。
在隱藏的用法裡，最重要的是 `display: none`。

```css
display: none;
```

它的效果是：

- 元素不顯示
- 元素不占位置
- 版面會像這個元素不存在一樣

### 範例

```css
.peppa {
  display: none;
  width: 200px;
  height: 200px;
  background-color: pink;
}

.george {
  width: 200px;
  height: 200px;
  background-color: skyblue;
}
```

```html
<div class="peppa">佩佩</div>
<div class="george">喬治</div>
```

### 顯示回來怎麼辦

如果原本是區塊元素，常見會寫回：

```css
display: block;
```

但要注意：

- 這只是範例中的還原方式
- 不代表所有元素都應該一律改成 `block`
- 如果原本是 `span`、`inline-block` 等元素，還原方式要依原本顯示型態決定

### 使用重點

- 想讓元素完全消失，且不保留位置，用 `display: none`
- `display` 比較像「把元素整個拿掉」
- 這也是前端互動中最常用的隱藏方式之一

---

## 3. `visibility`：看不見，但位置還在

`visibility` 用來控制元素是否可見。

```css
visibility: visible;
visibility: hidden;
```

### 效果

- `visibility: visible`
  - 元素正常顯示
- `visibility: hidden`
  - 元素看不見
  - 但原本占用的空間還保留

### 範例

```css
.baba {
  visibility: hidden;
  width: 200px;
  height: 200px;
  background-color: pink;
}

.mama {
  width: 200px;
  height: 200px;
  background-color: skyblue;
}
```

```html
<div class="baba">爸爸</div>
<div class="mama">媽媽</div>
```

### 使用重點

- 想保留原來位置，只是不讓它看見，用 `visibility: hidden`
- 這個屬性適合需要維持版面穩定，但暫時不顯示內容的情境

---

## 4. 三者比較

| 屬性 | 是否顯示 | 是否保留位置 | 主要用途 |
| --- | --- | --- | --- |
| `overflow` | 看內容是否超出盒子 | 會 | 控制溢出內容的顯示方式 |
| `display: none` | 否 | 否 | 完全隱藏元素 |
| `visibility: hidden` | 否 | 是 | 隱藏元素，但保留位置 |

## 5. 易錯點

- `overflow` 不是用來隱藏整個元素，而是控制溢出部分。
- `display: none` 和 `visibility: hidden` 不一樣，前者會讓元素不占位置，後者會保留位置。
- `display: block` 不是萬用的「顯示回來」寫法，只能當作某些元素的還原範例。
- `overflow: scroll` 會強制出現捲軸，`auto` 則是按需出現。

## 6. 一句話抓核心

`overflow` 管溢出，`display: none` 管整個消失，`visibility: hidden` 管看不見但位置還在。
