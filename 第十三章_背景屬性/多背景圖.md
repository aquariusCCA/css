## 本節導讀

CSS 允許同一個元素同時擁有**多張背景圖**。

最基本的寫法是：

```css
.box {
  background-image:
    url("./assets/badge.svg"),
    url("./assets/pattern-grid.svg");
}
```

也就是說：

> 一個元素，不只可以有一張背景圖，還可以有很多層背景。

這在實務上非常常見，像是：

- Hero 區塊：深色遮罩 + 背景圖
- 卡片：角落裝飾圖 + 底紋 + 漸層底色
- Banner：光暈 + 主背景 + 顏色底
- 漸層疊圖：線性漸層 + 圖片 + pattern
- UI 視覺修飾：陰影紋理 + 小 icon + 底色

不過多背景圖最容易讓人混亂的地方是：

1. 圖層順序到底怎麼看？
2. 第一張圖是在最上面，還是最下面？
3. `background-position`、`background-size`、`background-repeat` 要怎麼一一對應？
4. `background-color` 可以放在哪一層？
5. 用 `background` shorthand 時，逗號要怎麼分？
6. 為什麼有時後面的圖被前面的圖遮住？
7. 為什麼每一層可以有不同的 repeat / size / position？

所以本節真正要學會的核心是：

```text
多背景圖 = 多層背景依序堆疊
第一個在最上層
最後一個在最下層
背景色永遠在所有背景圖的最底下
其他背景相關屬性可用逗號逐層對應
```

## 正文

## 1. 多背景圖是什麼？

多背景圖就是：

```text
在同一個元素上，同時定義多個背景層。
```

語法上最常見的是在 `background-image` 中用逗號分隔：

```css
.box {
  background-image:
    url("./assets/badge.svg"),
    url("./assets/pattern-grid.svg"),
    linear-gradient(135deg, #eff6ff, #ffffff);
}
```

你可以把它想像成把好幾張透明膠片，從上到下疊在同一個元素上。

---

## 2. 圖層順序：第一個在最上層

這是本節最重要的觀念之一。

```css
.box {
  background-image:
    url("./assets/badge.svg"),
    url("./assets/pattern-grid.svg");
}
```

這裡：

- `badge.svg` 在最上層
- `pattern-grid.svg` 在下層

如果第一張背景有大片不透明區域，下面那張背景可能就完全看不到。

### 一句話記憶

```text
第一個最上面，最後一個最下面。
```

---

## 3. 背景色永遠在最底下

不管你有幾張背景圖，`background-color` 都是在所有背景圖的最底層。

```css
.hero {
  background-image:
    linear-gradient(rgb(15 23 42 / 70%), rgb(15 23 42 / 70%)),
    url("./assets/hero-scene.svg");
  background-color: #0f172a;
}
```

圖層順序是：

```text
第 1 層：linear-gradient(...)  最上層
第 2 層：hero-scene.svg       下層
最底層：#0f172a               背景色
```

所以 `background-color` 不是一張背景圖，不能像背景圖那樣插在中間。

---

## 4. 漸層也算一層背景

這是多背景圖非常重要、也非常實用的地方。

```css
.hero {
  background-image:
    linear-gradient(rgb(15 23 42 / 70%), rgb(15 23 42 / 70%)),
    url("./assets/hero-scene.svg");
}
```

這表示：

- 上層是半透明深色漸層
- 下層是圖片

這樣就能在不額外新增元素的情況下，做出：

- 深色遮罩
- 霧面遮罩
- 漸層遮罩
- 亮角 / 暗角效果

---

## 5. 各背景子屬性如何逐層對應？

多背景圖最容易讓人搞混的，是其他背景子屬性怎麼跟著對應。

答案是：

> **同樣用逗號分隔，按順序一一對應。**

例如：

```css
.card {
  background-image:
    url("./assets/badge.svg"),
    url("./assets/pattern-grid.svg");

  background-repeat:
    no-repeat,
    repeat;

  background-position:
    right 24px bottom 24px,
    left top;

  background-size:
    96px 96px,
    72px 72px;
}
```

對應關係是：

- 第 1 個 repeat 對第 1 張背景圖
- 第 2 個 repeat 對第 2 張背景圖

同理：

- 第 1 個 position 對第 1 張背景圖
- 第 2 個 position 對第 2 張背景圖

- 第 1 個 size 對第 1 張背景圖
- 第 2 個 size 對第 2 張背景圖

### 記法

```text
第一個對第一層
第二個對第二層
第三個對第三層
```

---

## 6. 如果層數不一致怎麼辦？

實務上常常會只寫一個值，讓多層共用。

例如：

```css
.card {
  background-image:
    url("./assets/badge.svg"),
    url("./assets/pattern-grid.svg");

  background-repeat: no-repeat;
}
```

這不代表只有第一層有 `no-repeat`。  
瀏覽器會用規則去補足未寫滿的部分，通常可以理解成：

```text
如果只寫一個值，所有背景層都使用這個值。
```

但在教學和維護上，我更建議你在多背景圖時盡量寫完整，避免誤解：

```css
background-repeat:
  no-repeat,
  repeat;
```

這樣最清楚。

---

## 7. 實務上最常見的組合：Hero 背景

多背景圖最常見的用途之一，就是 Hero 區塊。

```css
.hero {
  background:
    linear-gradient(rgb(15 23 42 / 68%), rgb(15 23 42 / 68%)),
    url("./assets/hero-scene.svg") center / cover no-repeat #0f172a;
}
```

這個寫法同時做了三件事：

1. 用第一層漸層遮罩壓暗背景
2. 用第二層圖片作為主背景
3. 用底色補足圖片沒覆蓋到的區域

優點：

- 不必額外加遮罩元素
- 視覺整潔
- 可讀性高
- 內容文字更容易閱讀

---

## 8. 實務案例：卡片裝飾圖 + 底紋 + 底色

```css
.card {
  background-image:
    url("./assets/badge.svg"),
    url("./assets/pattern-grid.svg"),
    linear-gradient(135deg, #eff6ff, #ffffff);

  background-repeat:
    no-repeat,
    repeat,
    no-repeat;

  background-position:
    right 24px bottom 24px,
    left top,
    center;

  background-size:
    96px 96px,
    72px 72px,
    auto;
}
```

圖層順序：

```text
第 1 層：badge.svg               右下角裝飾
第 2 層：pattern-grid.svg        平鋪底紋
第 3 層：linear-gradient(...)    底部漸層
```

這種寫法很適合：

- UI 卡片
- 儀表板區塊
- Landing page feature card
- 資訊模組區塊

## 9. `background` shorthand 中的多背景圖

`background` shorthand 也支援多背景圖。  
你可以把每一層都視為一組獨立的 background shorthand，再用逗號分隔。

```css
.hero {
  background:
    linear-gradient(rgb(15 23 42 / 68%), rgb(15 23 42 / 68%)),
    url("./assets/hero-scene.svg")
    center / cover
    no-repeat
    fixed
    #0f172a;
}
```

要注意：

- 每一層之間用逗號分隔
- `background-size` 要寫在 `background-position` 後面，並用 `/` 分隔
- `background-color` 只能放在最後一層

### 再看一個完整範例

```css
.card {
  background:
    url("./assets/badge.svg")
    right 24px bottom 24px / 96px 96px
    no-repeat,

    url("./assets/pattern-grid.svg")
    left top / 72px 72px
    repeat,

    linear-gradient(135deg, #eff6ff, #ffffff);
}
```

---

## 10. 多背景圖與 `origin / clip / attachment`

除了 `position`、`size`、`repeat`，其他背景子屬性也能逐層設定，例如：

```css
.card {
  background-image:
    url("./assets/badge.svg"),
    url("./assets/pattern-grid.svg");

  background-origin:
    content-box,
    border-box;

  background-clip:
    content-box,
    border-box;

  background-attachment:
    scroll,
    local;
}
```

這代表：

- 第 1 層角落徽章：定位與裁切都只在 `content-box`
- 第 2 層底紋：可以從 `border-box` 開始鋪

雖然不是每次都需要這麼細緻，但你要知道：

```text
多背景圖不是只有 image 可以多層，
其他背景子屬性也都可以逐層對應。
```

## 11. 常見錯誤 1：把圖層順序看反

錯誤觀念：

```text
最後一張背景在最上層
```

這是錯的。  
正確是：

```text
第一張在最上層
最後一張在最下層
```

---

## 12. 常見錯誤 2：背景色放在錯誤位置

錯誤觀念：

```css
background:
  #0f172a,
  url("./assets/hero-scene.svg");
```

這樣的概念就是錯的。  
背景色不是一層一般背景圖，應該放在最後一層 shorthand 的最後位置。

正確示意：

```css
background:
  linear-gradient(rgb(15 23 42 / 70%), rgb(15 23 42 / 70%)),
  url("./assets/hero-scene.svg") center / cover no-repeat #0f172a;
```

---

## 13. 常見錯誤 3：某層太不透明，蓋住下面所有背景

如果你的第一層是：

```css
linear-gradient(#000, #000)
```

這是完全不透明的純黑遮罩。  
那下面的背景圖就可能完全看不到。

修正方式：

```css
linear-gradient(rgb(0 0 0 / 50%), rgb(0 0 0 / 50%))
```

也就是改成半透明。

---

## 14. 常見錯誤 4：多背景圖寫太複雜，維護困難

雖然多背景圖很強，但如果你寫成：

- 4~6 層以上
- 每層都有很多子屬性
- 還要搭配複雜動畫

可讀性會快速下降。

這時要思考：

- 是否拆成 pseudo-element
- 是否使用額外 DOM
- 是否把某些裝飾轉成獨立元素
- 是否把內容圖片改回 `<img>`

### 原則

```text
視覺裝飾適合背景
有語意 / 有互動 / 需要替代文字的內容，不適合硬塞進背景
```

## 15. 什麼時候適合用多背景圖？

### 很適合

- Hero 遮罩 + 背景圖
- 卡片角落裝飾 + 底紋
- Banner 光暈 + 主圖 + 漸層底
- 裝飾性視覺疊圖
- 不需要語意的純裝飾圖層

### 不一定適合

- 需要 SEO / alt 的圖片
- 有互動的圖像內容
- 重要資訊圖表
- 太複雜的構圖（難維護）
- 需要獨立動畫控制的多個元素

## 16. 面試與讀碼常見問法

### 問題 1：CSS 可以同時設定多張背景圖嗎？

可以。  
做法是在背景相關屬性中使用逗號分隔多個值。

---

### 問題 2：多背景圖哪一張在最上面？

第一張在最上面。  
最後一張在最下面。

---

### 問題 3：`linear-gradient(...), url(...)` 誰在上面？

`linear-gradient(...)` 在上面，因為它寫在前面。

---

### 問題 4：多背景圖時，`background-position` 如何對應？

第一個 `background-position` 對第一張背景圖，第二個對第二張，以此類推。

---

### 問題 5：多背景圖時，背景色放哪裡？

背景色在所有背景圖的最底下。  
若用 shorthand，應放在最後一層的最後位置。

---

### 問題 6：多背景圖可以逐層設定 `background-size` 嗎？

可以。

```css
background-size:
  96px 96px,
  72px 72px;
```

```css
    url("./assets/badge.svg") right 24px bottom 24px / 96px 96px no-repeat,

    linear-gradient(135deg, #eff6ff, #ffffff);
}
```

### 練習 3：觀察圖層順序

建立兩個盒子：

```css
background-image:
  url("./assets/badge.svg"),
  url("./assets/halo.svg");
```

與：

```css
background-image:
  url("./assets/halo.svg"),
  url("./assets/badge.svg");
```

觀察順序交換後，哪一層被蓋住。

### 練習 4：逐層設定 size / repeat

建立兩張背景：

- 第一層不重複、尺寸 96px 96px
- 第二層重複、尺寸 72px 72px

用 `background-size`、`background-repeat` 逐層指定。
