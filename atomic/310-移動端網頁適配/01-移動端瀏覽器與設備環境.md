# 移動端瀏覽器與設備環境

## 瀏覽器現狀

移動端瀏覽器環境會隨平台而不同：iOS 主要基於 WebKit，Android 常見瀏覽器與 WebView 多基於 Chromium/Blink；實務上應依目標瀏覽器查詢兼容性。

![PC 與移動端常見瀏覽器對照](../../origin/310-移動端網頁適配/assets/images/mobile-basics-img-001-46a1e5.png)

<aside>
💡

**總結：兼容移動端主流瀏覽器時，不能只假設 WebKit；需要同時關注 iOS Safari、Android Chrome / WebView，以及具體 CSS 特性的支援狀況。**

</aside>

## 手機屏幕現狀

- 屏幕尺寸

    ![手機螢幕尺寸與像素密度示意圖](../../origin/310-移動端網頁適配/assets/images/mobile-basics-img-002-7e9531.png)

    - 屏幕對角線的長度，一般用英寸來度量。
- 移動端設備屏幕尺寸非常多，碎片化嚴重。
- `Android` 設備有多種分辨率：`480x800`、`480x854`、`540x960`、`720x1280`、`1080x1920` 等，還有傳說中的 2K，4k 屏。
- 近年來 iPhone 的碎片化也加劇了，其設備的主要分辨率有：`640x960`、`640x1136`、`750x1334`、`1242x2208` 等。

<aside>
💡

**作為開發者無需關注這些分辨率，因為我們常用的尺寸單位是 px。**

</aside>

## 移動端瀏覽器

![移動端瀏覽器基於 WebKit 內核示意圖](../../origin/310-移動端網頁適配/assets/images/mobile-basics-img-004-8c2de1.png)

- 移動端瀏覽器曾長期以 WebKit / Blink 系列為主，但實際兼容性仍需依目標環境確認。
- 可以使用現代 `HTML5` 標籤和 `CSS3` 樣式，但關鍵特性仍應查詢瀏覽器支援狀況。
- 瀏覽器私有前綴不應只固定考慮 `-webkit-`，應以實際 CSS 特性與目標瀏覽器需求為準。
