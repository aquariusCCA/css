# 浮動布局注意點與布局總結

浮動布局需要和標準流父盒子搭配：

- 先用標準流的父元素排列上下位置。
- 再讓內部子元素採取浮動排列左右位置。

一個元素浮動後，理論上其餘兄弟元素也要浮動。

如果一個盒子裡有多個盒子，其中一個盒子浮動了，其他兄弟盒子也應該浮動，以防止布局問題。

浮動盒子只會影響它後面的標準流，不會影響它前面的標準流。

網頁布局總結：

- 盒子模型可以幫助理解大部分 HTML 標籤都是盒子。
- CSS 浮動與定位可以讓每個盒子排列成網頁。
- 完整網頁通常由標準流、浮動、定位一起完成布局，每種方式都有專門用途。

三種傳統布局分工：

- 標準流：讓盒子上下排列或左右排列，垂直塊級盒子顯示就用標準流布局。
- 浮動：讓多個塊級元素一行顯示，或讓盒子左右對齊；多個塊級盒子水平顯示就用浮動布局。
- 定位：具有層疊概念，可以讓多個盒子前後疊壓顯示；元素需要在某個盒子內自由移動時使用定位布局。
