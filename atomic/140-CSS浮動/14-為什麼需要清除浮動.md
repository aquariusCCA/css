# 為什麼需要清除浮動

前面的浮動元素有一個標準流父元素，並且父元素都有高度。但實際布局中，所有父盒子不一定都方便設定高度。

![產品列表和新聞欄內容高度不固定的頁面示例](../../origin/140-CSS浮動/assets/images/clear-float-img-001-2d7043.png)

理想狀態是讓子盒子撐開父盒子，有多少孩子，父盒子就有多高。

如果父級盒子不方便給高度，而子盒子浮動後又不佔有位置，最後父級盒子高度會變成 `0`，進而影響下面的標準流盒子。

![子盒子浮動導致父盒子高度塌陷且下方盒子上移的示意圖](../../origin/140-CSS浮動/assets/images/clear-float-img-002-2d7402.png)

由於浮動元素不再以普通流方式佔據原本的位置，所以父級可能無法靠它自動撐高；同時，浮動元素仍會影響後續文字與行內內容的排版。

```css
.box {
  width: 800px;
  border: 1px solid blue;
  margin: 0 auto;
}

.damao {
  float: left;
  width: 300px;
  height: 200px;
  background-color: purple;
}

.ermao {
  float: left;
  width: 200px;
  height: 200px;
  background-color: pink;
}

.footer {
  height: 200px;
  background-color: black;
}
```

```html
<body>
  <div class="box">
    <div class="damao">大毛</div>
    <div class="ermao">二毛</div>
  </div>

  <div class="footer"></div>
</body>
```
