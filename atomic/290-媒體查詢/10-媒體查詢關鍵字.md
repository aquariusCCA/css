# 媒體查詢關鍵字

> 關鍵字將媒體類型或多個媒體特性連接到一起做為媒體查詢的條件。

## and

`and` 可以將多個媒體特性連接到一起，相當於「且」。

也就是說，一個媒體查詢中可以包含 0 到多個表達式，表達式又可以包含 0 到多個關鍵字，以及一種媒體類型。

```html
<!DOCTYPE html>
<html lang="en">

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
  <style>
    * {
      margin: 0;
      padding: 0;
    }

    h1 {
      height: 200px;
      background-color: gray;
      text-align: center;
      line-height: 200px;
      font-size: 100px;
    }

    /* 且運算 */
    @media screen and (min-width: 700px) and (max-width: 800px) {
      h1 {
        background-color: orange;
      }
    }
  </style>
</head>

<body>
  <h1>你好啊!!</h1>
</body>

</html>
```

## or

對於所有的連接選項，只要匹配成功一個就能應用規則。

原始筆記說明：將多個媒體查詢以逗號分隔放在一起，只要其中任何一個為真，整個媒體語句就返回真，相當於 or 操作符。

```html
<!DOCTYPE html>
<html lang="en">

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
  <style>
    * {
      margin: 0;
      padding: 0;
    }

    h1 {
      height: 200px;
      background-color: gray;
      text-align: center;
      line-height: 200px;
      font-size: 100px;
    }

    /* 或運算 */
    @media (max-width: 700px),
    (min-width: 800px) {
      h1 {
        background-color: orange;
      }
    }
  </style>
</head>

<body>
  <h1>你好啊!!</h1>
</body>

</html>
```

逗號分隔是相容性很好的 OR 寫法；Media Queries Level 4 也支援使用 `or` 組合媒體條件，但實務上仍需依目標瀏覽器相容性選擇寫法。

```css
@media (max-width: 700px) or (min-width: 800px) {
  h1 {
    background-color: orange;
  }
}
```

## not

`not` 操作符用來對一條媒體查詢的結果進行取反，相當於「非」。

> 注意：`not` 關鍵字僅能應用於整個查詢，而不能單獨應用於一個獨立的查詢。

```html
<!DOCTYPE html>
<html lang="en">

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
  <style>
    * {
      margin: 0;
      padding: 0;
    }

    h1 {
      height: 200px;
      background-color: gray;
      text-align: center;
      line-height: 200px;
      font-size: 100px;
    }

    /* 否定運算符 */
    @media not screen {
      h1 {
        background-color: orange;
      }
    }
  </style>
</head>

<body>
  <h1>你好啊!!</h1>
</body>

</html>
```

## only

`only` 操作符不會改變媒體查詢本身的匹配結果。

它主要用於讓非常舊、只理解媒體類型但不理解媒體特性的瀏覽器忽略該段樣式；現代開發通常不需要刻意加上 `only`。

```css
/* 肯定運算符 */
@media only screen and (width: 800px){
  h1 {
    background-color: orange;
  }
}
```

上面這行代碼，在老式瀏覽器中被解析為 `media="only"`，因為沒有一個叫 only 的設備，所以實際上老式瀏覽器不會應用樣式。

```css
@media screen and (width: 800px){
  h1 {
    background-color: orange;
  }
}
```

上面這行代碼，在老式瀏覽器中可能被解析為 `media="screen"`，並忽略後面的邏輯表達式，造成樣式被錯誤套用。這是 `only` 最初要處理的相容性問題；但這類瀏覽器在現代已極少見，因此 `only` 多半不是必要寫法。
