請依序套用以下提示詞：

- `prompts/system/base-role.md`
- `prompts/system/css-teacher-role.md`
- `prompts/workflows/notes-to-css-appendix.md`
- `prompts/formats/css-appendix-reference-format.md`

請根據以下設定產生 CSS 查表型資料。

### 必要設定

- 筆記路徑：`notes\第07章_元素的顯示與隱藏\元素的顯示與隱藏.md`
- 章節名稱：`第07章_元素的顯示與隱藏`
- 預期輸出目錄：`notes\第07章_元素的顯示與隱藏\`
- 查表類型：`全部類型`

### 任務要求

1. 先閱讀並理解指定筆記。
2. 在產生 appendix 前，先判斷本章主軸屬於屬性密集、值密集、概念判斷、語法結構、除錯流程或維護模型，並用判斷結果決定哪些查表類型真的適合產生。
3. 預設產生 `README.md` 作為章節 appendix 入口，讓讀者知道本章應先查哪份資料、核心對照是什麼、哪些預設查表檔未產生。
4. 判斷筆記中哪些內容可抽成：
   - 章節總覽入口。
   - CSS 屬性速查表。
   - 屬性值對照表。
   - 名詞表。
   - 問題對應屬性或檢查表。
   - 常見誤解速查。
   - 關鍵字索引。
5. 只整理筆記已支持的內容，不新增筆記沒有提到的新 CSS 知識。
6. 若某個欄位缺少來源依據，填入「待補來源」；若整個條目缺少來源依據，請刪除該條目。
7. 若某個查表類型只有少量範例材料，且不能幫助讀者理解本章主題，請省略該檔案，不要為了湊齊預設檔案而產生低價值表格。
8. 查表內容應短、準、可掃描，優先使用表格。
9. 不要把正式筆記重寫成摘要，也不要複製正式筆記的大段文字。
10. 每份 appendix 檔案開頭都要標明來源筆記路徑與用途；`README.md` 還要說明本章查表路徑。
11. 若筆記路徑是章節目錄，請整理該目錄下與章節主題相關的 `.md` 檔案。
12. 若本次只指定部分查表類型，只產生指定類型；只有在指定包含章節總覽入口或任務要求整章 appendix 時，才產生 `README.md`。
13. 預設輸出到 `appendix/<章節>/`，章節名稱應與 `notes/<章節>/` 保持一致。
14. 若預期輸出目錄被填成 `.md` 檔案路徑，請先指出欄位不一致；除非任務明確要求單檔輸出，否則改用該路徑的父層資料夾作為輸出目錄。
15. 每個含有 CSS 知識條目的表格都要包含 `來源` 欄，指向來源知識所在的最近標題，標題層級可為 H2-H6；純導航表不需要 `來源` 欄。
16. 產出前請檢查：
    - 每個條目是否能回到來源筆記。
    - 每列 `來源` 是否能定位到來源筆記中最直接支持該列的標題。
    - 是否避免加入來源筆記未支持的新知識。
    - 是否符合指定查表格式。
    - `README.md` 是否讓讀者知道本章先看哪份表。
    - 是否適合快速查找，而不是教學閱讀。
    - 是否清楚區分入口、屬性、屬性值、名詞、問題、誤解與關鍵字。
    - 屬性表與屬性值表是否真的有查表價值，而不是只收錄範例材料。
17. 最後將查表型資料產生到預期輸出目錄，並列出已省略的預設查表檔與原因。
