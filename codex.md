請依序套用以下提示詞：

- `prompts/system/base-role.md`
- `prompts/system/css-teacher-role.md`
- `prompts/system/css-demo-role.md`
- `prompts/workflows/notes-to-css-demo.md`

請根據以下設定產生 CSS demo 範例。

### 必要設定

- 來源路徑：`notes\第26章_Flex布局\flex佈局常見父項屬性_測試.md`
- 目標知識點：`flex佈局常見父項屬性`
- 目標輸出路徑：`demos\第26章_Flex布局`

### 任務要求

1. 先閱讀並理解來源筆記。
2. 確認來源路徑位於 `notes/` 下；若不是正式教學筆記，請停止產生並說明原因。
3. 確認目標輸出路徑位於 `demos/` 下；若不是 demo 目錄路徑，請停止產生並說明原因。
4. 不得修改來源筆記；`notes/` 只作為 demo 設計依據。
5. 根據目標知識點，從來源筆記中找出直接相關的段落、範例、問題情境、常見誤解與檢查重點。
6. 若目標知識點太大，請先收斂成一個可做成單一 demo 的主要概念，並說明收斂結果。
7. 決定 demo 型態：單一狀態、修改前 / 修改後、錯誤 / 正確，或少量互動切換。
8. 設計最小必要 HTML 與 CSS，讓學習者能清楚觀察：
   - CSS 的作用對象。
   - 主要 CSS 宣告。
   - 修改前後或不同狀態的畫面差異。
   - 對應的常見誤解或失效原因。
9. 優先產生可直接用瀏覽器開啟的靜態範例；只有互動能幫助觀察 CSS 行為時，才建立 `main.js`。
10. 不引入不必要的框架、外部 CDN、建置工具或與目標知識點無關的 CSS 技巧。
11. README 應說明來源筆記、目標知識點、開啟方式、操作方式、觀察重點與常見誤解。
12. 產出前請檢查：
    - demo 是否能直接用瀏覽器開啟。
    - demo 是否只聚焦一個主要知識點。
    - demo 是否能清楚觀察主要 CSS 宣告造成的畫面變化。
    - 是否沒有引用不存在的 CSS、JS、圖片或其他資產。
    - 是否沒有把正式筆記大段複製進 demo 頁面。
    - README 是否讓學習者知道要觀察什麼，而不是只列檔案清單。
13. 最後將 demo 檔案產生到目標輸出路徑。
