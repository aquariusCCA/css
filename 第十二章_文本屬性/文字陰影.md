# 文字陰影

> 返回章節首頁：[README.md](./README.md)
>
> `text-shadow` 屬性用於替文字添加陰影效果，可以讓文字產生立體感、發光感，或在複雜背景上提升可讀性。

![文字陰影](./images/image_1776231946898_b4e066e0.png)

## 導讀

- `text-shadow` 用來替文字加上陰影。
- 常見語法包含水平位移、垂直位移、模糊半徑與陰影顏色。
- 最基本格式是：`text-shadow: x-offset y-offset blur-radius color;`
- 水平位移控制陰影往左或往右。
- 垂直位移控制陰影往上或往下。
- 模糊半徑控制陰影邊緣的模糊程度。
- 顏色通常會使用 `rgba()` 或 `hsla()`，方便設定透明度。
- `text-shadow` 可以設定多組陰影，使用逗號分隔。
- `text-shadow` 不會改變元素本身的盒模型大小。
- 陰影可能會超出元素內容區域，如果父層有 `overflow: hidden;`，陰影可能被裁切。
- 實務上文字陰影要謹慎使用，過重的陰影會降低可讀性，也可能影響畫面質感。
- 如果只是為了讓圖片背景上的文字更清楚，通常還要搭配遮罩、漸層或背景層，不應只依賴文字陰影。

## 30 秒複習入口

- `text-shadow` 用來替文字添加陰影。
- 基本語法是：`text-shadow: 水平位移 垂直位移 模糊半徑 顏色;`
- `text-shadow: 5px 5px 6px rgba(0, 0, 0, .3);` 代表陰影往右下偏移，並帶有模糊效果。
- 第一個值是水平位移，正值往右，負值往左。
- 第二個值是垂直位移，正值往下，負值往上。
- 第三個值是模糊半徑，數值越大越模糊。
- 顏色建議明確指定，常用 `rgba()` 做半透明陰影。
- `text-shadow: none;` 可以取消文字陰影。
- 可以用逗號設定多重陰影。
- 多重陰影可以做出發光、霓虹、描邊等效果。
- `text-shadow` 是可繼承屬性，父元素設定後，子元素文字可能會跟著有陰影。
- `text-shadow` 不會影響排版尺寸，只是視覺效果。
- 過度使用文字陰影會讓文字變糊、變髒，降低閱讀體驗。

## 正文

`text-shadow` 用於替文字添加陰影效果。

最基本的寫法如下：

```css
div {
  text-shadow: 5px 5px 6px rgba(0, 0, 0, .3);
}
```

```html
<div>你是陰影，我是火影</div>
```

這段程式碼會讓文字產生一個往右下方偏移、帶有模糊效果的黑色半透明陰影。

## 基本語法

`text-shadow` 的常見語法如下：

```css
text-shadow: offset-x offset-y blur-radius color;
```

也可以理解成：

```css
text-shadow: 水平位移 垂直位移 模糊半徑 顏色;
```

例如：

```css
.title {
  text-shadow: 4px 4px 8px rgba(0, 0, 0, .4);
}
```

這裡：

| 值 | 說明 |
| --- | --- |
| `4px` | 水平位移，陰影往右移動 `4px` |
| `4px` | 垂直位移，陰影往下移動 `4px` |
| `8px` | 模糊半徑，陰影邊緣變模糊 |
| `rgba(0, 0, 0, .4)` | 黑色、透明度 40% 的陰影 |

## 水平位移 offset-x

第一個長度值是水平位移。

```css
.text {
  text-shadow: 5px 0 2px rgba(0, 0, 0, .4);
}
```

表示陰影往右移動 `5px`。

如果是負值：

```css
.text {
  text-shadow: -5px 0 2px rgba(0, 0, 0, .4);
}
```

表示陰影往左移動 `5px`。

簡單記：

```text
正值：往右
負值：往左
```

## 垂直位移 offset-y

第二個長度值是垂直位移。

```css
.text {
  text-shadow: 0 5px 2px rgba(0, 0, 0, .4);
}
```

表示陰影往下移動 `5px`。

如果是負值：

```css
.text {
  text-shadow: 0 -5px 2px rgba(0, 0, 0, .4);
}
```

表示陰影往上移動 `5px`。

簡單記：

```text
正值：往下
負值：往上
```

## 模糊半徑 blur-radius

第三個長度值是模糊半徑。

```css
.text {
  text-shadow: 2px 2px 6px rgba(0, 0, 0, .4);
}
```

`6px` 表示陰影會有模糊效果。

模糊半徑越大，陰影越柔和、越擴散。

```css
.text-1 {
  text-shadow: 2px 2px 0 rgba(0, 0, 0, .5);
}

.text-2 {
  text-shadow: 2px 2px 8px rgba(0, 0, 0, .5);
}
```

差異：

| 寫法 | 效果 |
| --- | --- |
| `blur-radius: 0` | 邊緣清楚、硬陰影 |
| `blur-radius: 8px` | 邊緣模糊、柔和陰影 |

模糊半徑不能是負值。

不合法：

```css
.text {
  text-shadow: 2px 2px -4px rgba(0, 0, 0, .4);
}
```

## 陰影顏色 color

最後一個值是陰影顏色。

```css
.text {
  text-shadow: 2px 2px 4px black;
}
```

實務上更常使用 `rgba()`，因為可以控制透明度。

```css
.text {
  text-shadow: 2px 2px 4px rgba(0, 0, 0, .35);
}
```

這樣陰影會比較自然，不會太死黑。

常見陰影顏色：

```css
text-shadow: 0 1px 2px rgba(0, 0, 0, .3);
text-shadow: 0 2px 4px rgba(0, 0, 0, .25);
text-shadow: 0 0 8px rgba(255, 255, 255, .8);
```

雖然某些情況可以省略顏色，但實務上建議明確寫出顏色，避免效果不夠可預期。

## `none`

`none` 表示沒有文字陰影。

```css
.text {
  text-shadow: none;
}
```

常見於覆蓋父元素或舊樣式。

例如：

```css
.banner {
  text-shadow: 0 2px 4px rgba(0, 0, 0, .4);
}

.banner .plain-text {
  text-shadow: none;
}
```

這樣 `.plain-text` 就可以取消繼承或外層造成的陰影效果。

## `text-shadow` 會繼承

`text-shadow` 是可繼承屬性。

例如：

```css
.hero {
  text-shadow: 0 2px 4px rgba(0, 0, 0, .4);
}
```

```html
<section class="hero">
  <h1>主標題</h1>
  <p>副標題文字</p>
</section>
```

如果 `h1` 和 `p` 沒有另外設定 `text-shadow`，它們可能會繼承 `.hero` 的文字陰影。

這在 banner 或 hero 區塊中很方便。

但如果你只想讓標題有陰影，不想讓段落也有陰影，建議精準設定：

```css
.hero-title {
  text-shadow: 0 2px 4px rgba(0, 0, 0, .4);
}
```

不要直接寫在太外層。

## `text-shadow` 不影響排版大小

`text-shadow` 是視覺效果，不會改變元素本身的寬度或高度。

例如：

```css
.title {
  text-shadow: 10px 10px 20px rgba(0, 0, 0, .5);
}
```

即使陰影看起來超出了文字範圍，元素的盒模型大小也不會因此變大。

這代表：

- 陰影不會撐開父元素
- 陰影不會影響排版流
- 陰影可能和其他元素重疊
- 父層如果有 `overflow: hidden;`，陰影可能被裁切

例如：

```css
.card {
  overflow: hidden;
}

.card-title {
  text-shadow: 0 4px 12px rgba(0, 0, 0, .5);
}
```

如果陰影超出 `.card` 的範圍，可能會被裁掉。

## 多重文字陰影

`text-shadow` 可以設定多組陰影，用逗號分隔。

```css
.title {
  text-shadow:
    1px 1px 2px rgba(0, 0, 0, .4),
    0 0 8px rgba(255, 255, 255, .8);
}
```

這表示同一段文字同時有兩層陰影：

1. 第一層：黑色偏移陰影
2. 第二層：白色發光效果

多重陰影常用於：

- 霓虹字
- 發光文字
- 立體文字
- 文字描邊模擬
- 複雜背景上的文字強化

## 發光效果

如果水平位移和垂直位移都設成 `0`，只設定模糊半徑與顏色，就會產生發光感。

```css
.glow {
  color: #fff;
  text-shadow: 0 0 8px rgba(255, 255, 255, .9);
}
```

如果想讓發光更強，可以疊加多層陰影：

```css
.neon {
  color: #fff;
  text-shadow:
    0 0 4px rgba(255, 255, 255, .9),
    0 0 8px rgba(0, 180, 255, .8),
    0 0 16px rgba(0, 180, 255, .6);
}
```

這種效果常見於：

- 霓虹招牌
- 科技風標題
- 遊戲 UI
- 活動頁視覺標題

但不適合大量正文，因為會影響閱讀。

## 立體文字效果

可以用多重陰影做出簡單的立體文字。

```css
.text-3d {
  color: #fff;
  text-shadow:
    1px 1px 0 #999,
    2px 2px 0 #777,
    3px 3px 0 #555;
}
```

這種效果會讓文字看起來像往右下方堆疊。

適合：

- 活動標題
- 海報文字
- 復古風設計

但要注意，陰影層數太多會讓文字變得厚重，容易降低可讀性。

## 模擬文字描邊

CSS 有 `-webkit-text-stroke` 可以做文字描邊，但也可以用多重 `text-shadow` 簡單模擬。

```css
.outline-text {
  color: #fff;
  text-shadow:
    1px 0 0 #000,
    -1px 0 0 #000,
    0 1px 0 #000,
    0 -1px 0 #000;
}
```

這樣可以在文字四周形成類似黑色描邊的效果。

如果要更完整的描邊，通常會使用：

```css
-webkit-text-stroke: 1px #000;
```

但 `text-shadow` 的好處是可以做出比較柔和或特殊的外圈效果。

## 圖片背景上的文字陰影

`text-shadow` 很常用於圖片背景上的文字。

```css
.hero-title {
  color: #fff;
  text-shadow: 0 2px 4px rgba(0, 0, 0, .6);
}
```

```html
<section class="hero">
  <h1 class="hero-title">探索前端世界</h1>
</section>
```

當背景圖片顏色複雜時，白色文字可能不夠清楚。

加上黑色半透明陰影後，文字邊緣會更明顯。

不過要注意：

> 文字陰影只能輔助提升可讀性，不能完全取代背景遮罩。

更穩定的做法通常是搭配遮罩：

```css
.hero {
  position: relative;
  background-image: url("./images/banner.jpg");
  background-size: cover;
}

.hero::before {
  content: "";
  position: absolute;
  inset: 0;
  background: rgba(0, 0, 0, .35);
}

.hero-title {
  position: relative;
  color: #fff;
  text-shadow: 0 2px 4px rgba(0, 0, 0, .5);
}
```

這樣文字可讀性會比只靠 `text-shadow` 更穩定。

## 和 `box-shadow` 的差異

`text-shadow` 是文字陰影。

`box-shadow` 是盒子陰影。

```css
.title {
  text-shadow: 0 2px 4px rgba(0, 0, 0, .4);
}
```

會影響文字視覺。

```css
.card {
  box-shadow: 0 4px 12px rgba(0, 0, 0, .15);
}
```

會影響整個盒子的陰影。

| 屬性 | 作用對象 | 常見用途 |
| --- | --- | --- |
| `text-shadow` | 文字 | 標題陰影、發光文字、圖片文字可讀性 |
| `box-shadow` | 元素盒子 | 卡片陰影、按鈕陰影、彈窗陰影 |

不要把兩者混在一起。

## 和 `filter: drop-shadow()` 的差異

`filter: drop-shadow()` 是濾鏡效果，可以根據元素的透明輪廓產生陰影。

```css
.icon {
  filter: drop-shadow(0 2px 4px rgba(0, 0, 0, .3));
}
```

它常用於圖片、SVG、PNG 透明圖等。

`text-shadow` 則主要用於文字。

```css
.text {
  text-shadow: 0 2px 4px rgba(0, 0, 0, .3);
}
```

簡單判斷：

| 需求 | 建議 |
| --- | --- |
| 文字陰影 | `text-shadow` |
| 盒子陰影 | `box-shadow` |
| 圖片或 SVG 輪廓陰影 | `filter: drop-shadow()` |

## 可讀性注意事項

文字陰影不是越明顯越好。

過重的陰影會讓文字變髒、變糊。

不建議：

```css
.bad {
  color: #333;
  text-shadow: 5px 5px 10px rgba(0, 0, 0, .8);
}
```

這可能讓文字邊緣很混濁。

比較適合：

```css
.good {
  color: #fff;
  text-shadow: 0 1px 2px rgba(0, 0, 0, .45);
}
```

尤其是正文文字，不建議加太強的陰影。

常見建議：

- 正文少用或不用陰影
- 標題可以使用輕微陰影
- 圖片背景上的文字可以使用陰影輔助
- 陰影顏色盡量半透明
- 模糊半徑不要過大
- 不要讓陰影降低文字和背景的對比

## 效能注意事項

一般少量文字陰影沒有太大問題。

但如果大量元素使用複雜陰影，可能會增加渲染成本。

例如：

```css
* {
  text-shadow:
    0 0 4px red,
    0 0 8px blue,
    0 0 16px purple;
}
```

這種寫法非常不建議。

可能造成：

- 畫面渲染負擔增加
- 滾動時不順
- 文字看起來混濁
- 維護困難

實務上建議只在少量重點文字上使用，例如：

- hero title
- banner slogan
- 活動頁大標
- 特殊狀態文字

不要整站所有文字都加陰影。

## 常見實務場景

### 標題輕微陰影

```css
.page-title {
  font-size: 40px;
  font-weight: 700;
  color: #fff;
  text-shadow: 0 2px 4px rgba(0, 0, 0, .35);
}
```

```html
<h1 class="page-title">前端學習筆記</h1>
```

適合大標題或圖片背景上的標題。

## 圖片背景文字

```css
.hero-title {
  color: #fff;
  font-size: 48px;
  font-weight: 700;
  text-shadow: 0 2px 6px rgba(0, 0, 0, .55);
}
```

```html
<section class="hero">
  <h1 class="hero-title">探索 CSS 世界</h1>
</section>
```

這可以提升文字在圖片背景上的辨識度。

## 發光文字

```css
.glow-text {
  color: #fff;
  text-shadow:
    0 0 4px rgba(255, 255, 255, .9),
    0 0 12px rgba(0, 150, 255, .8);
}
```

```html
<h2 class="glow-text">NEON TEXT</h2>
```

適合特殊視覺設計，不適合長篇文字。

## 錯誤或警告提示

```css
.error-text {
  color: #f53f3f;
  text-shadow: 0 1px 2px rgba(245, 63, 63, .25);
}
```

```html
<p class="error-text">資料格式錯誤，請重新確認。</p>
```

錯誤文字通常不一定需要陰影。

如果要加，也應該非常輕微，避免影響可讀性。

## Hover 文字效果

```css
.link {
  color: #1677ff;
  text-decoration: none;
  transition: text-shadow 0.2s;
}

.link:hover {
  text-shadow: 0 0 6px rgba(22, 119, 255, .45);
}
```

```html
<a class="link" href="#">查看詳情</a>
```

這可以讓連結 hover 時有微弱發光感。

但要避免太誇張，否則會影響整體 UI 質感。

## 常見錯誤

### 錯誤一：只寫一個長度值

錯誤寫法：

```css
.text {
  text-shadow: 5px;
}
```

`text-shadow` 至少需要水平位移和垂直位移。

正確寫法：

```css
.text {
  text-shadow: 5px 5px;
}
```

更完整：

```css
.text {
  text-shadow: 5px 5px 6px rgba(0, 0, 0, .3);
}
```

## 錯誤二：把第三個值當成陰影大小

```css
.text {
  text-shadow: 2px 2px 10px rgba(0, 0, 0, .4);
}
```

第三個值不是陰影往外擴張的大小，而是模糊半徑。

它會讓陰影邊緣變模糊。

`text-shadow` 沒有像 `box-shadow` 那樣的 spread radius。

## 錯誤三：以為 `text-shadow` 會撐開元素

```css
.title {
  text-shadow: 20px 20px 20px rgba(0, 0, 0, .5);
}
```

陰影可能看起來很大，但元素本身的盒模型不會變大。

如果父層有：

```css
overflow: hidden;
```

陰影可能會被裁切。

## 錯誤四：正文大量使用陰影

不建議：

```css
p {
  text-shadow: 1px 1px 2px rgba(0, 0, 0, .5);
}
```

正文加陰影容易讓長時間閱讀變累。

文字陰影比較適合標題、banner 或特殊效果，不適合大量正文。

## 錯誤五：陰影太重導致文字變髒

不建議：

```css
.title {
  color: #333;
  text-shadow: 4px 4px 8px rgba(0, 0, 0, .8);
}
```

比較自然：

```css
.title {
  color: #333;
  text-shadow: 0 1px 2px rgba(0, 0, 0, .2);
}
```

輕微陰影通常比厚重陰影更耐看。

## 錯誤六：只靠陰影解決圖片文字可讀性

如果背景圖片太複雜，只加文字陰影可能不夠。

不穩定：

```css
.hero-title {
  color: #fff;
  text-shadow: 0 2px 4px rgba(0, 0, 0, .6);
}
```

更穩定：

```css
.hero::before {
  content: "";
  position: absolute;
  inset: 0;
  background: rgba(0, 0, 0, .35);
}

.hero-title {
  position: relative;
  color: #fff;
  text-shadow: 0 2px 4px rgba(0, 0, 0, .5);
}
```

使用遮罩加陰影，可讀性更穩。

## 錯誤七：把 `text-shadow` 當成 `box-shadow`

錯誤理解：

```css
.card {
  text-shadow: 0 4px 12px rgba(0, 0, 0, .2);
}
```

這只會影響 `.card` 裡面的文字，不會讓卡片盒子有陰影。

如果要卡片陰影，應該用：

```css
.card {
  box-shadow: 0 4px 12px rgba(0, 0, 0, .2);
}
```

## 錯誤八：忽略繼承造成整個區塊都有陰影

```css
.section {
  text-shadow: 0 2px 4px rgba(0, 0, 0, .4);
}
```

如果 `.section` 裡面有很多文字，全部都可能繼承陰影。

如果只想讓標題有陰影，應該寫：

```css
.section-title {
  text-shadow: 0 2px 4px rgba(0, 0, 0, .4);
}
```

## 面試可能問法

### 問題一：`text-shadow` 是做什麼的？

`text-shadow` 用來替文字添加陰影效果。

常見寫法：

```css
.text {
  text-shadow: 2px 2px 4px rgba(0, 0, 0, .3);
}
```

它可以讓文字有陰影、發光或立體效果。

## 問題二：`text-shadow` 的四個值分別代表什麼？

```css
text-shadow: 2px 3px 4px rgba(0, 0, 0, .3);
```

分別代表：

| 值 | 說明 |
| --- | --- |
| `2px` | 水平位移，正值往右 |
| `3px` | 垂直位移，正值往下 |
| `4px` | 模糊半徑 |
| `rgba(0, 0, 0, .3)` | 陰影顏色 |

## 問題三：`text-shadow` 可以設定多個陰影嗎？

可以。

多個陰影用逗號分隔。

```css
.text {
  text-shadow:
    1px 1px 2px rgba(0, 0, 0, .4),
    0 0 8px rgba(255, 255, 255, .8);
}
```

可以做出多層陰影、發光、霓虹或立體效果。

## 問題四：`text-shadow` 會影響元素大小嗎？

不會。

`text-shadow` 是視覺效果，不會改變元素盒模型大小。

所以陰影不會撐開父元素，也不會影響其他元素的排版位置。

如果父層有 `overflow: hidden;`，陰影可能會被裁切。

## 問題五：`text-shadow` 和 `box-shadow` 差在哪？

`text-shadow` 是文字陰影。

```css
.title {
  text-shadow: 0 2px 4px rgba(0, 0, 0, .3);
}
```

`box-shadow` 是盒子陰影。

```css
.card {
  box-shadow: 0 4px 12px rgba(0, 0, 0, .15);
}
```

簡單記：

- 文字用 `text-shadow`
- 盒子用 `box-shadow`

## 問題六：如何做文字發光效果？

讓水平位移和垂直位移為 `0`，搭配模糊半徑和亮色陰影。

```css
.glow {
  color: #fff;
  text-shadow: 0 0 8px rgba(255, 255, 255, .9);
}
```

如果要更強烈，可以疊加多層陰影。

## 問題七：`text-shadow` 會繼承嗎？

會。

`text-shadow` 是可繼承屬性。

父元素設定後，子元素文字可能也會套用陰影。

如果只想讓某個標題有陰影，建議精準設定在該標題上，不要設定在太外層。

## 問題八：圖片背景上的文字只加 `text-shadow` 夠嗎？

不一定。

`text-shadow` 可以提升文字邊緣辨識度，但如果背景圖片太複雜，只靠陰影可能不夠。

更穩定的做法是搭配背景遮罩：

```css
.hero::before {
  background: rgba(0, 0, 0, .35);
}
```

再加上文字陰影：

```css
.hero-title {
  text-shadow: 0 2px 4px rgba(0, 0, 0, .5);
}
```

## 一句話總結

`text-shadow` 用來替文字添加陰影、發光或立體效果；基本語法是水平位移、垂直位移、模糊半徑與顏色，實務上要注意可讀性、繼承、父層裁切與效能，不建議在大量正文中濫用。
