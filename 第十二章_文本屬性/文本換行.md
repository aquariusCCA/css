# 文本換行

> 返回章節首頁：[README.md](./README.md)
>
> `white-space` 用於設定文本中的空白、換行字元，以及文字是否允許自動換行。

## 導讀

- `white-space` 主要控制文本中的空白處理、換行保留，以及是否自動換行。
- 它會影響空格、Tab、原始碼中的換行符號，以及文字超出容器時是否換行。
- 常見值有 `normal`、`nowrap`、`pre`、`pre-wrap`、`pre-line`。
- 現代 CSS 也常見 `break-spaces`，用來更完整保留空白並允許換行。
- `white-space` 常用於文章內容、程式碼區塊、聊天訊息、表格欄位、單行省略號。
- 單行省略號常會搭配 `white-space: nowrap;`、`overflow: hidden;`、`text-overflow: ellipsis;`。
- 如果只是想控制長單字或網址如何斷行，通常應該看 `overflow-wrap` 或 `word-break`，不一定只靠 `white-space`。
- 學 `white-space` 的核心是分清楚三件事：空白是否合併、換行是否保留、超出容器是否自動換行。

## 30 秒複習入口

- `white-space` 控制空白、換行，以及是否自動換行。
- `normal`：連續空白會合併，原始換行會當成空白，文字超出容器會自動換行。
- `nowrap`：連續空白會合併，原始換行會當成空白，文字不會自動換行。
- `pre`：保留空白與原始換行，不會自動換行，效果接近 `<pre>`。
- `pre-wrap`：保留空白與原始換行，超出容器時會自動換行。
- `pre-line`：保留原始換行，但連續空白會合併，超出容器時會自動換行。
- `break-spaces`：保留空白，並允許在保留的空白處換行。
- `white-space: nowrap;` 常用於單行文字、表格欄位、按鈕文字。
- 單行省略號通常需要 `white-space: nowrap;`、`overflow: hidden;`、`text-overflow: ellipsis;`。
- `white-space` 是可繼承屬性，父元素設定後，子元素可能會受到影響。
- `white-space` 不是專門解決長英文單字、網址爆版的屬性，那類問題常看 `overflow-wrap` 或 `word-break`。

## 核心判斷表

學 `white-space` 時，可以用三個問題來判斷：

1. 連續空白會不會合併？
2. 原始碼中的換行會不會保留？
3. 文字超出容器時會不會自動換行？

| 值 | 連續空白 | 原始換行 | 自動換行 |
| --- | --- | --- | --- |
| `normal` | 合併 | 不保留，視為空白 | 會 |
| `nowrap` | 合併 | 不保留，視為空白 | 不會 |
| `pre` | 保留 | 保留 | 不會 |
| `pre-wrap` | 保留 | 保留 | 會 |
| `pre-line` | 合併 | 保留 | 會 |
| `break-spaces` | 保留 | 保留 | 會，且保留空白也可成為換行點 |

## 速查區

| 寫法 | 說明 | 常見場景 |
| --- | --- | --- |
| `white-space: normal;` | 預設行為，空白合併，文字會自動換行 | 一般段落 |
| `white-space: nowrap;` | 強制文字不自動換行 | 單行標題、表格欄位、省略號 |
| `white-space: pre;` | 保留空白與換行，不自動換行 | 程式碼、原始格式文本 |
| `white-space: pre-wrap;` | 保留空白與換行，並自動換行 | 聊天訊息、使用者輸入內容 |
| `white-space: pre-line;` | 保留換行，合併連續空白，並自動換行 | 多行說明文字、公告內容 |
| `white-space: break-spaces;` | 更完整保留空白，並允許在空白處換行 | 需要精準保留空白的文本 |

## 正文

在 CSS 中，可以使用 `white-space` 屬性設定文本的空白與換行方式。

```css
p {
  white-space: normal;
}
```

`white-space` 不只是控制「要不要換行」。

它同時會影響：

- 空格是否合併
- Tab 是否保留
- 原始碼中的換行是否保留
- 文字超出容器時是否自動換行

所以它是一個和文字排版非常相關的屬性。

## 為什麼需要 `white-space`

HTML 預設會合併多數空白。

例如：

```html
<p>
  這是      一段
  文字
</p>
```

在一般情況下，瀏覽器不會把多個空格和原始碼換行完整顯示出來。

畫面可能看起來像：

```text
這是 一段 文字
```

如果你想保留原始輸入格式，就需要使用不同的 `white-space` 設定。

例如：

```css
.message {
  white-space: pre-wrap;
}
```

這樣就能保留使用者輸入的換行，同時避免文字超出容器。

## `normal`

`normal` 是 `white-space` 的預設值。

```css
p {
  white-space: normal;
}
```

它的行為是：

- 連續空白會被合併成一個空白
- 原始碼中的換行通常會被視為一個空白
- 文字超出容器時會自動換行

例如：

```html
<p class="normal">
  這是      一段
  文字
</p>
```

```css
.normal {
  white-space: normal;
}
```

畫面上不會保留很多空格，也不會保留原始碼中的換行。

`normal` 適合一般文章、段落、卡片描述、表單提示等常見文本。

## `nowrap`

`nowrap` 表示不自動換行。

```css
.title {
  white-space: nowrap;
}
```

它的行為是：

- 連續空白會合併
- 原始碼中的換行通常會被視為空白
- 文字不會因為容器寬度不足而自動換行

例如：

```css
.title {
  width: 200px;
  white-space: nowrap;
}
```

如果文字很長，它會一直往右延伸，可能超出容器。

`nowrap` 常見於：

- 單行標題
- 表格欄位
- 按鈕文字
- 導覽列項目
- 單行省略號

## 單行省略號

`nowrap` 最常見的實務用途之一，就是單行省略號。

```css
.ellipsis {
  width: 200px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

```html
<div class="ellipsis">
  這是一段很長很長的文字，超出寬度後會顯示省略號
</div>
```

三個條件通常都需要：

```css
white-space: nowrap;
overflow: hidden;
text-overflow: ellipsis;
```

其中：

- `white-space: nowrap;` 讓文字維持單行
- `overflow: hidden;` 隱藏超出的部分
- `text-overflow: ellipsis;` 用省略號表示被截斷的文字

另外，元素通常還需要有明確寬度、`max-width`，或受到父層寬度限制。

```css
.ellipsis {
  max-width: 240px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

如果沒有寬度限制，文字可能不會產生可判斷的溢出範圍，省略號就不會出現。

## `pre`

`pre` 會保留原始空白與換行。

```css
.code {
  white-space: pre;
}
```

它的行為是：

- 連續空白會保留
- 原始碼中的換行會保留
- 文字不會自動換行

效果接近 HTML 的 `<pre>` 標籤。

```html
<div class="code">function test() {
  console.log("Hello");
}</div>
```

```css
.code {
  white-space: pre;
}
```

畫面會保留換行與縮排。

`pre` 適合：

- 程式碼
- ASCII 圖形
- 需要保留原始格式的文本
- 日誌內容

但要注意，`pre` 不會自動換行，所以長行內容可能會超出容器。

因此實務上程式碼區塊常會搭配：

```css
.code {
  white-space: pre;
  overflow: auto;
}
```

讓使用者可以橫向捲動。

## `pre-wrap`

`pre-wrap` 會保留空白與換行，同時允許自動換行。

```css
.message {
  white-space: pre-wrap;
}
```

它的行為是：

- 連續空白會保留
- 原始碼中的換行會保留
- 文字超出容器時會自動換行

這是實務上非常常用的值。

例如聊天訊息：

```html
<div class="message">你好

這是第二行
    這裡有縮排</div>
```

```css
.message {
  white-space: pre-wrap;
}
```

畫面會保留使用者輸入的換行，也會在容器太窄時自動換行。

`pre-wrap` 常見於：

- 聊天訊息
- 留言內容
- 使用者輸入的多行文字
- 後台備註欄位
- 需要保留換行又不能爆版的文本

如果你的內容是使用者輸入的多行文字，`pre-wrap` 通常比 `pre` 更適合，因為它不會因為長行文字直接撐爆版面。

## `pre-line`

`pre-line` 會保留換行，但會合併連續空白。

```css
.notice {
  white-space: pre-line;
}
```

它的行為是：

- 連續空白會合併
- 原始碼中的換行會保留
- 文字超出容器時會自動換行

例如：

```html
<div class="notice">第一行

第二行      中間很多空白
第三行</div>
```

```css
.notice {
  white-space: pre-line;
}
```

畫面會保留「第一行、第二行、第三行」這種換行結構，但 `第二行      中間很多空白` 裡面的多個空格會被合併。

所以 `pre-line` 不是「空格被忽略」，而是：

> 連續空白會被合併成一個空白。

這點要特別注意。

`pre-line` 適合：

- 公告內容
- 多行說明文字
- 後台輸入的備註
- 需要保留段落換行，但不需要保留多個空白的內容

如果你只想保留換行，不想保留縮排或多個空格，`pre-line` 很適合。

## `pre-wrap` 和 `pre-line` 的差異

這兩個值很容易混淆。

| 值 | 保留換行 | 保留連續空白 | 自動換行 |
| --- | --- | --- | --- |
| `pre-wrap` | 會 | 會 | 會 |
| `pre-line` | 會 | 不會，會合併 | 會 |

簡單判斷：

如果你要保留使用者輸入的縮排與多個空格：

```css
white-space: pre-wrap;
```

如果你只想保留換行，不想保留多個空格：

```css
white-space: pre-line;
```

例如聊天訊息通常會用：

```css
.message {
  white-space: pre-wrap;
}
```

公告文字可以用：

```css
.notice {
  white-space: pre-line;
}
```

## `break-spaces`

`break-spaces` 是比較現代的值。

```css
.text {
  white-space: break-spaces;
}
```

它的行為接近 `pre-wrap`，但對空白的保留更完整。

簡單理解：

- 連續空白會保留
- 原始換行會保留
- 文字可以自動換行
- 保留的空白也會佔據空間
- 空白處也可以成為換行點

例如：

```html
<div class="text">文字      文字      文字</div>
```

```css
.text {
  white-space: break-spaces;
}
```

這些連續空白會被保留，並且在需要時也可以參與換行。

`break-spaces` 適合非常需要保留空白視覺效果的場景。

不過一般實務中，最常用的還是：

```css
normal
nowrap
pre
pre-wrap
pre-line
```

`break-spaces` 可以先理解，但不一定需要一開始就大量使用。

## `white-space` 會繼承

`white-space` 是可繼承屬性。

例如：

```css
.article {
  white-space: pre-line;
}
```

```html
<div class="article">
  <p>第一行
第二行</p>
</div>
```

如果子元素沒有重新設定，可能會受到父元素的 `white-space` 影響。

這在實務中要小心。

例如你在外層寫：

```css
.container {
  white-space: nowrap;
}
```

可能會讓裡面的標題、段落、按鈕文字都不換行。

如果只想讓某一個欄位不換行，應該精準設定：

```css
.table-cell.name {
  white-space: nowrap;
}
```

不要把 `nowrap` 放在太外層。

## 與 `overflow-wrap` 的差異

`white-space` 控制的是空白與換行規則。

`overflow-wrap` 控制的是當一個很長的單字、網址或連續字串撐爆容器時，是否允許在必要時斷開。

例如：

```css
.text {
  overflow-wrap: break-word;
}
```

常見於處理長網址：

```html
<p class="text">
  https://example.com/this-is-a-very-very-very-long-url-without-normal-break-points
</p>
```

```css
.text {
  overflow-wrap: break-word;
}
```

如果只有：

```css
white-space: normal;
```

長網址可能仍然撐爆容器，因為它中間沒有自然斷點。

這時候要搭配：

```css
overflow-wrap: break-word;
```

或：

```css
overflow-wrap: anywhere;
```

簡單判斷：

| 問題 | 常用屬性 |
| --- | --- |
| 要不要保留空白與換行 | `white-space` |
| 長單字或網址爆版 | `overflow-wrap` |
| CJK 或英文單字斷行規則 | `word-break` |
| 超出後顯示省略號 | `text-overflow` |

## 與 `word-break` 的差異

`word-break` 用來控制單字內部是否可以斷行。

例如：

```css
.text {
  word-break: break-all;
}
```

這會讓文字在任意位置斷開，常用於避免長字串爆版。

但它可能會讓英文單字被切得很難讀。

```css
.text {
  word-break: break-all;
}
```

可能會把：

```text
frontenddevelopment
```

切成：

```text
front
endde
velop
ment
```

如果只是要避免長網址爆版，通常更推薦先考慮：

```css
overflow-wrap: break-word;
```

或：

```css
overflow-wrap: anywhere;
```

而不是一開始就用 `word-break: break-all;`。

## 與 `text-overflow` 的差異

`text-overflow` 是控制文字溢出時如何顯示。

最常見的是省略號：

```css
.ellipsis {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

這裡三個屬性分工不同：

| 屬性 | 作用 |
| --- | --- |
| `white-space: nowrap;` | 強制文字不換行 |
| `overflow: hidden;` | 隱藏超出的部分 |
| `text-overflow: ellipsis;` | 超出部分用省略號表示 |

所以 `text-overflow` 通常不是單獨使用，而是和 `white-space`、`overflow` 搭配。

## 常見實務場景

### 一般段落

```css
.article p {
  white-space: normal;
  line-height: 1.8;
}
```

```html
<p>
  這是一段一般段落文字，瀏覽器會自動處理換行。
</p>
```

一般文章、卡片描述、表單提示，通常使用預設的 `normal` 就可以。

## 單行標題不換行

```css
.card-title {
  white-space: nowrap;
}
```

```html
<h3 class="card-title">
  這是一個不希望換行的標題
</h3>
```

如果標題太長，可能會超出容器。

因此常搭配省略號：

```css
.card-title {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

## 表格欄位不換行

```css
.table .name {
  white-space: nowrap;
}
```

```html
<td class="name">系統管理員帳號</td>
```

表格中常見欄位：

- 姓名
- 狀態
- 操作按鈕
- 日期時間
- 金額

這些通常不希望被拆成多行。

但描述欄位通常可以換行：

```css
.table .description {
  white-space: normal;
  line-height: 1.5;
}
```

## 聊天訊息保留換行

```css
.chat-message {
  white-space: pre-wrap;
  line-height: 1.6;
}
```

```html
<div class="chat-message">你好
這是第二行

這是隔一行後的文字</div>
```

這樣可以保留使用者輸入的換行，同時讓長文字自動換行。

聊天訊息、留言、備註欄位很常用這種設定。

## 公告內容保留換行但合併空白

```css
.notice-content {
  white-space: pre-line;
  line-height: 1.8;
}
```

```html
<div class="notice-content">第一點：請依規定辦理。
第二點：請於期限內完成。
第三點：如有問題請聯絡管理員。</div>
```

這樣會保留每一行的換行，但不會保留多個連續空白。

很適合公告、規章、說明文字。

## 程式碼區塊

```css
.code-block {
  white-space: pre;
  overflow: auto;
  font-family: Consolas, Monaco, monospace;
}
```

```html
<pre class="code-block">function hello() {
  console.log("Hello");
}</pre>
```

程式碼通常需要保留縮排，所以可以使用 `pre`。

如果希望長行也能自動換行，可以改用：

```css
.code-block {
  white-space: pre-wrap;
}
```

但程式碼自動換行有時會影響閱讀，因此要看需求決定。

## 後台備註欄位

```css
.remark {
  white-space: pre-wrap;
  line-height: 1.6;
}
```

```html
<div class="remark">客戶備註：
1. 需要電話通知
2. 文件需補件</div>
```

使用者輸入的備註通常需要保留換行，因此 `pre-wrap` 很適合。

## 長網址防爆版

```css
.content {
  white-space: normal;
  overflow-wrap: break-word;
}
```

```html
<p class="content">
  https://example.com/this-is-a-very-very-very-long-url-without-normal-break-points
</p>
```

這裡主要靠 `overflow-wrap` 處理長網址，而不是只靠 `white-space`。

## 常見錯誤

### 錯誤一：以為 `white-space` 只控制要不要換行

`white-space` 不只控制換行，也控制空白是否合併、原始換行是否保留。

例如：

```css
.text {
  white-space: pre-wrap;
}
```

這不只是「可以換行」，還代表會保留原始空白與原始換行。

## 錯誤二：把 `pre-line` 說成空格會被忽略

不精準的說法：

```text
pre-line 會忽略空格
```

比較正確的說法：

```text
pre-line 會保留換行，但連續空白會被合併。
```

也就是多個空格會變成一個空格，而不是所有空格都消失。

## 錯誤三：用 `nowrap` 後忘記處理溢出

```css
.title {
  white-space: nowrap;
}
```

如果文字太長，可能會超出容器。

常見補法：

```css
.title {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

## 錯誤四：單行省略號少寫寬度限制

不完整：

```css
.title {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

如果元素沒有寬度限制，省略號可能不會出現。

比較完整：

```css
.title {
  max-width: 240px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

## 錯誤五：用 `white-space` 解決所有爆版問題

如果是長網址或連續英文爆版，問題不一定是 `white-space`。

例如：

```text
aaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaaa
```

這種連續字串沒有自然斷點。

應該搭配：

```css
.content {
  overflow-wrap: break-word;
}
```

或：

```css
.content {
  overflow-wrap: anywhere;
}
```

## 錯誤六：把 `white-space: nowrap;` 設在太外層

不建議：

```css
.container {
  white-space: nowrap;
}
```

這可能導致整個容器裡面的文字都不能換行。

比較好的做法是只針對需要不換行的元素設定：

```css
.table-cell.status {
  white-space: nowrap;
}
```

## 錯誤七：程式碼區塊用 `normal`

```css
.code {
  white-space: normal;
}
```

這會讓程式碼縮排和換行被破壞。

程式碼區塊通常應該使用：

```css
.code {
  white-space: pre;
}
```

或：

```css
.code {
  white-space: pre-wrap;
}
```

## 實務範例：單行省略號元件

```css
.text-ellipsis {
  max-width: 240px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

```html
<div class="text-ellipsis">
  這是一段很長的文字，超過寬度後會顯示省略號
</div>
```

這是後台系統、列表、表格、卡片中非常常見的寫法。

## 實務範例：聊天訊息

```css
.chat-message {
  max-width: 320px;
  white-space: pre-wrap;
  overflow-wrap: break-word;
  line-height: 1.6;
  padding: 8px 12px;
}
```

```html
<div class="chat-message">你好
這是第二行

這是一個很長很長的訊息內容</div>
```

這裡：

- `white-space: pre-wrap;` 保留使用者輸入的換行
- `overflow-wrap: break-word;` 避免長字串爆版
- `line-height: 1.6;` 提升閱讀性

## 實務範例：公告內容

```css
.notice-content {
  white-space: pre-line;
  line-height: 1.8;
  color: #333;
}
```

```html
<div class="notice-content">系統公告：
本系統將於週末進行維護。
維護期間部分功能可能暫時無法使用。</div>
```

這適合保留換行，但不需要保留多個空格的正式文字。

## 實務範例：程式碼區塊

```css
.code-block {
  white-space: pre;
  overflow: auto;
  padding: 16px;
  font-family: Consolas, Monaco, monospace;
}
```

```html
<pre class="code-block">function sum(a, b) {
  return a + b;
}</pre>
```

如果想讓程式碼長行自動換行：

```css
.code-block {
  white-space: pre-wrap;
}
```

但如果你希望程式碼排版完全保留原貌，通常使用 `pre` 搭配橫向捲動。

## 實務範例：表格欄位

```css
.table .date,
.table .status,
.table .action {
  white-space: nowrap;
}

.table .description {
  white-space: normal;
  overflow-wrap: break-word;
}
```

```html
<table>
  <tr>
    <td class="date">2026-05-03 13:30</td>
    <td class="status">處理中</td>
    <td class="description">這是一段可以換行的描述文字。</td>
    <td class="action">查看詳情</td>
  </tr>
</table>
```

表格中不是所有欄位都應該不換行。

通常：

- 日期、狀態、操作欄：不換行
- 描述、備註欄：允許換行

## 面試可能問法

### 問題一：`white-space` 是做什麼的？

`white-space` 用來控制文本中的空白、換行字元，以及文字是否允許自動換行。

它不是只控制「換不換行」，也會影響空白是否合併、原始換行是否保留。

```css
p {
  white-space: normal;
}
```

## 問題二：`normal`、`nowrap`、`pre` 有什麼差異？

| 值 | 空白 | 原始換行 | 自動換行 |
| --- | --- | --- | --- |
| `normal` | 合併 | 不保留 | 會 |
| `nowrap` | 合併 | 不保留 | 不會 |
| `pre` | 保留 | 保留 | 不會 |

簡單記：

- `normal`：正常排版
- `nowrap`：強制單行
- `pre`：保留原格式

## 問題三：`pre-wrap` 和 `pre-line` 差在哪？

`pre-wrap` 會保留換行，也會保留連續空白。

`pre-line` 會保留換行，但會合併連續空白。

```css
.message {
  white-space: pre-wrap;
}
```

適合聊天訊息、留言、備註。

```css
.notice {
  white-space: pre-line;
}
```

適合公告、多行說明文字。

## 問題四：如何做單行省略號？

通常需要三個屬性：

```css
.ellipsis {
  width: 200px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

其中 `white-space: nowrap;` 用來讓文字保持單行。

## 問題五：`white-space: nowrap;` 會造成什麼問題？

如果文字太長，會超出容器，造成橫向溢出。

所以常常要搭配：

```css
overflow: hidden;
text-overflow: ellipsis;
```

或避免在太外層使用 `nowrap`。

## 問題六：長網址爆版要用 `white-space` 解決嗎？

不一定。

長網址或長英文單字爆版，通常要看 `overflow-wrap` 或 `word-break`。

例如：

```css
.content {
  overflow-wrap: break-word;
}
```

或：

```css
.content {
  overflow-wrap: anywhere;
}
```

`white-space` 主要處理空白、換行與自動換行規則，不是專門處理長單字斷行。

## 問題七：`white-space` 會繼承嗎？

會。

`white-space` 是可繼承屬性。父元素設定後，子元素可能會受到影響。

所以像 `white-space: nowrap;` 這種強限制不建議隨便設在很外層，否則可能造成整個區塊都不能換行。

## 一句話總結

`white-space` 用來控制文本中的空白、原始換行與自動換行行為；學習時要用「空白是否合併、換行是否保留、是否自動換行」三個角度判斷，實務上常用於單行省略號、聊天訊息、公告內容、程式碼區塊與表格欄位。
