## 導讀

- `text-align` 主要控制文字的水平對齊。
- 它不是讓元素本身水平移動，而是控制元素內部的文字或行內內容如何排列。
- 常見值有 `left`、`center`、`right`、`justify`。
- 現代 CSS 中也常見 `start`、`end`，它們會依照文字書寫方向自動決定靠左或靠右。
- `text-align` 是可繼承屬性，父元素設定後，子元素通常會繼承。
- 實務上常用於標題置中、表格內容對齊、按鈕文字置中、卡片內容排列等場景。
- 要理解 `text-align`，關鍵是分清楚「對齊元素本身」和「對齊元素裡面的內容」。

## 正文

`text-align` 屬性用於設定元素內部文字或行內內容的水平對齊方式。

最基本的寫法如下：

```css
h1 {
  text-align: center;
}
```

```html
<h1>居中對齊的標題</h1>
```

這段程式碼的意思是：讓 `h1` 盒子裡面的文字在水平方向置中。

注意，這裡不是讓 `h1` 這個元素本身置中，而是讓 `h1` 裡面的文字置中。

## `text-align` 到底控制誰？

`text-align` 控制的是元素內部的 inline-level content。

簡單來說，它會影響：

* 純文字
* `span`
* `a`
* `strong`
* `em`
* `img`
* `input`
* `button`
* `inline-block` 元素

例如：

```html
<div class="box">
  <span>我是 span</span>
</div>
```

```css
.box {
  text-align: center;
}
```

這時候會被置中的是 `.box` 裡面的 `span`，不是 `.box` 自己。

也就是說：

```css
.box {
  text-align: center;
}
```

意思是：

> 讓 `.box` 裡面的文字或行內內容置中。

不是：

> 讓 `.box` 這個盒子自己置中。

這是學 `text-align` 最重要的觀念。

## 常見值

### `left`

`left` 表示內容靠左對齊。

```css
p {
  text-align: left;
}
```

```html
<p>這是一段靠左對齊的文字。</p>
```

在中文和英文的一般排版中，靠左對齊很常見。

### `center`

`center` 表示內容置中對齊。

```css
.title {
  text-align: center;
}
```

```html
<h1 class="title">頁面標題</h1>
```

常見於：

* 標題
* 彈窗內容
* 空狀態提示
* 卡片標題
* 登入頁文字
* 按鈕文字

例如：

```css
.empty {
  text-align: center;
  color: #999;
}
```

```html
<div class="empty">
  目前沒有資料
</div>
```

### `right`

`right` 表示內容靠右對齊。

```css
.price {
  text-align: right;
}
```

```html
<div class="price">$ 1,280</div>
```

常見於：

* 金額
* 數字
* 統計欄位
* 表格中的數值欄位
* 操作按鈕區

例如後台表格常會這樣設計：

```css
.table-cell.amount {
  text-align: right;
}
```

金額靠右後，數字位數不同時會比較容易比較大小。

## `start` 與 `end`

`start` 和 `end` 是比較現代、也比較適合多語系的寫法。

```css
.article {
  text-align: start;
}
```

`start` 表示靠文字書寫方向的起點。

如果目前文字方向是由左到右，例如中文、英文：

```css
text-align: start;
```

效果接近：

```css
text-align: left;
```

如果文字方向是由右到左，例如阿拉伯文、希伯來文：

```css
text-align: start;
```

效果會接近：

```css
text-align: right;
```

### `end`

`end` 表示靠文字書寫方向的終點。

```css
.article {
  text-align: end;
}
```

在左到右語言中，`end` 接近靠右。

在右到左語言中，`end` 接近靠左。

### 為什麼實務上會用 `start` / `end`？

如果專案未來可能支援多語系，`start` 和 `end` 會比 `left` 和 `right` 更有彈性。

例如：

```css
.card-title {
  text-align: start;
}

.card-action {
  text-align: end;
}
```

這樣排版會根據語言方向自動調整。

## `justify`

`justify` 表示左右兩端對齊。

```css
.article {
  text-align: justify;
}
```

常見於文章、報告、正式文件排版。

```html
<p class="article">
  CSS 是用來描述 HTML 文件呈現方式的語言，它可以控制文字、顏色、間距、排版與響應式效果。
</p>
```

```css
.article {
  width: 400px;
  text-align: justify;
}
```

`justify` 會讓多行文字的左右邊界盡量對齊。

不過要注意：

* 通常只對多行文字有明顯效果。
* 最後一行通常不會被強制撐滿。
* 中文排版使用 `justify` 時，有些情況下字距可能看起來不自然。
* 如果容器太窄，文字間距可能被拉得太開。

所以 `justify` 適合文章排版，但不一定適合所有 UI 介面。

## `text-align` 與父子元素的關係

很多初學者會說：

> 要讓文字置中，要把 `text-align: center;` 設在父元素。

這句話大方向是對的，但要更精準一點：

> `text-align` 要設定在「包含文字或行內內容的那個元素」上。

例如：

```html
<div class="box">
  <span>文字</span>
</div>
```

```css
.box {
  text-align: center;
}
```

這裡 `.box` 是 `span` 的父元素，所以設定在 `.box` 上可以讓 `span` 置中。

但是下面這種情況：

```html
<h1>標題</h1>
```

```css
h1 {
  text-align: center;
}
```

也是正確的。

因為文字本身就在 `h1` 裡面，所以直接設定在 `h1` 上即可。

因此，不要死背「一定要設在父元素」，更精準的理解是：

> 設在包住文字或 inline-level content 的容器上。

## `text-align` 會繼承

`text-align` 是可繼承屬性。

也就是說，父元素設定後，子元素通常會繼承這個對齊方式。

```css
.article {
  text-align: center;
}
```

```html
<div class="article">
  <h2>標題</h2>
  <p>這是一段文字。</p>
</div>
```

在這個例子中，如果 `h2` 和 `p` 沒有另外設定 `text-align`，它們通常會繼承 `.article` 的 `center`。

如果你不希望子元素繼承，可以在子元素重新設定：

```css
.article {
  text-align: center;
}

.article p {
  text-align: left;
}
```

這樣標題可以置中，但段落維持靠左。

```html
<div class="article">
  <h2>置中標題</h2>
  <p>這段文字靠左，閱讀起來比較自然。</p>
</div>
```

這是文章頁面很常見的做法。

## `text-align` 不能讓 block 元素本身置中

這是最常見的錯誤。

例如：

```css
.box {
  width: 300px;
  text-align: center;
}
```

很多人會以為這樣 `.box` 自己會在頁面中置中。

但實際上不是。

這段 CSS 只會讓 `.box` 裡面的文字置中，不會讓 `.box` 這個元素本身置中。

如果要讓 block 元素本身水平置中，通常要使用：

```css
.box {
  width: 300px;
  margin: 0 auto;
}
```

```html
<div class="box">
  我是盒子裡的文字
</div>
```

如果想同時讓盒子本身置中，並讓裡面的文字也置中，可以這樣寫：

```css
.box {
  width: 300px;
  margin: 0 auto;
  text-align: center;
}
```

這裡：

```css
margin: 0 auto;
```

負責讓 `.box` 自己水平置中。

```css
text-align: center;
```

負責讓 `.box` 裡面的文字水平置中。

兩者控制的對象不同。

## 與 Flex 的差異

`text-align` 控制的是文字或行內內容的水平對齊。

Flex 控制的是 flex items 的排列。

例如：

```html
<div class="toolbar">
  <button>新增</button>
  <button>刪除</button>
</div>
```

如果你想讓兩個按鈕整組靠右，通常使用 Flex：

```css
.toolbar {
  display: flex;
  justify-content: flex-end;
}
```

而不是只用：

```css
.toolbar {
  text-align: right;
}
```

如果按鈕是 inline-block，`text-align: right;` 也可能有效，但在現代版面中，Flex 通常更適合控制一組元件的排列。

### 對比

```css
.text-box {
  text-align: center;
}
```

適合處理：

* 文字
* 連結
* 圖片
* inline-block 小元素

```css
.flex-box {
  display: flex;
  justify-content: center;
  align-items: center;
}
```

適合處理：

* 多個區塊
* 卡片
* 按鈕組
* 導覽列
* 複雜排版

## 與 `vertical-align` 的差異

`text-align` 控制水平對齊。

`vertical-align` 控制行內元素、行內塊元素或表格單元格內容的垂直對齊。

```css
.box {
  text-align: center;
}
```

表示水平置中。

```css
img {
  vertical-align: middle;
}
```

表示圖片與文字在垂直方向上對齊。

兩者不是同一件事。

## 常見錯誤

### 錯誤一：以為 `text-align` 可以讓盒子本身置中

錯誤理解：

```css
.box {
  width: 300px;
  text-align: center;
}
```

這不會讓 `.box` 自己置中，只會讓 `.box` 裡面的文字置中。

正確做法：

```css
.box {
  width: 300px;
  margin: 0 auto;
}
```

如果要文字也置中：

```css
.box {
  width: 300px;
  margin: 0 auto;
  text-align: center;
}
```

### 錯誤二：所有內容都用 `text-align: center`

置中雖然看起來整齊，但長段文字如果全部置中，閱讀體驗通常會變差。

不建議：

```css
.article {
  text-align: center;
}
```

如果是長文章，通常更適合：

```css
.article {
  text-align: left;
}
```

可以只讓標題置中：

```css
.article h1 {
  text-align: center;
}

.article p {
  text-align: left;
}
```

### 錯誤三：用 `text-align` 處理複雜版面

例如導覽列、工具列、卡片列表，通常不建議只靠 `text-align`。

比較推薦使用 Flex 或 Grid。

```css
.nav {
  display: flex;
  justify-content: space-between;
  align-items: center;
}
```

`text-align` 適合文字和行內內容；Flex / Grid 適合整體版面。

### 錯誤四：忽略繼承造成的影響

```css
.container {
  text-align: center;
}
```

這可能會讓裡面的標題、段落、按鈕全部置中。

如果只想讓標題置中，不應該設在太外層：

```css
.container h2 {
  text-align: center;
}
```

或者針對段落覆蓋：

```css
.container {
  text-align: center;
}

.container p {
  text-align: left;
}
```

### 錯誤五：混淆 `text-align` 和 `align-items`

```css
.box {
  text-align: center;
}
```

這是控制文字或 inline-level content 的水平對齊。

```css
.box {
  display: flex;
  align-items: center;
}
```

這是控制 flex items 在交叉軸上的對齊。

兩者不是同一種排版系統。

## 面試可能問法

### 問題一：`text-align` 是控制元素本身置中嗎？

不是。

`text-align` 控制的是元素內部文字或行內內容的水平對齊，不是控制元素本身的位置。

```css
.box {
  text-align: center;
}
```

這只會讓 `.box` 裡面的文字置中，不會讓 `.box` 自己置中。

如果要讓 `.box` 自己置中，通常使用：

```css
.box {
  width: 300px;
  margin: 0 auto;
}
```

### 問題二：`text-align` 要設定在父元素還是子元素？

要設定在「包含文字或 inline-level content 的容器」上。

例如：

```html
<div class="parent">
  <span>文字</span>
</div>
```

```css
.parent {
  text-align: center;
}
```

這裡設定在 `.parent` 上，會讓裡面的 `span` 置中。

但如果文字直接在 `h1` 裡：

```html
<h1>標題</h1>
```

那就可以直接寫：

```css
h1 {
  text-align: center;
}
```

### 問題三：`text-align` 會繼承嗎？

會。

`text-align` 是可繼承屬性。父元素設定後，子元素通常會繼承。

```css
.parent {
  text-align: center;
}
```

```html
<div class="parent">
  <p>這段文字通常會繼承置中對齊。</p>
</div>
```

如果不想繼承，可以在子元素重新設定：

```css
.parent p {
  text-align: left;
}
```

### 問題四：`left` 和 `start` 有什麼差別？

`left` 是固定靠左。

`start` 是依照文字方向決定靠起點。

在左到右語言中，`start` 通常等於 `left`。

在右到左語言中，`start` 通常等於 `right`。

所以多語系專案中，`start` 和 `end` 通常比 `left` 和 `right` 更有彈性。

### 問題五：什麼時候用 `text-align`，什麼時候用 Flex？

如果只是文字、連結、圖片、inline-block 內容的水平對齊，可以用 `text-align`。

如果是多個區塊、按鈕組、卡片、導覽列這類版面排列，通常用 Flex 或 Grid 更適合。

```css
.text {
  text-align: center;
}
```

```css
.layout {
  display: flex;
  justify-content: center;
  align-items: center;
}
```

## 一句話總結

`text-align` 用來控制元素內部文字或行內內容的水平對齊；它會繼承，但不會讓 block 元素本身置中，實務上要分清楚文字對齊、盒子置中與 Flex/Grid 排版的差異。
