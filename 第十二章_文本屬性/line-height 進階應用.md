## 30 秒複習入口
- `line-height: normal` 由瀏覽器決定，通常約略接近字級的 `1.2`
- 不帶單位的數字是倍數寫法，會乘上元素自己的 `font-size`

## 正文
`line-height` 決定一行文字在版面中佔據的垂直空間，也就是行盒高度。

它不只是「文字下面多一塊空白」，而是會影響文字上下方的留白分配。

## `line-height` 的幾種常見寫法

### `normal`
`normal` 是預設值，由瀏覽器和字型決定。

在很多瀏覽器中，它大約接近字級的 `1.2` 倍，但這不是固定保證。

### 不帶單位的數字
例如 `line-height: 1.8;`。

這種寫法會以元素自己的 `font-size` 為基準，乘上這個倍數來計算。

這也是 CSS 裡最推薦的寫法之一，因為它比較適合繼承。

### 帶單位的長度
例如 `line-height: 18px;`。

這會直接變成固定高度，不會跟著字級自動放大或縮小。

### 百分比
例如 `line-height: 180%;`。

它會先依元素自己的 `font-size` 算出對應高度，再轉成具體數值。

這種寫法可以用，但在繼承時比較容易出現預期外結果。

### `inherit`
`inherit` 代表直接繼承父元素的已計算值。

如果父元素使用的是單位數字，子元素通常仍能跟著自己的字級放大縮小。

如果父元素使用的是長度或百分比，子元素更容易拿到固定值，這是常見的排版陷阱。

## 為什麼通常推薦單位數字
假設：

```css
body {
  font-size: 35px;
  line-height: 1.8;
}

.main {
  font-size: 25px;
}
```

這時 `.main` 繼承到的會是 `1.8` 這個倍數，而不是某個固定像素值。

所以 `.main` 的實際行高會是：

```css
25px * 1.8 = 45px
```

這種行為通常比固定長度更適合做整站排版。

反過來，如果你用的是長度或百分比，子元素可能繼承到的是已經算好的固定值，字級變了，行高卻不一定跟著變，版面就容易失衡。

## 行盒與文字的關係
`line-height` 會影響 line box 的高度。

當行高大於字級時，多出來的空間會分配在文字上下，看起來就像文字被垂直安放在行中間。

當行高過小時，版面會變得很緊，文字看起來會擁擠，甚至在搭配固定高度或 `overflow` 時出現被裁切的視覺問題。

原文用 `inline box`、`content area` 來解釋這件事，可以幫助理解，但真正要記的是：`line-height` 會直接影響行盒高度，進而影響文字的上下留白。

## 常見混淆點
- `line-height: 1` 不是預設值，`normal` 才是預設值。
- `180%` 不等於「永遠是父元素字級的 180%」；它會先依元素自己的 `font-size` 算出值，再進入繼承流程。
- `inherit` 不是重新計算一次，而是直接沿用父元素的計算結果。
- `line-height` 不是專門用來做垂直置中，但在單行文字場景裡可以達到類似效果。

## 一句話總結
`line-height` 最穩定的用法通常是單位數字；`normal`、長度、百分比與 `inherit` 的差異，關鍵在於它們怎麼計算、怎麼繼承。
