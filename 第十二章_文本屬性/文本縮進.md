# 文本縮進

> 返回章節首頁：[README.md](./README.md)
>
> `text-indent` 屬性用於設定文本第一行的縮進距離，最常見於段落首行縮排。

## 導讀

- `text-indent` 主要控制「第一行文字」的縮進。
- 它常用於文章段落、新聞內容、正式文本等排版場景。
- 常見單位有 `px`、`em`、`rem`、`%`。
- 中文段落首行縮排常見寫法是 `text-indent: 2em;`，大約代表縮進兩個字的寬度。
- `text-indent` 是可繼承屬性，父元素設定後，子元素可能會繼承。
- `text-indent` 可以使用負值，但負縮進容易造成文字超出容器，實務上要小心。
- 它不是用來控制整段文字左右距離的屬性；如果要控制整個區塊的位置，應該使用 `margin` 或 `padding`。

- `text-indent: 2em;` 是中文段落首行縮排最常見寫法。

- `text-indent: 24px;` 表示第一行縮進固定 `24px`。
- `text-indent: 10%;` 的百分比通常會參考包含區塊的寬度。

## 正文

`text-indent` 用於設定文字第一行的縮進距離。

最基本的寫法如下：

```css
p {
  text-indent: 2em;
}
```

```html
<p>
  這是一段文章內容。設定 text-indent 之後，只有第一行會產生縮排。
</p>
```

這段程式碼會讓段落第一行往右縮進大約兩個字的寬度。

## `text-indent` 控制的是第一行

`text-indent` 最重要的觀念是：

> 它只控制文字排版後的第一行，不是控制整段文字。

例如：

```css
p {
  width: 300px;
  text-indent: 2em;
}
```

```html
<p>
  CSS 是用來描述 HTML 文件呈現方式的語言，它可以控制文字、顏色、間距、排版與響應式效果。
</p>
```

如果這段文字因為寬度限制而換成多行，只有第一行會縮進，第二行、第三行不會一起縮進。

如果你想讓整段文字都往右移，應該使用：

```css
p {
  padding-left: 2em;
}
```

或：

```css
p {
  margin-left: 2em;
}
```

不要用 `text-indent` 來做整段位移。

## 常見單位

### `px`

`px` 是固定像素值。

```css
p {
  text-indent: 24px;
}
```

這表示段落第一行固定縮進 `24px`。

優點是距離明確，缺點是不會跟著字體大小變化。

如果文字大小變大，`24px` 可能看起來太小；如果文字變小，`24px` 又可能看起來太大。

## `em`

`em` 是最常見的段落首行縮排單位。

```css
p {
  text-indent: 2em;
}
```

`em` 會根據目前元素的 `font-size` 計算。

例如：

```css
p {
  font-size: 16px;
  text-indent: 2em;
}
```

這時：

```text
2em = 16px * 2 = 32px
```

如果字體改成 `20px`：

```css
p {
  font-size: 20px;
  text-indent: 2em;
}
```

這時：

```text
2em = 20px * 2 = 40px
```

所以 `2em` 的好處是會跟著文字大小調整，適合文章段落排版。

中文文章中，首行縮排兩個字，通常就會寫成：

```css
.article p {
  text-indent: 2em;
}
```

## `rem`

`rem` 是根元素字體大小的倍數。

```css
p {
  text-indent: 2rem;
}
```

如果根元素 `html` 的字體大小是 `16px`：

```css
html {
  font-size: 16px;
}
```

那麼：

```text
2rem = 32px
```

`rem` 的特點是比較穩定，因為它不看當前元素的字體大小，而是看根元素字體大小。

如果你希望整個專案的縮排距離統一，可以使用 `rem`。

如果你希望縮排跟著段落自己的文字大小變化，則 `em` 更適合。

## `%`

`text-indent` 也可以使用百分比。

```css
p {
  text-indent: 10%;
}
```

百分比縮排通常會根據包含區塊的寬度來計算。

例如段落所在容器寬度是 `500px`：

```text
10% = 50px
```

所以第一行會縮進 `50px`。

百分比寫法在一般文章段落中較少見，因為容器變寬時，縮排也會變大，可能導致排版不穩定。

比較常見的首行縮排仍然是：

```css
text-indent: 2em;
```

## 負值縮進

`text-indent` 可以使用負值。

```css
p {
  text-indent: -2em;
}
```

負值會讓第一行往左移。

這種效果可以用於特殊排版，例如懸掛縮排。

```css
.note {
  padding-left: 2em;
  text-indent: -2em;
}
```

```html
<p class="note">
  注意：這是一段提示文字，第一行向左凸出，其餘行保持縮進。
</p>
```

這種排版的概念是：

```css
padding-left: 2em;
```

先讓整段文字往右移。

```css
text-indent: -2em;
```

再讓第一行往左拉回來。

這樣就能形成「第一行凸出，其餘行縮進」的效果。

常見於：

- 註解說明
- 條列文字
- 參考文獻
- 法規條文
- 特殊格式排版

不過負值縮排容易讓文字超出容器，所以實務上要小心使用。

## `text-indent` 會繼承

`text-indent` 是可繼承屬性。

例如：

```css
.article {
  text-indent: 2em;
}
```

```html
<div class="article">
  <p>第一段文字。</p>
  <p>第二段文字。</p>
</div>
```

如果子元素沒有另外設定 `text-indent`，可能會繼承 `.article` 的縮排設定。

但實務上更建議明確設定在段落元素上：

```css
.article p {
  text-indent: 2em;
}
```

這樣語意更清楚，也比較不容易影響到不該縮排的元素。

例如：

```html
<div class="article">
  <h2>標題不需要縮排</h2>
  <p>段落才需要首行縮排。</p>
</div>
```

推薦寫法：

```css
.article h2 {
  text-indent: 0;
}

.article p {
  text-indent: 2em;
}
```

## `text-indent` 與 `padding-left` 的差異

`text-indent` 和 `padding-left` 都會讓文字看起來往右，但控制對象不同。

| 屬性 | 影響範圍 | 常見用途 |
| --- | --- | --- |
| `text-indent` | 只影響第一行 | 段落首行縮排 |
| `padding-left` | 影響整個內容區域 | 整段文字往右移 |
| `margin-left` | 影響整個元素的位置 | 元素本身往右移 |

例如：

```css
.text-indent {
  text-indent: 2em;
}

.padding-left {
  padding-left: 2em;
}

.margin-left {
  margin-left: 2em;
}
```

差異如下：

- `text-indent`：只有第一行往右。
- `padding-left`：每一行內容都往右。
- `margin-left`：整個元素盒子往右。

所以如果你只是想做中文段落首行縮排，使用 `text-indent`。

如果你想讓整段內容都縮進，使用 `padding-left`。

如果你想讓整個區塊離左邊遠一點，使用 `margin-left`。

## 文章段落首行縮排

中文文章排版中，首行縮排兩個字很常見。

```css
.article {
  line-height: 1.8;
  color: #333;
}

.article p {
  text-indent: 2em;
  margin-bottom: 1em;
}
```

```html
<article class="article">
  <p>
    CSS 是用來描述 HTML 文件呈現方式的語言，它可以控制文字、顏色、間距、排版與響應式效果。
  </p>
  <p>
    在文章排版中，text-indent 常用於段落首行縮排，讓閱讀節奏更接近傳統中文排版。
  </p>
</article>
```

這種寫法適合：

- 文章
- 部落格
- 新聞內容
- 書籍排版
- 長篇說明文字

但在後台管理系統、表單、卡片、列表等 UI 介面中，不一定需要首行縮排。

## UI 介面中不一定需要首行縮排

雖然 `text-indent: 2em;` 很適合文章，但不代表所有文字都要縮排。

例如下面這些場景通常不需要首行縮排：

- 表單 label
- 按鈕文字
- 表格內容
- 卡片標題
- 導覽列文字
- 錯誤提示
- 空狀態提示

不建議：

```css
body {
  text-indent: 2em;
}
```

這樣可能會讓整個頁面的文字都受到影響，包含標題、按鈕、表格、表單提示等。

比較好的做法是限制範圍：

```css
.article p {
  text-indent: 2em;
}
```

只讓文章段落縮排。


## 第一行不縮排，第二行之後縮排

嚴格來說，`text-indent` 本身控制的是第一行。

如果你想做「第一行不縮排，第二行之後縮排」，通常會使用懸掛縮排技巧：

```css
.hanging {
  padding-left: 2em;
  text-indent: -2em;
}
```

```html
<p class="hanging">
  注意事項：這是一段很長的說明文字，當文字換行後，第二行之後會維持縮進。
</p>
```

這種寫法在表單說明、錯誤提示、條列描述中很有用。

## 常見錯誤

### 錯誤一：以為 `text-indent` 會讓整段文字縮進

錯誤理解：

```css
p {
  text-indent: 2em;
}
```

以為整段文字每一行都會往右移。

實際上，只有第一行會縮進。

如果要整段文字都往右移，應該使用：

```css
p {
  padding-left: 2em;
}
```

或：

```css
p {
  margin-left: 2em;
}
```

## 錯誤二：把 `text-indent` 設在太外層

不建議：

```css
body {
  text-indent: 2em;
}
```

這樣可能會影響整個頁面，造成標題、表格、按鈕、表單文字都出現不必要的縮排。

比較好的做法：

```css
.article p {
  text-indent: 2em;
}
```

只針對文章段落設定。

## 錯誤三：用 `px` 寫中文首行縮排

可以寫：

```css
p {
  text-indent: 32px;
}
```

但如果字體大小改變，`32px` 不一定剛好等於兩個字。

更彈性的寫法是：

```css
p {
  text-indent: 2em;
}
```

因為 `em` 會跟著目前元素的字體大小變動。

## 錯誤四：負值縮排造成文字被切掉

```css
p {
  text-indent: -9999px;
}
```

早期有人會用這種方式把文字移出畫面，達到隱藏文字的效果。

但這種技巧現在不建議濫用，因為可能造成可維護性、可訪問性與版面問題。

如果要隱藏文字，應根據情境選擇更合適的方法，例如：

```css
.visually-hidden {
  position: absolute;
  width: 1px;
  height: 1px;
  overflow: hidden;
  white-space: nowrap;
  clip-path: inset(50%);
}
```

## 錯誤五：在 UI 元件中濫用首行縮排

例如：

```css
.card p {
  text-indent: 2em;
}
```

如果卡片文字很短，首行縮排可能反而讓版面不整齊。

UI 元件通常更重視對齊、間距和資訊層級，不一定需要傳統段落縮排。

文章內容才比較適合使用 `text-indent`。

```css
.article {
  max-width: 720px;
  margin: 0 auto;
  color: #333;

}

.article h1 {
```

## 面試可能問法

### 問題一：`text-indent` 是做什麼的？

`text-indent` 用來設定文本第一行的縮進距離，最常見用途是段落首行縮排。

```css
p {
  text-indent: 2em;
}
```

這會讓段落第一行縮進大約兩個字的寬度。

## 問題二：`text-indent: 2em;` 是什麼意思？

`2em` 表示目前元素字體大小的 2 倍。

如果段落字體大小是 `16px`：

```text
2em = 32px
```

如果段落字體大小變成 `20px`：

```text
2em = 40px
```

所以 `2em` 很適合中文段落首行縮排。

## 問題三：`text-indent` 會影響整段文字嗎？

不會。

`text-indent` 只影響第一行。

如果文字換行，第二行之後不會跟著縮排。

如果要讓整段文字都往右移，應該使用 `padding-left` 或 `margin-left`。

## 問題四：`text-indent` 可以是負值嗎？

可以。

```css
p {
  text-indent: -2em;
}
```

負值會讓第一行往左移，常見於懸掛縮排等特殊排版。

通常會搭配 `padding-left` 使用：

```css
.note {
  padding-left: 2em;
  text-indent: -2em;
}
```

## 問題五：`text-indent` 會繼承嗎？

會。

`text-indent` 是可繼承屬性。父元素設定後，子元素可能會繼承。

但實務上通常建議直接設定在需要縮排的段落上：

```css
.article p {
  text-indent: 2em;
}
```

避免影響標題、表格、按鈕或其他 UI 元素。

## 一句話總結

`text-indent` 用來控制文字第一行的縮進，最常見寫法是 `text-indent: 2em;`；它適合文章段落首行縮排，但不適合拿來控制整段文字位移，實務上要分清楚它和 `padding-left`、`margin-left` 的差異。
