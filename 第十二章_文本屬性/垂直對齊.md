# 垂直對齊

> 返回章節首頁：[README.md](./README.md)
>
> `vertical-align` 用於設定行內元素、行內塊元素或表格單元格內容的垂直對齊方式。

## 導讀

- `vertical-align` 不是通用的「垂直置中」屬性。
- 它主要作用在 inline、inline-block、inline-table、table-cell 這類元素上。
- 在一般 block 元素上設定 `vertical-align`，通常不會產生你期待的垂直置中效果。
- `vertical-align` 最常見的場景是圖片、icon、表單元素與文字對齊。
- 圖片底部出現空白縫隙，常常和 inline 元素的 baseline 對齊有關。
- `vertical-align: middle;` 常用來讓圖片、icon、input 和旁邊文字看起來垂直居中。
- 在 `table-cell` 中，`vertical-align` 可以控制儲存格內容的垂直位置。
- 如果要做一般區塊的垂直置中，通常應該使用 Flex 或 Grid，而不是 `vertical-align`。
- 學 `vertical-align` 的核心是理解：它不是控制元素在父層中的垂直位置，而是控制行內排版或表格單元格中的垂直對齊。

## 正文

`vertical-align` 用來設定元素在垂直方向上的對齊方式。

最基本的寫法如下：

```css
img {
  vertical-align: middle;
}
```

```html
<img src="./images/img.jpg" alt=""> 這是一段文字
```

這段程式碼會讓圖片和旁邊的文字在垂直方向上看起來比較接近居中。

但是要注意：

> `vertical-align` 不是所有元素都有效。

它主要作用在：

- inline 元素
- inline-block 元素
- inline-table 元素
- table-cell 元素

對一般 block 元素，例如 `div`，直接寫：

```css
div {
  vertical-align: middle;
}
```

通常不會讓這個 `div` 在父元素裡面垂直置中。

## `vertical-align` 到底控制誰？

`vertical-align` 主要控制兩種情況：

### 情況一：行內排版中的垂直對齊

例如：

```html
<span>文字</span>
<img src="./images/img.jpg" alt="">
<input type="text">
```

這些元素在同一行裡面排版時，瀏覽器需要決定它們在垂直方向上怎麼對齊。

這時候 `vertical-align` 就會發揮作用。

### 情況二：表格單元格內容的垂直對齊

例如：

```html
<table>
  <tr>
    <td>內容</td>
  </tr>
</table>
```

在 `td` 或 `th` 上設定：

```css
td {
  vertical-align: middle;
}
```

可以控制儲存格內部內容在垂直方向上的位置。

## 它不能控制一般 block 元素本身置中

很多初學者會這樣寫：

```css
.parent {
  height: 200px;
}

.child {
  vertical-align: middle;
}
```

期待 `.child` 在 `.parent` 裡面垂直置中。

但這通常不會生效。

因為 `.child` 如果是普通 block 元素，`vertical-align` 不會用來控制它在父層中的垂直位置。

正確做法通常是使用 Flex：

```css
.parent {
  height: 200px;
  display: flex;
  align-items: center;
}
```

如果水平也要置中：

```css
.parent {
  height: 200px;
  display: flex;
  align-items: center;
  justify-content: center;
}
```

或使用 Grid：

```css
.parent {
  height: 200px;
  display: grid;
  place-items: center;
}
```

所以要記住：

> 一般區塊垂直置中，用 Flex 或 Grid；行內元素和表格單元格對齊，才常看 `vertical-align`。

## 認識 baseline

`vertical-align` 的預設值是：

```css
vertical-align: baseline;
```

baseline 中文通常稱為「基線」。

在文字排版中，英文字母大多是站在一條看不見的線上，這條線就是 baseline。

例如：

```text
Hello world
-----------
baseline
```

像 `a`、`x`、`n` 這些字母大多站在 baseline 上。

但像 `g`、`p`、`y` 這些字母有下伸部，會往 baseline 下方延伸。

瀏覽器處理圖片、input、inline-block 元素時，預設也會把它們放進同一個行盒裡，並用 baseline 做對齊。

這就是為什麼圖片底部可能會出現一點空白。

## 圖片底部為什麼會有空白？

圖片預設是 inline-level 元素。

當圖片放在一行文字中時，它會像文字一樣參與 baseline 對齊。

例如：

```html
<div class="box">
  <img src="./images/img.jpg" alt="">
</div>
```

```css
.box {
  border: 1px solid red;
}
```

你可能會看到圖片底部和外層 `div` 之間有一點空白。

這不是圖片本身有 margin，也不是 border 出錯。

而是因為 inline-level 元素會預留給文字下伸部的空間，例如 `g`、`p`、`y` 下方需要的空間。

## 解決圖片底部空白

### 方法一：設定 `vertical-align`

```css
img {
  vertical-align: middle;
}
```

或：

```css
img {
  vertical-align: top;
}
```

或：

```css
img {
  vertical-align: bottom;
}
```

這些都可以改變圖片和 baseline 的對齊方式，進而消除或改善底部空白。

### 方法二：把圖片改成 block

```css
img {
  display: block;
}
```

圖片變成 block 元素後，就不再以 inline-level 的方式參與文字基線對齊，底部空白通常就會消失。

這是實務中很常見也很乾淨的解法。

### 方法三：父層設定 `line-height: 0`

```css
.image-box {
  line-height: 0;
}
```

這也可能消除圖片底部空白，因為它壓縮了行盒高度。

但這種方式會影響父層內其他文字排版，使用時要小心。

一般情況下，最常用的是：

```css
img {
  display: block;
}
```

或：

```css
img {
  vertical-align: middle;
}
```

## 常見值

### `baseline`

`baseline` 是預設值，表示元素和父元素的基線對齊。

```css
img {
  vertical-align: baseline;
}
```

這是圖片底部容易出現空白的原因之一。

`baseline` 適合一般文字排版，但如果圖片或 icon 和文字對齊不自然，就可以改用其他值。

## `middle`

`middle` 表示元素中線和文字中線附近對齊。

```css
img {
  vertical-align: middle;
}
```

常見於圖片、icon、表單元素與文字對齊。

例如：

```html
<img class="icon" src="./images/icon.png" alt="">
<span>設定</span>
```

```css
.icon {
  width: 16px;
  height: 16px;
  vertical-align: middle;
}
```

這樣 icon 和文字看起來會比較自然。

要注意的是，`vertical-align: middle;` 在 inline 排版中不是嚴格的「父容器正中央」，而是基於文字基線與 x-height 的對齊效果。

實務上可以理解成：

> 讓行內元素和旁邊文字看起來接近垂直居中。

如果要非常精準地讓 icon 和文字置中，現代專案通常會用 Flex：

```css
.icon-text {
  display: inline-flex;
  align-items: center;
  gap: 4px;
}
```

## `top`

`top` 表示元素頂部和所在行盒的頂部對齊。

```css
img {
  vertical-align: top;
}
```

常見於多個 inline-block 元素高度不同時，希望它們頂部對齊。

例如：

```html
<div class="card">短內容</div>
<div class="card">很多很多內容<br>第二行內容</div>
```

```css
.card {
  display: inline-block;
  width: 160px;
  vertical-align: top;
}
```

如果沒有設定 `vertical-align: top;`，inline-block 預設會 baseline 對齊，可能導致卡片底部或文字基線對齊，看起來上方不整齊。

設定後，每個 `.card` 的頂部會對齊。

## `bottom`

`bottom` 表示元素底部和所在行盒的底部對齊。

```css
img {
  vertical-align: bottom;
}
```

常見於圖片或 inline-block 元素希望底部貼齊時。

例如：

```css
img {
  vertical-align: bottom;
}
```

有時也可以用來消除圖片底部空白。

## `text-top`

`text-top` 會讓元素頂部和父元素文字內容區的頂部對齊。

```css
.icon {
  vertical-align: text-top;
}
```

它和 `top` 不完全一樣。

簡單理解：

- `top` 比較偏向和整個 line box 頂部對齊
- `text-top` 比較偏向和文字本身的頂部對齊

這類值在一般 UI 中比較少用，通常在精細文字排版時才會碰到。

## `text-bottom`

`text-bottom` 會讓元素底部和父元素文字內容區的底部對齊。

```css
.icon {
  vertical-align: text-bottom;
}
```

它和 `bottom` 也不完全一樣。

簡單理解：

- `bottom` 比較偏向和整個 line box 底部對齊
- `text-bottom` 比較偏向和文字本身的底部對齊

一般實務中，最常用的仍然是：

```css
vertical-align: middle;
vertical-align: top;
vertical-align: bottom;
```

## `sub` 和 `super`

`sub` 表示下標。

```css
.sub {
  vertical-align: sub;
}
```

`super` 表示上標。

```css
.super {
  vertical-align: super;
}
```

例如：

```html
H<span class="sub">2</span>O
```

```css
.sub {
  vertical-align: sub;
  font-size: 0.75em;
}
```

會形成類似：

```text
H₂O
```

再例如平方：

```html
x<span class="super">2</span>
```

```css
.super {
  vertical-align: super;
  font-size: 0.75em;
}
```

會形成類似：

```text
x²
```

不過 HTML 本身也有語意標籤：

```html
H<sub>2</sub>O
x<sup>2</sup>
```

如果是語意上的上標或下標，優先使用 `sup` 和 `sub` 會更合適。

## 長度值與百分比

`vertical-align` 也可以使用長度值做微調。

```css
.icon {
  vertical-align: -2px;
}
```

負值會讓元素稍微往下移。

正值會讓元素稍微往上移。

這常用於 icon 和文字的精準對齊。

例如：

```css
.icon {
  display: inline-block;
  vertical-align: -0.125em;
}
```

有些圖示字體或 SVG icon 會用這種方式微調。

但要注意，手動微調雖然有效，卻比較依賴字體、icon 尺寸和設計稿。

如果是元件化開發，很多時候更推薦用 Flex：

```css
.icon-text {
  display: inline-flex;
  align-items: center;
}
```

## 圖片和文字對齊

圖片和文字放在同一行時，預設可能不會看起來完全居中。

```html
<img class="avatar" src="./images/avatar.png" alt="">
<span>使用者名稱</span>
```

```css
.avatar {
  width: 24px;
  height: 24px;
  vertical-align: middle;
}
```

這樣圖片會和文字比較自然地對齊。

如果圖片和文字是元件的一部分，也可以用 Flex：

```css
.user {
  display: inline-flex;
  align-items: center;
  gap: 8px;
}
```

```html
<span class="user">
  <img src="./images/avatar.png" alt="">
  <span>使用者名稱</span>
</span>
```

Flex 的優點是更穩定，也更適合複雜內容。

## Icon 和文字對齊

icon 和文字對齊是 `vertical-align` 很常見的場景。

```html
<span class="menu-item">
  <span class="icon">✓</span>
  <span>完成</span>
</span>
```

```css
.icon {
  vertical-align: middle;
}
```

如果 icon 是 SVG：

```html
<span class="menu-item">
  <svg class="icon" viewBox="0 0 16 16"></svg>
  <span>完成</span>
</span>
```

```css
.icon {
  width: 16px;
  height: 16px;
  vertical-align: middle;
}
```

不過現代專案更常用：

```css
.menu-item {
  display: inline-flex;
  align-items: center;
  gap: 4px;
}
```

原因是 Flex 對 icon、文字、不同尺寸內容的對齊更穩定。

## 表單元素和文字對齊

表單元素預設也常和文字對齊不自然。

例如：

```html
<input type="checkbox"> 我同意使用條款
```

可以寫：

```css
input {
  vertical-align: middle;
}
```

讓 checkbox 和文字看起來更接近居中。

或者：

```html
<label class="checkbox-label">
  <input type="checkbox">
  <span>我同意使用條款</span>
</label>
```

```css
.checkbox-label {
  display: inline-flex;
  align-items: center;
  gap: 4px;
}
```

現代 UI 中，表單控制項和文字的對齊通常更推薦使用 Flex。

## Textarea 和文字對齊

例如：

```html
<textarea></textarea> 請輸入備註
```

如果沒有設定，textarea 和文字可能以 baseline 對齊，看起來不自然。

可以寫：

```css
textarea {
  vertical-align: middle;
}
```

但如果是完整表單排版，通常會用 block 或 Flex / Grid 結構處理，而不是只靠 `vertical-align`。

## inline-block 元素對齊

`vertical-align` 對 inline-block 很常用。

例如：

```html
<div class="item">短內容</div>
<div class="item">多行內容<br>第二行</div>
<div class="item">短內容</div>
```

```css
.item {
  display: inline-block;
  width: 160px;
  vertical-align: top;
}
```

這樣每個 inline-block 的頂部會對齊。

如果沒有設定：

```css
vertical-align: top;
```

inline-block 預設會 baseline 對齊，看起來可能上下不齊。

這在早期不用 Flex 的版面中很常見。

現代如果是多個卡片排列，通常會使用 Flex 或 Grid：

```css
.list {
  display: flex;
  align-items: flex-start;
  gap: 16px;
}
```

或：

```css
.list {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 16px;
}
```

## table-cell 中的 `vertical-align`

`vertical-align` 在表格儲存格中也很重要。

```css
td {
  vertical-align: middle;
}
```

```html
<table>
  <tr>
    <td>名稱</td>
    <td>這是一段比較長的內容<br>第二行</td>
  </tr>
</table>
```

在 `td` 或 `th` 中，`vertical-align` 控制的是儲存格內容在儲存格內部的垂直位置。

常見值：

```css
td {
  vertical-align: top;
}

td {
  vertical-align: middle;
}

td {
  vertical-align: bottom;
}
```

### 表格常見用法

```css
.table th,
.table td {
  padding: 8px 12px;
  vertical-align: middle;
}
```

這樣表格內容會在儲存格中垂直置中，看起來比較整齊。

如果某些欄位內容很多，也可以設定頂部對齊：

```css
.table .description {
  vertical-align: top;
}
```

後台系統表格很常會遇到這種需求。

## `vertical-align` 和 `text-align` 的差異

`text-align` 控制水平對齊。

```css
.box {
  text-align: center;
}
```

表示讓元素內部文字或行內內容水平置中。

`vertical-align` 控制垂直方向上的行內對齊或表格儲存格內容對齊。

```css
img {
  vertical-align: middle;
}
```

表示圖片和旁邊文字在垂直方向上對齊。

兩者不是同一種屬性。

| 屬性 | 控制方向 | 常見用途 |
| --- | --- | --- |
| `text-align` | 水平方向 | 文字水平置中、靠左、靠右 |
| `vertical-align` | 垂直方向 | 圖片與文字對齊、表格內容垂直對齊 |

## `vertical-align` 和 `line-height` 的關係

`line-height` 控制一行文字的行盒高度。

`vertical-align` 控制行內元素在行盒中的垂直對齊位置。

例如圖片底部空白問題，就和這兩件事都有關：

- 圖片是 inline-level 元素
- inline-level 元素會參與 line box
- 預設 baseline 對齊
- line box 會預留文字下伸部空間

因此圖片底部可能出現空白。

解法可以是：

```css
img {
  vertical-align: bottom;
}
```

或：

```css
img {
  display: block;
}
```

## `vertical-align` 和 Flex 的差異

`vertical-align` 適合處理：

- 圖片和文字在一行內對齊
- icon 和文字對齊
- input 和文字對齊
- inline-block 元素對齊
- table-cell 內容對齊

Flex 適合處理：

- 父容器內子元素垂直置中
- 多個區塊排列
- icon + text 元件
- 按鈕內容置中
- 卡片、工具列、導覽列排版

例如：

```css
.button {
  display: inline-flex;
  align-items: center;
  justify-content: center;
}
```

比單純依賴：

```css
vertical-align: middle;
```

更適合現代元件排版。

簡單判斷：

| 需求 | 建議 |
| --- | --- |
| 圖片和文字在同一行對齊 | 可用 `vertical-align` |
| icon 和文字組成元件 | 優先用 Flex |
| 表格儲存格內容垂直置中 | 用 `vertical-align` |
| block 元素在父層中垂直置中 | 用 Flex 或 Grid |
| 多個卡片排列 | 用 Flex 或 Grid |

## 常見錯誤

### 錯誤一：以為 `vertical-align` 可以讓 block 元素垂直置中

錯誤寫法：

```css
.parent {
  height: 200px;
}

.child {
  vertical-align: middle;
}
```

如果 `.child` 是普通 block 元素，這通常不會讓它在 `.parent` 中垂直置中。

正確做法：

```css
.parent {
  height: 200px;
  display: flex;
  align-items: center;
}
```

如果水平也要置中：

```css
.parent {
  height: 200px;
  display: flex;
  align-items: center;
  justify-content: center;
}
```

## 錯誤二：圖片底部空白以為是 margin

圖片底部空白常常不是 margin，而是 baseline 對齊與 line box 造成的。

常見解法：

```css
img {
  display: block;
}
```

或：

```css
img {
  vertical-align: bottom;
}
```

## 錯誤三：以為 `middle` 是父容器絕對垂直置中

```css
img {
  vertical-align: middle;
}
```

這不是讓圖片在父容器中絕對垂直置中，而是讓圖片在行內排版中和文字中線附近對齊。

如果要讓元素在容器中垂直置中，用 Flex：

```css
.parent {
  display: flex;
  align-items: center;
}
```

## 錯誤四：在複雜元件中硬用 `vertical-align`

例如 icon + text + badge + button 的複雜元件，不建議只靠 `vertical-align` 微調。

不建議：

```css
.icon,
.text,
.badge {
  vertical-align: middle;
}
```

更推薦：

```css
.item {
  display: inline-flex;
  align-items: center;
  gap: 8px;
}
```

Flex 更穩定，也更容易維護。

## 錯誤五：忽略 inline-block 預設 baseline 對齊

多個 inline-block 元素高度不同時，如果不設定 `vertical-align`，可能會出現上方不齊。

修正：

```css
.item {
  display: inline-block;
  vertical-align: top;
}
```

或直接使用 Flex / Grid 排版。

## 錯誤六：在表格中忘記 `vertical-align`

表格中如果某些欄位內容有多行，預設垂直位置可能不符合需求。

可以明確設定：

```css
td {
  vertical-align: middle;
}
```

或：

```css
td {
  vertical-align: top;
}
```

後台資料表中常需要根據欄位類型設定不同垂直對齊。

## 錯誤七：用大量像素微調 icon

```css
.icon {
  vertical-align: -3px;
}
```

這種寫法短期有效，但如果字體、字號、icon 尺寸改變，對齊可能又跑掉。

如果是穩定的文字 icon 元件，更推薦：

```css
.icon-text {
  display: inline-flex;
  align-items: center;
}
```

## 面試可能問法

### 問題一：`vertical-align` 是做什麼的？

`vertical-align` 用來設定行內元素、行內塊元素或表格單元格內容的垂直對齊方式。

常見場景是圖片和文字對齊、icon 和文字對齊、input 和文字對齊、表格儲存格內容垂直對齊。

## 問題二：`vertical-align` 可以讓普通 div 在父元素中垂直置中嗎？

通常不可以。

如果 `div` 是普通 block 元素，`vertical-align` 不會讓它在父元素中垂直置中。

正確做法通常是 Flex：

```css
.parent {
  display: flex;
  align-items: center;
}
```

或 Grid：

```css
.parent {
  display: grid;
  place-items: center;
}
```

## 問題三：圖片底部為什麼會有空白？

因為圖片預設是 inline-level 元素，會參與文字的 baseline 對齊。

瀏覽器會為文字下伸部保留空間，所以圖片底部可能會出現一點空白。

常見解法：

```css
img {
  display: block;
}
```

或：

```css
img {
  vertical-align: bottom;
}
```

## 問題四：`vertical-align: middle;` 是真正的垂直置中嗎？

不完全是。

在 inline 排版中，`middle` 是讓元素和文字中線附近對齊，看起來接近垂直居中。

但它不是讓元素在父容器中絕對置中。

如果要讓元素在父容器中垂直置中，應該使用 Flex 或 Grid。

## 問題五：`vertical-align` 對哪些元素有效？

主要對以下元素有效：

- inline 元素
- inline-block 元素
- inline-table 元素
- table-cell 元素

對普通 block 元素本身通常不會產生垂直置中效果。

## 問題六：表格中如何讓內容垂直置中？

可以在 `td` 或 `th` 上設定：

```css
td,
th {
  vertical-align: middle;
}
```

如果希望內容靠上：

```css
td {
  vertical-align: top;
}
```

## 問題七：`vertical-align` 和 Flex 的差異是什麼？

`vertical-align` 主要處理行內排版或表格儲存格內容的垂直對齊。

Flex 是一種佈局模型，可以控制父容器內子元素的排列和對齊。

簡單判斷：

- 圖片和文字在同一行對齊：可以用 `vertical-align`
- block 元素在父層中垂直置中：用 Flex 或 Grid
- icon + text 元件：通常優先用 Flex
- 表格儲存格內容垂直對齊：用 `vertical-align`

## 一句話總結

`vertical-align` 不是通用垂直置中工具，它主要用於行內元素、行內塊元素與表格單元格內容的垂直對齊；實務上常用來處理圖片、icon、表單元素與文字對齊，以及圖片底部空白問題，一般區塊置中則應使用 Flex 或 Grid。
