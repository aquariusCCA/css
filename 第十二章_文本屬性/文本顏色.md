# 文本顏色

> 返回章節首頁：[README.md](./README.md)
>
> `color` 屬性用於設定元素內文字的前景顏色，是 CSS 中最常見、也最基礎的文本屬性之一。

## 導讀

- `color` 主要用來設定文字顏色。
- `color` 是可繼承屬性，子元素通常會繼承父元素的文字顏色。
- 常見顏色寫法有命名色、HEX、RGB、RGBA、HSL。
- 實務開發中，通常會搭配設計規範、CSS 變數或主題色系來管理顏色。
- `color` 不只影響文字本身，也會影響部分依賴文字顏色的效果，例如 `currentColor`。
- 寫文字顏色時，除了顏色是否好看，也要注意可讀性與對比度。

## 正文

`color` 屬性用於設定元素內文字的顏色。

```css
div {
  color: red;
}
```

```html
<div>正在努力學習前端知識中 ...</div>
```

設定後，`div` 裡面的文字會顯示成紅色。

## 常見顏色寫法

### 命名色

命名色是直接使用英文顏色名稱。

```css
p {
  color: red;
}
```

常見命名色有：

```css
color: red;
color: blue;
color: green;
color: black;
color: white;
color: gray;
```

命名色的優點是簡單直覺，缺點是不夠精準，也不太適合大型專案維護。

所以命名色比較適合教學、測試或臨時除錯。

## HEX 寫法

HEX 是實務中最常見的顏色寫法之一。

```css
p {
  color: #cc00ff;
}
```

HEX 使用十六進位表示紅、綠、藍三個通道。

```css
/* 完整寫法 */
color: #333333;

/* 簡寫 */
color: #333;
```

`#333` 等同於 `#333333`。

常見灰階文字會這樣寫：

```css
.title {
  color: #222;
}

.content {
  color: #666;
}

.desc {
  color: #999;
}
```

在實務中，灰階顏色經常用來區分文字層級：

* 標題文字顏色較深
* 正文文字顏色適中
* 輔助文字顏色較淺

## RGB 寫法

RGB 使用紅、綠、藍三個數值表示顏色。

```css
p {
  color: rgb(255, 0, 255);
}
```

每個數值通常介於 `0` 到 `255` 之間。

```css
color: rgb(255, 0, 0);     /* 紅色 */
color: rgb(0, 255, 0);     /* 綠色 */
color: rgb(0, 0, 255);     /* 藍色 */
color: rgb(0, 0, 0);       /* 黑色 */
color: rgb(255, 255, 255); /* 白色 */
```

RGB 的優點是可以很清楚地看出紅、綠、藍三個通道的數值。

## RGBA 寫法

RGBA 是 RGB 加上透明度。

```css
p {
  color: rgba(255, 0, 255, 0.5);
}
```

最後一個值是透明度：

```css
color: rgba(0, 0, 0, 1);   /* 完全不透明 */
color: rgba(0, 0, 0, 0.5); /* 半透明 */
color: rgba(0, 0, 0, 0);   /* 完全透明 */
```

實務上，半透明文字要小心使用，因為背景不同時，可讀性可能會變差。

## HSL 寫法

HSL 使用色相、飽和度、亮度來描述顏色。

```css
p {
  color: hsl(300, 100%, 50%);
}
```

HSL 的三個值分別是：

| 值          | 說明                       |
| ---------- | ------------------------ |
| Hue        | 色相，通常是 `0deg` 到 `360deg` |
| Saturation | 飽和度，數值越高顏色越鮮豔            |
| Lightness  | 亮度，數值越高越亮                |

例如：

```css
color: hsl(0, 100%, 50%);   /* 紅色 */
color: hsl(120, 100%, 50%); /* 綠色 */
color: hsl(240, 100%, 50%); /* 藍色 */
```

HSL 的優點是調整顏色深淺時比較直覺，尤其適合建立同一色系的不同狀態。

```css
.button {
  color: hsl(210, 100%, 40%);
}

.button:hover {
  color: hsl(210, 100%, 30%);
}
```

## `color` 的繼承特性

`color` 是可繼承屬性。

也就是說，父元素設定了 `color` 後，子元素通常會自動繼承這個文字顏色。

```css
.article {
  color: #333;
}
```

```html
<div class="article">
  <h2>標題</h2>
  <p>這是一段文字。</p>
  <span>這是補充說明。</span>
</div>
```

在這個例子中，如果 `h2`、`p`、`span` 沒有另外設定 `color`，它們通常都會繼承 `.article` 的 `#333`。

這也是為什麼實務中常常會在外層容器設定整體文字顏色。

```css
body {
  color: #333;
}
```

這樣整個頁面的預設文字顏色就會比較一致。

## 子元素覆蓋父元素顏色

雖然 `color` 會繼承，但子元素可以重新設定自己的顏色。

```css
.article {
  color: #333;
}

.article .warning {
  color: red;
}
```

```html
<div class="article">
  <p>一般文字</p>
  <p class="warning">警告文字</p>
</div>
```

這時一般文字會繼承 `.article` 的 `#333`，但 `.warning` 會改成紅色。

## `currentColor`

`currentColor` 是 CSS 中一個很實用的關鍵字，代表目前元素的 `color` 值。

```css
button {
  color: #1677ff;
  border: 1px solid currentColor;
}
```

這表示：

```css
border-color: #1677ff;
```

因為 `currentColor` 會拿目前元素的 `color` 當作顏色值。

### 實務用途

當你希望文字、邊框、圖示顏色保持一致時，可以使用 `currentColor`。

```css
.icon-button {
  color: #1677ff;
  border: 1px solid currentColor;
}

.icon-button svg {
  fill: currentColor;
}
```

這樣只要改 `.icon-button` 的 `color`，文字、邊框、SVG 圖示就可以一起變色。

## 搭配 CSS 變數

大型專案通常不會到處硬寫顏色，而是會用 CSS 變數統一管理。

```css
:root {
  --text-primary: #1f2329;
  --text-regular: #4e5969;
  --text-secondary: #86909c;
  --text-disabled: #c9cdd4;
  --color-danger: #f53f3f;
  --color-success: #00b42a;
}

.title {
  color: var(--text-primary);
}

.content {
  color: var(--text-regular);
}

.desc {
  color: var(--text-secondary);
}

.error {
  color: var(--color-danger);
}
```

這種寫法的優點是：

* 顏色有統一來源
* 修改主題比較方便
* 比較符合設計系統
* 避免專案中到處散落不同的顏色值

## 實務常見文字顏色層級

在實際頁面中，文字通常會有不同層級。

```css
.title {
  color: #1f2329;
  font-weight: 700;
}

.content {
  color: #4e5969;
}

.description {
  color: #86909c;
}

.disabled {
  color: #c9cdd4;
}
```

常見層級如下：

| 類型   | 說明        |
| ---- | --------- |
| 主要文字 | 標題、重要內容   |
| 常規文字 | 一般正文      |
| 次要文字 | 補充說明、提示文字 |
| 禁用文字 | 不可操作、失效狀態 |
| 危險文字 | 錯誤、刪除、警告  |
| 成功文字 | 成功狀態、通過狀態 |

## 狀態文字範例

```css
.message {
  font-size: 14px;
}

.message.success {
  color: #00b42a;
}

.message.warning {
  color: #ff7d00;
}

.message.error {
  color: #f53f3f;
}

.message.info {
  color: #1677ff;
}
```

```html
<p class="message success">操作成功</p>
<p class="message warning">請注意資料格式</p>
<p class="message error">提交失敗</p>
<p class="message info">這是一段提示訊息</p>
```

這種寫法常見於表單驗證、後台管理系統、提示訊息和狀態標籤。

## 與 `background-color` 的差異

`color` 是文字前景色。

`background-color` 是背景色。

```css
.box {
  color: white;
  background-color: black;
}
```

```html
<div class="box">白色文字，黑色背景</div>
```

兩者經常一起搭配使用。

寫文字顏色時，不能只看 `color` 本身，還要看它所在區塊的背景顏色。

## 可讀性與對比度

文字顏色最重要的不只是好不好看，而是能不能清楚閱讀。

例如下面這種寫法可讀性較差：

```css
.bad {
  color: #cccccc;
  background-color: #ffffff;
}
```

因為文字太接近背景色，使用者閱讀時會很吃力。

比較好的做法是讓文字與背景有足夠差異：

```css
.good {
  color: #333333;
  background-color: #ffffff;
}
```

實務上，尤其是表單錯誤訊息、按鈕文字、提示文字，都要注意顏色對比。

## 常見錯誤

### 錯誤一：以為 `color` 會改背景色

```css
div {
  color: yellow;
}
```

這只會讓文字變黃色，不會改變背景。

如果要改背景，要使用：

```css
div {
  background-color: yellow;
}
```

### 錯誤二：到處硬編碼顏色

```css
.card-title {
  color: #333;
}

.form-label {
  color: #333;
}

.table-title {
  color: #333;
}
```

這樣短期可以用，但大型專案會不好維護。

比較好的方式是：

```css
:root {
  --text-primary: #333;
}

.card-title,
.form-label,
.table-title {
  color: var(--text-primary);
}
```

### 錯誤三：顏色太淡，導致文字難讀

```css
p {
  color: #dddddd;
  background-color: #ffffff;
}
```

這種文字對比太低，不適合作為主要內容。

### 錯誤四：只靠顏色表達狀態

```html
<span class="status error">失敗</span>
<span class="status success">成功</span>
```

如果只靠紅色、綠色區分狀態，對部分使用者可能不夠清楚。

比較好的方式是同時搭配文字、圖示或狀態描述。

```html
<span class="status error">錯誤：提交失敗</span>
<span class="status success">成功：資料已送出</span>
```

### 錯誤五：忽略連結的預設顏色

瀏覽器中的 `a` 標籤通常有預設顏色。

```html
<a href="#">這是一個連結</a>
```

實務中常會重新設定：

```css
a {
  color: #1677ff;
  text-decoration: none;
}

a:hover {
  color: #0958d9;
  text-decoration: underline;
}
```

## 實務範例：文章區塊文字顏色

```css
.article {
  color: #333;
  background-color: #fff;
}

.article h1 {
  color: #111;
}

.article p {
  color: #444;
}

.article .meta {
  color: #999;
}

.article .danger {
  color: #f53f3f;
}
```

```html
<article class="article">
  <h1>CSS 文本顏色</h1>
  <p class="meta">更新時間：2026-05-03</p>
  <p>color 屬性可以設定文字顏色。</p>
  <p class="danger">這是一段警告文字。</p>
</article>
```

這種寫法可以讓整個區塊有預設文字色，也能針對標題、輔助文字、警告文字做差異化。

## 實務範例：按鈕文字與邊框同步

```css
.button {
  color: #1677ff;
  border: 1px solid currentColor;
  background-color: transparent;
}

.button:hover {
  color: #0958d9;
}
```

```html
<button class="button">查看詳情</button>
```

因為 `border` 使用 `currentColor`，所以按鈕 hover 時，只要改 `color`，文字和邊框就會一起變色。

## 面試可能問法

### 問題一：`color` 會不會繼承？

會。

`color` 是可繼承屬性。父元素設定 `color` 後，子元素如果沒有自己設定顏色，通常會繼承父元素的文字顏色。

```css
.parent {
  color: #333;
}
```

```html
<div class="parent">
  <p>這段文字會繼承 #333</p>
</div>
```

### 問題二：`color` 和 `background-color` 差在哪？

`color` 設定文字前景色。

`background-color` 設定背景色。

```css
.box {
  color: white;
  background-color: black;
}
```

### 問題三：`currentColor` 是什麼？

`currentColor` 代表目前元素的 `color` 值。

```css
.box {
  color: red;
  border-color: currentColor;
}
```

這裡的 `border-color` 會等於 `red`。

### 問題四：實務中為什麼不建議到處寫死顏色？

因為大型專案顏色需要統一管理。

如果每個地方都硬寫 `#333`、`#666`、`#999`，之後要改設計風格或做主題切換會很麻煩。

比較好的方式是使用 CSS 變數或設計 token。

```css
:root {
  --text-primary: #333;
}

.title {
  color: var(--text-primary);
}
```

## 一句話總結

`color` 是設定文字前景色的屬性，會被子元素繼承；實務上除了掌握命名色、HEX、RGB 等基本寫法，也要理解 `currentColor`、CSS 變數、顏色層級與可讀性。
