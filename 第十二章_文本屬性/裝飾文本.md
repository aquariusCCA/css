# 裝飾文本

> 返回章節首頁：[README.md](./README.md)
>
> `text-decoration` 用於設定文字裝飾效果，例如下劃線、刪除線、上劃線，也可以搭配顏色、線條樣式與粗細一起控制。

![裝飾文本](./images/image_1776231946898_867e6aea.png)

## 導讀

- `text-decoration` 最常見的用途是控制文字底線、刪除線與上劃線。
- 常見值有 `underline`、`line-through`、`overline`、`none`。
- `a` 標籤在瀏覽器中通常預設會有底線，實務上常會用 `text-decoration: none;` 取消，再依設計需求重新定義。
- `text-decoration` 不只是單一屬性，也可以視為 shorthand，可以同時設定線的位置、顏色、樣式與粗細。
- 現代 CSS 可以搭配 `text-underline-offset` 調整底線與文字的距離。
- 文字裝飾常用於連結、錯誤狀態、刪除價格、標記重點與互動提示。
- 使用文字裝飾時，要注意可讀性與可用性，不建議把所有連結底線都無條件移除。

- `text-decoration: line-through;` 代表刪除線。
- `text-decoration: overline;` 代表上劃線。
- `text-decoration: none;` 代表取消文字裝飾。
- `a` 標籤常用 `text-decoration: none;` 取消瀏覽器預設底線。
- `text-decoration` 可以同時設定線條位置、樣式、顏色與粗細。
- `text-decoration-line` 控制線的位置，例如 `underline`、`line-through`。
- `text-decoration-style` 控制線條樣式，例如 `solid`、`dashed`、`wavy`。
- `text-decoration-color` 控制裝飾線顏色。
- `text-decoration-thickness` 控制裝飾線粗細。
- `text-underline-offset` 可以調整底線和文字之間的距離。

## 正文

`text-decoration` 屬性用於設定文字裝飾效果。

最基本的寫法如下：

```css
div {
  text-decoration: underline;
}
```

```html
<div>這是一段有下劃線的文字</div>
```

這段程式碼會讓 `div` 裡面的文字出現下劃線。

## 常見值

### `underline`

`underline` 表示下劃線。

```css
.text {
  text-decoration: underline;
}
```

```html
<p class="text">這是一段有下劃線的文字</p>
```

下劃線最常見的用途是表示連結。

```css
a {
  text-decoration: underline;
}
```

```html
<a href="#">查看詳情</a>
```

在使用者介面中，下劃線通常會讓人聯想到「可以點擊」。

所以如果文字不是連結，使用下劃線時要小心，避免讓使用者誤以為可以操作。

## `line-through`

`line-through` 表示刪除線。

```css
.old-price {
  text-decoration: line-through;
}
```

```html
<span class="old-price">$1,280</span>
<span class="new-price">$999</span>
```

刪除線常見於：

- 商品原價
- 已刪除內容
- 已失效文字
- 任務完成狀態

例如待辦事項完成後，可以用刪除線表示已完成。

```css
.todo-item.done {
  color: #999;
  text-decoration: line-through;
}
```

```html
<li class="todo-item done">完成 CSS 文本屬性筆記</li>
```

## `overline`

`overline` 表示上劃線。

```css
.text {
  text-decoration: overline;
}
```

```html
<p class="text">這是一段有上劃線的文字</p>
```

上劃線在一般 UI 中比較少見，通常用於特殊視覺設計或特定排版需求。

實務上最常用的仍然是：

```css
text-decoration: underline;
text-decoration: line-through;
text-decoration: none;
```

## `none`

`none` 表示取消文字裝飾。

```css
a {
  text-decoration: none;
}
```

```html
<a href="#">這是一個沒有底線的連結</a>
```

瀏覽器中的 `a` 標籤通常預設會有底線。

實務開發時，常會先取消預設底線，再依照設計稿重新定義連結樣式。

```css
a {
  color: #1677ff;
  text-decoration: none;
}

a:hover {
  text-decoration: underline;
}
```

這樣平常不顯示底線，滑鼠移入時才出現底線。

## `text-decoration` 是 shorthand

`text-decoration` 可以視為一個 shorthand，也就是可以一次設定多個相關屬性。

它常見對應到以下子屬性：

| 子屬性 | 作用 |
| --- | --- |
| `text-decoration-line` | 設定線的位置 |
| `text-decoration-color` | 設定線的顏色 |
| `text-decoration-style` | 設定線的樣式 |
| `text-decoration-thickness` | 設定線的粗細 |

例如：

```css
.link {
  text-decoration-line: underline;
  text-decoration-color: red;
  text-decoration-style: wavy;
  text-decoration-thickness: 2px;
}
```

也可以簡寫成：

```css
.link {
  text-decoration: underline wavy red 2px;
}
```

這表示：

- 有下劃線
- 線條是波浪線
- 線條顏色是紅色
- 線條粗細是 `2px`

## `text-decoration-line`

`text-decoration-line` 用來設定裝飾線的位置。

```css
.text {
  text-decoration-line: underline;
}
```

常見值如下：

| 值 | 說明 |
| --- | --- |
| `none` | 無裝飾線 |
| `underline` | 下劃線 |
| `overline` | 上劃線 |
| `line-through` | 刪除線 |

也可以同時設定多個線條。

```css
.text {
  text-decoration-line: underline overline;
}
```

這樣文字會同時有下劃線和上劃線。

不過實務上同時使用多條線容易影響閱讀，通常不建議濫用。

## `text-decoration-style`

`text-decoration-style` 用來設定線條樣式。

```css
.text {
  text-decoration-line: underline;
  text-decoration-style: wavy;
}
```

常見值如下：

| 值 | 說明 |
| --- | --- |
| `solid` | 實線 |
| `double` | 雙線 |
| `dotted` | 點線 |
| `dashed` | 虛線 |
| `wavy` | 波浪線 |

例如：

```css
.error {
  text-decoration-line: underline;
  text-decoration-style: wavy;
  text-decoration-color: red;
}
```

這種波浪底線常見於錯誤提示或拼字檢查的視覺效果。

## `text-decoration-color`

`text-decoration-color` 用來設定裝飾線的顏色。

```css
.link {
  color: #1677ff;
  text-decoration-line: underline;
  text-decoration-color: #f53f3f;
}
```

這裡文字顏色是藍色，但底線顏色是紅色。

如果沒有特別設定 `text-decoration-color`，裝飾線通常會使用目前文字的顏色。

也就是說，下面這段：

```css
.link {
  color: #1677ff;
  text-decoration: underline;
}
```

底線通常會跟著文字一樣是藍色。

## `text-decoration-thickness`

`text-decoration-thickness` 用來設定裝飾線的粗細。

```css
.title {
  text-decoration-line: underline;
  text-decoration-thickness: 3px;
}
```

常見寫法：

```css
text-decoration-thickness: auto;
text-decoration-thickness: 1px;
text-decoration-thickness: 0.1em;
```

實務上，如果文字較大，底線太細可能看不清楚，可以稍微增加底線粗細。

```css
.section-title {
  font-size: 32px;
  font-weight: 700;
  text-decoration-line: underline;
  text-decoration-thickness: 4px;
}
```

## `text-underline-offset`

`text-underline-offset` 不是 `text-decoration` shorthand 的一部分，但常常搭配下劃線使用。

它可以調整底線與文字之間的距離。

```css
.link {
  text-decoration: underline;
  text-underline-offset: 4px;
}
```

如果底線太貼近文字，閱讀起來可能會覺得擁擠。

加上 `text-underline-offset` 後，底線和文字之間會比較有呼吸感。

```css
.nav-link {
  color: #333;
  text-decoration: underline;
  text-underline-offset: 6px;
}
```

這在導覽列、標題連結、文章連結中都很常見。

## `text-decoration-skip-ink`

`text-decoration-skip-ink` 用來控制底線遇到文字筆畫時，是否避開字形。

```css
.link {
  text-decoration: underline;
  text-decoration-skip-ink: auto;
}
```

常見值：

| 值 | 說明 |
| --- | --- |
| `auto` | 瀏覽器自動處理，通常會避開字形筆畫 |
| `none` | 不避開，底線直接穿過文字筆畫 |

例如英文中的 `g`、`p`、`y` 這些字母有下伸部，如果底線直接穿過，可能會影響閱讀。

`text-decoration-skip-ink: auto;` 可以讓底線視覺上更自然。

## 連結樣式設計

連結是 `text-decoration` 最常見的使用場景。

瀏覽器預設通常會讓連結有底線。

```html
<a href="#">查看詳情</a>
```

如果直接取消底線：

```css
a {
  text-decoration: none;
}
```

雖然畫面可能更乾淨，但也可能降低使用者辨識連結的能力。

比較好的做法是保留某種互動提示。

例如：

```css
a {
  color: #1677ff;
  text-decoration: none;
}

a:hover {
  text-decoration: underline;
}
```

或是：

```css
a {
  color: #1677ff;
  text-decoration: underline;
  text-underline-offset: 3px;
}
```

如果連結出現在大量文字中，通常建議保留底線，因為只靠顏色有時不夠明顯。

```css
.article a {
  color: #1677ff;
  text-decoration: underline;
  text-underline-offset: 3px;
}
```

## `text-decoration` 和 `border-bottom` 的差異

有時候你會看到有人用 `border-bottom` 做類似底線的效果。

```css
.link {
  border-bottom: 1px solid currentColor;
}
```

這和 `text-decoration: underline;` 很像，但本質不同。

| 寫法 | 特點 |
| --- | --- |
| `text-decoration: underline;` | 真正的文字底線，會跟文字排版更緊密 |
| `border-bottom` | 是盒模型邊框，可控制性高，但不是文字裝飾線 |

### `text-decoration` 適合

- 一般文字底線
- 文章連結
- 表示語意上的文字裝飾

### `border-bottom` 適合

- 需要更明顯的底線效果
- 需要動畫效果
- 需要完整控制距離、粗細、長度

例如導覽列 hover 動畫常用 `border-bottom` 或偽元素。

```css
.nav-link {
  position: relative;
  text-decoration: none;
}

.nav-link::after {
  content: "";
  position: absolute;
  left: 0;
  bottom: -4px;
  width: 0;
  height: 2px;
  background-color: currentColor;
  transition: width 0.2s;
}

.nav-link:hover::after {
  width: 100%;
}
```

這種效果就不是單純 `text-decoration` 容易完成的。

## 關於繼承的常見誤解

`text-decoration` 本身不是繼承屬性。

也就是說，子元素不會像 `color` 那樣直接繼承父元素的 `text-decoration` 值。

但是要注意，父元素的文字裝飾線可能會延伸到子元素文字上。

例如：

```css
.parent {
  text-decoration: underline;
}

.child {
  text-decoration: none;
}
```

```html
<p class="parent">
  父元素文字
  <span class="child">子元素文字</span>
</p>
```

你可能會發現，`.child` 看起來仍然有底線。

這不是因為 `.child` 繼承了 `text-decoration`，而是因為父元素的底線裝飾延伸到了子元素的文字上。

如果你真的想讓某一段文字不要被父層底線影響，通常需要調整結構或避免在太外層設定底線。

## 常見錯誤

### 錯誤一：以為 `text-decoration` 只會設定底線

`text-decoration` 不只可以設定底線，也可以設定刪除線、上劃線、線條顏色、樣式與粗細。

```css
.text {
  text-decoration: underline wavy red 2px;
}
```

### 錯誤二：移除所有連結底線，卻沒有其他提示

```css
a {
  color: #333;
  text-decoration: none;
}
```

這樣可能讓使用者看不出哪些文字可以點擊。

比較好的做法：

```css
a {
  color: #1677ff;
  text-decoration: underline;
  text-underline-offset: 3px;
}
```

或至少保留 hover 狀態：

```css
a {
  color: #1677ff;
  text-decoration: none;
}

a:hover {
  text-decoration: underline;
}
```

### 錯誤三：把 `text-decoration: none;` 寫在子元素，期待取消父元素底線

```css
.parent {
  text-decoration: underline;
}

.child {
  text-decoration: none;
}
```

這不一定能讓 `.child` 視覺上完全沒有底線，因為父元素的文字裝飾可能會延伸到子元素文字上。

比較好的方式是避免在太外層設定底線，改成只針對需要底線的文字設定。

```css
.need-underline {
  text-decoration: underline;
}
```

### 錯誤四：用刪除線表達不清楚的狀態

```css
.item {
  text-decoration: line-through;
}
```

如果沒有搭配文字或狀態說明，使用者可能不知道這代表「已完成」、「已刪除」還是「不可用」。

比較好的做法：

```html
<span class="item done">已完成：整理 CSS 筆記</span>
```

```css
.item.done {
  color: #999;
  text-decoration: line-through;
}
```

### 錯誤五：過度使用波浪線或多重裝飾線

```css
.text {
  text-decoration: underline overline wavy red;
}
```

這種效果雖然能做出來，但很容易影響閱讀。

文字裝飾的目的通常是輔助辨識，不應該搶走內容本身的注意力。

## 實務範例：文章連結樣式

```css
.article {
  color: #333;
  line-height: 1.8;
}

.article a {
  color: #1677ff;
  text-decoration: underline;
  text-underline-offset: 3px;
  text-decoration-thickness: 1px;
}

.article a:hover {
  color: #0958d9;
  text-decoration-thickness: 2px;
}
```

```html
<article class="article">
  <p>
    你可以閱讀
    <a href="#">CSS 文本屬性</a>
    來了解更多文字相關樣式。
  </p>
</article>
```

這種設計讓連結在文章中清楚可見，也保留 hover 互動感。

## 實務範例：商品價格

```css
.product-price {

.product-price .old {

  <span class="old">$1,280</span>
  <span class="new">$999</span>
</div>
```

刪除線表示原價已被取代，新價格用更明顯的顏色與字重強調。

## 實務範例：表單錯誤提示

```css

}

.form-error .keyword {
  text-decoration-line: underline;
  text-decoration-style: wavy;
  text-decoration-color: currentColor;
  text-underline-offset: 3px;
}
```

```html
<p class="form-error">
  請確認 <span class="keyword">Email 格式</span> 是否正確。
</p>
```

這裡用 `currentColor` 讓底線顏色跟錯誤文字顏色保持一致。

## 面試可能問法

### 問題一：`text-decoration` 常見值有哪些？

常見值有：

```css
text-decoration: none;
text-decoration: underline;
text-decoration: line-through;
text-decoration: overline;
```

其中：

- `none`：取消文字裝飾
- `underline`：下劃線
- `line-through`：刪除線
- `overline`：上劃線

## 問題二：如何取消 `a` 標籤預設底線？

可以使用：

```css
a {
  text-decoration: none;
}
```

但實務上建議保留 hover 或其他互動提示：

```css
a {
  color: #1677ff;
  text-decoration: none;
}

a:hover {
  text-decoration: underline;
}
```

## 問題三：`text-decoration` 是不是繼承屬性？

不是。

`text-decoration` 本身不是繼承屬性。

但父元素設定的文字裝飾可能會延伸到子元素文字上，所以看起來有時候會像是子元素也有底線。

## 問題四：`text-decoration` 和 `border-bottom` 有什麼差別？

`text-decoration` 是文字裝飾線，語意上就是文字的底線、刪除線或上劃線。

`border-bottom` 是盒模型的底部邊框，可以模擬底線，但它本質上不是文字裝飾。

簡單判斷：

- 一般連結底線：優先用 `text-decoration`
- 需要動畫或高度客製化底線：可以考慮 `border-bottom` 或偽元素

## 問題五：`text-underline-offset` 是做什麼的？

`text-underline-offset` 用來調整底線和文字之間的距離。

```css
a {
  text-decoration: underline;
  text-underline-offset: 4px;
}
```

它可以讓底線不要太貼近文字，提升可讀性。

## 一句話總結

`text-decoration` 用來控制文字裝飾線，常見值有 `none`、`underline`、`line-through`、`overline`；實務上除了會取消連結底線，也要理解它的 shorthand 本質、子屬性、底線偏移、可讀性與連結可用性。
