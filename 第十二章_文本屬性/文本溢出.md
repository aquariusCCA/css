# 文本溢出

> 返回章節首頁：[README.md](./README.md)
>
> `text-overflow` 用於設定文本超出容器時的顯示方式，最常見的效果是單行文字超出後顯示省略號。

## 導讀

- `text-overflow` 用來控制文本溢出容器時的顯示方式。
- 它本身不會讓文字溢出，也不會自動隱藏內容，通常需要搭配 `overflow` 和 `white-space`。
- 最常見的值是 `clip` 和 `ellipsis`。
- 單行省略號通常需要同時設定：寬度限制、`white-space: nowrap;`、`overflow: hidden;`、`text-overflow: ellipsis;`。
- 多行省略號不是單靠 `text-overflow` 完成，常見做法是搭配 `-webkit-line-clamp`。
- 在 Flex 或 Grid 佈局中，單行省略號常常需要額外設定 `min-width: 0;`。
- 在表格中使用省略號時，常需要固定欄寬或使用 `table-layout: fixed;`。
- `text-overflow` 解決的是「超出後怎麼顯示」，不是「長單字怎麼斷行」。
- 長網址、長英文、連續字串爆版時，通常要看 `overflow-wrap` 或 `word-break`。

## 30 秒複習入口

- `text-overflow` 用來設定文本溢出時如何顯示。
- `clip` 是預設值，代表直接裁剪超出的文字。
- `ellipsis` 代表超出部分用省略號表示。
- `text-overflow` 單獨寫通常不會有效果。
- 單行省略號通常需要：`white-space: nowrap;`、`overflow: hidden;`、`text-overflow: ellipsis;`。
- 元素還需要有明確寬度、`max-width`，或受到父容器寬度限制。
- 如果元素是普通 `span`，可能需要改成 `display: inline-block;` 或 `block` 才容易生效。
- Flex 子元素中省略號不生效時，常要補 `min-width: 0;`。
- Grid 子元素中省略號不生效時，也常要補 `min-width: 0;`。
- 多行省略號常用 `display: -webkit-box;`、`-webkit-line-clamp`、`-webkit-box-orient: vertical;`、`overflow: hidden;`。
- `text-overflow` 不負責長單字斷行；長網址爆版通常搭配 `overflow-wrap: break-word;` 或 `overflow-wrap: anywhere;`。
- `text-overflow` 常用於表格、卡片標題、列表、麵包屑、檔名、後台欄位。

## 正文

`text-overflow` 用於規定當文字內容超出容器時，應該如何顯示。

最常見的寫法是：

```css
.text {
  text-overflow: ellipsis;
}
```

但是只寫這一行通常不會出現省略號。

因為 `text-overflow` 只負責：

> 超出的文字要怎麼呈現。

它不負責：

- 限制元素寬度
- 強制文字單行
- 隱藏超出的內容
- 讓文字真的產生溢出

所以實務上一定要搭配其他屬性。

## 常見值

### `clip`

`clip` 是預設值，表示直接裁剪超出的文字。

```css
.text {
  text-overflow: clip;
}
```

當文字超出容器時，超出的部分會被直接截掉，不會顯示省略號。

完整範例：

```css
.text {
  width: 160px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: clip;
}
```

```html
<div class="text">
  這是一段很長很長的文字
</div>
```

這時文字超出容器後會被裁剪。

## `ellipsis`

`ellipsis` 表示用省略號表示被截斷的文字。

```css
.text {
  text-overflow: ellipsis;
}
```

完整範例：

```css
.text {
  width: 160px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

```html
<div class="text">
  這是一段很長很長的文字
</div>
```

當文字超過 `160px` 時，超出的部分會用省略號表示。

畫面大概會像：

```text
這是一段很長很...
```

實務上 `ellipsis` 是 `text-overflow` 最常用的值。

## 單行省略號的必要條件

單行省略號不是只靠 `text-overflow`。

通常需要同時滿足以下條件：

```css
.ellipsis {
  width: 200px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

### 條件一：有寬度限制

```css
width: 200px;
```

或：

```css
max-width: 200px;
```

或元素受到父容器寬度限制。

如果沒有寬度限制，元素可能會跟著內容一起變寬，不會產生可判斷的溢出範圍。

例如下面這樣可能不會有效：

```css
.text {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

因為沒有寬度限制。

比較完整：

```css
.text {
  max-width: 240px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

## 條件二：強制文字單行

```css
white-space: nowrap;
```

這會讓文字不自動換行。

如果沒有這個屬性，文字可能會換到下一行，而不是在單行中被截斷。

## 條件三：隱藏超出內容

```css
overflow: hidden;
```

如果沒有隱藏超出的內容，文字可能會直接超出容器顯示，省略號不會出現。

## 條件四：設定省略號

```css
text-overflow: ellipsis;
```

這才是控制「超出後顯示省略號」的屬性。

## 最完整的單行省略號模板

```css
.text-ellipsis {
  max-width: 240px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

```html
<div class="text-ellipsis">
  這是一段很長很長的文字，超出寬度後會顯示省略號
</div>
```

這是最常見、最實用的模板。

## 為什麼只寫 `text-overflow: ellipsis` 沒有效？

錯誤寫法：

```css
.text {
  text-overflow: ellipsis;
}
```

這樣通常不會有效。

原因是：

- 沒有限制寬度
- 沒有禁止換行
- 沒有隱藏超出內容
- 文字不一定真的產生溢出

正確寫法：

```css
.text {
  width: 200px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

## 對 inline 元素的影響

如果元素是普通 `span`，直接設定省略號可能不如預期。

```html
<span class="text">這是一段很長的文字</span>
```

```css
.text {
  width: 200px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

普通 inline 元素通常不容易直接套用寬度，所以省略號可能失效。

可以改成：

```css
.text {
  display: inline-block;
  max-width: 200px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

或：

```css
.text {
  display: block;
  width: 200px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

簡單判斷：

| 元素狀態 | 注意事項 |
| --- | --- |
| `block` | 通常較容易設定寬度與省略號 |
| `inline-block` | 可保留行內排列，又能設定寬度 |
| `inline` | 寬度控制有限，省略號容易不生效 |

## Flex 中的省略號

Flex 佈局中，單行省略號很常遇到失效問題。

例如：

```html
<div class="card">
  <div class="title">
    這是一個很長很長的標題文字
  </div>
  <button>操作</button>
</div>
```

```css
.card {
  display: flex;
  align-items: center;
}

.title {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

有時候 `.title` 不會正常縮小，導致省略號失效。

這時通常要補：

```css
.title {
  min-width: 0;
}
```

完整寫法：

```css
.card {
  display: flex;
  align-items: center;
  gap: 8px;
}

.title {
  flex: 1;
  min-width: 0;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

原因是 Flex item 預設可能不願意小於內容本身的最小寬度。

`min-width: 0;` 可以允許它被壓縮，省略號才有機會生效。

## Grid 中的省略號

Grid 佈局中也可能遇到類似問題。

```css
.row {
  display: grid;
  grid-template-columns: 1fr auto;
  gap: 8px;
}

.title {
  min-width: 0;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

如果省略號在 Grid item 中失效，也可以先檢查：

```css
min-width: 0;
```

是否有設定。

## 表格中的省略號

表格欄位也常需要文字溢出省略。

例如後台資料表中：

- 使用者名稱
- 文件名稱
- 備註
- 地址
- 描述
- 錯誤訊息

可能會很長。

常見寫法：

```css
.table {
  width: 100%;
  table-layout: fixed;
}

.table .ellipsis {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

```html
<table class="table">
  <tr>
    <td class="ellipsis">
      這是一段很長很長的表格內容
    </td>
  </tr>
</table>
```

`table-layout: fixed;` 可以讓表格欄位寬度更穩定，省略號更容易生效。

如果欄寬不固定，表格可能會被內容撐開，導致省略號不出現。

## 多行文本省略號

`text-overflow: ellipsis;` 主要是單行省略號常用。

多行省略號通常需要使用 `line-clamp` 相關寫法。

常見寫法：

```css
.multi-ellipsis {
  display: -webkit-box;
  -webkit-line-clamp: 3;
  -webkit-box-orient: vertical;
  overflow: hidden;
}
```

```html
<div class="multi-ellipsis">
  這是一段很長很長的文字，會顯示最多三行，超過三行後會被截斷並顯示省略效果。
</div>
```

這裡：

```css
-webkit-line-clamp: 3;
```

代表最多顯示三行。

```css
overflow: hidden;
```

代表超出的內容隱藏。

```css
display: -webkit-box;
-webkit-box-orient: vertical;
```

是這種寫法需要搭配的舊式伸縮盒模型設定。

## 多行省略號模板

```css
.text-line-clamp-2 {
  display: -webkit-box;
  -webkit-line-clamp: 2;
  -webkit-box-orient: vertical;
  overflow: hidden;
}

.text-line-clamp-3 {
  display: -webkit-box;
  -webkit-line-clamp: 3;
  -webkit-box-orient: vertical;
  overflow: hidden;
}
```

使用：

```html
<p class="text-line-clamp-2">
  這是一段最多顯示兩行的文字，超過兩行後會被截斷。
</p>
```

多行省略號常見於：

- 卡片描述
- 商品介紹
- 文章摘要
- 通知內容
- 搜尋結果摘要
- 留言預覽

## 單行省略號和多行省略號比較

| 類型 | 核心寫法 | 適合場景 |
| --- | --- | --- |
| 單行省略號 | `white-space: nowrap; overflow: hidden; text-overflow: ellipsis;` | 標題、表格欄位、檔名 |
| 多行省略號 | `-webkit-line-clamp` 搭配 `overflow: hidden` | 卡片描述、文章摘要 |

單行省略號的重點是：

```css
white-space: nowrap;
```

多行省略號的重點是：

```css
-webkit-line-clamp
```

兩者不是同一套寫法。

## 多行省略號注意事項

多行省略號雖然常用，但要注意：

- 不適合需要完整閱讀的正式內容
- 被截斷的內容仍可能存在於 HTML 中
- 如果內容很重要，應提供「展開」或「查看更多」
- 不同瀏覽器或不同排版情況下，效果可能有細微差異
- 行高、字體、容器寬度都會影響最後可顯示的文字數量

實務上如果文字很重要，不建議只靠截斷。

可以搭配：

```html
<button>展開更多</button>
```

或：

```html
<a href="#">查看完整內容</a>
```

## `text-overflow` 和 `overflow` 的差異

`overflow` 控制超出容器的內容要不要顯示。

```css
overflow: hidden;
```

表示超出的內容隱藏。

`text-overflow` 控制被隱藏的文字要如何提示。

```css
text-overflow: ellipsis;
```

表示用省略號提示文字被截斷。

兩者常搭配使用。

| 屬性 | 作用 |
| --- | --- |
| `overflow` | 控制超出內容是否可見 |
| `text-overflow` | 控制文字被截斷時的提示方式 |

## `text-overflow` 和 `white-space` 的差異

`white-space` 控制空白、換行與是否自動換行。

```css
white-space: nowrap;
```

表示文字不自動換行。

`text-overflow` 控制文字溢出後如何顯示。

```css
text-overflow: ellipsis;
```

表示顯示省略號。

單行省略號中兩者分工如下：

```css
.ellipsis {
  white-space: nowrap;      /* 讓文字保持單行 */
  overflow: hidden;         /* 隱藏超出內容 */
  text-overflow: ellipsis;  /* 超出部分顯示省略號 */
}
```

## `text-overflow` 和 `overflow-wrap` 的差異

`text-overflow` 是截斷文字。

`overflow-wrap` 是讓長單字或長網址可以斷行。

例如長網址：

```html
<p class="url">
  https://example.com/this-is-a-very-very-very-long-url-without-normal-break-points
</p>
```

如果希望它不要爆版，而是必要時斷行：

```css
.url {
  overflow-wrap: break-word;
}
```

如果希望它保持單行，超出顯示省略號：

```css
.url {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

兩種需求不同：

| 需求 | 寫法 |
| --- | --- |
| 長文字超出後截斷 | `text-overflow: ellipsis;` |
| 長單字或網址必要時斷行 | `overflow-wrap: break-word;` |
| 更積極允許任意位置斷行 | `overflow-wrap: anywhere;` |
| 強制單字內部斷行 | `word-break: break-all;` |

## `text-overflow` 和 `word-break` 的差異

`word-break` 控制單字內部能不能斷開。

```css
.text {
  word-break: break-all;
}
```

這會讓文字可以在幾乎任意位置斷開。

但它可能會破壞英文單字的閱讀性。

`text-overflow` 則是不斷行，而是截斷後顯示省略號。

```css
.text {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

簡單判斷：

- 要截斷：用 `text-overflow`
- 要斷行：用 `overflow-wrap` 或 `word-break`

## 常見實務場景

### 卡片標題單行省略

```css
.card-title {
  max-width: 280px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

```html
<h3 class="card-title">
  這是一個很長很長的卡片標題
</h3>
```

卡片標題常常需要保持單行，避免破壞卡片高度。

## 卡片描述多行省略

```css
.card-desc {
  display: -webkit-box;
  -webkit-line-clamp: 2;
  -webkit-box-orient: vertical;
  overflow: hidden;
  line-height: 1.6;
}
```

```html
<p class="card-desc">
  這是一段卡片描述文字，最多顯示兩行，超出後會被截斷。
</p>
```

適合列表卡片、商品卡片、文章摘要。

## 表格欄位省略

```css
.table {
  width: 100%;
  table-layout: fixed;
}

.table td {
  padding: 8px 12px;
}

.table .filename {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

```html
<table class="table">
  <tr>
    <td class="filename">
      2026年度非常非常長的報表檔案名稱.xlsx
    </td>
  </tr>
</table>
```

後台系統中非常常見。

## Flex 卡片標題省略

```css
.item {
  display: flex;
  align-items: center;
  gap: 8px;
}

.item-title {
  flex: 1;
  min-width: 0;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}

.item-action {
  flex-shrink: 0;
}
```

```html
<div class="item">
  <div class="item-title">
    這是一個很長很長的標題文字
  </div>
  <button class="item-action">操作</button>
</div>
```

這裡 `min-width: 0;` 是關鍵。

如果少了它，`.item-title` 可能不會正常縮小。

## 檔名省略

```css
.file-name {
  max-width: 320px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

```html
<span class="file-name">
  這是一個非常非常長的檔案名稱.pdf
</span>
```

如果想保留副檔名，單純省略號不一定夠用，可能需要更進階的結構拆分。

例如：

```html
<span class="file">
  <span class="file-main">這是一個非常非常長的檔案名稱</span>
  <span class="file-ext">.pdf</span>
</span>
```

```css
.file {
  display: inline-flex;
  max-width: 320px;
}

.file-main {
  min-width: 0;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}

.file-ext {
  flex-shrink: 0;
}
```

這樣可以讓主檔名省略，但副檔名保留。

## 麵包屑省略

```css
.breadcrumb {
  display: flex;
  align-items: center;
  gap: 4px;
}

.breadcrumb-item {
  min-width: 0;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}

.breadcrumb-item.current {
  flex: 1;
}
```

```html
<nav class="breadcrumb">
  <span>首頁</span>
  <span>/</span>
  <span class="breadcrumb-item current">
    這是一個很長很長的目前頁面名稱
  </span>
</nav>
```

麵包屑常需要防止目前頁面名稱太長撐爆版面。

## 常見錯誤

### 錯誤一：只寫 `text-overflow: ellipsis;`

錯誤寫法：

```css
.text {
  text-overflow: ellipsis;
}
```

這通常不會出現省略號。

正確寫法：

```css
.text {
  width: 200px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

## 錯誤二：忘記設定寬度限制

錯誤寫法：

```css
.text {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

如果元素沒有寬度限制，可能不會產生溢出。

正確寫法：

```css
.text {
  max-width: 240px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

## 錯誤三：忘記 `white-space: nowrap;`

錯誤寫法：

```css
.text {
  width: 200px;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

文字可能會換行，而不是單行截斷。

正確寫法：

```css
.text {
  width: 200px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

## 錯誤四：在 Flex 裡省略號失效，卻不知道要補 `min-width: 0`

錯誤寫法：

```css
.item {
  display: flex;
}

.title {
  flex: 1;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

可能失效。

建議補上：

```css
.title {
  flex: 1;
  min-width: 0;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

## 錯誤五：普通 `span` 設寬度但沒效果

錯誤寫法：

```css
span {
  width: 200px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

普通 inline 元素寬度不容易生效。

修正：

```css
span {
  display: inline-block;
  max-width: 200px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

## 錯誤六：把多行省略號當成 `text-overflow` 單獨能解決

錯誤理解：

```css
.text {
  text-overflow: ellipsis;
}
```

就能做多行省略。

多行省略通常要用：

```css
.text {
  display: -webkit-box;
  -webkit-line-clamp: 3;
  -webkit-box-orient: vertical;
  overflow: hidden;
}
```

## 錯誤七：所有長文字都用省略號

省略號會隱藏部分內容。

如果內容很重要，例如：

- 錯誤訊息
- 法規條文
- 使用者必須閱讀的說明
- 表單驗證原因
- 金融資料
- 系統警告

不建議只用省略號截斷。

應提供完整查看方式，例如 Tooltip、展開更多、詳情頁。

## 錯誤八：把文字爆版問題都交給 `text-overflow`

如果問題是長網址或長英文沒有斷點，可能應該使用：

```css
overflow-wrap: break-word;
```

或：

```css
overflow-wrap: anywhere;
```

而不是直接省略。

要先判斷需求：

- 要保留完整內容並換行：用斷行方案
- 要限制空間並截斷內容：用省略號方案

## 實務範例：後台列表標題

```css
.list-item {
  display: flex;
  align-items: center;
  gap: 8px;
}

.list-title {
  flex: 1;
  min-width: 0;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}

.list-status {
  flex-shrink: 0;
}
```

```html
<div class="list-item">
  <div class="list-title">
    這是一筆很長很長的資料標題
  </div>
  <span class="list-status">處理中</span>
</div>
```

這種寫法適合列表項目，右側狀態固定，左側標題可省略。

## 實務範例：文章卡片

```css
.article-card {
  width: 280px;
}

.article-card-title {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}

.article-card-desc {

  -webkit-line-clamp: 2;

  line-height: 1.6;
```

```html
<div class="article-card">
  <h3 class="article-card-title">
    這是一個很長很長的文章標題
  </h3>

  <p class="article-card-desc">
    這是一段文章摘要，最多顯示兩行，超出之後會被截斷。
  </p>
</div>
```

標題通常做單行省略，摘要通常做多行省略。

## 實務範例：表格檔名欄位

```css
.file-table {
```

```html
<table class="file-table">
```

適合後台報表、文件管理、附件清單。

## 實務範例：Tooltip 顯示完整內容

```css
.text-ellipsis {

  overflow: hidden;
  text-overflow: ellipsis;
}
```

```html
<div
  class="text-ellipsis"
```

簡單情況下可以用 `title` 顯示完整內容。

## 面試可能問法

### 問題一：`text-overflow` 是做什麼的？

`text-overflow` 用來設定文字超出容器時的呈現方式。

常見值有：

```css
text-overflow: clip;
text-overflow: ellipsis;
```

其中 `ellipsis` 會讓超出的文字用省略號表示。

## 問題二：為什麼只寫 `text-overflow: ellipsis;` 沒有效？

因為 `text-overflow` 只控制溢出後的顯示方式。

它還需要搭配：

```css
width: 200px;
white-space: nowrap;
overflow: hidden;
text-overflow: ellipsis;
```

也就是要有寬度限制、強制單行、隱藏超出內容，省略號才會正常出現。

## 問題三：單行省略號怎麼寫？

```css
.ellipsis {
  width: 200px;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

如果是 `span`，可能還要加：

```css
display: inline-block;
```

如果在 Flex 中，可能還要加：

```css
min-width: 0;
```

## 問題四：多行省略號怎麼寫？

常見寫法：

```css
.multi-ellipsis {
  display: -webkit-box;
  -webkit-line-clamp: 3;
  -webkit-box-orient: vertical;
  overflow: hidden;
}
```

這表示最多顯示三行，超出後截斷。

## 問題五：Flex 裡面省略號失效怎麼辦？

常見原因是 Flex item 不願意縮小。

可以在需要省略的元素上加：

```css
min-width: 0;
```

完整寫法：

```css
.title {
  flex: 1;
  min-width: 0;
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

## 問題六：`text-overflow` 和 `overflow-wrap` 差在哪？

`text-overflow` 是截斷文字並用省略號提示。

`overflow-wrap` 是允許長單字或長網址在必要時斷行。

如果你要省略：

```css
text-overflow: ellipsis;
```

如果你要避免長網址爆版：

```css
overflow-wrap: break-word;
```

## 問題七：表格中如何讓文字省略？

常見寫法：

```css
table {
  width: 100%;
  table-layout: fixed;
}

td {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
```

`table-layout: fixed;` 可以讓欄寬更穩定，避免內容把表格撐開。

## 一句話總結

`text-overflow` 用來控制文字溢出後的顯示方式，最常見是 `ellipsis`；但它必須搭配寬度限制、`overflow: hidden` 和 `white-space: nowrap` 才能做出單行省略號，多行省略號則通常使用 `-webkit-line-clamp`，在 Flex、Grid、表格中還要特別注意容器能否被壓縮。
