# 第六章 選擇器

## 章節定位

本章主要整理 CSS 選擇器的基本概念、常見用法與實務回查重點。

CSS 選擇器的核心作用是：

```text
先找到要套用樣式的 HTML 元素，再對這些元素設定外觀。
```

也就是說，選擇器負責「找誰」，CSS 宣告負責「改成什麼樣子」。

本章會先建立選擇器的基本心智模型，再依序學習基礎選擇器與複合選擇器。

## 本章學習目標

讀完本章後，你應該能理解：

- CSS 選擇器在 CSS 規則中扮演什麼角色
- 基礎選擇器有哪些，以及各自適合什麼情境
- class 和 id 的差異
- 後代選擇器和子選擇器的差異
- 相鄰兄弟選擇器和通用兄弟選擇器的差異
- 屬性選擇器、偽類選擇器、偽元素選擇器的常見用法
- `nth-child` 和 `nth-of-type` 的差異
- 實務上如何避免選擇器寫得太深、太脆弱、太難維護

## 常見問題索引

## Q1：為什麼我的 CSS 沒有效果？

優先檢查：

1. 選擇器有沒有寫錯
2. HTML 中有沒有符合條件的元素
3. class 名稱有沒有拼錯
4. `.` 和 `#` 有沒有用錯
5. 是否被其他 CSS 規則覆蓋

建議回查：

- [選擇器的作用](./選擇器的作用.md)
- [基礎選擇器](./基礎選擇器.md)

## Q2：`.box` 和 `#box` 差在哪裡？

`.box` 是 class 選擇器。

```css
.box {
  color: red;
}
```

對應：

```html
<div class="box">內容</div>
```

`#box` 是 id 選擇器。

```css
#box {
  color: red;
}
```

對應：

```html
<div id="box">內容</div>
```

建議回查：

- [基礎選擇器](./基礎選擇器.md)

## Q3：空格和 `>` 差在哪裡？

空格代表後代選擇器。

```css
.nav a {
  color: red;
}
```

會選中 `.nav` 裡面所有層級的 `a`。

`>` 代表子選擇器。

```css
.nav > a {
  color: red;
}
```

只會選中 `.nav` 的直接子層 `a`。

建議回查：

- [複合選擇器](./複合選擇器.md)

## Q4：`+` 和 `~` 差在哪裡？

`+` 只選緊接在後面的第一個兄弟元素。

```css
h2 + p {
  color: red;
}
```

`~` 會選後面所有符合條件的兄弟元素。

```css
h2 ~ p {
  color: red;
}
```

建議回查：

- [複合選擇器](./複合選擇器.md)

## Q5：`:nth-child` 和 `:nth-of-type` 差在哪裡？

簡單記法：

```text
nth-child：看所有子元素的第幾個。
nth-of-type：看同類型元素的第幾個。
```

建議回查：

- [複合選擇器](./複合選擇器.md)

## Q6：`::before` 和 `::after` 為什麼沒出現？

通常是因為忘記寫 `content`。

```css
.box::before {
  content: "";
}
```

建議回查：

- [複合選擇器](./複合選擇器.md)

## 實務學習建議

學選擇器時，不建議只背語法。

每學一個選擇器，都應該問自己四個問題：

```text
1. 它會選中誰？
2. 它不會選中誰？
3. 它適合用在什麼情境？
4. 它容易和哪個選擇器搞混？
```

例如學到：

```css
.card > .title {
  color: red;
}
```

你應該能說出：

```text
它會選中 .card 的直接子層 .title。
它不會選中更深層的 .title。
它適合用在只想控制第一層結構的情境。
它容易和 .card .title 搞混。
```

## 實務開發提醒

## 1. 優先使用 class 作為樣式主體

實務上最常用的是 class 選擇器。

例如：

```css
.card {
  padding: 16px;
}

.card-title {
  font-size: 20px;
}
```

原因是 class：

- 可重複使用
- 適合元件化
- 比 id 更容易覆蓋
- 比深層結構選擇器更好維護

## 2. 避免選擇器寫得太深

不建議：

```css
.page .main .content .card .header .title span {
  color: red;
}
```

比較建議：

```css
.card-title {
  color: red;
}
```

或：

```css
.card .title {
  color: red;
}
```

選擇器越深，越依賴 HTML 結構，也越容易壞掉。

## 3. 不要濫用 id 寫樣式

id 選擇器優先級比較高。

如果大量用 id 寫 CSS，後續要覆蓋樣式會比較麻煩。

比較建議：

```css
.form-input {
  border: 1px solid #ccc;
}
```

而不是：

```css
#formInput {
  border: 1px solid #ccc;
}
```

## 4. 關係選擇器要注意影響範圍

例如：

```css
.article p {
  line-height: 1.8;
}
```

這會影響 `.article` 裡面所有層級的 `p`。

如果只想選第一層段落，應該寫：

```css
.article > p {
  line-height: 1.8;
}
```

## 5. 偽類可以處理很多互動狀態

很多簡單互動不需要 JavaScript。

例如：

```css
.button:hover {
  opacity: 0.8;
}

.input:focus {
  border-color: blue;
}

.checkbox:checked + .label {
  color: red;
}
```

這些都可以直接用 CSS 處理。

## 建議練習方式

學完本章後，可以用下面方式練習：

## 練習 1：判斷會選中誰

看到一段 CSS 和 HTML，先不要急著看答案。

先自己判斷：

```text
這個選擇器會選中哪些元素？
哪些元素不會被選中？
```

## 練習 2：比較相似選擇器

特別練習下面幾組：

```text
空格 vs >
+ vs ~
.class vs #id
:nth-child vs :nth-of-type
:偽類 vs ::偽元素
[class="box"] vs .box
```

## 練習 3：改寫太深的選擇器

例如把：

```css
.page .main .content .card .header .title {
  color: red;
}
```

改成：

```css
.card-title {
  color: red;
}
```

或更符合元件語意的 class 命名。

CSS 選擇器的作用是找到要套用樣式的元素；學習選擇器時，重點不只是背語法，而是要能判斷「會選中誰、不會選中誰、適合什麼情境、是否容易維護」。
