# 選擇器的作用

## 本節導讀

CSS 選擇器的核心作用，是**把想要套用樣式的 HTML 元素找出來**。

也就是說，CSS 在設定樣式之前，必須先知道：

> 這一組樣式到底要作用在哪些元素身上？

因此，選擇器可以理解成 CSS 的「查找條件」。

先用選擇器找到目標元素，再對這些元素設定外觀，這就是 CSS 選擇器最基本的用途。

## CSS 規則的基本結構

一段 CSS 通常可以分成兩個部分：

```css
選擇器 {
  屬性: 屬性值;
}
```

例如：

```css
p {
  color: red;
}
```

這段 CSS 的意思是：

```text
找到所有 p 標籤，然後把文字顏色設定成紅色。
```

其中：

```css
p
```

就是選擇器。

```css
color: red;
```

就是樣式宣告。

## 選擇器在做什麼？

選擇器就是根據條件，把符合條件的 HTML 元素選出來。

可以把它理解成兩個步驟：

1. 找到要處理的元素
2. 對這些元素套用樣式

例如：

```css
.title {
  color: blue;
}
```

這段 CSS 的意思是：

```text
找到所有 class 名稱包含 title 的元素，然後把文字顏色設定成藍色。
```

所以選擇器負責的是「找誰」，樣式宣告負責的是「改成什麼樣子」。

## 用生活例子理解

可以把 HTML 頁面想像成一間教室，裡面有很多學生。

CSS 樣式像是老師要下指令，例如：

```text
請穿紅色衣服。
```

但是老師不能只說「請穿紅色衣服」，還要先說清楚：

```text
第一排的學生，請穿紅色衣服。
```

這裡的「第一排的學生」就像 CSS 選擇器。

它負責先找到目標對象，後面的指令才知道要套用到誰身上。

## 選擇器和樣式宣告的關係

選擇器和樣式宣告的關係可以這樣理解：

```text
選擇器：決定誰會被影響
樣式宣告：決定被影響後長什麼樣子
```

例如：

```css
.box {
  width: 100px;
  height: 100px;
  background-color: pink;
}
```

這段可以拆成：

```text
.box：
找到 class 名稱包含 box 的元素

width: 100px;
height: 100px;
background-color: pink;
把這些元素設定成寬高 100px，背景色粉紅色
```

## 選擇器不是在「建立元素」

選擇器只負責選中已經存在的 HTML 元素。

它不會建立新的 HTML 標籤。

例如：

```css
.card {
  border: 1px solid #ccc;
}
```

這段 CSS 不會自動產生一個 `.card` 元素。

它只會去找 HTML 裡面已經存在的元素：

```html
<div class="card">內容</div>
```

然後把樣式套用上去。

## 選擇器和 HTML 的關係

CSS 選擇器能不能生效，取決於 HTML 裡有沒有符合條件的元素。

例如：

```css
.warning {
  color: red;
}
```

如果 HTML 是這樣：

```html
<p class="warning">注意事項</p>
```

那這個 `p` 會變成紅色。

但如果 HTML 是這樣：

```html
<p class="notice">注意事項</p>
```

那 `.warning` 就選不到任何元素，樣式也不會生效。

## 與選擇器分類的關係

CSS 選擇器可以先分成兩大類：

- 基礎選擇器
- 複合選擇器

## 基礎選擇器

基礎選擇器是用最單純的條件來選元素。

常見的基礎選擇器包括：

- 標籤選擇器
- 類選擇器
- id 選擇器
- 通配符選擇器

例如：

```css
p {
  color: red;
}

.title {
  color: blue;
}

#main {
  width: 100%;
}
```

這些都是用單一條件來選元素。

## 複合選擇器

複合選擇器是在基礎選擇器的基礎上，再透過元素關係、屬性、狀態或結構來更精準地選取元素。

例如：

```css
.nav li a {
  color: orange;
}

.card > .title {
  font-size: 20px;
}

input[type="text"] {
  border: 1px solid #ccc;
}
```

這些選擇器不只是單純找某一種元素，而是加入了更多條件。

例如：

```css
.nav li a
```

意思是：

```text
找到 class 為 nav 的元素裡面，li 裡面的所有 a 標籤。
```

## 常見混淆點

### 1. 選擇器不是樣式本身

選擇器只負責找元素。

真正改變外觀的是大括號 `{}` 裡面的樣式宣告。

```css
p {
  color: red;
}
```

其中：

```text
p 是選擇器
color: red; 是樣式宣告
```

### 2. 選擇器選不到元素時，樣式不會生效

例如：

```css
.box {
  color: red;
}
```

如果 HTML 裡面沒有任何元素使用 `class="box"`，這段 CSS 就不會影響任何元素。

### 3. 不同選擇器可能選到同一個元素

例如：

```css
p {
  color: red;
}

.text {
  font-size: 20px;
}
```

```html
<p class="text">內容</p>
```

這個 `p` 同時被 `p` 和 `.text` 選中，所以它會同時套用兩組樣式。

### 4. 多個選擇器選中同一個元素時，可能會有優先級問題

例如：

```css
p {
  color: red;
}

.text {
  color: blue;
}
```

```html
<p class="text">內容</p>
```

這時候同一個元素同時被 `p` 和 `.text` 選中，而且兩者都設定了 `color`。

這種情況就會牽涉到 CSS 的選擇器優先級。

這部分會在後續筆記中再深入整理。

## 實務上為什麼要學選擇器？

在實際開發中，選擇器會影響：

- 樣式能不能正確套用
- 樣式會不會影響到不該影響的元素
- CSS 是否容易維護
- 組件樣式是否容易覆蓋
- 頁面結構改變時，樣式是否容易壞掉

例如：

```css
div p span a {
  color: red;
}
```

這種選擇器雖然可以選到目標，但層級太深，後續 HTML 結構只要稍微改變，樣式就可能失效。

所以學選擇器不只是學語法，更重要的是學會：

```text
如何用清楚、精準、可維護的方式找到元素。
```

