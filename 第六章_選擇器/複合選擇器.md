## 本節導讀

複合選擇器是在基礎選擇器之上，透過元素之間的關係、屬性、狀態或結構來選取目標。  
和基礎選擇器相比，它更精準，也更常用來處理實務上的頁面樣式。

## 關鍵字

- 複合選擇器
- 後代選擇器
- 子選擇器
- 相鄰兄弟選擇器

- 群組選擇器
- 屬性選擇器
- 偽類選擇器
- 偽元素選擇器

## 什麼是複合選擇器

複合選擇器不是單靠一個最基礎的條件來選元素，而是把元素之間的關係或元素本身的特徵組合起來使用。

常見類型包括：

- 後代選擇器
- 子選擇器
- 相鄰兄弟選擇器
- 通用兄弟選擇器
- 群組選擇器
- 屬性選擇器
- 偽類選擇器
- 偽元素選擇器

## 後代選擇器

後代選擇器用空格表示。

範例：[demo/後代選擇器.html](./demo/後代選擇器.html)

```css
父元素 子元素 {
  屬性: 值;
}
```

它會選中某個元素裡面的所有後代元素，不論中間隔了幾層。

### 例子

```css
ol li {
  color: pink;
}

ol li a {
  color: red;
}

.nav li a {
  color: orange;
}
```

```html
<ol>
  <li>ol 的子元素</li>
  <li>ol 的子元素</li>
  <li>ol 的子元素</li>
  <li><a href="#">連結</a></li>
</ol>

<ul>
  <li>ul 的子元素</li>
  <li>ul 的子元素</li>
  <li>ul 的子元素</li>
  <li><a href="#">連結</a></li>
</ul>

<ul class="nav">
  <li>ul 的子元素</li>
  <li>ul 的子元素</li>
  <li>ul 的子元素</li>
  <li><a href="#">連結</a></li>
  <li><a href="#">連結</a></li>
</ul>
```

### 重點

- 空格代表後代關係
- 只要是後代都能選到
- 選取範圍比子選擇器更大

## 子選擇器

子選擇器用 `>` 表示。

範例：[demo/子選擇器.html](./demo/子選擇器.html)

```css
父元素 > 子元素 {
  屬性: 值;
}
```

它只選中直接子元素，不會往更深一層找。

### 例子

```css
div > p {
  color: pink;
}

.nav > a {
  color: red;
}
```

```html
<div class="nav">
  <a href="#">直接子元素</a>
  <p>
    <a href="#">不是直接子元素</a>
  </p>
</div>
```

### 重點

- `>` 表示直接子層
- 只能選到第一層子元素
- 比後代選擇器更精確

## 相鄰兄弟選擇器

相鄰兄弟選擇器用 `+` 表示。

範例：[demo/相鄰兄弟選擇器.html](./demo/相鄰兄弟選擇器.html)

```css
元素1 + 元素2 {
  屬性: 值;
}
```

它會選中緊接在前一個元素後面的第一個同層兄弟元素。

### 例子

```css
h1 + p {
  margin-top: 50px;
  color: red;
}

li + li {
  color: red;
}
```

```html
<p>This is paragraph.</p>
<h1>This is a heading.</h1>
<p>This is paragraph.</p>
<p>This is paragraph.</p>

<div>
  <ul>
    <li>List item 1</li>
    <li>List item 2</li>
    <li>List item 3</li>
  </ul>
</div>
```

### 重點

- `+` 只選緊接著的下一個兄弟元素
- 兩個元素必須同層
- 常用來處理相鄰項目的樣式差異

## 通用兄弟選擇器

通用兄弟選擇器用 `~` 表示。

範例：[demo/通用兄弟選擇器.html](./demo/通用兄弟選擇器.html)

```css
元素1 ~ 元素2 {
  屬性: 值;
}
```

它會選中同一層級中，位於前一個元素之後的所有兄弟元素。

### 例子

```css
h1 ~ p {
  color: red;
}
```

```html
<p>1</p>
<h1>2</h1>
<p>3</p>
<p>4</p>
<p>5</p>
```

### 重點

- `~` 也是同層關係
- 會選中後面所有兄弟元素
- 比相鄰兄弟選擇器範圍更大

## 群組選擇器

群組選擇器用逗號 `,` 分隔。

範例：[demo/群組選擇器.html](./demo/群組選擇器.html)

```css
選擇器1, 選擇器2 {
  屬性: 值;
}
```

它的意思不是「合併關係」，而是「這幾個選擇器共用同一組樣式」。

### 例子

```css
ul, div {
  color: pink;
}

div, p, .pig li {
  color: pink;
}
```

```html
<div>內容</div>

<p>內容</p>

<span>不受影響</span>

<ul class="pig">
  <li>內容</li>
  <li>內容</li>
  <li>內容</li>
</ul>
```

### 重點

- 逗號表示分組，不表示層級
- 選擇器彼此之間沒有父子或兄弟關係
- 適合共用相同樣式

## 屬性選擇器

屬性選擇器是根據元素是否具有某個屬性，或屬性的值是否符合條件來選取元素。

範例：[demo/屬性選擇器.html](./demo/屬性選擇器.html)

### 基本寫法

```css
元素[屬性] {
  屬性: 值;
}
```

### 常見寫法

```css
input[value] {
  color: pink;
}

input[type=text] {
  color: pink;
}

div[class^=icon] {
  color: pink;
}

section[class$=data] {
  color: pink;
}
```

### 例子

```html
<input type="text" value="請輸入內容">
<input type="text">

<input type="text" name="" id="">
<input type="password" name="" id="">

<div class="icon1">內容</div>
<div class="icon2">內容</div>
<div class="icon3">內容</div>
<div class="icon4">內容</div>

<section class="icon1-data">1</section>
<section class="icon2-data">2</section>
<section class="icon3-data">3</section>
```

### 重點

- `[attr]`：只要有這個屬性就選中
- `[attr=value]`：屬性值必須完全相同
- `[attr^=value]`：以指定字串開頭
- `[attr$=value]`：以指定字串結尾

## 偽類選擇器

偽類選擇器用來表示元素的某種狀態，或元素在結構中的某個位置。

範例：[demo/偽類選擇器.html](./demo/偽類選擇器.html)

### 常見狀態偽類

```css
a:link {
  color: #333;
  text-decoration: none;
}

a:visited {
  color: orange;
}

a:hover {
  color: skyblue;
}

a:active {
  color: green;
}

input:focus {
  background-color: yellow;
}
```

### 說明

- `:link`：尚未被點擊過的連結
- `:visited`：已經訪問過的連結
- `:hover`：滑鼠移到元素上
- `:active`：元素被按下時
- `:focus`：元素取得焦點時，常見於 `input`

### 順序提醒

連結相關的偽類常用 `LVHA` 順序：

1. `:link`
2. `:visited`
3. `:hover`
4. `:active`

### 結構偽類

```css
ul li:first-child {
  background-color: pink;
}

ul li:last-child {
  background-color: pink;
}

ul li:nth-child(2) {
  background-color: pink;
}

ul li:nth-child(even) {
  background-color: #ccc;
}

ul li:nth-child(odd) {
  background-color: gray;
}

ul li:first-of-type {
  background-color: pink;
}

ul li:last-of-type {
  background-color: pink;
}

ul li:nth-last-child(2) {
  background-color: purple;
}
```

### `first-child`

`first-child` 會選中某個父元素裡的第一個子元素。

```css
ul li:first-child {
  background-color: pink;
}
```

重點：

- 它看的是「父元素中的第一個孩子」
- 不會只看 `li`，而是先看是不是整個第一個子元素

### `last-child`

`last-child` 會選中某個父元素裡的最後一個子元素。

```css
ul li:last-child {
  background-color: pink;
}
```

重點：

- 它看的是「父元素中的最後一個孩子」
- 只要符合最後一個子元素，就會被選中

### `nth-child(n)`

`nth-child(n)` 會根據元素在父元素中的順序來選取第 `n` 個子元素。

```css
ul li:nth-child(2) {
  background-color: pink;
}

ul li:nth-child(even) {
  background-color: #ccc;
}

ul li:nth-child(odd) {
  background-color: gray;
}
```

重點：

- `n` 代表位置序號
- `nth-child(2)` 就是選第 2 個
- `even` 表示偶數位置
- `odd` 表示奇數位置
- `n+3` 表示從第 3 個開始往後選
- `-n+3` 表示選前 3 個

### `first-of-type`

`first-of-type` 會選中同類型元素中的第一個。

```css
ul li:first-of-type {
  background-color: pink;
}
```

重點：

- 它只看同一種類型的元素
- 不管前面混了其他標籤，只要在同類型裡是第一個就會被選中

### `last-of-type`

`last-of-type` 會選中同類型元素中的最後一個。

```css
ul li:last-of-type {
  background-color: pink;
}
```

重點：

- 它只看同一種類型的元素
- 在同類型中排最後的元素會被選中

### `nth-last-child(n)`

`nth-last-child(n)` 是從最後一個子元素往前數，第 `n` 個就會被選中。

```css
ul li:nth-last-child(2) {
  background-color: purple;
}
```

重點：

- 它是「倒著數」
- `nth-last-child(1)` 就是最後一個
- `nth-last-child(2)` 就是倒數第 2 個

### `nth-child` 和 `nth-of-type` 的差別

```css
section div:nth-child(1) {
  background-color: red;
}

section div:nth-of-type(1) {
  background-color: blue;
}
```

```html
<section>
  <p>內容</p>
  <div>內容</div>
  <div>內容</div>
</section>
```

- `nth-child` 是在所有子元素中計算順序
- `nth-of-type` 是在同類型元素中計算順序
- 如果前面混了其他標籤，兩者結果可能不同

### 快速區分

- 想看父元素的第幾個孩子，用 `nth-child`
- 想看同類型元素中的第幾個，用 `nth-of-type`
- 想看第一個或最後一個孩子，用 `first-child` 和 `last-child`
- 想看同類型的第一個或最後一個，用 `first-of-type` 和 `last-of-type`
- 想倒著數，就用 `nth-last-child`

## 偽元素選擇器

偽元素不是實際存在的 HTML 標籤，而是由 CSS 創造出來的效果。

範例：[demo/偽元素選擇器.html](./demo/偽元素選擇器.html)

### 常見偽元素

```css
p::first-letter {
  font-size: 30px;
  color: blueviolet;
}

p::first-line {
  color: blue;
}

p::selection {
  color: aqua;
}
```

### `::before` 和 `::after`

`::before` 和 `::after` 常用來在元素前後插入裝飾性內容。

```css
div {
  width: 200px;
  height: 200px;
  background-color: pink;
}

div::before {
  content: '';
}

div::after {
  content: '補充文字';
}
```

```html
<div>內容</div>
```

### 重點

- `::before` 和 `::after` 必須搭配 `content`
- 這兩個偽元素常用來做圖示、裝飾、清除浮動等效果
- 它們不是額外的 HTML 標籤

## 常見混淆點

- 後代選擇器是空格，不是 `>`
- 子選擇器只選直接子元素
- `+` 只選緊接的下一個兄弟元素
- `~` 會選後面所有兄弟元素
- 群組選擇器用逗號，不表示層級
- `nth-child` 和 `nth-of-type` 的計算方式不同
- `::before`、`::after` 一定要有 `content`

## 一句話總結

複合選擇器就是用元素關係、屬性、狀態和結構來精準選取目標，比基礎選擇器更靈活，也更常用。
