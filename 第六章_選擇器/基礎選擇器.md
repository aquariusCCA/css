## 本節導讀

基礎選擇器是最常見、最基礎的 CSS 選擇器。  
它們的共同點是：都只靠單一條件來選出元素，適合用來建立最基本的樣式規則。

## 關鍵字

- 基礎選擇器
- 標籤選擇器
- 類選擇器
- id 選擇器
- 通配符選擇器
- class
- selector

## 什麼是基礎選擇器

基礎選擇器是由單個選擇器組成的。

常見的基礎選擇器包括：

- 標籤選擇器
- 類選擇器
- id 選擇器
- 通配符選擇器

## 標籤選擇器

標籤選擇器是用 HTML 標籤名稱作為選擇器，根據標籤名稱統一指定樣式。

### 作用

- 可以一次選中同一種標籤
- 適合替頁面中同類元素設定共通樣式

### 優點

- 能快速統一同類標籤的樣式

### 缺點

- 不能做細部差異化設定
- 一旦選中某種標籤，會影響同類型的所有元素

### 寫法

```css
標籤名 {
  屬性名: 屬性值;
}
```

### 例子

```css
p {
  color: green;
}

div {
  color: pink;
}
```

```html
<p>男生</p>
<p>男生</p>
<p>男生</p>
<div>女生</div>
<div>女生</div>
<div>女生</div>
```

## 類選擇器

如果想針對某一類元素做差異化選擇，可以使用類選擇器。

類選擇器在 HTML 中用 `class` 屬性表示，在 CSS 中用 `.` 來表示。

### 作用

- 可以選出某一個元素
- 也可以選出某幾個元素
- 適合做重複使用的樣式分類

### 命名注意

- 類名可以由數字、字母、下劃線、中劃線組成
- 長名稱或詞組可以用中橫線連接
- 不要用純數字或中文命名
- 類名最好有意義，看到就能理解用途

### 寫法

```css
.類名 {
  屬性名: 屬性值;
}
```

### 例子

```css
.red {
  color: red;
}

.star-sing {
  color: green;
}

.memeda {
  color: pink;
}
```

```html
<ul>
  <li class="red">冰雨</li>
  <li class="red">來生緣</li>
  <li>李香蘭</li>
  <li class="memeda">生僻字</li>
  <li class="star-sing">江南style</li>
</ul>

<div class="red">我也想變紅色</div>
```

## 多類名

一個標籤可以同時擁有多個類名。

### 作用

- 把共通樣式抽成一個類
- 再搭配元素自己的類名
- 方便重用，也方便統一修改

### 注意點

- 多個類名寫在同一個 `class` 屬性中
- 類名之間要用空格隔開
- 這個元素會同時套用這些類名對應的樣式

### 例子

```css
.red {
  color: red;
}

.font35 {
  font-size: 35px;
}
```

```html
<div class="red font35">Kevin</div>
```

## id 選擇器

id 選擇器是用 HTML 元素的 `id` 屬性來指定樣式。

### 作用

- 為特定的唯一元素指定樣式
- 常用在頁面中只會出現一次的元素上

### 命名與使用

- HTML 用 `id` 屬性設定
- CSS 中以 `#` 表示
- 一個 id 在同一份 HTML 文件中應該只出現一次

### 和類選擇器的差別

- 類選擇器像名字，一個元素可以有多個類名
- id 選擇器像身分證，一個頁面中應該唯一
- 類選擇器通常更常用
- id 選擇器常和 JavaScript 搭配使用

### 例子

```css
#pink {
  color: pink;
}
```

```html
<div id="pink">蕭士麟</div>
```

## 通配符選擇器

通配符選擇器使用 `*` 表示，會選中頁面中的所有元素。

### 作用

- 套用到所有標籤
- 不需要手動逐一呼叫

### 寫法

```css
* {
  屬性1: 屬性值1;
}
```

### 例子

```css
* {
  color: red;
}
```

```html
<div>我的</div>
<span>我的</span>
<ul>
  <li>還是我的</li>
</ul>
```

### 常見用途

- 統一設定頁面樣式
- 清除元素預設的內外邊距

```css
/* 清除內外邊距 */
* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}
```

## 一句話總結

基礎選擇器就是用最單純的方式把元素選出來，再對它們套用樣式。

## 什麼是基礎選擇器？

基礎選擇器是由單一條件組成的選擇器。

常見的基礎選擇器包括：

| 選擇器 | 寫法 | 選取條件 | 常見用途 |
|---|---|---|---|
| 標籤選擇器 | `p` | 選中指定標籤 | 統一設定同類標籤樣式 |
| 類選擇器 | `.box` | 選中指定 class | 最常用，適合重複使用樣式 |
| id 選擇器 | `#main` | 選中指定 id | 適合唯一元素，較少大量用於 CSS |
| 通配符選擇器 | `*` | 選中所有元素 | 常用於 reset 或基礎初始化 |

## 標籤選擇器

標籤選擇器是用 HTML 標籤名稱作為選擇器。

它會選中頁面中所有指定的標籤。

## 一句話理解

> 標籤選擇器會根據 HTML 標籤名稱選元素。

## 語法

```css
標籤名 {
  屬性名: 屬性值;
}
```

例如：

```css
p {
  color: green;
}
```

這段 CSS 的意思是：

```text
選中頁面中所有 p 標籤，並把文字顏色設定成綠色。
```

## 範例

```css
p {
  color: green;
}

div {
  color: pink;
}
```

```html
<p>男生</p>
<p>男生</p>
<p>男生</p>

<div>女生</div>
<div>女生</div>
<div>女生</div>
```

## 會選中誰？

```css
p {
  color: green;
}
```

會選中：

```html
<p>男生</p>
<p>男生</p>
<p>男生</p>
```

因為它們都是 `p` 標籤。

```css
div {
  color: pink;
}
```

會選中：

```html
<div>女生</div>
<div>女生</div>
<div>女生</div>
```

因為它們都是 `div` 標籤。

## 不會選中誰？

```css
p {
  color: green;
}
```

不會選中：

```html
<div>女生</div>
<span>其他文字</span>
```

因為它們不是 `p` 標籤。

## 適合使用的情境

標籤選擇器適合用在：

- 統一設定某一類標籤的基礎樣式
- 設定文章段落、標題、列表等預設樣式
- 做全站基礎排版

例如：

```css
body {
  font-family: Arial, sans-serif;
}

p {
  line-height: 1.8;
}

a {
  text-decoration: none;
}
```

## 優點

- 寫法簡單
- 可以快速統一同類標籤的樣式
- 適合設定基礎排版

## 缺點

- 範圍比較大
- 容易影響所有同類型標籤
- 不適合做細部差異化設定

## 常見錯誤

### 錯誤：以為標籤選擇器只會影響某一個元素

```css
p {
  color: red;
}
```

這段不是只影響其中一個 `p`。

它會影響頁面中所有 `p` 標籤。

如果只想改其中一部分，應該使用 class。

```html
<p class="warning">注意事項</p>
```

```css
.warning {
  color: red;
}
```

## 類選擇器

類選擇器是根據 HTML 元素上的 `class` 屬性來選取元素。

它是實務開發中最常用的 CSS 選擇器。

## 一句話理解

> 類選擇器會選中所有擁有指定 class 名稱的元素。

## 語法

HTML 中使用 `class`：

```html
<div class="box">內容</div>
```

CSS 中使用 `.`：

```css
.box {
  屬性名: 屬性值;
}
```

## 範例

```css
.red {
  color: red;
}

.star-sing {
  color: green;
}

.memeda {
  color: pink;
}
```

```html
<ul>
  <li class="red">冰雨</li>
  <li class="red">來生緣</li>
  <li>李香蘭</li>
  <li class="memeda">生僻字</li>
  <li class="star-sing">江南style</li>
</ul>

<div class="red">我也想變紅色</div>
```

## 會選中誰？

```css
.red {
  color: red;
}
```

會選中所有 `class` 中包含 `red` 的元素：

```html
<li class="red">冰雨</li>
<li class="red">來生緣</li>
<div class="red">我也想變紅色</div>
```

## 不會選中誰？

```css
.red {
  color: red;
}
```

不會選中：

```html
<li>李香蘭</li>
<li class="memeda">生僻字</li>
<li class="star-sing">江南style</li>
```

因為它們的 `class` 中沒有 `red`。

## 類選擇器的重點

類選擇器不是只能選一個元素。

如果只有一個元素使用這個 class，看起來就像只選到一個元素。

如果多個元素使用同一個 class，它們都會被選中。

例如：

```html
<p class="note">第一段</p>
<p class="note">第二段</p>
<p class="note">第三段</p>
```

```css
.note {
  color: blue;
}
```

這三個 `p` 都會變成藍色。

## 適合使用的情境

類選擇器適合用在：

- 可重複使用的樣式
- 元件樣式
- 頁面局部區塊
- 狀態樣式
- 工具類樣式

例如：

```css
.card {
  border: 1px solid #ddd;
  border-radius: 8px;
}

.error-text {
  color: red;
}

.text-center {
  text-align: center;
}
```

## 命名注意

class 名稱建議：

- 使用有意義的英文命名
- 長名稱可以用中橫線 `-` 連接
- 不建議使用純數字開頭
- 不建議使用中文命名
- 不建議命名過度依賴外觀，除非它是工具類

例如，較推薦：

```html
<p class="error-message">錯誤訊息</p>
```

不太推薦：

```html
<p class="red">錯誤訊息</p>
```

原因是：

```text
red 只描述顏色。
error-message 描述這個元素的語意。
```

如果未來錯誤訊息不再是紅色，`error-message` 仍然合理，但 `red` 就會變得不準確。

不過，如果你是在寫工具類，例如 Tailwind CSS 的概念，像 `text-red`、`text-center` 這類名稱就可以接受，因為它本來就是以功能或樣式為目的。

## 常見錯誤

### 錯誤 1：CSS 忘記加 `.`

錯誤寫法：

```css
red {
  color: red;
}
```

正確寫法：

```css
.red {
  color: red;
}
```

HTML：

```html
<p class="red">文字</p>
```

說明：

```text
.red 才是 class 選擇器。
red 會被當成標籤選擇器。
```

### 錯誤 2：HTML 中 class 加了 `.`

錯誤寫法：

```html
<p class=".red">文字</p>
```

正確寫法：

```html
<p class="red">文字</p>
```

說明：

```text
CSS 裡面要寫 .red。
HTML 的 class 屬性裡只寫 red。
```

## 多類名

一個 HTML 元素可以同時擁有多個 class 名稱。

這種寫法稱為多類名。

## 一句話理解

> 多類名可以讓一個元素同時套用多組 class 樣式。

## 語法

```html
<div class="類名1 類名2 類名3">內容</div>
```

多個 class 名稱之間要用空格隔開。

## 範例

```css
.red {
  color: red;
}

.font35 {
  font-size: 35px;
}
```

```html
<div class="red font35">Kevin</div>
```

這個 `div` 會同時套用：

```css
.red {
  color: red;
}
```

以及：

```css
.font35 {
  font-size: 35px;
}
```

所以結果是：

```text
文字顏色是紅色，字體大小是 35px。
```

## 會選中誰？

```css
.red {
  color: red;
}
```

會選中：

```html
<div class="red font35">Kevin</div>
```

因為它的 class 名稱中包含 `red`。

```css
.font35 {
  font-size: 35px;
}
```

也會選中：

```html
<div class="red font35">Kevin</div>
```

因為它的 class 名稱中包含 `font35`。

## 多類名的用途

多類名適合用來拆分樣式。

例如：

```css
.btn {
  padding: 8px 16px;
  border-radius: 4px;
}

.btn-primary {
  background-color: blue;
  color: white;
}

.large {
  font-size: 20px;
}
```

```html
<button class="btn btn-primary large">送出</button>
```

這樣可以把樣式拆成：

```text
btn：按鈕共通樣式
btn-primary：主要按鈕樣式
large：尺寸樣式
```

## 常見錯誤

### 錯誤 1：類名之間沒有空格

錯誤寫法：

```html
<div class="redfont35">Kevin</div>
```

這不是兩個 class。

這是一個叫做 `redfont35` 的 class。

正確寫法：

```html
<div class="red font35">Kevin</div>
```

### 錯誤 2：以為 class 的先後順序一定決定樣式結果

例如：

```html
<div class="red blue">文字</div>
```

如果 CSS 是：

```css
.red {
  color: red;
}

.blue {
  color: blue;
}
```

結果通常會是藍色。

但這不是因為 HTML 中 `blue` 寫在後面，而是因為 CSS 中 `.blue` 這條規則寫在 `.red` 後面。

也就是說：

```text
同樣優先級時，後出現的 CSS 規則會覆蓋前面的 CSS 規則。
```

## id 選擇器

id 選擇器是根據 HTML 元素上的 `id` 屬性來選取元素。

id 在同一份 HTML 文件中應該是唯一的。

## 一句話理解

> id 選擇器通常用來選中頁面中唯一的元素。

## 語法

HTML 中使用 `id`：

```html
<div id="main">主要內容</div>
```

CSS 中使用 `#`：

```css
#main {
  屬性名: 屬性值;
}
```

## 範例

```css
#pink {
  color: pink;
}
```

```html
<div id="pink">蕭士麟</div>
```

## 會選中誰？

```css
#pink {
  color: pink;
}
```

會選中：

```html
<div id="pink">蕭士麟</div>
```

因為它的 id 是 `pink`。

## 不會選中誰？

```css
#pink {
  color: pink;
}
```

不會選中：

```html
<div class="pink">內容</div>
```

因為 `#pink` 找的是 `id="pink"`，不是 `class="pink"`。

如果要選 class，應該寫：

```css
.pink {
  color: pink;
}
```

## id 選擇器的特點

id 選擇器有幾個重要特點：

- 一個元素只能有一個 `id`
- 同一份 HTML 文件中，同一個 `id` 應該只出現一次
- id 選擇器的優先級比 class 選擇器高
- id 常用於 JavaScript 操作、錨點定位、表單 label 對應等場景
- 實務上不建議大量用 id 來寫 CSS 樣式

## class 和 id 的差別

| 比較項目 | class | id |
|---|---|---|
| HTML 屬性 | `class` | `id` |
| CSS 寫法 | `.box` | `#main` |
| 是否可重複 | 可以重複使用 | 同一頁應該唯一 |
| 一個元素可有幾個 | 可以有多個 class | 只能有一個 id |
| 常見用途 | 樣式重用、元件樣式 | 唯一定位、JS、錨點 |
| CSS 使用頻率 | 很常用 | 較少大量使用 |
| 優先級 | 較低 | 較高 |

## 為什麼實務上 class 比 id 更常用？

因為 class 比較適合重複使用，也比較容易維護。

例如：

```html
<div class="card">卡片 1</div>
<div class="card">卡片 2</div>
<div class="card">卡片 3</div>
```

```css
.card {
  border: 1px solid #ddd;
  padding: 16px;
}
```

這樣可以讓多個元素共用同一組樣式。

但 id 通常只適合唯一元素：

```html
<div id="app">應用程式入口</div>
```

如果大量使用 id 寫樣式，後續想覆蓋樣式時會比較麻煩，因為 id 選擇器優先級較高。

## 常見錯誤

### 錯誤 1：CSS 忘記加 `#`

錯誤寫法：

```css
main {
  width: 100%;
}
```

這會被當成標籤選擇器，代表選中 `<main>` 標籤。

如果 HTML 是：

```html
<div id="main">內容</div>
```

正確 CSS 應該是：

```css
#main {
  width: 100%;
}
```

### 錯誤 2：把 class 和 id 混用

HTML：

```html
<div class="box">內容</div>
```

錯誤 CSS：

```css
#box {
  color: red;
}
```

這樣選不到元素。

正確 CSS：

```css
.box {
  color: red;
}
```

### 錯誤 3：同一頁重複使用同一個 id

不建議：

```html
<div id="card">卡片 1</div>
<div id="card">卡片 2</div>
<div id="card">卡片 3</div>
```

建議改成：

```html
<div class="card">卡片 1</div>
<div class="card">卡片 2</div>
<div class="card">卡片 3</div>
```

## 通配符選擇器

通配符選擇器使用 `*` 表示。

它會選中頁面中的所有元素。

## 一句話理解

> 通配符選擇器會選中所有 HTML 元素。

## 語法

```css
* {
  屬性名: 屬性值;
}
```

## 範例

```css
* {
  color: red;
}
```

```html
<div>我的</div>
<span>我的</span>
<ul>
  <li>還是我的</li>
</ul>
```

## 會選中誰？

```css
* {
  color: red;
}
```

會選中頁面中所有元素，例如：

```html
<div>我的</div>
<span>我的</span>
<ul>
  <li>還是我的</li>
</ul>
```

所以 `div`、`span`、`ul`、`li` 都會被選中。

## 適合使用的情境

通配符選擇器最常見的用途是做基礎初始化。

例如清除預設內外邊距：

```css
* {
  margin: 0;
  padding: 0;
}
```

更常見的 box-sizing 寫法：

```css
*,
*::before,
*::after {
  box-sizing: border-box;
}
```

這樣可以讓元素寬高計算更直覺。

## 不適合使用的情境

通配符選擇器不適合隨便設定一般樣式。

例如：

```css
* {
  color: red;
}
```

這會讓頁面中幾乎所有文字都變紅，影響範圍太大，後續也不好維護。

如果只是想設定文章文字，應該使用更精準的選擇器。

例如：

```css
.article {
  color: #333;
}
```

## 常見錯誤

### 錯誤：用通配符選擇器處理太多樣式

不建議：

```css
* {
  color: red;
  font-size: 20px;
  background-color: yellow;
}
```

這樣會讓整個頁面所有元素都受到影響。

通配符選擇器比較適合做少量、全域性的初始化設定。

## 基礎選擇器優先級初步理解

當多個選擇器同時選中同一個元素，而且設定了同一個 CSS 屬性時，就會產生覆蓋問題。

例如：

```css
p {
  color: red;
}

.text {
  color: blue;
}

#main-text {
  color: green;
}
```

```html
<p id="main-text" class="text">內容</p>
```

這個元素同時被三個選擇器選中：

```text
p
.text
#main-text
```

如果它們都設定 `color`，通常會以 `#main-text` 的結果為主，因為 id 選擇器優先級比較高。

基礎選擇器的優先級大致可以先這樣記：

```text
id 選擇器 > class 選擇器 > 標籤選擇器 > 通配符選擇器
```

也就是：

```text
#main > .text > p > *
```

這只是初步理解。  
完整的 CSS 優先級會在後續筆記中再深入整理。

## 四種基礎選擇器快速比較

| 選擇器 | 範例 | 選中範圍 | 優點 | 風險 |
|---|---|---|---|---|
| 標籤選擇器 | `p` | 所有指定標籤 | 簡單、適合基礎樣式 | 容易影響太多元素 |
| 類選擇器 | `.box` | 所有指定 class | 最常用、可重用 | 命名需要有規劃 |
| id 選擇器 | `#app` | 指定唯一 id | 定位明確 | 優先級高，不適合大量寫樣式 |
| 通配符選擇器 | `*` | 所有元素 | 適合初始化 | 影響範圍最大 |

## 實務建議

實務開發中可以先這樣理解：

```text
想設定整類標籤的基礎樣式：用標籤選擇器
想寫可重複使用的樣式：用類選擇器
想定位唯一元素：用 id
想做全域初始化：用通配符選擇器
```

最常用的是：

```text
類選擇器
```

因為它兼顧：

- 可重用
- 可維護
- 可組合
- 適合元件化開發

## 常見混淆點整理

### 1. `.box` 和 `#box` 不一樣

```css
.box {
  color: red;
}
```

選的是：

```html
<div class="box">內容</div>
```

```css
#box {
  color: red;
}
```

選的是：

```html
<div id="box">內容</div>
```

### 2. `class="red font35"` 是兩個 class

```html
<div class="red font35">內容</div>
```

代表這個元素同時擁有：

```text
red
font35
```

不是一個叫做 `red font35` 的 class。

### 3. id 應該唯一，class 可以重複

```html
<div id="header">頁首</div>
```

id 適合唯一元素。

```html
<div class="card">卡片 1</div>
<div class="card">卡片 2</div>
<div class="card">卡片 3</div>
```

class 適合重複使用。

### 4. 通配符選擇器影響範圍最大

```css
* {
  margin: 0;
  padding: 0;
}
```

這會影響所有元素，所以要小心使用。

## 回查重點

如果之後忘記基礎選擇器，可以先記這幾句：

```text
標籤選擇器：用標籤名稱選元素，例如 p、div、a。
類選擇器：用 class 選元素，例如 .box，最常用。
id 選擇器：用 id 選元素，例如 #app，通常用於唯一元素。
通配符選擇器：用 * 選中所有元素，常用於初始化。
```
