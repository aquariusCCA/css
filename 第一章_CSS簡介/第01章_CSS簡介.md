## 本節導讀

這一章在說明 CSS 的用途、基本語法，以及常見的程式碼風格。

第一次閱讀時，建議先抓住 HTML 與 CSS 的分工，再看 CSS 的基本語法，最後用程式碼風格補齊實作上的閱讀習慣。

核心觀念只有一個：

- HTML 負責結構與語意
- CSS 負責外觀與版面
- 結構和樣式分開，網頁才比較好維護

## 你會在這篇學到什麼

- 為什麼只靠 HTML 不足以完成網頁樣式設計
- CSS 的角色、用途與基本定位
- CSS 基本語法怎麼組成
- 展開風格與緊湊風格分別適合什麼情境

## 1. HTML 的侷限性

HTML 主要用來描述內容的結構和語意。

例如：

- `<h1>` 表示這是一個大標題
- `<p>` 表示這是一個段落

HTML 可以處理基本結構，但如果想把頁面做得更漂亮，光靠 HTML 會很受限。

如果把樣式也混進 HTML，程式碼容易變得臃腫、難維護，也不利於後續調整版面。

## 2. CSS 是什麼

CSS 是層疊樣式表的簡稱，英文是 Cascading Style Sheets，也常譯為級聯樣式表。

> CSS 不是標記語言，而是樣式表語言。

CSS 主要用來設定 HTML 頁面的外觀，例如：

- 文字顏色
- 字體大小
- 圖片外型
- 區塊大小與間距
- 版面布局
- 整體顯示樣式

簡單來說，CSS 的作用就是讓 HTML 更好看，也讓版面控制更有彈性。

## 3. CSS 語法規範

CSS 的基本結構由兩部分組成：

1. 選擇器
2. 一條或多條聲明

![CSS語法規範.png](./images/image_1776158990520_ba8d013e.png)

```css
選擇器 {
    屬性名: 屬性值;
    ...
}
```

重點如下：

- 屬性和屬性值會組成一組「鍵值對」
- 屬性是指定要設定的樣式項目，例如字體大小、文字顏色
- 屬性名和屬性值之間用英文 `:` 分隔
- 多個鍵值對之間用英文 `;` 分隔

## 4. CSS 程式碼風格

CSS 常見有兩種寫法：

### 展開風格

開發時通常會用展開風格，因為比較容易閱讀與維護。

```css
h1 {
    color: red;
    font-size: 40px;
}
```

### 緊湊風格

上線時通常會轉成緊湊風格，因為可以減少檔案體積。

```css
h1{color:red;font-size:40px;}
```

## 5. 總結

- HTML 主要負責結構與內容語意
- CSS 主要負責外觀與版面
- CSS 的基本語法是「選擇器 + 聲明」
- 開發常用展開風格，上線常用緊湊風格
- 結構和樣式分離，是 HTML + CSS 最重要的基礎觀念
