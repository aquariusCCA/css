## 本節導讀

這一章是 CSS 的入門章節，主要說明 CSS 的用途、基本語法，以及常見的程式碼風格。

第一次閱讀時，建議先抓住一個核心觀念：

> HTML 負責網頁的結構與語意，CSS 負責網頁的外觀與版面。

也就是說：

- HTML 決定「這是什麼內容」
- CSS 決定「這個內容長什麼樣子」

例如：

```html
<h1>我的第一個網頁</h1>
```

這段 HTML 表示「這是一個一級標題」。

如果想讓這個標題變成紅色、字體變大，就可以使用 CSS：

```css
h1 {
    color: red;
    font-size: 40px;
}
```

所以可以先用一句話理解：

> HTML 管結構，CSS 管樣式。

# 1. HTML 的侷限性

HTML 主要用來描述網頁內容的「結構」與「語意」。

例如：

```html
<h1>文章標題</h1>
<p>這是一段文章內容。</p>
```

這段 HTML 表示：

* `<h1>` 是一級標題
* `<p>` 是段落

HTML 很適合描述內容本身，但它不適合負責大量樣式控制。

例如，如果我們想控制：

* 文字顏色
* 字體大小
* 背景顏色
* 元素寬度與高度
* 元素之間的距離
* 頁面排版
* 響應式布局

這些都不是 HTML 最擅長的工作。

---

## 1.1 如果把樣式都寫在 HTML 會怎樣？

假設我們把樣式直接寫進 HTML：

```html
<h1 style="color: red; font-size: 40px;">文章標題</h1>
<p style="color: gray; font-size: 18px;">這是一段文章內容。</p>
```

這樣雖然可以改變外觀，但問題是：

* HTML 會變得很臃腫
* 樣式分散在各個標籤上，不容易管理
* 如果很多地方都要改顏色，需要一個一個修改
* 結構與樣式混在一起，後續維護成本很高

因此實務開發中，通常會把結構和樣式分開管理：

```html
<h1>文章標題</h1>
<p>這是一段文章內容。</p>
```

```css
h1 {
    color: red;
    font-size: 40px;
}

p {
    color: gray;
    font-size: 18px;
}
```

這樣 HTML 負責內容結構，CSS 負責畫面樣式，兩者分工會比較清楚。

---

# 2. CSS 是什麼

CSS 是 **Cascading Style Sheets** 的縮寫，中文通常翻譯為：

* 層疊樣式表
* 級聯樣式表

CSS 不是標記語言，而是樣式表語言。

HTML 是用標籤描述內容結構，例如：

```html
<p>這是一段文字</p>
```

CSS 則是用規則描述這些內容應該如何顯示，例如：

```css
p {
    color: blue;
    font-size: 16px;
}
```

這段 CSS 的意思是：

> 找到所有 `<p>` 元素，讓文字變成藍色，字體大小變成 16px。

---

## 2.1 CSS 可以控制哪些樣式？

CSS 可以控制 HTML 頁面的外觀，例如：

* 文字顏色
* 字體大小
* 字型
* 背景顏色
* 邊框
* 圓角
* 陰影
* 寬度與高度
* 內距與外距
* 元素排列方式
* 頁面布局
* 動畫效果
* 響應式設計

簡單來說：

> CSS 的作用就是控制網頁的視覺呈現。

---

## 2.2 為什麼叫「層疊」樣式表？

CSS 的英文全名是 **Cascading Style Sheets**。

其中 Cascading 可以先理解為「層疊」。

所謂層疊，簡單來說就是：

> 當多個 CSS 規則同時影響同一個元素時，瀏覽器會根據一定的規則，決定最後要套用哪一個樣式。

例如：

```css
p {
    color: red;
}

p {
    color: blue;
}
```

這兩段 CSS 都在設定 `<p>` 的文字顏色。

如果沒有其他特殊條件，後面的 `color: blue;` 會覆蓋前面的 `color: red;`，所以最後文字會變成藍色。

不過，真正的 CSS 層疊規則還會受到以下因素影響：

* 樣式來源
* 選擇器權重
* 書寫順序
* `!important`

這些內容後續會在「CSS 層疊與權重」相關章節詳細說明。

本章只需要先知道：

> CSS 之所以叫層疊樣式表，是因為多個樣式規則可能同時作用在同一個元素上，瀏覽器會依照規則決定最終樣式。

---

# 3. CSS 基本語法

CSS 的基本語法由兩個主要部分組成：

1. 選擇器
2. 聲明區塊

基本格式如下：

```css
選擇器 {
    屬性名: 屬性值;
}
```

例如：

```css
h1 {
    color: red;
    font-size: 40px;
}
```

這段 CSS 的意思是：

> 選中所有 `<h1>` 元素，將文字顏色設定為紅色，字體大小設定為 40px。

---

## 3.1 選擇器是什麼？

選擇器用來告訴瀏覽器：

> 我要選中哪些 HTML 元素？

例如：

```css
h1 {
    color: red;
}
```

其中 `h1` 就是選擇器。

它的意思是：

> 找到頁面中所有 `<h1>` 元素。

例如 HTML 是：

```html
<h1>主標題</h1>
<h1>另一個主標題</h1>
```

CSS 是：

```css
h1 {
    color: red;
}
```

那麼這兩個 `<h1>` 的文字都會變成紅色。

---

## 3.2 聲明區塊是什麼？

選擇器後面的 `{}` 稱為聲明區塊。

```css
h1 {
    color: red;
    font-size: 40px;
}
```

其中這一段就是聲明區塊：

```css
{
    color: red;
    font-size: 40px;
}
```

聲明區塊裡面可以放一條或多條聲明。

每一條聲明都表示一個樣式設定。

---

## 3.3 聲明是什麼？

一條聲明由兩部分組成：

1. 屬性名
2. 屬性值

例如：

```css
color: red;
```

其中：

* `color` 是屬性名
* `red` 是屬性值
* `:` 用來分隔屬性名和屬性值
* `;` 表示這條聲明結束

所以：

```css
color: red;
```

可以理解為：

> 把文字顏色設定為紅色。

另一個例子：

```css
font-size: 40px;
```

可以理解為：

> 把字體大小設定為 40px。

---

## 3.4 CSS 語法拆解範例

完整範例：

```css
h1 {
    color: red;
    font-size: 40px;
}
```

可以拆成：

| 部分   | 範例            | 說明                   |
| ---- | ------------- | -------------------- |
| 選擇器  | `h1`          | 選中 HTML 中的 `<h1>` 元素 |
| 聲明區塊 | `{ ... }`     | 放置多條樣式聲明             |
| 聲明   | `color: red;` | 一組完整的樣式設定            |
| 屬性名  | `color`       | 要設定的樣式項目             |
| 屬性值  | `red`         | 該樣式項目的具體值            |
| 冒號   | `:`           | 分隔屬性名與屬性值            |
| 分號   | `;`           | 結束一條聲明               |

---

## 3.5 HTML 與 CSS 對照理解

HTML：

```html
<h1>CSS 簡介</h1>
<p>CSS 用來控制網頁外觀。</p>
```

CSS：

```css
h1 {
    color: red;
    font-size: 40px;
}

p {
    color: gray;
    font-size: 18px;
}
```

可以這樣理解：

| HTML 元素 | HTML 負責   | CSS 負責    |
| ------- | --------- | --------- |
| `<h1>`  | 表示這是一個主標題 | 控制標題顏色與大小 |
| `<p>`   | 表示這是一個段落  | 控制段落顏色與大小 |

所以 HTML 與 CSS 的關係是：

> HTML 決定內容的結構，CSS 決定內容的外觀。

# 4. CSS 程式碼風格

CSS 常見有兩種程式碼風格：

1. 展開風格
2. 緊湊風格

這兩種寫法的功能一樣，只是格式不同。

---

## 4.1 展開風格

展開風格會把每一條聲明分行書寫。

```css
h1 {
    color: red;
    font-size: 40px;
}
```

這種寫法的優點是：

* 結構清楚
* 容易閱讀
* 容易修改
* 適合多人協作
* 適合開發階段使用

因此開發時通常會使用展開風格。

---

## 4.2 緊湊風格

緊湊風格會把空白、換行盡量移除。

```css
h1{color:red;font-size:40px;}
```

這種寫法的優點是：

* 檔案體積較小
* 適合上線環境
* 可以減少瀏覽器下載 CSS 檔案的大小

不過，實務開發中通常不會手動撰寫緊湊風格。

比較常見的做法是：

1. 開發時使用展開風格
2. 打包時透過工具自動壓縮 CSS
3. 上線後使用壓縮後的 CSS 檔案

例如使用：

* Vite
* Webpack
* PostCSS
* CSS Minifier
* CleanCSS
* cssnano

所以可以記成：

> 開發時寫給人看，上線時壓縮給瀏覽器用。

---

## 4.3 展開風格與緊湊風格比較

| 類型   | 範例   | 適合情境 | 優點        |
| ---- | ---- | ---- | --------- |
| 展開風格 | 多行書寫 | 開發階段 | 易讀、易維護    |
| 緊湊風格 | 單行壓縮 | 上線階段 | 檔案較小、載入較快 |

---

# 5. 為什麼要讓 HTML 與 CSS 分離？

HTML 與 CSS 分離，是前端開發中非常重要的基礎觀念。

原因是：

* HTML 專心描述內容結構
* CSS 專心控制外觀樣式
* 結構變動時，不一定要改樣式
* 樣式變動時，不一定要改 HTML
* 多個頁面可以共用同一份 CSS
* 專案變大後會比較容易維護

例如：

```html
<h1>產品介紹</h1>
<p>這是產品內容。</p>
```

```css
h1 {
    color: red;
}

p {
    color: gray;
}
```

如果之後想把標題改成藍色，只需要修改 CSS：

```css
h1 {
    color: blue;
}
```

HTML 不需要改。

這就是結構與樣式分離的好處。

# 6. 初學者常見誤解

## 6.1 誤解一：HTML 也可以寫樣式，所以不需要 CSS

HTML 確實可以透過 `style` 屬性直接寫樣式：

```html
<h1 style="color: red;">標題</h1>
```

但這種寫法不適合大量使用。

因為它會讓樣式分散在 HTML 裡面，後續很難統一管理。

比較推薦的做法是把樣式寫到 CSS：

```html
<h1>標題</h1>
```

```css
h1 {
    color: red;
}
```

---

## 6.2 誤解二：CSS 只是讓網頁變漂亮

CSS 不只是讓網頁變漂亮。

它還負責：

* 版面布局
* 元素排列
* 響應式設計
* 動畫效果
* 使用者介面視覺狀態
* 不同裝置上的顯示效果

例如手機版和電腦版畫面不同，很多時候就是透過 CSS 控制。

---

## 6.3 誤解三：緊湊風格比較專業，所以開發時應該手寫緊湊風格

這是不建議的。

開發時應該優先考慮可讀性與可維護性。

緊湊風格通常是工具在打包時自動產生，不是人手動維護的主要格式。
