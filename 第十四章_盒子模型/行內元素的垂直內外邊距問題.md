# 行內元素的垂直內外邊距問題

> 所屬章節：[第十四章 盒子模型](./README.md)  
> 關鍵字：行內元素、inline、margin-top、margin-bottom、padding-top、padding-bottom、line-height、inline-block  
> 建議回查情境：替 `span`、`a` 設定上下 `margin` 沒反應時；行內元素上下 `padding` 看起來怪怪的時；想讓文字垂直置中或讓連結有上下留白時

## 本節導讀

這一節說明行內元素在垂直方向上設定 `margin` 和 `padding` 時，為什麼常常不像區塊元素那樣影響布局。初學時很容易把 `span`、`a` 這類行內元素當成一般盒子來調上下距離，結果發現 `margin-top`、`margin-bottom` 幾乎沒有想要的效果。

第一次閱讀時，先分清楚「水平內外距」和「垂直內外距」的差異，再看應該用 `line-height`、`display: inline-block;` 還是改由外層元素控制間距。

## 你會在這篇學到什麼

- 行內元素和區塊元素在布局上的差異。
- 行內元素的左右 `margin` / `padding` 為什麼通常能正常生效。
- 行內元素的上下 `margin` 為什麼不能用來推開垂直距離。
- 行內元素的上下 `padding` 為什麼可能有視覺效果，但不適合控制垂直布局。
- 什麼情況下應該使用 `line-height`。
- 什麼情況下應該把行內元素轉成 `inline-block` 或 `block`。

## 先講結論

普通行內元素，例如 `span`、一般文字中的 `a`，主要參與文字行內的排版。它的左右內外距通常能正常影響同一行中的水平距離；但上下內外距不適合拿來控制垂直布局。

可以先記成：

| 屬性 | 對普通行內元素的常見效果 |
| --- | --- |
| `margin-left` / `margin-right` | 會影響左右距離。 |
| `margin-top` / `margin-bottom` | 通常不會把上下元素推開。 |
| `padding-left` / `padding-right` | 會影響左右內距，也會影響可見背景範圍。 |
| `padding-top` / `padding-bottom` | 可能產生可見背景或可點擊範圍，但不會像區塊元素那樣穩定撐開行距。 |
| `line-height` | 會影響文字所在行盒的高度，常用來控制文字垂直節奏。 |

所以，如果想穩定控制垂直方向，不要只靠行內元素本身的上下 `margin` 或 `padding`。應該依情境改用 `line-height`、`display: inline-block;`、`display: block;`，或由外層區塊元素控制距離。

## 為什麼行內元素的垂直間距容易失效

行內元素會跟文字一起排在同一行，例如：

```html
<span>span</span>
<span>span</span>
```

這些 `span` 會像文字一樣排在同一條文字行裡，而不是像 `div` 那樣一個盒子獨占一行。

因此，當你寫：

```css
span {
  margin: 100px;
}
```

它的左右 `margin` 會讓兩個 `span` 之間的水平距離變大；但 `margin-top` 和 `margin-bottom` 不會像區塊元素一樣，把上下方向的內容推開。

這就是原文想提醒的核心：不要期待普通行內元素的上下 `margin` 能控制垂直位置。

## `padding-top` 和 `padding-bottom` 的精確理解

原文寫「行內標籤的 `padding-top` 和 `padding-bottom` 不生效」。這句需要修正得更精確。

對普通行內元素來說，上下 `padding` 可能會讓背景或可點擊範圍在視覺上變大：

```css
span {
  padding: 20px;
  background-color: skyblue;
}
```

但是它不會像區塊元素那樣穩定推開上下行，也不適合用來控制垂直布局。結果常會變成：

- 背景看起來往上、往下延伸。
- 文字行之間沒有按預期被推開。
- 上下內容可能被視覺區域覆蓋或擠在一起。

因此，更穩定的說法是：

> 行內元素的上下 `padding` 可能有視覺效果，但不適合拿來控制垂直位置或上下間距。

## 錯誤示範：直接用 margin 或 padding 調整行內元素垂直位置

```css
span {
  margin: 100px;
  padding: 100px;
  background-color: pink;
}
```

```html
<span>span</span>
<span>span</span>
```

這段程式碼會讓水平距離和背景範圍變得很誇張，但不會得到穩定的垂直布局效果。

如果你的目的有以下幾種，要改用不同做法：

- 想控制文字行高：用 `line-height`。
- 想讓行內元素有穩定的上下內距：改成 `inline-block`。
- 想讓元素上下和其他區塊拉開距離：使用區塊元素或由外層容器設定 `margin`。

## 解法一：用 `line-height` 控制文字行高

如果目標是調整文字所在行的高度，或讓單行文字在固定高度中垂直置中，可以使用 `line-height`。

```css
.tag {
  line-height: 40px;
}
```

```html
<span class="tag">span</span>
```

`line-height` 控制的是文字行盒高度。當只有單行文字時，常用來讓文字在固定高度的區域中看起來垂直置中。

但要注意：`line-height` 不是萬用解法。它適合控制文字行高，不適合替所有元素解決上下間距問題。

## 解法二：轉成 `inline-block`

如果你想讓 `a`、`span` 這類元素既能和文字排在同一行，又能穩定設定寬高、上下 `padding`，可以轉成行內塊元素。

```css
.tag {
  display: inline-block;
  padding: 8px 16px;
  background-color: skyblue;
}
```

```html
<span class="tag">標籤</span>
<span class="tag">按鈕感文字</span>
```

`inline-block` 的特點是：

- 外部排列像行內元素，可以和其他內容排在同一行。
- 內部盒子行為比較像區塊元素，可以設定寬高與上下 `padding`。
- 很適合做標籤、按鈕、導航連結等元件。

## 解法三：轉成區塊元素或由外層控制距離

如果你的目標是讓元素和上下內容拉開距離，通常不該使用普通行內元素本身的上下 `margin`。

可以把元素轉成區塊元素：

```css
.note {
  display: block;
  margin: 20px 0;
}
```

```html
<span class="note">這段內容需要上下距離</span>
```

也可以更常見地讓外層區塊負責間距：

```css
p {
  margin-bottom: 16px;
}
```

```html
<p>這是一段文字，裡面可以有 <span>行內元素</span>。</p>
<p>下一段文字。</p>
```

布局距離通常交給區塊元素處理，會比硬調行內元素的上下 `margin` 更穩定。

## 常見混淆點

### 行內元素不是不能設定所有 margin 和 padding

行內元素的左右 `margin` / `padding` 通常可以正常影響水平距離。真正容易出問題的是垂直方向，尤其是想用 `margin-top`、`margin-bottom` 推開上下內容時。

### `padding-top` 和 `padding-bottom` 不是完全沒有任何效果

它們可能讓背景或可點擊範圍看起來變大，但不會像區塊元素的上下 `padding` 一樣穩定撐開布局，所以不建議用它們控制垂直間距。

### `line-height` 解決的是文字行高問題

如果只是想讓單行文字在一個高度裡看起來垂直置中，`line-height` 很適合。  
如果想讓整個元素有上下留白或和其他區塊拉開距離，通常應改用 `inline-block`、`block` 或外層容器的間距。

### 圖片和表單元素可能不是這篇討論的普通行內元素

`img`、`input` 這類元素雖然預設也可能以行內方式排列，但它們是替換元素，尺寸行為和普通文字型行內元素不同。這篇主要討論的是 `span`、文字中的 `a` 這類普通行內元素。
