# 看透網頁布局的本質

> 所屬章節：[第十四章 盒子模型](./README.md)  
> 關鍵字：CSS 盒子、盒子模型、網頁布局、元素盒子、內容區域  
> 建議回查情境：分不清 HTML 元素、盒子模型與 CSS 布局關係時；開始學 `width`、`height`、`padding`、`border`、`margin` 前

## 本節導讀

這一節先建立盒子模型章節的核心視角：網頁布局不是在憑空移動文字和圖片，而是在控制一個個由 HTML 元素產生的矩形盒子。

第一次閱讀時，先理解「元素基本都可以被看成盒子」，再看布局過程如何從 HTML 結構、CSS 樣式、盒子位置與內容呈現一路串起來。後續學 `content`、`padding`、`border`、`margin`、`box-sizing` 時，都可以回到這個視角理解。

## 你會在這篇學到什麼

- 為什麼可以把網頁布局理解成「用 CSS 擺盒子」。
- HTML 元素、CSS 樣式與盒子模型之間的關係。
- 網頁布局的基本拆解流程。
- 盒子模型在後續屬性學習中的位置。
- 初學時容易混淆的幾個觀念。

## 先講結論

網頁布局的本質可以理解成：

> 把 HTML 元素當成一個個矩形盒子，再用 CSS 控制盒子的尺寸、外觀、間距與位置，最後讓內容在盒子中呈現。

這裡的「盒子」不一定是畫面上看得到的方框。即使元素沒有背景色、沒有邊框，只要它參與頁面排版，瀏覽器通常都會為它建立一個可計算的矩形區域。

## 為什麼說網頁元素基本都是盒子

HTML 負責描述頁面結構，例如標題、段落、圖片、列表、區塊等。瀏覽器在渲染這些元素時，會把它們轉換成頁面上的一個個區域，再依照 CSS 規則計算這些區域的大小、間距、位置與外觀。

所以在學布局時，可以先把每個元素想成一個盒子：

- 標題是一個盒子。
- 段落是一個盒子。
- 圖片是一個盒子。
- `div`、`section`、`header` 這類容器元素也是盒子。

這個視角能讓布局問題變得更具體：不是「這段文字為什麼跑到那裡」，而是「這個盒子的尺寸、間距、排列方式或所在位置出了什麼問題」。

## 網頁布局的基本過程

網頁布局可以拆成以下幾步來理解：

1. 準備 HTML 元素
   先根據頁面內容建立必要的 HTML 結構。這些元素是後續布局的基礎。

2. 把元素理解成盒子
   每個元素在頁面上都可以先視為一個矩形區域。後續要控制的，不是抽象的內容，而是這些矩形區域的大小、間距與排列。

3. 使用 CSS 設定盒子樣式
   透過 CSS 控制盒子的寬高、背景、邊框、內邊距、外邊距等外觀與空間。

4. 把盒子放到合適的位置
   依需求決定盒子要在正常文檔流中排列，還是透過其他布局方式調整位置與對齊方式。

5. 讓內容在盒子中正確呈現
   文字、圖片或其他子元素會放在盒子內部。內容可能影響盒子的尺寸；盒子的尺寸也可能限制內容的顯示。

這個順序主要是學習布局時的拆解方式，不代表實際開發時一定要先寫完 CSS 再放內容。實務上常見流程是先建立 HTML 結構，再逐步補 CSS 樣式，最後根據內容效果來調整布局。

## 盒子模型與布局的關係

盒子模型把一個盒子拆成幾個層次：

- `content`：內容區域，文字、圖片或子元素主要放在這裡。
- `padding`：內邊距，控制內容和邊框之間的距離。
- `border`：邊框，包住內容區域與內邊距。
- `margin`：外邊距，控制這個盒子和其他盒子之間的距離。

理解這四個部分後，很多布局問題都可以被重新描述成更具體的問題：

- 盒子太大或太小，通常要檢查 `width`、`height`、內容量與 `box-sizing`。
- 內容離邊框太近，通常要檢查 `padding`。
- 盒子之間距離不對，通常要檢查 `margin`。
- 想要顯示邊界或分隔感，通常會用到 `border`。

## 簡單例子

下面這段 HTML 建立了一個卡片區塊：

```html
<div class="card">
  <h2>標題</h2>
  <p>這是一段內容文字。</p>
</div>
```

CSS 則負責把這個區塊變成一個具體的盒子：

```css
.card {
  width: 300px;
  padding: 16px;
  border: 1px solid #ddd;
  margin: 20px auto;
  background-color: #fff;
}
```

可以這樣拆解：

- `.card` 對應頁面上的一個盒子。
- `width` 控制盒子的寬度。
- `padding` 讓內容不要貼著盒子邊緣。
- `border` 讓盒子的邊界被看見。
- `margin` 控制盒子和外部元素的距離。
- `background-color` 控制盒子的背景色。

## 常見混淆點

### 盒子不等於看得見的邊框

盒子是瀏覽器用來排版的矩形區域。沒有設定 `border` 時，盒子仍然存在，只是邊界不一定看得見。

### 布局不只是調整位置

布局包含尺寸、間距、排列、對齊與內容呈現。只移動位置，通常不足以解決真正的排版問題。

### 內容與盒子會互相影響

內容放在盒子裡，但內容量可能影響盒子的高度；如果盒子尺寸被固定，內容也可能出現溢出問題。這些都需要透過盒子模型相關屬性來判斷。

### 盒子模型是後續布局知識的基礎

後面學到的 `content`、`padding`、`border`、`margin`、`box-sizing`，都是在回答同一件事：盒子的尺寸與空間到底怎麼算。
