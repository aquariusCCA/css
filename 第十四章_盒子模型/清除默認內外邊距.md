# 清除默認內外邊距

> 所屬章節：[第十四章 盒子模型](./README.md)  
> 關鍵字：默認樣式、margin、padding、CSS reset、通配符、瀏覽器預設樣式  
> 建議回查情境：頁面一開始有空白邊距時；不同瀏覽器顯示間距不一致時；切版前想統一元素的初始 `margin` 和 `padding` 時

## 本節導讀

這一節說明為什麼切版前常會先清除元素的默認內外邊距。很多 HTML 元素本身就帶有瀏覽器預設樣式，例如 `body` 有默認外邊距，`ul`、`ol` 有默認內外距，標題與段落也常有默認上下間距。

第一次閱讀時，先理解「為什麼要清除」，再看兩種常見 reset 寫法。後續如果遇到頁面明明沒寫間距卻出現空白，可以回來檢查是不是瀏覽器默認樣式造成的。

## 你會在這篇學到什麼

- 為什麼瀏覽器會讓元素自帶 `margin` 或 `padding`。
- 為什麼切版前常要清除默認內外邊距。
- 通配符 reset 的寫法與特點。
- 指定元素列表 reset 的寫法與特點。
- 清除默認樣式後，為什麼仍然可以按需求重新設定間距。
- 行內元素設定上下內外邊距時要注意什麼。

## 先講結論

不同瀏覽器會替部分 HTML 元素加上默認樣式。為了讓頁面布局更可控，切版前常會先把常見元素的默認 `margin` 和 `padding` 清除。

最簡單的寫法是：

```css
* {
  margin: 0;
  padding: 0;
}
```

這樣可以讓所有元素先回到沒有內外邊距的狀態。之後需要間距時，再由我們自己明確設定。

## 為什麼要清除默認內外邊距

HTML 元素不是完全沒有樣式。瀏覽器會提供一套預設樣式，讓沒有寫 CSS 的頁面也能正常閱讀。

例如：

- `body` 預設可能有外邊距。
- `h1`、`h2`、`p` 預設可能有上下外邊距。
- `ul`、`ol` 預設可能有縮排與列表樣式。
- `button`、`input` 等表單元素在不同瀏覽器中也可能有不同默認外觀。

這些預設樣式對純文字頁面有幫助，但在切版時可能造成干擾：

- 頁面邊緣出現沒有預期的空白。
- 元素之間的距離不是自己設定的。
- 不同瀏覽器顯示出來的間距略有差異。
- 設計稿量好的距離和實際畫面對不上。

所以在正式布局前，常會先做一次 reset，把常見的默認內外距清掉。

## 寫法一：通配符清除

通配符 `*` 會選中頁面中的所有元素。

```css
* {
  margin: 0;
  padding: 0;
}
```

這種寫法的優點是簡單、好記，初學切版時很常見。

可以理解成：

- 所有元素的默認外邊距都先歸零。
- 所有元素的默認內邊距都先歸零。
- 後續需要留白時，再自己寫 `margin` 或 `padding`。

如果專案也希望盒子尺寸更容易控制，常會搭配 `box-sizing`：

```css
* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}
```

`box-sizing: border-box;` 的作用不是清除內外距，而是讓 `width` 和 `height` 包含 `padding` 與 `border`。這部分可以回到 [box-sizing](./box-sizing.md) 複習。

## 寫法二：指定元素列表清除

另一種寫法是只清除常見元素，而不是用 `*` 選中全部元素。

```css
blockquote,
body,
button,
dd,
dl,
dt,
fieldset,
form,
h1,
h2,
h3,
h4,
h5,
h6,
hr,
input,
legend,
li,
ol,
p,
pre,
td,
textarea,
th,
ul {
  margin: 0;
  padding: 0;
}
```

這種寫法的重點是：只針對常見會有默認內外距、或在項目中需要統一處理的元素重置。

它和通配符寫法的差異：

| 寫法 | 特點 | 適合情境 |
| --- | --- | --- |
| `*` 通配符 | 簡單直接，所有元素都歸零 | 初學、練習、小型頁面 |
| 指定元素列表 | 控制範圍更明確 | 較完整的專案 reset、需要保留部分元素默認行為時 |

初學階段先會用通配符即可。等專案變大，再理解更完整的 reset 或 normalize 寫法。

## 清除後要自己重新設定間距

清除默認內外邊距不代表頁面不需要間距，而是把「瀏覽器預設的間距」改成「開發者明確設定的間距」。

例如，清除後可以重新設定段落間距：

```css
p {
  margin-bottom: 16px;
}
```

也可以重新設定列表縮排：

```css
ul {
  padding-left: 20px;
}
```

這樣做的好處是：頁面間距來自自己的 CSS，而不是來自瀏覽器的默認樣式。

## 行內元素的上下間距注意事項

原文提到：「行內元素為了照顧兼容性，盡量只設定左右內外邊距，不要設定上下內外邊距；轉換為塊級或行內塊元素就可以。」

這句要放在正確上下文理解：它不是在說 reset 時不能清除行內元素的 `margin` / `padding`，而是在說後續自己設定間距時，行內元素在垂直方向上的表現比較特殊。

常見情況是：

- 行內元素的 `margin-left`、`margin-right` 通常可以正常影響水平距離。
- 行內元素的 `margin-top`、`margin-bottom` 通常不會像塊級元素那樣推開上下元素。
- 行內元素的上下 `padding` 可能產生視覺效果，但不適合拿來穩定控制垂直布局。

如果想穩定控制上下間距，常見做法是把元素轉成塊級或行內塊：

```css
a {
  display: inline-block;
  padding: 8px 16px;
}
```

或者使用 `line-height` 控制文字行高：

```css
a {
  line-height: 40px;
}
```

更完整的行內元素垂直方向問題，可以看 [行內元素的垂直內外邊距問題](./行內元素的垂直內外邊距問題.md)。

## 常見混淆點

### 清除默認樣式不是刪掉所有間距設計

reset 只是清掉瀏覽器自帶的 `margin` 和 `padding`。真正需要的頁面留白，仍然要在後續 CSS 中自己加回來。

### `margin: 0; padding: 0;` 不會清除所有默認樣式

這兩行只處理內外邊距。像列表符號、字體、表單控件外觀、邊框等其他默認樣式，還需要另外設定。

例如要移除列表符號，通常還會寫：

```css
ul,
ol {
  list-style: none;
}
```

### 通配符 reset 和完整 reset 不是一回事

`* { margin: 0; padding: 0; }` 是最基本的清除內外距。完整專案可能還會處理 `box-sizing`、列表樣式、圖片顯示、表單樣式、字體繼承等更多細節。

### 清除默認內外距和外邊距合併是不同問題

清除默認 `margin` 可以減少瀏覽器預設間距造成的干擾，但不代表 CSS 的外邊距合併規則不存在。自己重新設定上下 `margin` 時，仍然可能遇到垂直外邊距合併。
