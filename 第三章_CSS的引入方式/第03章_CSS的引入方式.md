# 第03章 CSS 的引入方式

> 所屬章節：[第三章 CSS 的引入方式](./README.md)  
> 課程入口：[CSS 筆記總覽](../README.md)  
> 關鍵字：行內樣式、內部樣式表、外部樣式表、`style`、`<style>`、`<link>`、結構與樣式分離、樣式複用、瀏覽器快取  
> 建議回查情境：忘了三種 CSS 引入方式差異、想判斷正式開發該用哪種方式、CSS 沒有生效時、忘了 `<style>` 或 `<link>` 的基本寫法時

## 本節導讀

這一章說明 CSS 在 HTML 中常見的三種引入方式：

1. **行內樣式**：直接寫在 HTML 標籤的 `style` 屬性中
2. **內部樣式表**：寫在 HTML 文件中的 `<style>` 標籤內
3. **外部樣式表**：寫在獨立的 `.css` 檔案中，再用 `<link>` 引入 HTML

這三種方式使用的 CSS 語法本質上是一樣的，差別主要在於：

- CSS 寫在哪裡
- 作用範圍有多大
- 是否容易複用
- 是否適合長期維護
- 是否適合正式專案與團隊協作

第一次閱讀時，建議先看「先講結論」和「三種方式怎麼選」，先建立判斷框架，再回頭看每一種寫法。

## 你會在這篇學到什麼

- CSS 三種引入方式各自是什麼
- `style` 屬性、`<style>` 標籤、`<link>` 標籤的基本寫法
- 三種方式的適用場景與限制
- 為什麼正式開發通常優先使用外部樣式表
- 為什麼行內樣式優先權較高，但不代表它最適合專案維護
- CSS 沒有生效時，可以先檢查哪些地方

## 先講結論

如果只想先記住使用場景，可以先記這一句：

> 單一元素用行內，單一頁面用內部，多頁面與正式專案用外部。

更具體來說：

- **行內樣式**：適合臨時測試或單一元素微調，不適合大量使用。
- **內部樣式表**：適合單頁練習、教學範例、小型 demo。
- **外部樣式表**：最適合正式開發，因為可複用、好維護，也更符合結構與樣式分離的做法。

實務上可以用三個問題判斷：

1. 這個樣式只影響一個元素，還是會影響很多元素？
2. 這個樣式只在單一頁面使用，還是會被多個頁面共用？
3. 這段樣式之後是否需要長期維護或讓其他人一起修改？

如果答案開始涉及「很多元素」、「多頁共用」、「長期維護」、「多人協作」，通常就應該優先考慮外部樣式表。

## 1. 行內樣式

行內樣式是直接把 CSS 寫在 HTML 標籤的 `style` 屬性裡。

```html
<p style="color: pink; font-size: 20px;">給我一個粉紅的回憶</p>
```

在這個例子中：

- `style` 是 HTML 標籤的屬性
- `color: pink;` 是一段 CSS 宣告
- `font-size: 20px;` 也是一段 CSS 宣告
- 多個 CSS 宣告通常用分號 `;` 分隔

### 特點

- 樣式只作用在當前這一個元素
- 寫法直接，修改單一元素時很快
- 在很多情況下，行內樣式的覆蓋能力會比一般選擇器更強
- HTML 結構與 CSS 樣式會混在一起，可讀性與可維護性較差

### 適合什麼情況

- 臨時測試某個樣式
- 只想快速調整單一標籤
- 寫教學範例時，為了讓效果一眼看懂
- 某些特殊情境下，由 JavaScript 或前端框架動態產生單一元素的樣式值

例如：拖曳、動畫、即時寬度、高度、座標、旋轉角度等動態值，有時可能會透過行內樣式控制。

### 不適合什麼情況

- 頁面有很多樣式要管理
- 多個元素要使用相同樣式
- 團隊協作或正式開發
- 樣式之後可能會反覆調整
- 希望 HTML 負責結構、CSS 負責樣式，兩者保持清楚分工

### 為什麼不建議大量使用行內樣式？

假設頁面上有很多錯誤訊息都寫成這樣：

```html
<p style="color: red;">帳號不可為空</p>
<p style="color: red;">密碼不可為空</p>
<p style="color: red;">驗證碼不可為空</p>
```

如果之後設計師要求錯誤訊息全部改成深紅色，你就要一個一個修改。

比較好的做法是把樣式抽出來：

```html
<p class="error-message">帳號不可為空</p>
<p class="error-message">密碼不可為空</p>
<p class="error-message">驗證碼不可為空</p>
```

```css
.error-message {
    color: darkred;
}
```

這樣之後只要改 CSS 一處，所有使用 `.error-message` 的元素就會一起更新。

### 小提醒

- 行內樣式優先權通常較高，但這不是推薦大量使用它的理由。
- 真正選擇 CSS 引入方式時，應該優先考慮「是否好維護」、「是否可複用」、「是否符合專案結構」。
- 可以把行內樣式理解成「方便快速，但不適合管理大量樣式」的工具。

## 2. 內部樣式表

內部樣式表是把 CSS 集中寫在 HTML 文件中的 `<style>` 標籤內，通常放在 `<head>` 裡。

```html
<!DOCTYPE html>
<html lang="zh-Hant">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>內部樣式表</title>
    <style>
        div {
            color: pink;
        }
    </style>
</head>
<body>
    <div>這段文字會套用同一頁面中的內部樣式。</div>
</body>
</html>
```

在這個例子中：

- CSS 寫在 `<style>` 標籤內
- `div { color: pink; }` 會影響同一份 HTML 文件中的 `div` 元素
- 樣式不再直接寫在單一標籤上，而是集中管理在同一個區塊

### 特點

- 樣式集中在同一個 HTML 檔案中
- 可以一次控制當前頁面的多個元素
- 比行內樣式更清楚
- HTML 和 CSS 還是放在同一個檔案中，尚未完全分離
- 樣式通常只能服務當前這一頁，不能方便地跨多個頁面共用

### 適合什麼情況

- HTML、CSS 初學練習
- 教學範例
- 單頁 demo
- 小型測試頁面
- 想快速展示某一頁的樣式效果

### 不適合什麼情況

- 多個頁面需要共用同一份樣式
- 專案樣式越來越多，需要拆分與管理
- 多人協作開發
- 樣式需要長期維護

### 小提醒

- `<style>` 理論上可以出現在文件中不同位置，但實務上通常放在 `<head>`，結構較清楚。
- 內部樣式表可以讓同一頁面內的多個元素共用樣式，但不適合拿來管理整個網站的共用樣式。
- 如果你的 CSS 內容開始變長，或多個頁面都需要相同樣式，就應該考慮改成外部樣式表。

## 3. 外部樣式表

外部樣式表是把 CSS 獨立寫在 `.css` 檔案中，再用 `<link>` 標籤把它引入 HTML。

引入外部樣式表通常分成兩步：

1. 建立 CSS 檔案
2. 在 HTML 的 `<head>` 中使用 `<link>` 引入 CSS 檔案

`style.css`

```css
div {
    color: pink;
}
```

`index.html`

```html
<!DOCTYPE html>
<html lang="zh-Hant">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>外部樣式表</title>
    <link rel="stylesheet" href="./style.css">
</head>
<body>
    <div>來呀，快活呀，反正有大把時間。</div>
</body>
</html>
```

在這個例子中：

- `style.css` 負責放 CSS
- `index.html` 負責放 HTML 結構
- `<link rel="stylesheet" href="./style.css">` 負責把 CSS 檔案引入 HTML

### `<link>` 的重點

```html
<link rel="stylesheet" href="./style.css">
```

- `link`：用來連結外部資源
- `rel="stylesheet"`：告訴瀏覽器這個資源是一份樣式表
- `href="./style.css"`：指定 CSS 檔案路徑

如果 `href` 路徑寫錯，瀏覽器就找不到 CSS 檔案，樣式也不會生效。

### 特點

- HTML 與 CSS 分成不同檔案，結構清楚
- 同一份 CSS 可以被多個 HTML 頁面共用
- 樣式量變多時，仍然比較容易維護
- 適合多人協作與長期開發
- 瀏覽器可快取 CSS 檔案，後續載入通常更有效率

### 適合什麼情況

- 正式開發
- 多頁網站
- 團隊協作
- 需要長期維護的專案
- 需要多個頁面共用同一套樣式
- 想把 HTML 結構與 CSS 樣式清楚分離

### 優勢

- **可複用**：多個頁面可以共用同一份 CSS。
- **好維護**：修改一個 CSS 檔案，就能影響所有有引入它的頁面。
- **結構清楚**：HTML 負責內容結構，CSS 負責畫面樣式。
- **方便分工**：多人協作時，比較容易分開處理 HTML 和 CSS。
- **有利快取**：瀏覽器可以快取外部 CSS 檔案，減少重複載入成本。

### 可能遇到的問題

外部樣式表本身很適合正式開發，但初學時常見問題是「CSS 沒有成功引入」。常見原因包括：

- `href` 路徑寫錯
- CSS 檔名寫錯
- CSS 檔案副檔名不是 `.css`
- `<link>` 沒有放在正確的 HTML 文件中
- 忘記寫 `rel="stylesheet"`
- 瀏覽器快取導致看起來沒有更新
- CSS 選擇器沒有選到目標元素

## 4. 三種方式怎麼選

| 方式 | CSS 寫在哪裡 | 作用範圍 | 優點 | 缺點 | 適合情境 |
| --- | --- | --- | --- | --- | --- |
| 行內樣式 | HTML 標籤的 `style` 屬性 | 當前元素 | 修改直接、測試快速 | 難維護、難複用、HTML 與 CSS 混在一起 | 臨時測試、單一元素微調、特殊動態樣式 |
| 內部樣式表 | HTML 的 `<style>` 標籤 | 當前頁面 | 同頁可集中管理，比行內清楚 | 不方便跨頁共用，CSS 多了會讓 HTML 變長 | 單頁練習、教學 demo、小型測試頁 |
| 外部樣式表 | 獨立 `.css` 檔案，再用 `<link>` 引入 | 多個頁面 | 可複用、易維護、結構清楚、利於協作 | 需要正確引入檔案，路徑錯會失效 | 正式專案、多頁網站、長期維護 |

### 實務判斷流程

可以用下面這個流程快速判斷：

```text
只是臨時測試單一元素？
→ 可以用行內樣式

只是寫一個單頁 HTML 練習或 demo？
→ 可以用內部樣式表

是正式專案、多頁網站、多人協作、需要長期維護？
→ 優先使用外部樣式表
```

### 更接近實務的判斷方式

- 如果樣式會被重複使用，盡量不要寫成行內樣式。
- 如果 CSS 開始變多，盡量不要全部塞在 `<style>` 裡。
- 如果是正式專案，通常會把樣式放在外部 CSS、CSS Module、SCSS、Vue SFC 的 `<style>` 區塊，或其他由建構工具管理的樣式檔案中。
- 不管使用哪一種工具，核心原則都是：**讓樣式有結構、可維護、可複用**。

## 5. 常見混淆點

### 5.1 行內樣式的優先權高，是不是代表最好用？

不是。

優先權高只代表它在樣式覆蓋上比較強，不代表它適合長期維護。

正式開發選擇樣式寫法時，不應該只看優先權，而要看：

- 是否需要複用
- 是否容易修改
- 是否容易搜尋
- 是否會讓 HTML 變得混亂
- 是否符合團隊的樣式管理方式

所以行內樣式可以用，但不應該大量用來管理專案樣式。

### 5.2 內部樣式表能不能複用？

可以，但通常只限於「同一個頁面內」複用。

例如同一頁有很多 `div`、`p` 或同一種 class，都可以被 `<style>` 裡的 CSS 控制。

但如果是多個 HTML 頁面都要使用同一套樣式，外部樣式表會更合適。

### 5.3 外部樣式表是不是一定比較慢？

不一定。

外部樣式表需要額外載入 CSS 檔案，這是它的成本。但在正式專案中，它通常有更大的維護優勢，而且瀏覽器可以快取外部 CSS 檔案。

在現代前端專案中，Vite、Webpack 等建構工具也常會對 CSS 進行打包、壓縮、拆分或快取處理。

所以不能只用「多一個檔案」來判斷外部樣式表不好。正式開發更重要的是整體可維護性。

### 5.4 `<style>` 和 `<link>` 差在哪裡？

| 寫法 | 作用 |
| --- | --- |
| `<style>` | 直接在 HTML 文件內寫 CSS |
| `<link>` | 從外部載入獨立的 CSS 檔案 |

簡單記法：

> `<style>` 是「直接寫 CSS」，`<link>` 是「連到 CSS 檔案」。

### 5.5 Vue 裡的 `<style>` 算哪一種？

在 Vue 單檔元件 `.vue` 裡，常會看到這種寫法：

```vue
<template>
    <div class="card">內容</div>
</template>

<style scoped>
.card {
    padding: 16px;
}
</style>
```

它看起來像內部樣式表，但在 Vue 專案中，這段樣式通常會被 Vite 或其他建構工具處理後再輸出到瀏覽器。

所以學習 HTML 階段可以先把它理解成「元件內的樣式區塊」，但實務上它已經不是單純手寫 HTML 裡的 `<style>` 用法。

這裡先記住核心原則即可：

> 不管是一般 HTML、Vue、React 或其他框架，樣式都應該被妥善組織，避免大量散落在行內樣式中。

## 6. CSS 沒有生效時的檢查清單

如果使用外部樣式表後，畫面沒有套用 CSS，可以先依序檢查：

1. HTML 是否有正確寫入 `<link rel="stylesheet" href="...">`
2. `href` 路徑是否正確
3. CSS 檔案名稱是否拼錯
4. CSS 檔案是否真的存在
5. CSS 檔案副檔名是否為 `.css`
6. `<link>` 是否放在你目前開啟的 HTML 文件中
7. CSS 選擇器是否真的選到目標元素
8. 是否被其他 CSS 規則覆蓋
9. 瀏覽器是否因快取而沒有更新
10. 開發者工具 Network 面板中，CSS 檔案是否成功載入

初學時最常見的是：**路徑寫錯** 或 **選擇器沒有選到元素**。

## 7. 30 秒複習

- 行內樣式：寫在 `style` 屬性中，只影響當前元素，適合臨時測試，不適合大量使用。
- 內部樣式表：寫在 `<style>` 標籤中，適合同一頁面的練習與 demo。
- 外部樣式表：寫在獨立 `.css` 檔案中，再用 `<link>` 引入，最適合正式專案。

如果只記一句話：

> 能用外部樣式表管理的樣式，通常就不要大量寫進 HTML 裡。

## 8. 小練習

### 練習一：比較三種寫法

建立一個 `index.html`，放三段文字：

```html
<p>第一段文字</p>
<p>第二段文字</p>
<p>第三段文字</p>
```

分別用以下三種方式讓文字變成粉紅色：

1. 行內樣式
2. 內部樣式表
3. 外部樣式表

完成後思考：

- 哪一種最快？
- 哪一種最容易重複使用？
- 如果要把三段文字全部改成藍色，哪一種最容易維護？

### 練習二：故意讓外部 CSS 失效

先正確引入 CSS：

```html
<link rel="stylesheet" href="./style.css">
```

再故意把路徑改錯：

```html
<link rel="stylesheet" href="./styles/style.css">
```

觀察畫面變化，並用瀏覽器開發者工具確認 CSS 檔案是否載入失敗。

這個練習可以幫你建立一個重要觀念：

> CSS 沒有生效時，不一定是 CSS 寫錯，也可能是檔案根本沒有成功載入。

## 本章總結

本章的重點不是背三個名詞，而是理解 CSS 管理方式的差異。

真正的判斷核心是：

- 樣式是否只影響單一元素？
- 樣式是否需要在同一頁重複使用？
- 樣式是否需要跨頁共用？
- 專案是否需要長期維護？
- 是否需要讓 HTML 結構與 CSS 樣式保持分離？

對初學者來說，三種方式都要會；但對正式開發來說，通常應該優先使用外部樣式表或由前端工具管理的樣式檔案。
