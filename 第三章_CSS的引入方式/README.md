# 第三章 CSS 的引入方式

本章整理 CSS 在 HTML 中常見的三種使用方式：**行內樣式**、**內部樣式表**、**外部樣式表**。

這一章的重點不是死記名稱，而是回答一個實務問題：

> 樣式應該寫在哪裡，之後才好維護、好複用、好回查？

## 本章核心結論

- **行內樣式**：適合臨時測試或單一元素微調，不適合大量使用。
- **內部樣式表**：適合單頁練習、教學範例或小型 demo。
- **外部樣式表**：最適合正式開發，因為樣式可以複用、檔案結構清楚，也較方便多人協作。

一句話記法：

> 單一元素用行內，單一頁面用內部，多頁面與正式專案用外部。

## 本章和整體學習地圖的關係

- 前面學 CSS 基礎觀念時，主要是在理解「CSS 負責控制畫面樣式」。
- 本章進一步回答「CSS 要放在哪裡」以及「HTML 如何載入 CSS」。
- 先理解引入方式，再學選擇器、屬性、盒模型、排版時，會更清楚樣式是如何被組織與管理的。
- 這一章也是從「能寫出效果」走向「能維護專案」的重要起點。

## 適合快速回查的情境

- 忘了行內樣式、內部樣式表、外部樣式表的差異
- 忘了 `<style>` 和 `<link>` 的基本寫法
- 想判斷正式開發中應該使用哪一種 CSS 引入方式
- CSS 沒有生效時，想檢查是不是 `<link>` 路徑或檔案引入錯誤
- 想回想「為什麼不建議大量使用行內樣式」
- 想直接打開範例比較三種引入方式時

## 學完本章後應該具備的能力

讀完本章後，你應該能做到：

- 看懂 `style="..."`、`<style>`、`<link rel="stylesheet">` 分別代表什麼
- 根據使用場景判斷該用哪一種 CSS 引入方式
- 說明為什麼正式專案通常優先使用外部樣式表
- 知道行內樣式優先權較高，但不代表它最適合維護
- 遇到 CSS 沒有生效時，能先檢查檔案路徑、引入位置與選擇器是否正確
