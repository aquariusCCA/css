# 問題對應檢查表

來源筆記：`notes/第05章_瀏覽器的私有前綴/瀏覽器的私有前綴.md`

用途：遇到私有前綴閱讀、排序或維護問題時，快速判斷應優先檢查哪個條件。

| 畫面問題 | 優先檢查 | 建議檢查或語法 | 判斷理由 | 來源 |
|---|---|---|---|---|
| 同一個圓角效果出現好幾條相似宣告 | 哪些是前綴版本，哪一條是標準版本 | `-moz-border-radius`、`-webkit-border-radius`、`border-radius` | 前綴是 property name 的一部分，通常服務同一功能的相容寫法 | [先看問題](../../notes/第05章_瀏覽器的私有前綴/瀏覽器的私有前綴.md#先看問題同一個圓角效果為什麼會出現好幾條相似宣告) |
| 不知道某條前綴宣告控制哪個功能 | 前綴後面的標準 property name 是什麼 | `-webkit-transform` → `transform` | 前綴指出廠商或瀏覽器家族，後面的屬性名稱才指出 CSS 功能 | [第三課](../../notes/第05章_瀏覽器的私有前綴/瀏覽器的私有前綴.md#第三課看到--webkit--moz--ms--o--時應該先判斷什麼) |
| 需要同時保留前綴和標準寫法 | 標準屬性是否放在最後 | 前綴版本 → 標準版本 | 標準寫法應成為最後且主要的宣告 | [第二課](../../notes/第05章_瀏覽器的私有前綴/瀏覽器的私有前綴.md#第二課加前綴時為什麼標準寫法要放在最後) |
| 只寫標準屬性是否足夠 | 目標瀏覽器是否已支援標準寫法 | `border-radius`、`transform`、`transition` | 已支援標準寫法的瀏覽器可優先使用標準屬性 | [第一課](../../notes/第05章_瀏覽器的私有前綴/瀏覽器的私有前綴.md#第一課為什麼新-css-功能曾經需要私有前綴) |
| 舊環境可能看不到預期效果 | 是否需要特定前綴版本 | `-webkit-*`、`-moz-*`、`-ms-*`、`-o-*` | 某些舊版瀏覽器可能只理解前綴版本 | [第二課](../../notes/第05章_瀏覽器的私有前綴/瀏覽器的私有前綴.md#第二課加前綴時為什麼標準寫法要放在最後) |
| CSS 裡的前綴很多，想判斷是否同一功能 | 各宣告的值與標準屬性是否描述同一視覺效果 | `-webkit-transform`、`-ms-transform`、`transform` | 若值類似，可判斷是一組相容性候選寫法 | [第三課](../../notes/第05章_瀏覽器的私有前綴/瀏覽器的私有前綴.md#第三課看到--webkit--moz--ms--o--時應該先判斷什麼) |
| 不想每天手動查所有前綴 | 是否只是日常撰寫，還是有舊版支援需求 | 標準 CSS、建構工具、目標瀏覽器 | 來源方向是日常先寫標準 CSS，有需求再處理前綴 | [第四課](../../notes/第05章_瀏覽器的私有前綴/瀏覽器的私有前綴.md#第四課什麼時候可以不用手動處理私有前綴) |
| 想移除舊專案中的前綴 | 是否確認目前專案支援範圍 | 目標瀏覽器、標準屬性支援條件 | 前綴可能曾經服務特定舊環境，不能只因看見就刪 | [誤解五](../../notes/第05章_瀏覽器的私有前綴/瀏覽器的私有前綴.md#誤解五看到前綴就可以直接刪掉) |
| 不確定前綴要保留、補上或移除 | 是否依流程先看標準屬性與支援需求 | 標準屬性 → 同功能前綴 → 書寫順序 → 目標瀏覽器 | 本章流程以標準寫法為主線，相容寫法按需求保留 | [判斷流程](../../notes/第05章_瀏覽器的私有前綴/瀏覽器的私有前綴.md#私有前綴的判斷流程) |
| 想知道 `border-radius` 範例是否能當固定模板 | 範例目的是順序，還是現代相容需求 | `border-radius`、前綴版本、標準版本 | 來源範例用來說明順序，不代表現代專案一定要加前綴 | [誤解四](../../notes/第05章_瀏覽器的私有前綴/瀏覽器的私有前綴.md#誤解四來源範例中的-border-radius-代表現在一定要加前綴) |
