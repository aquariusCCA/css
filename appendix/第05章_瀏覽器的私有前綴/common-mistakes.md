| 誤解或錯誤寫法 | 錯誤原因 | 修正方式 | 相關概念 | 來源 |
|---|---|---|---|---|
| 私有前綴是 CSS 的必背語法清單 | 把相容性標記當成日常主要任務 | 先理解前綴目的，需要相容舊環境時再處理 | 私有前綴、標準屬性 | [誤解一](../../notes/第05章_瀏覽器的私有前綴/瀏覽器的私有前綴.md#誤解一私有前綴是-css-的必背語法清單) |
| 每個 CSS 屬性都要寫好幾個前綴版本 | 把前綴範例誤當固定模板 | 只有確定需要支援特定前綴版本時才補 | 前綴版本、目標瀏覽器 | [第二課](../../notes/第05章_瀏覽器的私有前綴/瀏覽器的私有前綴.md#第二課加前綴時為什麼標準寫法要放在最後) |
| 加了所有前綴就一定比較好 | 前綴不是越多越安全，會降低可讀性 | 依目標瀏覽器支援範圍決定是否保留 | 維護取捨、支援範圍 | [誤解二](../../notes/第05章_瀏覽器的私有前綴/瀏覽器的私有前綴.md#誤解二加了所有前綴就一定比較好) |
| 標準屬性可以省略，只寫前綴版本 | CSS 的主要意圖會變得不清楚 | 讓標準屬性成為最後且主要的寫法 | 標準版本、書寫順序 | [誤解三](../../notes/第05章_瀏覽器的私有前綴/瀏覽器的私有前綴.md#誤解三標準屬性可以省略只寫前綴版本) |
| `border-radius` 範例代表現在一定要加前綴 | 混淆範例目的與現代專案需求 | 把範例重點放在順序，是否加前綴另依支援條件判斷 | `border-radius`、前綴順序 | [誤解四](../../notes/第05章_瀏覽器的私有前綴/瀏覽器的私有前綴.md#誤解四來源範例中的-border-radius-代表現在一定要加前綴) |
| 看到 `-webkit-` 就代表 CSS 只能在 Safari 或 Chrome 使用 | 把前綴對應關係誤讀成唯一使用範圍 | 仍需看標準屬性、實際版本與專案支援資料 | `-webkit-`、標準屬性 | [第三課](../../notes/第05章_瀏覽器的私有前綴/瀏覽器的私有前綴.md#第三課看到--webkit--moz--ms--o--時應該先判斷什麼) |
| 看到前綴就可以直接刪掉 | 未確認前綴是否仍服務舊環境 | 先確認專案目標瀏覽器，再決定移除 | 舊專案、支援範圍 | [誤解五](../../notes/第05章_瀏覽器的私有前綴/瀏覽器的私有前綴.md#誤解五看到前綴就可以直接刪掉) |
| 不用關注前綴等於永遠不用做相容性判斷 | 忽略舊版瀏覽器或實驗性功能需求 | 有明確支援需求時仍要查目標環境 | 目標瀏覽器、實驗性功能 | [第四課](../../notes/第05章_瀏覽器的私有前綴/瀏覽器的私有前綴.md#第四課什麼時候可以不用手動處理私有前綴) |
