# 常見誤解速查

來源筆記：`notes/第03章_CSS的引入方式/CSS的引入方式.md`

用途：快速排查本章常見誤解、錯誤原因與修正方向。

| 誤解或錯誤寫法 | 錯誤原因 | 修正方式 | 相關概念 | 來源 |
|---|---|---|---|---|
| 以為三種引入方式只是寫法不同 | 忽略作用範圍與維護成本不同 | 依單一元素、單頁或多頁共用需求選擇 | 行內樣式表、內部樣式表、外部樣式表 | [誤解一](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#誤解一三種方式只是寫法不同效果都一樣) |
| 把行內樣式當成日常大量使用方式 | 樣式會分散在 HTML 各處，重複且難改 | 只在臨時測試或單一元素簡單修改時使用 | `style` attribute、維護成本 | [誤解四](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#誤解四行內樣式方便所以適合大量使用) |
| 以為行內樣式最直接就最適合開發 | 直接性會讓結構與視覺設定混在一起 | 長期維護時優先考慮外部樣式表 | 行內樣式表、結構與樣式分離 | [第一課](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第一課只想改一個標籤時行內樣式表如何工作) |
| 把 `<style>` 和 `style` 混在一起 | 一個是 HTML 標籤，一個是 HTML attribute | 用 `style="..."` 改單一元素；用 `<style>` 放頁面 CSS 規則 | `style` attribute、`<style>` 標籤 | [第二課](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第二課想控制整個頁面時內部樣式表如何集中樣式) |
| 以為內部樣式表已經完全分離 HTML 和 CSS | CSS 仍寫在同一份 HTML 文件內 | 多頁共用或獨立維護時改用外部樣式表 | 內部樣式表、外部樣式表 | [誤解二](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#誤解二內部樣式表已經完全分離-html-和-css) |
| 以為內部樣式表會自動套用到其他頁面 | `<style>` 只屬於當前 HTML 文件 | 需要跨頁共用時，讓多頁引入同一份 `.css` 檔案 | `<style>`、作用範圍 | [第二課](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第二課想控制整個頁面時內部樣式表如何集中樣式) |
| 把 `<link>` 放在 `<body>` 內容中當一般連結 | 本章教學與實務上樣式表通常放在 `<head>` | 在 `<head>` 中寫 `<link rel="stylesheet" href="...">` | `<link>`、`<head>` | [第三課](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第三課多個頁面要共用樣式時外部樣式表如何分離結構與樣式) |
| 外部 CSS 沒效果就只改 CSS 規則 | 可能是 CSS 檔案根本沒有被 HTML 載入 | 檢查 `<link>`、`rel="stylesheet"` 與 `href` 路徑 | 外部樣式表、載入檢查 | [誤解三](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#誤解三外部-css-沒效果一定是-css-規則寫錯) |
| 以為外部樣式表會自動讓畫面更漂亮 | 引入方式只決定 CSS 放哪裡與如何載入 | 把外部樣式表視為維護與共用方式，設計品質仍取決於規則 | 外部樣式表、CSS 規則 | [誤解五](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#誤解五外部樣式表一定會讓畫面比較漂亮) |
| 只檢查 selector 和 property，不檢查 CSS 是否被讀到 | 忽略引入方式本身也可能失敗 | 先確認樣式在哪裡、作用範圍與 HTML 是否成功讀取 CSS | 載入檢查、作用範圍 | [實務使用情境](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#實務使用情境) |
