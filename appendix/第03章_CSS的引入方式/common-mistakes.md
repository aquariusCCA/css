# 常見誤解速查

來源筆記：`notes/第03章_CSS的引入方式/CSS的引入方式.md`

用途：快速排查本章常見錯誤理解、失效原因與修正方向。

| 誤解或錯誤寫法 | 錯誤原因 | 修正方式 | 相關概念 | 來源 |
|---|---|---|---|---|
| 三種引入方式只是寫法不同 | 作用範圍與維護結果不同 | 依單一元素、單頁或多頁共用來選擇 | 行內樣式表、內部樣式表、外部樣式表 | [誤解一](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#誤解一三種方式只是寫法不同效果都一樣) |
| 內部樣式表已經完全分離 HTML 和 CSS | CSS 仍寫在同一份 HTML 文件內 | 需要跨頁共用或獨立維護時改用外部樣式表 | 內部樣式表、外部樣式表 | [誤解二](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#誤解二內部樣式表已經完全分離-html-和-css) |
| 外部 CSS 沒效果一定是 CSS 規則寫錯 | 外部樣式表需要先被 HTML 成功載入 | 檢查 `<link>`、`rel="stylesheet"` 與 `href` 路徑 | 外部樣式表、`rel`、`href` | [誤解三](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#誤解三外部-css-沒效果一定是-css-規則寫錯) |
| 行內樣式方便，所以適合大量使用 | 樣式會分散在 HTML 各處，重複且難維護 | 只在臨時測試或單一元素局部修改時使用 | 行內樣式表、CSS 優先級 | [誤解四](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#誤解四行內樣式方便所以適合大量使用) |
| 外部樣式表一定會讓畫面比較漂亮 | 引入方式不會自動改善設計品質 | 把外部樣式表理解成維護方式，不是設計效果保證 | 外部樣式表、維護性 | [誤解五](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#誤解五外部樣式表一定會讓畫面比較漂亮) |
| 把 `<style>` 和 `style` 當成同一件事 | 一個是 HTML 標籤，一個是 HTML attribute | 用作用範圍區分：`style` 影響單一元素，`<style>` 影響當前頁面 | `style` attribute、`<style>` | [內部樣式表](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第二課想控制整個頁面時內部樣式表如何集中樣式) |
| 把 `<link>` 寫在 `<body>` 當一般連結 | 樣式表載入語法通常放在 `<head>` | 在 `<head>` 使用 `<link rel="stylesheet" href="...">` | `<link>`、外部樣式表 | [外部樣式表](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第三課多個頁面要共用樣式時外部樣式表如何分離結構與樣式) |
| 只檢查 selector，不檢查 `href` 路徑 | 路徑錯誤時 CSS 檔案根本沒有被讀取 | 先確認 HTML 是否成功載入那份 CSS 檔案 | `href`、外部樣式表 | [外部樣式表](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第三課多個頁面要共用樣式時外部樣式表如何分離結構與樣式) |
