## 先從這裡查

| 讀者想查 | 先看檔案 | 適合查什麼 |
|---|---|---|
| 不知道該用哪一種 CSS 引入方式 | `decision-table.md` | 從單一元素、單頁或多頁共用需求反查選擇方式 |
| 看不懂 `style`、`<style>` 與 `<link>` 差異 | `concept-glossary.md` | 查三種語法角色、作用範圍與維護差異 |
| CSS 寫好了但頁面沒有變 | `decision-table.md` | 反查 CSS 是否被 HTML 讀到、作用範圍是否正確 |
| 擔心把方便寫法用成長期方案 | `common-mistakes.md` | 查常見誤解、錯誤原因與修正方向 |
| 想用關鍵字快速回找資料 | `keyword-index.md` | 從語法、問題詞或同義詞定位查表檔 |

## 核心總覽

| 核心項目 | 角色或作用範圍 | 適合情境 | 注意或限制 | 來源 |
|---|---|---|---|---|
| 行內樣式表 | CSS 寫在單一 HTML 標籤的 `style` attribute | 臨時測試或單一元素的簡單局部修改 | 不適合大量撰寫或長期維護 | [第一課](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第一課只想改一個標籤時行內樣式表如何工作) |
| 內部樣式表 | CSS 寫在同一份 HTML 文件的 `<style>` 標籤中 | 單頁練習、簡單範例或只服務當前頁面 | 不會自動分享給其他 HTML 文件 | [第二課](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第二課想控制整個頁面時內部樣式表如何集中樣式) |
| 外部樣式表 | CSS 寫在獨立 `.css` 檔案，HTML 用 `<link>` 引入 | 多個頁面共用樣式或長期維護 | 要確認 `rel` 與 `href` 路徑讓 CSS 成功載入 | [第三課](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第三課多個頁面要共用樣式時外部樣式表如何分離結構與樣式) |
| 引入方式判斷 | 依 CSS 和 HTML 的距離選擇方式 | 決定樣式放在元素、頁面或外部檔案 | 不只看眼前畫面，也要看維護成本 | [判斷流程](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#三種引入方式的判斷流程) |
| 載入檢查 | 確認 CSS 是否真的進入 HTML 頁面 | CSS 寫了但畫面沒有變時 | 外部樣式表要多檢查 `<link>`、`rel` 與 `href` | [誤解三](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#誤解三外部-css-沒效果一定是-css-規則寫錯) |
