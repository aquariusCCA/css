# 問題對應檢查表

來源筆記：`notes/第03章_CSS的引入方式/CSS的引入方式.md`

用途：遇到引入方式選擇或 CSS 載入問題時，快速判斷應優先檢查哪個語法角色或條件。

| 畫面問題 | 優先檢查 | 建議檢查或語法 | 判斷理由 | 來源 |
|---|---|---|---|---|
| 只想快速改一個元素 | 是否只影響單一標籤 | `style="color: pink; font-size: 20px;"` | 行內樣式表只作用在寫了 `style` 的元素上 | [第一課](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第一課只想改一個標籤時行內樣式表如何工作) |
| 同一頁很多元素要套用同樣樣式 | 是否只服務當前 HTML 文件 | `<style>...</style>` | 內部樣式表能在同一頁集中撰寫規則 | [第二課](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第二課想控制整個頁面時內部樣式表如何集中樣式) |
| 多個頁面要共用同一套樣式 | 是否需要跨頁共用與長期維護 | `<link rel="stylesheet" href="./style.css">` | 外部樣式表可讓多個 HTML 引入同一份 CSS | [第三課](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第三課多個頁面要共用樣式時外部樣式表如何分離結構與樣式) |
| 不想在每個 `<div>` 重複寫 `style` | 是否可以集中規則 | `<style>` 中寫 `div { ... }` | 內部樣式表可讓當前頁面符合 selector 的元素一起套用 | [第二課](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第二課想控制整個頁面時內部樣式表如何集中樣式) |
| CSS 寫好了但頁面沒有變 | CSS 是否真的進入 HTML 頁面 | 樣式放置位置、作用範圍、HTML 是否讀到 CSS | 本章提醒先確認 CSS 被載入，再查 selector 與 property | [先看問題](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#先看問題css-寫好了頁面為什麼沒有變) |
| 外部 CSS 沒有效果 | `<link>` 是否存在且設定正確 | `<link>`、`rel="stylesheet"`、`href="./style.css"` | 外部樣式表多了載入步驟，路徑錯就不會套用 | [誤解三](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#誤解三外部-css-沒效果一定是-css-規則寫錯) |
| HTML 內容和樣式混在一起太多 | 是否使用大量行內樣式 | 改用內部或外部樣式表 | 行內樣式大量使用會重複、難找、難改 | [第一課](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第一課只想改一個標籤時行內樣式表如何工作) |
| 單頁練習想讓 HTML 和 CSS 放一起觀察 | 是否只是課堂練習或簡單範例 | `<style>` 放在 `<head>` | 內部樣式表常用於單頁練習，方便觀察範例 | [實務使用情境](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#實務使用情境) |
| 正式專案需要整理、搜尋、重用 CSS | 是否應獨立成 `.css` 檔案 | 外部樣式表 | 筆記指出正式專案通常使用外部樣式表 | [實務使用情境](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#實務使用情境) |
| 不確定三種方式差在哪裡 | 比較 CSS 和 HTML 的距離 | 行內 → 內部 → 外部 | 三種方式差在作用範圍、分離程度與維護結果 | [判斷流程](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#三種引入方式的判斷流程) |
