# 問題對應屬性表

來源筆記：`notes/第03章_CSS的引入方式/CSS的引入方式.md`

用途：遇到 CSS 引入、作用範圍或載入問題時，快速判斷應優先檢查哪個條件。

| 畫面問題 | 優先檢查 | 建議屬性 | 判斷理由 | 來源 |
|---|---|---|---|---|
| CSS 寫了但頁面沒有變 | CSS 是否真的進入 HTML 頁面 | `style` attribute、`<style>`、`<link>` | 瀏覽器要先讀到 CSS，才可能套用樣式 | [先看問題](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#先看問題css-寫好了頁面為什麼沒有變) |
| 只想快速修改一個元素 | 是否只有單一標籤需要局部樣式 | `style` attribute | 行內樣式直接作用在寫了 `style` 的元素上 | [行內樣式表](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第一課只想改一個標籤時行內樣式表如何工作) |
| 同一頁多個 `<div>` 都要變色 | 是否只服務同一份 HTML 文件 | `<style>`、selector | 內部樣式表可在本頁集中寫一次規則 | [內部樣式表](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第二課想控制整個頁面時內部樣式表如何集中樣式) |
| 多個頁面要共用同一套樣式 | 各頁是否能引入同一份 `.css` 檔案 | `<link rel="stylesheet" href="...">` | 外部樣式表可讓多個 HTML 頁面共用樣式 | [外部樣式表](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第三課多個頁面要共用樣式時外部樣式表如何分離結構與樣式) |
| 外部 CSS 沒有效果 | `<link>` 是否存在，`rel` 與 `href` 是否正確 | `rel`、`href` | 外部樣式表多了載入步驟，路徑錯誤會導致 CSS 沒被讀到 | [誤解三](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#誤解三外部-css-沒效果一定是-css-規則寫錯) |
| 文字沒有變粉紅或變大 | CSS 是否已載入並作用到目標元素 | `color`、`font-size` | 本章範例用這兩個 declaration 改變文字顏色與大小 | [行內樣式表](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第一課只想改一個標籤時行內樣式表如何工作) |
| 樣式之後可能要跨頁維護 | 是否需要共用、搜尋與集中修改 | `<link rel="stylesheet" href="...">` | 外部 CSS 檔案比逐頁或逐元素修改更可靠 | [實務使用情境](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#實務使用情境) |
