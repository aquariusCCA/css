# 名詞表

來源筆記：`notes/第03章_CSS的引入方式/CSS的引入方式.md`

用途：快速查找本章重要名詞、CSS 引入語法角色與初學者應掌握的判斷方式。

| 名詞 | 簡短定義 | 初學者理解方式 | 相關屬性或概念 | 來源 |
|---|---|---|---|---|
| CSS 引入方式 | 讓 CSS 進入 HTML 頁面並套用到元素上的方式 | 先問 CSS 放在哪裡，再問作用範圍多大 | 行內樣式表、內部樣式表、外部樣式表 | [學習目標](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#學習目標) |
| 行內樣式表 | CSS 寫在單一元素的 `style` attribute 中 | 樣式直接貼在目標標籤上，只影響那個元素 | `style` attribute、當前標籤 | [第一課](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第一課只想改一個標籤時行內樣式表如何工作) |
| `style` attribute | HTML 標籤上用來放行內 CSS 宣告的 attribute | 看 `style="..."` 裡是否放了 CSS 宣告 | 行內樣式表、declaration | [第一課](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第一課只想改一個標籤時行內樣式表如何工作) |
| 內部樣式表 | CSS 寫在同一份 HTML 文件的 `<style>` 標籤中 | 把當前頁面的 CSS 集中在文件裡 | `<style>`、當前頁面 | [第二課](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第二課想控制整個頁面時內部樣式表如何集中樣式) |
| `<style>` 標籤 | HTML 文件中用來放一整段 CSS 規則的標籤 | 通常放在 `<head>`，服務當前 HTML 文件 | 內部樣式表、`<head>` | [第二課](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第二課想控制整個頁面時內部樣式表如何集中樣式) |
| 外部樣式表 | CSS 寫在獨立 `.css` 檔案，再由 HTML 引入 | 多個 HTML 頁面可以共用同一份 CSS | `.css` 檔案、`<link>` | [第三課](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第三課多個頁面要共用樣式時外部樣式表如何分離結構與樣式) |
| `<link>` 標籤 | HTML 中用來連結外部資源的標籤 | 本章用它在 `<head>` 中引入外部 CSS 檔案 | `rel="stylesheet"`、`href` | [第三課](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第三課多個頁面要共用樣式時外部樣式表如何分離結構與樣式) |
| `rel="stylesheet"` | 表示外部資源是一份樣式表 | 檢查 HTML 是否把外部資源當成 CSS 載入 | `<link>`、外部樣式表 | [第三課](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第三課多個頁面要共用樣式時外部樣式表如何分離結構與樣式) |
| `href` | 在 `<link>` 中指出 CSS 檔案位置的 attribute | CSS 沒載入時，先看路徑是否指向正確檔案 | `<link>`、路徑檢查 | [第三課](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第三課多個頁面要共用樣式時外部樣式表如何分離結構與樣式) |
| 作用範圍 | 一段 CSS 會影響哪些 HTML 元素或頁面 | 行內看單一元素，內部看當前頁面，外部可跨頁共用 | 當前標籤、當前頁面、多個頁面 | [先看問題](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#先看問題css-寫好了頁面為什麼沒有變) |
| 結構與樣式分離 | 讓 HTML 專注內容結構，CSS 專注樣式規則 | 越接近外部樣式表，越利於多頁共用與維護 | 外部樣式表、維護性 | [第三課](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第三課多個頁面要共用樣式時外部樣式表如何分離結構與樣式) |
| 載入檢查 | 確認 HTML 是否成功讀到 CSS 的排查步驟 | CSS 沒效果時，不只查 selector，也查 CSS 是否被載入 | `<link>`、`rel`、`href` | [誤解三](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#誤解三外部-css-沒效果一定是-css-規則寫錯) |
