## 核心總覽

| 核心項目 | 角色或作用範圍 | 適合情境 | 注意或限制 | 來源 |
|---|---|---|---|---|
| 層疊性 | 同權重規則發生同屬性衝突時的取捨 | 判斷後寫的宣告是否覆蓋前面宣告 | 只處理衝突屬性，未衝突宣告仍會保留 | [第一課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第一課同一個選擇器重複設定時後面的規則為什麼會勝出) |
| 繼承性 | 子元素沒有直接設定時沿用父元素部分樣式 | 設定基礎文字樣式，減少重複 CSS | 文字相關屬性常繼承，盒模型與版面屬性通常不繼承 | [第二課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第二課子元素沒有自己設定樣式時為什麼會跟著父元素變化) |
| 直接指定 | CSS selector 直接選到目標元素並設定屬性 | 判斷元素自己是否已有競爭值 | 直接作用在元素上的樣式通常比繼承值優先 | [第三課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第三課看起來沒有繼承時真的代表繼承失效嗎) |
| specificity | 不同 selector 同時選到同一元素時的權重比較 | 判斷 ID、類、標籤等 selector 誰勝出 | 權重用四組位置比較，不是普通十進位加法 | [第四課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第四課不同選擇器同時選到同一元素時誰的優先級比較高) |
| 複合選擇器權重 | 將 selector 各部分依類型分組累加 | 判斷 `.nav li`、`ul li`、`li` 等規則競爭 | 累加不會進位，selector 長度不等於權重高 | [第五課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第五課複合選擇器的權重如何疊加) |
| 繼承值權重 | 子元素從父元素取得的值可先理解為權重 0 | 父元素 selector 很強但子元素仍用自己樣式時 | 父元素權重不會搬到子元素身上 | [第六課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第六課父元素權重很高為什麼子元素還是使用自己的樣式) |
| 整體判斷流程 | 從直接指定、特殊競爭、specificity、來源順序到繼承逐步判斷 | 除錯任何「最後樣式不是預期值」問題 | 不要只背 selector 順序，要先確認是否選到目標元素 | [整體判斷流程](../../notes/第04章_CSS三大特性/CSS三大特性.md#三大特性的整體判斷流程) |
