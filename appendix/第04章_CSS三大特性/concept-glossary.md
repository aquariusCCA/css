# 名詞表

來源筆記：`notes/第04章_CSS三大特性/CSS三大特性.md`

用途：快速查找本章重要名詞、規則競爭概念與初學者應掌握的判斷方式。

| 名詞 | 簡短定義 | 初學者理解方式 | 相關屬性或概念 | 來源 |
|---|---|---|---|---|
| CSS 三大特性 | 層疊性、繼承性與優先級的合稱 | 用來判斷同一元素最後採用哪個 CSS 值 | 層疊性、繼承性、優先級 | [學習目標](../../notes/第04章_CSS三大特性/CSS三大特性.md#學習目標) |
| 層疊性 | 同一元素同一屬性有多個候選值時的取捨規則 | 權重相同且屬性衝突時，後寫宣告勝出 | 來源順序、同權重衝突 | [第一課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第一課同一個選擇器重複設定時後面的規則為什麼會勝出) |
| 繼承性 | 子元素可從父元素取得部分 CSS 屬性的值 | 子元素自己沒設定時，先看屬性能不能從父元素沿用 | `color`、`font-*`、`line-height` | [第二課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第二課子元素沒有自己設定樣式時為什麼會跟著父元素變化) |
| 優先級 | 多個 selector 同時直接選到同一元素時的勝負比較 | 先比 selector 權重，再看來源順序 | specificity、selector | [第四課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第四課不同選擇器同時選到同一元素時誰的優先級比較高) |
| specificity | selector 的權重計算方式 | 用四組數字比較：行內、ID、類/屬性/偽類、標籤/偽元素 | ID 選擇器、類選擇器、標籤選擇器 | [第四課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第四課不同選擇器同時選到同一元素時誰的優先級比較高) |
| 直接指定 | CSS 規則直接匹配目標元素並設定屬性 | 先問這條 selector 選到的是不是目標元素自己 | 繼承值、selector 匹配 | [第三課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第三課看起來沒有繼承時真的代表繼承失效嗎) |
| 繼承值 | 子元素從父元素或祖先元素取得的 CSS 值 | 元素自己沒被直接設定時才回頭看繼承 | 繼承性、權重 0 | [第六課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第六課父元素權重很高為什麼子元素還是使用自己的樣式) |
| 來源順序 | CSS declaration 在來源中出現的先後位置 | 同權重、同屬性衝突時才看誰寫在後面 | 層疊性、後寫覆蓋 | [第一課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第一課同一個選擇器重複設定時後面的規則為什麼會勝出) |
| 可繼承屬性 | 子元素通常可沿用父元素值的屬性 | 本章以文字相關屬性作為主要記憶方向 | `color`、`font-*`、`text-*`、`line-height` | [第二課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第二課子元素沒有自己設定樣式時為什麼會跟著父元素變化) |
| 不常繼承屬性 | 子元素通常不會自動沿用父元素值的屬性 | 盒模型與版面屬性若自動繼承會讓版面失控 | `border`、`background`、`padding`、`margin`、`width`、`height` | [第二課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第二課子元素沒有自己設定樣式時為什麼會跟著父元素變化) |
| 無單位 `line-height` | 不帶單位的行高比例 | 繼承比例，子元素用自己的 `font-size` 重新計算 | `line-height`、`font-size` | [行高的繼承](../../notes/第04章_CSS三大特性/CSS三大特性.md#行高的繼承) |
| 瀏覽器預設樣式 | 瀏覽器直接套用在部分元素上的預設規則 | `a` 和 `h1` 看似沒繼承時，先檢查是否有直接預設樣式 | `a`、`h1`、直接指定 | [第三課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第三課看起來沒有繼承時真的代表繼承失效嗎) |
| 複合選擇器 | 由多個 selector 部分組成的 selector | 各部分權重分組累加，再由左到右比較 | `.nav li`、`ul li`、不進位 | [第五課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第五課複合選擇器的權重如何疊加) |
| `!important` | 會改變 declaration 競爭層級的標記 | 本章只作入門提醒，不當成 selector 權重本身 | 優先級、declaration | [第四課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第四課不同選擇器同時選到同一元素時誰的優先級比較高) |
