# 常見誤解速查

來源筆記：`notes/第04章_CSS三大特性/CSS三大特性.md`

用途：快速排查本章常見誤解、錯誤原因與修正方向。

| 誤解或錯誤寫法 | 錯誤原因 | 修正方式 | 相關概念 | 來源 |
|---|---|---|---|---|
| 後寫的 CSS 一定會覆蓋前面的 CSS | 忽略 specificity 與競爭條件 | 先比 specificity，同權重再看來源順序 | 層疊性、優先級 | [誤解一](../../notes/第04章_CSS三大特性/CSS三大特性.md#誤解一後寫的-css-一定會覆蓋前面的-css) |
| 後面的同 selector 會讓前面整段規則失效 | 把層疊誤解成整段規則覆蓋 | 只比較同一元素、同一屬性的衝突值 | 層疊性、來源順序 | [第一課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第一課同一個選擇器重複設定時後面的規則為什麼會勝出) |
| 父元素權重高，子元素就一定跟著使用 | 把父元素 specificity 搬到子元素身上 | 確認 CSS 是否直接選到目標子元素 | 繼承值、直接指定 | [誤解二](../../notes/第04章_CSS三大特性/CSS三大特性.md#誤解二父元素權重高子元素就一定會跟著使用) |
| 父元素有 `border`，子元素也會自動有邊框 | 誤以為所有屬性都會繼承 | 先判斷屬性是否通常可繼承 | 繼承性、盒模型屬性 | [第二課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第二課子元素沒有自己設定樣式時為什麼會跟著父元素變化) |
| 所有文字看起來相關的屬性都一定會繼承 | 把入門口訣當成完整規格表 | 不確定時查該屬性的繼承特性 | 可繼承屬性、文字屬性 | [誤解三](../../notes/第04章_CSS三大特性/CSS三大特性.md#誤解三所有文字看起來相關的屬性都一定會繼承) |
| `a` 和 `h1` 是不能繼承的元素 | 忽略瀏覽器預設樣式可能直接作用在元素上 | 直接選到 `a` 或 `h1` 設定需要的屬性 | 直接指定、瀏覽器預設樣式 | [誤解四](../../notes/第04章_CSS三大特性/CSS三大特性.md#誤解四a-和-h1-是不能繼承的元素) |
| 權重可以像普通數字一樣進位 | 把四組 specificity 當成十進位加總 | 由左到右比較四組位置 | specificity、不進位 | [誤解五](../../notes/第04章_CSS三大特性/CSS三大特性.md#誤解五權重會像普通數字一樣進位) |
| selector 寫得越長就一定越強 | 忽略 selector 類型所在位置 | 分別累加 ID、類、標籤等位置 | 複合選擇器、specificity | [第五課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第五課複合選擇器的權重如何疊加) |
| selector 權重高就一定有作用 | 忘記 selector 必須先匹配元素 | 先確認 HTML 結構和 selector 是否相符 | selector 匹配、優先級 | [誤解六](../../notes/第04章_CSS三大特性/CSS三大特性.md#誤解六selector-權重高就一定有作用) |
| 看到 `!important` 就以為父元素一定控制子元素 | 混淆父元素直接指定與子元素繼承值 | 若要控制子元素，寫能直接選到子元素的 selector | `!important`、繼承值 | [第六課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第六課父元素權重很高為什麼子元素還是使用自己的樣式) |
| 只看誰寫在下面，不看是否直接選到元素 | 漏掉 selector 匹配是參與競爭的前提 | 先確認目標元素是否被直接選中 | 直接指定、selector | [整體判斷流程](../../notes/第04章_CSS三大特性/CSS三大特性.md#三大特性的整體判斷流程) |
