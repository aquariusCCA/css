# 問題對應檢查表

來源筆記：`notes/第04章_CSS三大特性/CSS三大特性.md`

用途：遇到最終樣式不符合預期時，快速判斷應優先檢查哪個 CSS 條件或規則競爭點。

| 畫面問題 | 優先檢查 | 建議檢查或語法 | 判斷理由 | 來源 |
|---|---|---|---|---|
| 同一段文字最後不是你以為的顏色 | 目標元素是否被多條規則影響 | 直接指定、specificity、來源順序、繼承 | 瀏覽器會替每個元素、每個屬性決定最後值 | [先看問題](../../notes/第04章_CSS三大特性/CSS三大特性.md#先看問題為什麼同一段文字最後不是你以為的顏色) |
| 同一個 selector 寫兩次，後面顏色生效 | 是否同權重且設定同一屬性 | 相同 selector、`color`、來源順序 | 權重相同且同屬性衝突時，後寫宣告勝出 | [第一課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第一課同一個選擇器重複設定時後面的規則為什麼會勝出) |
| 後面規則生效但前面某些樣式還在 | 哪些屬性真的發生衝突 | 比較同一元素、同一屬性 | 層疊只取捨衝突屬性，未衝突宣告仍保留 | [第一課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第一課同一個選擇器重複設定時後面的規則為什麼會勝出) |
| 子元素文字跟著父元素變色或變大 | 子元素是否沒有自己設定可繼承屬性 | `color`、`font-size`、父子結構 | 文字相關屬性通常可從父元素繼承 | [第二課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第二課子元素沒有自己設定樣式時為什麼會跟著父元素變化) |
| 父元素有邊框，子元素沒有同樣邊框 | 屬性是否屬於盒模型或版面類型 | `border`、`padding`、`margin` | 盒模型與版面屬性通常不會自動繼承 | [第二課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第二課子元素沒有自己設定樣式時為什麼會跟著父元素變化) |
| 不同字級的子元素行高仍保持比例 | 父元素是否使用無單位 `line-height` | `line-height: 1.5;`、子元素 `font-size` | 無單位行高繼承比例，子元素用自己的字級計算 | [行高的繼承](../../notes/第04章_CSS三大特性/CSS三大特性.md#行高的繼承) |
| 父元素設定紅色，但連結仍像預設連結色 | 目標元素是否已有直接樣式 | `div a { color: red; }` | 直接作用在 `a` 上的樣式會比繼承值更優先 | [第三課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第三課看起來沒有繼承時真的代表繼承失效嗎) |
| 父元素設定小字，但 `h1` 還是很大 | `h1` 是否已有直接字級規則 | `div h1 { font-size: 12px; }` | `h1` 通常有瀏覽器預設標題字級直接作用在自己身上 | [第三課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第三課看起來沒有繼承時真的代表繼承失效嗎) |
| 類選擇器和標籤選擇器都選到元素 | selector specificity 誰比較高 | `.text` 對 `div` | 類選擇器權重高於標籤選擇器 | [第四課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第四課不同選擇器同時選到同一元素時誰的優先級比較高) |
| ID、類與標籤都設定同一個顏色 | 是否有 ID 選擇器直接選到元素 | `#demo`、`.text`、`div` | ID 選擇器權重高於類與標籤選擇器 | [第四課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第四課不同選擇器同時選到同一元素時誰的優先級比較高) |
| 很長的 selector 沒有贏過一個 class | 權重是否在不同位置比較 | `div div div...` 對 `.one` | 權重分組比較，不會從標籤位置進位到類位置 | [第五課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第五課複合選擇器的權重如何疊加) |
| `.nav li` 贏過 `ul li` 和 `li` | 複合 selector 含有哪些 selector 類型 | `.nav li`、`ul li`、`li` | 類選擇器所在位置高於標籤選擇器累加 | [第五課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第五課複合選擇器的權重如何疊加) |
| 父元素用 ID 加 `!important`，子元素仍是自己的顏色 | CSS 直接選到父元素還是子元素 | `#father` 對 `p` | 子元素自己的直接宣告會勝過從父元素來的繼承值 | [第六課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第六課父元素權重很高為什麼子元素還是使用自己的樣式) |
| 想讓父元素底下的 `p` 明確變色 | selector 是否直接匹配子元素 | `#father p { color: red; }` | 要控制子元素，應寫出能匹配子元素的 selector | [第六課](../../notes/第04章_CSS三大特性/CSS三大特性.md#第六課父元素權重很高為什麼子元素還是使用自己的樣式) |
| 不確定最終樣式該怎麼判斷 | 是否依序檢查競爭條件 | 直接指定 → `!important` → specificity → 來源順序 → 繼承 | 本章提供完整判斷流程 | [整體判斷流程](../../notes/第04章_CSS三大特性/CSS三大特性.md#三大特性的整體判斷流程) |
