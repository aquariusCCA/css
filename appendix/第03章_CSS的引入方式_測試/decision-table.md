# 問題對應檢查表

來源筆記：`notes/第03章_CSS的引入方式/CSS的引入方式.md`

用途：遇到 CSS 引入、作用範圍或載入問題時，用這份表快速決定先檢查哪裡。

| 讀者問題 | 優先檢查 | 建議檢查或語法 | 判斷理由 | 來源 |
|---|---|---|---|---|
| CSS 寫好了但頁面沒有變 | CSS 是否真的進入 HTML 頁面 | `style` attribute、`<style>`、`<link>` | 瀏覽器要先讀到 CSS，才可能套用樣式 | [先看問題](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#先看問題css-寫好了頁面為什麼沒有變) |
| 只想快速改一個元素 | 是否只有單一標籤需要局部樣式 | `style="..."` | 行內樣式直接作用在寫了 `style` 的元素上 | [行內樣式表](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第一課只想改一個標籤時行內樣式表如何工作) |
| 同一頁很多元素要套同一套樣式 | 是否只服務同一份 HTML 文件 | `<style>...</style>` | 內部樣式表可以在本頁集中寫一次規則 | [內部樣式表](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第二課想控制整個頁面時內部樣式表如何集中樣式) |
| 多個頁面要共用樣式 | 各頁是否能引入同一份 `.css` 檔案 | `<link rel="stylesheet" href="...">` | 外部樣式表能讓多個 HTML 頁面共用樣式 | [外部樣式表](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第三課多個頁面要共用樣式時外部樣式表如何分離結構與樣式) |
| 外部 CSS 沒有效果 | `<link>` 是否存在，`rel` 與 `href` 是否正確 | `rel="stylesheet"`、`href="./style.css"` | 外部樣式表多了載入步驟，路徑錯誤會導致 CSS 沒被讀到 | [誤解三](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#誤解三外部-css-沒效果一定是-css-規則寫錯) |
| 不確定三種方式怎麼選 | 樣式要影響單一元素、當前頁面或多個頁面 | 行內樣式表、內部樣式表、外部樣式表 | 三種方式的差異在作用範圍與維護結果 | [判斷流程](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#三種引入方式的判斷流程) |
| 樣式之後可能要長期維護 | 是否需要共用、搜尋與集中修改 | 外部樣式表 | 修改獨立 CSS 檔案比逐頁或逐元素修改可靠 | [實務使用情境](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#實務使用情境) |
