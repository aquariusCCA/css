# 名詞表

來源筆記：`notes/第03章_CSS的引入方式/CSS的引入方式.md`

用途：想確認本章引入方式、作用範圍或相關 HTML 語法角色時查這份。

| 名詞 | 簡短定義 | 初學者理解方式 | 相關屬性或概念 | 來源 |
|---|---|---|---|---|
| CSS 引入方式 | 讓 CSS 進入 HTML 頁面的方式 | 先問 CSS 放在哪裡，再問它影響哪些元素或頁面 | 行內樣式表、內部樣式表、外部樣式表 | [本章範圍](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#本章範圍) |
| 行內樣式表 | 寫在單一 HTML 標籤 `style` attribute 裡的 CSS | 樣式直接貼在目標元素上，只影響該標籤 | `style` attribute、當前標籤 | [行內樣式表](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第一課只想改一個標籤時行內樣式表如何工作) |
| 內部樣式表 | 寫在同一份 HTML 文件 `<style>` 標籤中的 CSS | 可以集中管理同一頁的規則，但仍留在該 HTML 裡 | `<style>`、selector、當前頁面 | [內部樣式表](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第二課想控制整個頁面時內部樣式表如何集中樣式) |
| 外部樣式表 | 寫在獨立 `.css` 檔案，再由 HTML 引入的 CSS | 多個 HTML 頁面可以共用同一份樣式檔 | `<link>`、`rel`、`href`、`.css` | [外部樣式表](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第三課多個頁面要共用樣式時外部樣式表如何分離結構與樣式) |
| `style` attribute | 寫在 HTML 開始標籤上的樣式 attribute | 不需要 selector，但只作用在寫了 `style` 的元素 | 行內樣式表、當前標籤 | [行內樣式表](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第一課只想改一個標籤時行內樣式表如何工作) |
| `<style>` | 在 HTML 文件中放置 CSS 規則的標籤 | 通常放在 `<head>`，用來管理當前頁面的 CSS | 內部樣式表、selector | [內部樣式表](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第二課想控制整個頁面時內部樣式表如何集中樣式) |
| `<link rel="stylesheet">` | 讓 HTML 文件載入外部 CSS 檔案的標籤 | HTML 透過它找到外部 `.css` 檔案並套用規則 | 外部樣式表、`rel`、`href` | [外部樣式表](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第三課多個頁面要共用樣式時外部樣式表如何分離結構與樣式) |
| `rel="stylesheet"` | 表示外部資源是一份樣式表的 attribute 值 | 外部 CSS 載入語法中需要確認的角色標示 | `<link>`、外部樣式表 | [外部樣式表](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第三課多個頁面要共用樣式時外部樣式表如何分離結構與樣式) |
| `href` | 指出外部 CSS 檔案位置的 attribute | 路徑錯誤時，頁面會像沒有套用樣式 | `<link>`、外部樣式表、路徑 | [外部樣式表](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第三課多個頁面要共用樣式時外部樣式表如何分離結構與樣式) |
| 作用範圍 | 一段 CSS 會影響哪些元素或頁面 | 行內看當前標籤，內部看當前頁面，外部可跨多頁共用 | 當前標籤、當前頁面、多個頁面 | [先看問題](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#先看問題css-寫好了頁面為什麼沒有變) |
| 結構與樣式分離 | HTML 保留內容結構，CSS 獨立管理視覺規則 | 外部樣式表最符合這個維護方向 | 外部樣式表、維護性 | [外部樣式表](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第三課多個頁面要共用樣式時外部樣式表如何分離結構與樣式) |
