## 先從這裡查

| 讀者想查 | 先看檔案 | 適合查什麼 |

| 不知道現在該選哪種寫法 | `decision-table.md` | 從單一元素、單頁或多頁共用反查選擇方向 |
| CSS 寫了但頁面沒有變 | `decision-table.md` | 先確認 CSS 是否真的被 HTML 讀到 |
| 想確認 `style`、`<style>`、`<link>` 的角色 | `concept-glossary.md` | 查名詞、作用範圍與相關語法 |
| 擔心常見誤判 | `common-mistakes.md` | 查錯誤理解、失效原因與修正方向 |
| 只記得某個關鍵字 | `keyword-index.md` | 用關鍵字反查本章查表檔案 |

## 核心總覽

| 核心項目 | 角色或作用範圍 | 適合情境 | 注意或限制 | 來源 |
|---|---|---|---|---|
| 行內樣式表 | CSS 寫在單一 HTML 標籤的 `style` attribute，作用在當前標籤 | 臨時測試或單一元素的簡單修改 | 不適合大量使用，容易重複且難維護 | [行內樣式表](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第一課只想改一個標籤時行內樣式表如何工作) |
| 內部樣式表 | CSS 寫在同一份 HTML 的 `<style>` 標籤中，作用在當前頁面 | 單頁練習、簡單範例或只服務當前頁面的樣式 | 不會自動分享給其他 HTML 文件 | [內部樣式表](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第二課想控制整個頁面時內部樣式表如何集中樣式) |
| 外部樣式表 | CSS 寫在獨立 `.css` 檔案，再由 HTML 用 `<link>` 引入 | 多頁共用樣式或希望長期維護 | 要確認 `<link>`、`rel` 與 `href` 路徑是否正確 | [外部樣式表](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#第三課多個頁面要共用樣式時外部樣式表如何分離結構與樣式) |
| CSS 沒生效檢查 | 先確認 CSS 是否進入 HTML，再看作用對象 | 畫面沒有照預期變化時 | 不要一開始只檢查 selector 或 property | [先看問題](../../notes/第03章_CSS的引入方式/CSS的引入方式.md#先看問題css-寫好了頁面為什麼沒有變) |
