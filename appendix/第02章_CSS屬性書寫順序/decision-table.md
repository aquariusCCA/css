# 問題對應屬性表

來源筆記：`notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md`

用途：遇到 CSS 閱讀、維護或畫面排查問題時，快速判斷應優先檢查哪類屬性。

| 畫面問題 | 優先檢查 | 建議屬性 | 判斷理由 | 來源 |
|---|---|---|---|---|
| 一段 CSS 可以生效但很難讀 | declaration 是否依固定分類排序 | 待補來源 | 固定順序可降低來回掃描成本 | [先看問題](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#先看問題一段-css-能生效但很難閱讀) |
| 不確定元素如何參與版面 | 是否先看布局定位屬性 | `display`、`position`、`float`、`clear` | 這組屬性影響元素與其他元素互動 | [第一課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第一課先寫布局定位先判斷元素如何參與版面) |
| 不確定元素是否可見或內容是否溢出 | 是否檢查可見與溢出條件 | `visibility`、`overflow` | 筆記將可見與溢出列入布局定位判斷 | [第一課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第一課先寫布局定位先判斷元素如何參與版面) |
| 卡片太寬或大小不對 | 是否檢查自身盒子尺寸 | `width`、`height` | 自身屬性描述元素盒子大小 | [第二課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第二課再寫自身屬性建立元素盒子的大小與外觀底層) |
| 內容太貼邊 | 是否檢查內容與邊界距離 | `padding` | `padding` 控制內容和邊界之間的距離 | [第二課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第二課再寫自身屬性建立元素盒子的大小與外觀底層) |
| 卡片之間太擠 | 是否檢查元素外部距離 | `margin` | `margin` 控制元素外部距離 | [第二課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第二課再寫自身屬性建立元素盒子的大小與外觀底層) |
| 背景或邊框沒有出現 | 是否檢查自身屬性區段 | `background`、`border` | 背景與邊框屬於元素自身可視範圍 | [第二課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第二課再寫自身屬性建立元素盒子的大小與外觀底層) |
| 文字顏色或字體不對 | 是否檢查文本屬性區段 | `color`、`font`、`font-size` | 文本屬性處理文字呈現 | [第三課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第三課接著寫文本屬性處理內容文字的呈現) |
| 文字對齊或換行不如預期 | 是否先看容器，再看文字設定 | `text-align`、`white-space` | 文字結果可能受容器寬度、內距與溢出影響 | [第三課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第三課接著寫文本屬性處理內容文字的呈現) |
| 圓角、陰影或游標細節不對 | 是否檢查其他屬性區段 | `border-radius`、`box-shadow`、`cursor` | 這些屬性多作為裝飾或互動細節 | [第四課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第四課最後寫其他屬性處理裝飾互動與特殊內容) |
| 不確定屬性應放在哪一段 | 是否依排序表回查分類 | 待補來源 | 回查表提供四段排序模型 | [屬性書寫順序回查](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#屬性書寫順序回查) |
