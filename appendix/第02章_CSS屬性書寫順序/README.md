# 本章查表入口

來源筆記：`notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md`

用途：快速判斷本章 appendix 應該先查哪份資料。

## 先從這裡查

| 讀者想查 | 先看檔案 | 適合查什麼 |
|---|---|---|
| 不知道一段 CSS 應該先看哪裡 | `decision-table.md` | 從閱讀或除錯問題反查優先檢查區段 |
| 想查四大屬性群組與排序 | `properties-reference.md` | 查布局定位、自身、文本與其他屬性的歸類 |
| 想確認本章重要名詞 | `concept-glossary.md` | 查書寫順序、維護慣例、顯示模式與元素盒子 |
| CSS 可以生效但很難維護 | `decision-table.md` | 查如何把零散宣告整理成固定閱讀路徑 |
| 擔心把排序誤解成語法規則 | `common-mistakes.md` | 查常見誤解、錯誤原因與修正方向 |
| 想用關鍵字快速回找資料 | `keyword-index.md` | 從屬性名、問題詞或同義詞定位查表檔 |

## 核心總覽

| 核心項目 | 角色或作用範圍 | 適合情境 | 注意或限制 | 來源 |
|---|---|---|---|---|
| 屬性書寫順序 | 同一個 declaration block 內的閱讀與維護路徑 | declaration 數量變多、需要快速修改或除錯時 | 是維護慣例，不是瀏覽器要求的語法格式 | [小結](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#小結) |
| 布局定位屬性 | 先判斷元素如何顯示、定位、浮動、可見或處理溢出 | 位置、排列、顯示模式或內容裁切不如預期 | `display` 建議優先寫，但排序不會修正錯誤的屬性值 | [第一課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第一課先寫布局定位先判斷元素如何參與版面) |
| 自身屬性 | 描述元素盒子的寬高、內外距、邊框與背景 | 尺寸、間距、背景或邊框需要調整時 | `margin` 與 `padding` 同組，但效果不同 | [第二課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第二課再寫自身屬性建立元素盒子的大小與外觀底層) |
| 文本屬性 | 處理文字顏色、字體、裝飾、對齊與換行 | 文字閱讀體驗或呈現結果需要調整時 | 文字結果也可能受容器寬度、內距與溢出條件影響 | [第三課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第三課接著寫文本屬性處理內容文字的呈現) |
| 其他屬性 | 放置裝飾、互動提示或特殊內容相關宣告 | 圓角、陰影、游標或特殊效果需要調整時 | 不是不重要，也不是可以隨便亂放 | [第四課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第四課最後寫其他屬性處理裝飾互動與特殊內容) |
| 排序限制 | 區分維護順序與真正的 CSS 競爭規則 | 畫面結果與排序期待不一致時 | cascade、specificity、inheritance 與 source order 需另章深入 | [完整範例](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#完整範例把零散宣告整理成可維護順序) |
