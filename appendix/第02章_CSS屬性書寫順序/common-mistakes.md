# 常見誤解速查

來源筆記：`notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md`

用途：快速排查本章常見誤解、錯誤原因與修正方向。

| 誤解或錯誤寫法 | 錯誤原因 | 修正方式 | 相關概念 | 來源 |
|---|---|---|---|---|
| 以為屬性順序一定會改變畫面 | 本章排序多半是可讀性規範 | 把排序當成閱讀與維護工具 | 維護慣例、可讀性 | [誤解一](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#誤解一屬性順序一定會改變畫面) |
| 以為把 `display` 放前面就一定看得出差異 | 元素預設顯示模式可能已符合設定 | 用 `display` 先建立版面模型，不把它當成必然視覺變化 | `display`、顯示模式 | [第一課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第一課先寫布局定位先判斷元素如何參與版面) |
| 以為排序可以解決所有排版問題 | 排序只讓問題更容易被看見 | 屬性值用錯時，仍要回到該屬性的規則修正 | 布局定位屬性、除錯路徑 | [第一課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第一課先寫布局定位先判斷元素如何參與版面) |
| 分類後就不用理解各屬性 | 分類只幫助定位，不等於理解效果 | 查到區段後，仍要回到屬性本身的規則 | 屬性群組、屬性效果 | [誤解二](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#誤解二把屬性分類後就不用理解各屬性) |
| 把 `margin` 和 `padding` 當成同一種距離 | 兩者都和元素盒子有關，但作用空間不同 | 用外部距離理解 `margin`，用內容與邊界距離理解 `padding` | 自身屬性、元素盒子 | [第二課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第二課再寫自身屬性建立元素盒子的大小與外觀底層) |
| 以為 `background` 只屬於裝飾 | 本章來源把 `background` 放在自身屬性中 | 先把背景視為附著在元素盒子上的條件 | `background`、元素盒子 | [第二課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第二課再寫自身屬性建立元素盒子的大小與外觀底層) |
| 以為文本屬性只作用在文字節點本身 | 筆記提醒文字樣式可能套在元素上，再影響內部文字 | 檢查套用元素、容器寬度、內距與溢出條件 | 文本屬性、容器條件 | [第三課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第三課接著寫文本屬性處理內容文字的呈現) |
| 把 `vertical-align` 當成任何元素都能垂直置中 | 來源未展開其完整適用情境 | 本章只把它列入文本相關分類，不當作完整置中教學 | `vertical-align`、文本屬性 | [第三課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第三課接著寫文本屬性處理內容文字的呈現) |
| 以為其他屬性可以隨便放 | 「其他」只表示不屬於前面主結構判斷 | 仍可依裝飾、互動或特殊內容再細分 | 其他屬性、視覺裝飾 | [誤解四](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#誤解四其他屬性可以隨便放) |
| 把 `break-word` 或 `background:liner-gradient...` 當成完整語法教學 | 來源沒有提供完整上下文或穩定語法依據 | 只保留分類意圖；完整語法需另行確認 | 來源限制、待確認語法 | [誤解五](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#誤解五來源中的每個名稱都能直接當成完整語法) |
| 把屬性排序當成 CSS 生效順序 | 真正影響樣式競爭的是其他 CSS 規則 | 把本章排序視為維護順序，另學 cascade、specificity、inheritance 與 source order | 維護順序、source order | [完整範例](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#完整範例把零散宣告整理成可維護順序) |
