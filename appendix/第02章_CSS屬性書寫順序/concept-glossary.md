# 名詞表

來源筆記：`notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md`

用途：快速查找本章重要名詞、排序分類與初學者應掌握的判斷方式。

| 名詞 | 簡短定義 | 初學者理解方式 | 相關屬性或概念 | 來源 |
|---|---|---|---|---|
| CSS 屬性書寫順序 | 同一個 selector 中 declaration 的排列慣例 | 先用固定路線讀 CSS，而不是來回找屬性 | declaration、可讀性 | [本章範圍](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#本章範圍) |
| declaration | CSS 規則中的一條樣式宣告 | 本章討論的是多條 declaration 如何排序 | property、value | [學習目標](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#學習目標) |
| 布局定位屬性 | 先判斷元素如何參與版面的屬性群組 | 先看顯示模式、定位、浮動、可見與溢出 | `display`、`overflow` | [第一課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第一課先寫布局定位先判斷元素如何參與版面) |
| 顯示模式 | 元素如何在版面中呈現的基本角色 | 先看 `display`，再判斷尺寸與排列 | `display` | [第一課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第一課先寫布局定位先判斷元素如何參與版面) |
| 自身屬性 | 描述元素盒子大小、間距、邊框與背景的屬性群組 | 看元素多大、離外界多遠、內容離邊界多遠 | `width`、`margin`、`padding` | [第二課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第二課再寫自身屬性建立元素盒子的大小與外觀底層) |
| 元素盒子 | 元素自身佔用與呈現的範圍 | 用寬高、內外距、邊框與背景觀察 | `width`、`border`、`background` | [第二課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第二課再寫自身屬性建立元素盒子的大小與外觀底層) |
| 文本屬性 | 處理文字顏色、字體、裝飾、對齊與換行的屬性群組 | 先確認容器，再看容器內文字如何呈現 | `color`、`font`、`text-align` | [第三課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第三課接著寫文本屬性處理內容文字的呈現) |
| 其他屬性 | 裝飾、互動或特殊內容相關的補充屬性群組 | 在版面、盒子、文字清楚後再看細節 | `border-radius`、`box-shadow`、`cursor` | [第四課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第四課最後寫其他屬性處理裝飾互動與特殊內容) |
| 維護順序 | 方便人閱讀與修改的排序方式 | 不等於瀏覽器要求的語法順序 | 可讀性、source order | [完整範例](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#完整範例把零散宣告整理成可維護順序) |
| source order | 後出現的宣告可能影響結果的規則線索 | 本章只提醒，不展開完整 cascade | cascade、重複宣告 | [本章範圍](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#本章範圍) |
