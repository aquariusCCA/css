# 名詞表

來源筆記：`notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md`

用途：快速查找本章重要名詞、排序模型與初學者應掌握的判斷方式。

| 名詞 | 簡短定義 | 初學者理解方式 | 相關屬性或概念 | 來源 |
|---|---|---|---|---|
| 屬性書寫順序 | 同一個 CSS 規則中 declaration 的固定排列方式 | 用固定路線讀 CSS：先版面，再盒子，再文字，最後細節 | declaration、維護慣例 | [小結](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#小結) |
| 維護慣例 | 幫助人閱讀與修改 CSS 的約定 | 排序主要服務開發者，不是瀏覽器語法要求 | 可讀性、團隊協作 | [本章範圍](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#本章範圍) |
| declaration block | `{ ... }` 中放置多條 CSS 宣告的區塊 | 本章討論的是同一個區塊內宣告如何排列 | declaration、selector | [前置知識](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#前置知識) |
| 布局定位屬性 | 先判斷元素如何參與版面的屬性群組 | 第一眼先看元素如何顯示、定位、浮動與處理溢出 | `display`、`position`、`overflow` | [第一課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第一課先寫布局定位先判斷元素如何參與版面) |
| 自身屬性 | 描述元素盒子自身條件的屬性群組 | 看元素多大、離外界多遠、內容離邊界多遠 | `width`、`margin`、`padding` | [第二課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第二課再寫自身屬性建立元素盒子的大小與外觀底層) |
| 文本屬性 | 處理元素內文字呈現的屬性群組 | 容器條件清楚後，再看文字顏色、字體、對齊與換行 | `color`、`font`、`text-align` | [第三課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第三課接著寫文本屬性處理內容文字的呈現) |
| 其他屬性 | 放在排序後段的裝飾、互動或特殊內容宣告 | 先確定版面、盒子與文字，再看細節效果 | `border-radius`、`box-shadow`、`cursor` | [第四課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第四課最後寫其他屬性處理裝飾互動與特殊內容) |
| 顯示模式 | 元素以何種模式參與畫面排版 | 讀到 `display` 時，先建立元素的版面角色 | `display`、block、inline | [第一課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第一課先寫布局定位先判斷元素如何參與版面) |
| 元素盒子 | 元素自身大小、距離、邊框與背景的觀察對象 | 查尺寸與間距時，集中看自身屬性區段 | `width`、`height`、`border`、`background` | [第二課](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#第二課再寫自身屬性建立元素盒子的大小與外觀底層) |
| source order | 後出現宣告可能影響結果的 CSS 競爭相關概念 | 本章只提醒它存在，不把排序模型當成完整 cascade 教學 | cascade、specificity、inheritance | [完整範例](../../notes/第02章_CSS屬性書寫順序/CSS屬性書寫順序.md#完整範例把零散宣告整理成可維護順序) |
