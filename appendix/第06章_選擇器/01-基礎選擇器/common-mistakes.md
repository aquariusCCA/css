# 常見誤解速查

來源筆記：`notes/第06章_選擇器/01-基礎選擇器.md`

用途：快速排查本章常見 `selector` 誤解、選錯範圍與 HTML 對應錯誤。

| 誤解或錯誤寫法 | 錯誤原因 | 修正方式 | 相關概念 | 來源 |
|---|---|---|---|---|
| 只看 `color: red;`，不看前面的 `selector` | 宣告區塊不決定樣式套到誰身上 | 先問這條規則有沒有選到目標元素 | `selector`、`declaration block` | [常見誤解](../../../notes/第06章_選擇器/01-基礎選擇器.md#常見誤解與修正方式) |
| 以為 `p` 只會選到第一個 `<p>` | 誤解標籤選擇器的選取範圍 | 記住 `p` 會選到所有 `<p>` 元素 | 標籤選擇器 | [第一課](../../../notes/第06章_選擇器/01-基礎選擇器.md#第一課想一次改同一種標籤) |
| 只想改幾個 `<li>`，卻寫 `li { ... }` | 標籤選擇器會選到所有同標籤元素 | 替目標項目加 class，再用類選擇器 | 標籤選擇器、類選擇器 | [常見誤解](../../../notes/第06章_選擇器/01-基礎選擇器.md#常見誤解與修正方式) |
| HTML 寫 `class="important"`，CSS 寫 `important { ... }` | 少了 `.`，CSS 會把 `important` 當成標籤名稱 | 改寫成 `.important { ... }` | 類選擇器、`class` | [第二課](../../../notes/第06章_選擇器/01-基礎選擇器.md#第二課想只改某幾個元素) |
| 以為 class 只能用一次 | 混淆 class 與唯一元素的概念 | 同一個 class 可以被多個元素共用 | reusable style、類選擇器 | [第二課](../../../notes/第06章_選擇器/01-基礎選擇器.md#第二課想只改某幾個元素) |
| `class="red, large-text"` | HTML 的多個 class 不是用逗號分隔 | 改成 `class="red large-text"` | 多類名、`class` attribute | [第三課](../../../notes/第06章_選擇器/01-基礎選擇器.md#第三課想把共用樣式拆開重組) |
| 以為 HTML 中 class 的順序一定決定最後效果 | 把 class 列表順序當成本章主線判斷 | 先掌握多類名用空格組合；層疊、權重與來源順序另章處理 | 多類名、cascade、specificity | [第三課](../../../notes/第06章_選擇器/01-基礎選擇器.md#第三課想把共用樣式拆開重組) |
| 把 id 當成可重複樣式名稱 | id 用來表示頁面中的唯一元素 | 需要重複套用外觀時改用 class | id 選擇器、類選擇器 | [第四課](../../../notes/第06章_選擇器/01-基礎選擇器.md#第四課想選頁面中的唯一元素) |
| HTML 寫 `id="main-title"`，CSS 寫 `.main-title` | `.` 是 class 選擇器，不是 id 選擇器 | 改用 `#main-title` | id 選擇器、類選擇器 | [第四課](../../../notes/第06章_選擇器/01-基礎選擇器.md#第四課想選頁面中的唯一元素) |
| 把 `*` 當成方便的萬用解法 | 通配符選擇器會影響所有元素 | 只改局部時改用更精準的 `selector` | 通配符選擇器、過度選取 | [第五課](../../../notes/第06章_選擇器/01-基礎選擇器.md#第五課想對所有元素套用基礎規則) |
