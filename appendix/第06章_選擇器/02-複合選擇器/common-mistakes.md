# 常見誤解速查

來源筆記：`notes/第06章_選擇器/02-複合選擇器.md`

用途：快速排查本章常見 `selector` 符號誤解、選取範圍錯誤與維護風險。

| 誤解或錯誤寫法 | 錯誤原因 | 修正方式 | 相關概念 | 來源 |
|---|---|---|---|---|
| 把 `selector` 裡的空格看成可有可無 | 空格在 CSS `selector` 中有語意 | 區分 `.nav a`、`.nav > a` 與 `p.box` | 後代選擇器、子選擇器、交集選擇器 | [常見誤解](../../../notes/第06章_選擇器/02-複合選擇器.md#常見誤解與修正方式) |
| 以為 `.nav a` 只選直接子元素 | 空格表示後代關係，不限直接下一層 | 只選直接下一層時改用 `.nav > a` | 後代選擇器、子選擇器 | [第一課](../../../notes/第06章_選擇器/02-複合選擇器.md#第一課想選到某個區塊裡的所有後代元素) |
| 所有結構問題都用後代選擇器 | 後代選擇器範圍很大，可能選到深層元素 | 只想選第一層時使用 `>` | 後代選擇器、過度選取 | [常見誤解](../../../notes/第06章_選擇器/02-複合選擇器.md#常見誤解與修正方式) |
| `.menu a` 和 `.menu > a` 當成同一件事 | 混淆「在裡面」與「父元素正好是」 | 先看 HTML 是否多包一層，再選空格或 `>` | 子選擇器、直接父子關係 | [第二課](../../../notes/第06章_選擇器/02-複合選擇器.md#第二課想只選直接子元素) |
| 以為 `+` 或 `~` 可以往前選 | 兄弟選擇器只會從左邊元素往後找 | 只能選後方同層元素，不能影響前面元素 | 相鄰兄弟、通用兄弟 | [第三課](../../../notes/第06章_選擇器/02-複合選擇器.md#第三課想選同一層中後面的元素) |
| 忽略兄弟元素必須同一層 | 不同父元素下的元素不是兄弟關係 | 檢查兩個元素是否有同一個父元素 | 兄弟元素、同一父元素 | [第三課](../../../notes/第06章_選擇器/02-複合選擇器.md#第三課想選同一層中後面的元素) |
| `h2 p` 用來表示 `h2` 和 `p` | 忘記逗號會把 `selector` 變成後代關係 | 寫成 `h2, p` 才是並集選擇器 | 並集選擇器、後代選擇器 | [第四課](../../../notes/第06章_選擇器/02-複合選擇器.md#第四課想讓多種元素共用同一組樣式) |
| `p .box` 用來表示 `p` 同時有 `box` `class` | 加空格會變成選 `p` 裡面的 `.box` 後代 | 寫成 `p.box`，中間不要加空格 | 交集選擇器、後代選擇器 | [第五課](../../../notes/第06章_選擇器/02-複合選擇器.md#第五課想選同時符合多個條件的元素) |
| 交集選擇器寫成 `選擇器1選擇器2:` | 冒號不屬於交集選擇器語法 | 選擇器緊貼後直接接 `{ ... }` | 交集選擇器、語法修正 | [第五課](../../../notes/第06章_選擇器/02-複合選擇器.md#第五課想選同時符合多個條件的元素) |
| `selector` 越長就越精準、越好 | 過長 `selector` 增加維護成本，也容易受 HTML 結構變動影響 | 只寫必要條件，不把整條 DOM 路徑都寫入 CSS | `selector` 可讀性、維護性 | [常見誤解](../../../notes/第06章_選擇器/02-複合選擇器.md#常見誤解與修正方式) |
