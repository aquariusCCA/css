# 問題對應檢查表

來源筆記：`notes/第06章_選擇器/02-複合選擇器.md`

用途：遇到複合選擇器選取需求或樣式失效時，快速判斷應優先檢查哪種 HTML 關係或 `selector` 寫法。

| 畫面問題 | 優先檢查 | 建議檢查或語法 | 判斷理由 | 來源 |
|---|---|---|---|---|
| 單一基礎選擇器選太多或太少 | 需求是否需要描述結構、關係或多個條件 | 複合選擇器 | 複合選擇器用來讓 CSS 更精準描述要選誰 | [先看問題](../../../notes/第06章_選擇器/02-複合選擇器.md#先看問題基礎選擇器不一定夠精準) |
| 只想改導覽列中的連結 | 目標 `a` 是否位在 `.nav` 裡 | `.nav a` | 後代選擇器會選外層內所有層級的目標元素 | [第一課](../../../notes/第06章_選擇器/02-複合選擇器.md#第一課想選到某個區塊裡的所有後代元素) |
| 後代選擇器選到太深層的元素 | 是否只想選直接下一層 | `.menu > a` | 空格看內部任意層級，`>` 才看直接父子關係 | [第二課](../../../notes/第06章_選擇器/02-複合選擇器.md#第二課想只選直接子元素) |
| 只想選 `.menu` 第一層連結 | 目標元素的父元素是否正好是 `.menu` | `.menu > a` | 子選擇器只選直接子層，不選被其他元素包住的深層連結 | [第二課](../../../notes/第06章_選擇器/02-複合選擇器.md#第二課想只選直接子元素) |
| 想選標題後緊接的第一段 | `p` 是否緊接在 `h2` 後面且同層 | `h2 + p` | `+` 只選緊接在左邊元素後面的第一個符合元素 | [第三課](../../../notes/第06章_選擇器/02-複合選擇器.md#第三課想選同一層中後面的元素) |
| 想選標題後方所有同層段落 | 目標段落是否在 `h2` 後方且同層 | `h2 ~ p` | `~` 選左邊元素後方所有同層符合元素 | [第三課](../../../notes/第06章_選擇器/02-複合選擇器.md#第三課想選同一層中後面的元素) |
| 兄弟選擇器沒有選到元素 | 是否同一父元素，且目標是否在左邊元素後方 | `+`、`~` | 兄弟選擇器不能往前選，也不跨層選 | [第三課](../../../notes/第06章_選擇器/02-複合選擇器.md#第三課想選同一層中後面的元素) |
| 多種元素要共用同一組樣式 | 共同點是否是同一組宣告，而非結構關係 | `h2, p, .note li` | 並集選擇器讓多個 `selector` 共用同一個宣告區塊 | [第四課](../../../notes/第06章_選擇器/02-複合選擇器.md#第四課想讓多種元素共用同一組樣式) |
| `h2` 和 `p` 應一起選，卻變成找 `h2` 裡的 `p` | 是否漏掉逗號 | `h2, p` | `h2 p` 是後代選擇器，`h2, p` 才是並集選擇器 | [第四課](../../../notes/第06章_選擇器/02-複合選擇器.md#第四課想讓多種元素共用同一組樣式) |
| 只想選 `<p class="box">`，不要選 `<div class="box">` | 條件是否要套在同一個元素上 | `p.box` | 交集選擇器要求同一元素同時符合標籤與 `class` 條件 | [第五課](../../../notes/第06章_選擇器/02-複合選擇器.md#第五課想選同時符合多個條件的元素) |
| `p.box` 寫了空格後選取結果不同 | 是否把交集誤寫成後代選擇器 | `p.box`，不是 `p .box` | `p.box` 選同一元素；`p .box` 選 `p` 裡面的 `.box` 後代 | [第五課](../../../notes/第06章_選擇器/02-複合選擇器.md#第五課想選同時符合多個條件的元素) |
| `selector` 很長但仍不好維護 | 是否把整條 DOM 路徑都寫進 CSS | 只保留必要條件 | 來源提醒複合選擇器應表達必要條件，而不是追求看起來很精準 | [常見誤解](../../../notes/第06章_選擇器/02-複合選擇器.md#常見誤解與修正方式) |
