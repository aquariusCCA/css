# 從 CSS 筆記產生查表型資料

## 使用情境

當需要根據 `notes/<章節>/*.md` 中的正式教學筆記，產生 `appendix/<章節>/` 下的 CSS 查表型資料時，套用這份可重複流程。

本檔案負責「正式筆記 → 查表型資料」的處理流程，不負責定義角色，也不負責固定輸出格式。

---

## 必要輸入

- 筆記路徑：正式筆記所在路徑，可以是一個 `.md` 檔案或章節目錄。
- 章節名稱：本次 appendix 所屬章節名稱，應與 `notes/<章節>/` 保持一致。
- 預期輸出目錄：查表型資料要產生的位置，預設為 `appendix/<章節>/`。
- 查表類型：本次要產生的資料類型，預設先判斷章節內容後，產生全部適合類型。

預設查表類型包括：

- 章節總覽入口。
- CSS 屬性速查表。
- 屬性值對照表。
- 名詞表。
- 問題對應屬性或檢查表。
- 常見誤解速查。
- 關鍵字索引。

---

## 輸入檢查

1. 檢查筆記路徑、章節名稱、預期輸出目錄與查表類型是否完整。
2. 若必要輸入不足，先指出缺少的項目，不產生查表型資料。
3. 若筆記路徑是章節目錄，應讀取該目錄下與本章主題相關的 `.md` 檔案。
4. 若預期輸出目錄以 `.md` 結尾，應先指出輸入欄位與路徑型態不一致；除非任務明確要求單檔輸出，否則使用該路徑的父層資料夾作為輸出目錄。
5. 若筆記路徑無法讀取、內容不足或不是正式筆記，應停止產生，並說明原因。
6. 若查表類型包含未知類型，應忽略未知類型，並列出已忽略的項目。

---

## 來源限制

appendix 查表型資料只能整理 `notes/` 中已建立的知識。

- 可以根據正式筆記中的段落、表格、程式碼、常見誤解、自我檢查與來源說明抽取條目。
- 可以重組、去重、壓縮與交叉索引正式筆記內容。
- 不應補寫新的教學正文。
- 不應加入正式筆記沒有支持的新 CSS 規格、瀏覽器支援資訊、歷史脈絡或工具建議。
- 若某個條目需要補充才能成立，但筆記沒有支持，應標示為「待補來源」或略過。
- `origin/` 可作為追溯來源，但 appendix 的直接整理依據仍應是 `notes/`。

---

## 來源分析流程

正式產生查表型資料前，先完成來源分析。此分析可以作為內部工作步驟，不一定要完整輸出，除非任務明確要求。

### 1. 讀取筆記結構

讀取筆記後，辨識：

- 標題與章節層級。
- 學習目標。
- 本章範圍。
- 屬性、屬性值與語法片段。
- 名詞、心智模型與判斷流程。
- 範例中的 HTML 與 CSS。
- 常見誤解與修正方式。
- 實務使用情境。
- 自我檢查題。
- 來源與補充說明。
- 每個可抽取知識點所在的最近標題，標題層級可為 H2-H6。
- 辨識來源標題時，H1 視為整篇筆記主題標題，不作為單一條目的來源定位；每個候選條目應記錄正文中最直接支持該知識點的 H2-H6 標題。

### 2. 判斷章節查表適配

建立候選條目前，先判斷本章最適合哪些查表入口。不要假設每一章都必須產生所有查表檔。

依筆記主軸選擇一個或多個主題類型：

| 主題類型 | 判斷線索 | 優先查表 |
|---|---|---|
| 屬性密集型 | 章節主軸是多個 CSS 屬性的用途、條件與差異 | 章節總覽入口、CSS 屬性速查表、屬性值對照表、問題對應屬性表 |
| 值密集型 | 章節主軸是同一屬性下多個值的效果與差異 | 章節總覽入口、屬性值對照表、CSS 屬性速查表、常見誤解速查 |
| 概念判斷型 | 章節主軸是名詞、心智模型、選擇方式或作用範圍 | 章節總覽入口、名詞表、問題對應檢查表、常見誤解速查、關鍵字索引 |
| 語法結構型 | 章節主軸是語法部位、檔案結構或寫法組成 | 章節總覽入口、名詞表、問題對應檢查表、常見誤解速查 |
| 除錯流程型 | 章節主軸是失效原因、排查順序或判斷流程 | 章節總覽入口、問題對應檢查表、常見誤解速查、關鍵字索引 |
| 維護模型型 | 章節主軸是閱讀順序、組織方式、共用與長期維護 | 章節總覽入口、名詞表、問題對應檢查表、常見誤解速查 |

適配規則：

1. `README.md` 是章節 appendix 的預設入口。只要本次是產生整章 appendix，就應產生。
2. 若 CSS 屬性或屬性值只是範例材料，不是本章要讀者查找或比較的主軸，應省略 `properties-reference.md` 或 `values-reference.md`，或只在產出說明中指出未產生原因。
3. 若某個查表類型只能產生一兩筆範例條目，且這些條目不能幫助讀者理解本章主題，應省略該檔案。
4. 若使用者明確指定某個查表類型，但來源內容不足，應說明「來源不足以產生有查表價值的檔案」，不要為了湊齊檔案而填入低價值條目。
5. 概念判斷型、語法結構型、除錯流程型與維護模型型章節，應優先讓讀者先看總覽、判斷表與誤解表，再補名詞索引；不要把屬性表和值表當成固定主入口。
6. 每次省略預設查表類型時，應在最終回覆列出省略檔案與理由。

### 3. 建立候選條目

依照查表類型抽取候選內容：

| 查表類型 | 候選內容 |
|---|---|
| 章節總覽入口 | 本章查表路徑、核心對照、建議先看哪份表、各 appendix 檔用途 |
| CSS 屬性速查表 | CSS 屬性、作用對象、用途、常見值、預設值、注意事項 |
| 屬性值對照表 | 屬性值、所屬屬性、畫面效果、適用條件、容易混淆的值 |
| 名詞表 | CSS 名詞、版面模型詞彙、角色名稱、判斷概念 |
| 問題對應屬性或檢查表 | 畫面問題、除錯線索、優先檢查項目、建議屬性、建議檢查語法 |
| 常見誤解速查 | 錯誤理解、失效原因、正確修正方式 |
| 關鍵字索引 | 屬性名、值、名詞、常見問題、排版任務與同義搜尋詞 |

### 4. 清理與去重

整理候選條目時應：

1. 合併意思相同或高度重複的條目。
2. 保留最符合查表用途的短句。
3. 將教學段落壓縮成可快速掃描的表格或短條目。
4. 避免同一段教學內容在多份查表中完整重複。
5. 若同一資訊適合多份表，放在最有查找價值的位置，其他表只用關聯欄位指向。
6. 為每個候選條目記錄最直接支持它的來源標題。

### 5. 回溯來源

每個條目都應能回到來源筆記。

- 每份 appendix 檔案開頭應標明來源筆記路徑。
- 每個含有 CSS 知識條目的表格應包含 `來源` 欄，指向來源知識所在的最近標題；純導航表不需要 `來源` 欄。
- `來源` 欄應使用相對 Markdown 連結，連到 H2-H6 標題錨點。
- 若同一條目可對應多個標題，只選最直接支持該條目的標題。
- 若最近標題是重複或過度通用的標題，且錨點不易穩定判斷，應改用最近的唯一父標題。
- 條目內容應使用筆記已出現或已支持的概念。
- 若內容主要來自筆記中的「補充說明」，可在注意事項中標示。
- 若筆記沒有提供足夠依據，應標示「待補來源」或不輸出該條目。

---

## 產生流程

1. 依輸入讀取指定筆記。
2. 判斷章節主題類型與查表適配結果。
3. 決定本次真正要產生哪些查表類型，並記錄省略類型與理由。
4. 從筆記抽取屬性、值、名詞、問題、誤解、核心對照與關鍵字。
5. 對每種適合的查表類型建立候選條目。
6. 合併重複條目，刪除沒有來源依據或沒有查表價值的條目。
7. 為每個保留條目指定一個主要來源標題。
8. 依固定格式產生 `README.md` 與各 appendix 檔案。
9. 檢查每份檔案是否短、準、可掃描，且讀者能從入口檔知道先看哪裡。
10. 檢查產出內容是否沒有變成教學正文、摘要筆記或來源全文重排。
11. 將完成內容寫入預期輸出目錄。

---

## 檔案命名

預設輸出目錄：

```text
appendix/<章節>/
```

預設檔案名稱：

```text
README.md
properties-reference.md
values-reference.md
concept-glossary.md
decision-table.md
common-mistakes.md
keyword-index.md
```

若本次只產生部分查表類型，只建立對應檔案，並在最終回覆說明哪些預設類型被省略與原因。

## 產出前品質檢查

### 來源一致性

- 每份檔案是否標明來源筆記路徑。
- 每個含有 CSS 知識條目的表格是否包含 `來源` 欄。
- 每列 `來源` 是否能連回來源筆記中最直接支持該列的標題。
- 每個條目是否能由來源筆記支持。
- 是否避免加入筆記沒有提到的新知識。
- 來源不足的條目是否已標示「待補來源」或被移除。

### 查表品質

- `README.md` 是否讓讀者能先理解本章 appendix 的查找路徑。
- 章節核心對照是否放在最適合的入口，而不是分散到多份表格中。
- 內容是否適合快速查找，而不是重新教學。
- 表格欄位是否短而清楚。
- 條目是否避免長段落。
- 相近概念是否有清楚區分。
- 常見混淆是否放在容易回查的位置。
- 屬性表與屬性值表是否真的有查表價值，而不是只收錄範例材料。

### 分類品質

- 章節總覽、核心對照與查表路徑是否放在章節總覽入口。
- 屬性是否放在屬性速查表。
- 屬性值是否放在屬性值對照表。
- 名詞與心智模型是否放在名詞表。
- 畫面問題與判斷流程是否放在問題對應屬性或檢查表。
- 錯誤理解與修正方式是否放在常見誤解速查。
- 搜尋入口與同義詞是否放在關鍵字索引。

---

## 輸出

將完成的查表型資料寫入本次指定的預期輸出目錄。

若環境無法實際寫入檔案，應輸出完整內容，並明確標示建議寫入路徑。
