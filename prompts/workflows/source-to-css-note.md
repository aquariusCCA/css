# 從來源資料產生 CSS 教學筆記

## 使用情境

當需要把指定來源資料整理成 `notes/` 中的 CSS 正式教學筆記時，套用這份可重複流程。

本檔案負責「來源資料 → 教學筆記」的處理流程，不負責定義角色，也不負責固定輸出格式。

---

## 必要輸入

- 來源路徑：原始資料所在路徑。
- 目標主題：本次筆記要聚焦的 CSS 主題。
- 預期輸出路徑：正式筆記要產生的位置。

---

## 輸入檢查

1. 檢查來源路徑、目標主題與預期輸出路徑是否完整。
2. 若必要輸入不足，先指出缺少的項目，不產生正式筆記。
3. 若目標主題的邊界不清楚，應依來源內容與目標主題做合理收斂，並在筆記中標示本章範圍。
4. 若來源資料無法讀取或內容不足，應在「來源與補充說明」中標示限制。

---

## 來源分析流程

正式撰寫前，先完成來源分析。此分析可以作為內部工作步驟，不一定要完整輸出，除非任務明確要求。

### 1. 讀取與辨識來源結構

讀取來源路徑中的資料，辨識：

- 標題與段落。
- 清單與表格。
- 程式碼區塊。
- 圖片、連結或外部引用。
- 來源中的範例與註解。
- 可能的雜訊，例如導覽列、廣告、重複標題或與主題無關的內容。

### 2. 建立來源內容分區

將來源內容分成下列類型：

| 類型 | 處理方式 |
|---|---|
| 直接支持內容 | 應納入筆記主線。 |
| 背景內容 | 可簡短補充，但不可搶走主題焦點。 |
| 延伸內容 | 標示為可另開章節，不在本章深入展開。 |
| 不相關內容 | 不納入正式筆記。 |
| 來源不足內容 | 標示待確認，不寫成肯定結論。 |
| 來源範例 | 評估是否保留、簡化、修正或重寫。 |

### 3. 建立主題邊界

明確判斷：

- 本章要深入說明什麼。
- 哪些內容只需要簡短提及。
- 哪些內容應留到其他章節。
- 來源資料是否足以支撐本章學習目標。
- 需要哪些補充說明才能讓初學者理解。

---

## 主題類型判斷

判斷本章主要屬於哪一種 CSS 主題，並用判斷結果安排教學順序、範例設計、常見誤解與自我檢查。

1. 依 `prompts/system/css-teacher-role.md` 的「CSS 主題判斷規則」選擇教學重點。
2. 若來源同時橫跨多個類型，選一個主線類型；其餘只作為背景、必要前置知識或可另開章節。
3. 類型判斷不一定要寫進正式筆記，但必須影響章節順序、例子選擇與誤解檢查。

---

## 筆記生成流程

1. 依來源分析結果決定本章學習目標。
2. 根據目標主題建立教學順序：
   - 先提出學習者會遇到的具體問題或畫面狀況。
   - 再建立修改前 baseline：目前 HTML/CSS 條件、原本畫面結果、以及造成問題的既有規則或排版模型。
   - 接著只引入解決當前問題所需的 CSS 規則。
   - 再用小型範例展示修改前後的差異。
   - 最後抽象成語法、判斷流程、常見錯誤、實務注意事項與小結。
3. 若目標主題包含多個屬性、規則或觀念，應拆成一連串課程段落；每一課用讀者問題或學習任務命名，並在課內完成「問題情境、畫面觀察、最小修改、結果解釋、誤判修正、小整理」。
4. 每個主要 CSS 解法都應明確回答：修改前發生什麼、為什麼會那樣、加入哪個最小 CSS 宣告、修改後畫面如何改變。
5. 將來源內容重組成教科書式說明，不要只重排來源文字。
6. 若來源內容是屬性清單、值表或速查型資料，先轉換成一組教學問題，再安排成課程段落；不要直接逐條改寫成筆記主線，也不要先集中列完整語法表。
7. 轉換來源清單時，先為每組屬性或值找出讀者會遇到的畫面問題與修改前 baseline，再決定最小範例；屬性表或值表只放在該問題講完後，作為回查整理。
8. 優先使用來源直接支持的內容作為主線，但主線應以學習者理解順序安排，而不是照來源順序機械排列。
9. 「核心概念」、「語法、規則或心智模型」、「範例與拆解」是要被覆蓋的教學功能，可以融入每一課，不應為了符合格式而切成固定上層章節。
10. 需要補充來源以外的背景時，必須標示為「補充說明」。
11. 若某段內容主要來自一般 CSS 背景知識，而非來源，應降低篇幅，不得喧賓奪主。
12. 若來源中有範例，先評估是否適合初學者；必要時可簡化或修正，但必須保留原本教學目的。
13. 若來源範例有明顯錯誤，修正後應說明修正原因。
14. 若來源出現其他 CSS 主題，只在理解本章必要時簡短提及，並標示可另開章節。

---

## 產出前品質檢查

正式輸出前，確認筆記符合 `prompts/formats/css-teaching-note-format.md` 的格式與品質要求。

詳細格式見 `prompts/formats/css-teaching-note-format.md`。

---

## 輸出

完成後，將正式筆記寫入本次指定的預期輸出路徑；若無法寫入檔案，輸出完整筆記內容並標示建議寫入路徑。
