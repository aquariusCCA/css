# 從來源資料產生 CSS 教學筆記

## 使用情境

當需要把指定來源資料整理成 `notes/` 中的 CSS 正式教學筆記時，套用這份可重複流程。

本檔案負責「來源資料 → 教學筆記」的處理流程，不負責定義角色，也不負責固定輸出格式。

---

## 必要輸入

- 來源路徑：原始資料所在路徑。
- 目標主題：本次筆記要聚焦的 CSS 主題。
- 預期輸出路徑：正式筆記要產生的位置。

---

## 輸入檢查

1. 檢查來源路徑、目標主題與預期輸出路徑是否完整。
2. 若必要輸入不足，先指出缺少的項目，不產生正式筆記。
3. 若目標主題的邊界不清楚，應依來源內容與目標主題做合理收斂，並在筆記中標示本章範圍。
4. 若來源資料無法讀取或內容不足，應在「來源與補充說明」中標示限制。

---

## 來源分析流程

正式撰寫前，先完成來源分析。此分析可以作為內部工作步驟，不一定要完整輸出，除非任務明確要求。

### 1. 讀取與辨識來源結構

讀取來源路徑中的資料，辨識：

- 標題與段落。
- 清單與表格。
- 程式碼區塊。
- 圖片、連結或外部引用。
- 來源中的範例與註解。
- 可能的雜訊，例如導覽列、廣告、重複標題或與主題無關的內容。

### 2. 建立來源內容分區

將來源內容分成下列類型：

| 類型 | 處理方式 |
|---|---|
| 直接支持內容 | 應納入筆記主線。 |
| 背景內容 | 可簡短補充，但不可搶走主題焦點。 |
| 延伸內容 | 標示為可另開章節，不在本章深入展開。 |
| 不相關內容 | 不納入正式筆記。 |
| 來源不足內容 | 標示待確認，不寫成肯定結論。 |
| 來源範例 | 評估是否保留、簡化、修正或重寫。 |

### 3. 建立主題邊界

明確判斷：

- 本章要深入說明什麼。
- 哪些內容只需要簡短提及。
- 哪些內容應留到其他章節。
- 來源資料是否足以支撐本章學習目標。
- 需要哪些補充說明才能讓初學者理解。

---

## 主題類型判斷

判斷本章主要屬於哪一種 CSS 主題，並依類型安排教學順序。

### selector 類主題

重視：

- 匹配規則。
- HTML 結構與選取範圍。
- specificity、source order 與覆蓋。
- 常見誤選與漏選。

### property 類主題

重視：

- 屬性用途。
- 常見值與效果差異。
- 適用情境與限制。
- 常見失效原因。
- 相近屬性比較。

### box model 類主題

重視：

- content、padding、border、margin 的視覺位置。
- width、height 與實際佔用空間。
- `box-sizing`。
- 尺寸計算與常見誤解。

### typography 類主題

重視：

- font、font-size、line-height、letter-spacing 等文字樣式。
- 可讀性、繼承與排版節奏。
- 單位與實務使用情境。

### color 與 background 類主題

重視：

- color 與 background 的作用差異。
- 背景圖片、尺寸、位置與重複方式。
- 對比、可讀性與狀態表達。

### layout、display 與 normal flow 類主題

重視：

- 元素流向。
- 盒模型與可用空間。
- 父子元素關係。
- display 模型差異。
- 版面除錯線索。

### positioning 類主題

重視：

- 定位模式差異。
- containing block。
- 偏移、堆疊與覆蓋。
- 定位造成的版面影響。

### flexbox 類主題

重視：

- container 與 item 的角色。
- main axis 與 cross axis。
- 空間分配、對齊、換行。
- 父子屬性位置。

### grid 類主題

重視：

- container 與 item 的角色。
- 欄列、軌道、格線與區域。
- 項目放置與空間分配。
- grid 與 flexbox 的情境差異。

### cascade、inheritance 與 specificity 類主題

重視：

- 規則競爭與覆蓋。
- 繼承與非繼承。
- specificity 計算與 source order。
- computed style 與原始 CSS 的差異。

### responsive 類主題

重視：

- 流式設計。
- media query 條件。
- 斷點設計。
- 內容優先順序與實際版面變化。

### transition 與 animation 類主題

重視：

- 狀態變化。
- 時間、節奏與觸發條件。
- transition 與 animation 的差異。
- 效能與可讀性。

### debugging 類主題

重視：

- 是否選到元素。
- 是否被覆蓋。
- 屬性是否適用。
- 父元素條件是否成立。
- 如何用 computed style 與 box model 觀察問題。

## 筆記生成流程

1. 依來源分析結果決定本章學習目標。
2. 根據目標主題建立教學順序：
   - 先提出學習者會遇到的具體問題或畫面狀況。
   - 再帶讀者觀察父子關係、元素狀態、可用空間或規則衝突。
   - 接著只引入解決當前問題所需的 CSS 規則。
   - 再用小型範例展示修改前後的差異。
   - 最後抽象成語法、判斷流程、常見錯誤、實務注意事項與小結。
3. 若目標主題包含多個屬性、規則或觀念，應拆成一連串課程段落；每一課用讀者問題或學習任務命名，並在課內完成「問題情境、畫面觀察、最小修改、結果解釋、誤判修正、小整理」。
4. 將來源內容重組成教科書式說明，不要只重排來源文字。
5. 若來源內容是屬性清單、值表或速查型資料，先轉換成一組教學問題，再安排成課程段落；不要直接逐條改寫成筆記主線，也不要先集中列完整語法表。
6. 優先使用來源直接支持的內容作為主線，但主線應以學習者理解順序安排，而不是照來源順序機械排列。
7. 「核心概念」、「語法、規則或心智模型」、「範例與拆解」是要被覆蓋的教學功能，可以融入每一課，不應為了符合格式而切成固定上層章節。
8. 需要補充來源以外的背景時，必須標示為「補充說明」。
9. 若某段內容主要來自一般 CSS 背景知識，而非來源，應降低篇幅，不得喧賓奪主。
10. 若來源中有範例，先評估是否適合初學者；必要時可簡化或修正，但必須保留原本教學目的。
11. 若來源範例有明顯錯誤，修正後應說明修正原因。
12. 若來源出現其他 CSS 主題，只在理解本章必要時簡短提及，並標示可另開章節。

## 產出前品質檢查

正式輸出前，必須檢查以下項目：

### 來源一致性

- 主要教學內容是否由來源直接支持，或已標示為「補充說明」。
- 是否避免捏造來源沒有支持的規格、瀏覽器支援、歷史脈絡或版本資訊。
- 來源不足之處是否標示為「待確認」。
- 是否區分直接支持內容、補充內容與可另開章節內容。

### 教學品質

- 筆記主體是否讀起來像一連串課程，而不是格式欄位的展開。
- 是否先建立概念，再進入語法或規則。
- 是否先提出具體問題或畫面狀況，再說明 CSS 解法。
- 是否有清楚的學習目標與前置知識。
- 是否使用段落說明，而不是只有條列摘要。
- 是否能讓初學者理解「為什麼會產生這個 CSS 結果」。
- 是否避免一開始就用屬性表或值表取代教學推進。
- 是否避免把「核心概念」、「語法」與「範例」拆成互不連續的固定大章。
- 是否避免一次引入太多無關概念。

### 範例品質

- 範例是否小而完整。
- 範例是否包含最小必要 HTML 與 CSS。
- 範例是否直接支撐本章核心觀念。
- 範例前是否先說明要解決的具體排版問題。
- 範例是否說明修改前與修改後的畫面差異。
- 範例拆解是否說明作用對象、作用條件、視覺結果與常見誤解。
- 錯誤範例與正確範例是否清楚標示。

### 主題邊界

- 是否只深入處理目標主題。
- 背景概念是否沒有搶走主題焦點。
- 延伸內容是否標示為可另開章節。

### 自我檢查

- 自我檢查題是否能驗證學習目標。
- 問題是否具體，不是只問「是否理解」。
- 若有練習提示，是否能讓學習者實際操作或觀察 CSS 結果。

---

## 輸出

將完成的正式筆記寫入本次指定的預期輸出路徑。

若環境無法實際寫入檔案，應輸出完整筆記內容，並明確標示建議寫入路徑。
