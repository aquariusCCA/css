# 產生 CSS Demo 範例

## Goal

根據指定 `notes/` 正式教學筆記，分析筆記內容並產生一個或多個可直接在瀏覽器開啟、具教學性的 CSS demo 範例；每個 demo 應聚焦單一主要概念。

產出內容應根據來源筆記設計，不修改來源筆記，不加入與本次範圍無關的 CSS 知識，也不做成大型綜合頁面。

---

## Use When

當你要把 `notes/<章節>/*.md` 中的 CSS 正式教學筆記，轉換成 `demos/` 下的小型可執行教學範例時，使用這份請求模板。

適合產生：

- 單一 CSS 屬性或值的觀察 demo。
- 修改前 / 修改後的對照 demo。
- 錯誤 / 正確的常見誤解 demo。
- 少量互動切換的 CSS 行為觀察 demo。

---

## Apply

請依序套用以下提示詞：

## Inputs

必要輸入：

- 來源路徑：`{{source_path}}`
- 目標輸出路徑：`{{output_path}}`

選填輸入：

- 目標知識點或範圍限制：`{{target_scope}}`

若任務仍使用 `目標知識點` 欄位，視同 `目標知識點或範圍限制`。

輸入限制：

- 來源路徑應指向 `notes/<章節>/*.md` 或 `notes/<章節>/`。
- 目標輸出路徑必須位於 `demos/` 下，且應是目錄路徑。
- 若提供 `{{target_scope}}`，它可以是單一知識點，也可以是較大的章節範圍；較大範圍應先拆成多個 demo。

## Prompt Template

請根據以下設定產生 CSS demo 範例。

### 必要設定

- 來源路徑：`{{source_path}}`
- 目標輸出路徑：`{{output_path}}`

### 選填設定

- 目標知識點或範圍限制：`{{target_scope}}`

### 任務要求

1. 先閱讀並理解來源筆記。
2. 確認來源路徑位於 `notes/` 下；若不是正式教學筆記，請停止產生並說明原因。
3. 確認目標輸出路徑位於 `demos/` 下；若不是 demo 目錄路徑，請停止產生並說明原因。
4. 不得修改來源筆記；`notes/` 只作為 demo 設計依據。
5. 根據來源筆記建立內容地圖，找出可做成 demo 的段落、範例、問題情境、常見誤解與檢查重點。
6. 若提供 `{{target_scope}}`，請將它作為範圍限制；若它是整章或大型主題，不要停止產生，應先拆成多個單一概念 demo。
7. 若未提供 `{{target_scope}}`，請根據整份來源筆記建立 demo 候選清單。
8. 每個 demo 候選應包含 demo 名稱、主要概念、來源標題、建議 demo 型態、觀察重點與輸出子目錄。
9. 刪除缺少來源支持、範圍外或不適合做成可觀察範例的候選。
10. 為每個保留候選決定 demo 型態：單一狀態、修改前 / 修改後、錯誤 / 正確，或少量互動切換。
11. 逐一設計最小必要 HTML 與 CSS，讓學習者能清楚觀察：
   - CSS 的作用對象。
   - 主要 CSS 宣告。
   - 修改前後或不同狀態的畫面差異。
   - 對應的常見誤解或失效原因。
12. 優先產生可直接用瀏覽器開啟的靜態範例；只有互動能幫助觀察 CSS 行為時，才建立 `main.js`。
13. 不引入不必要的框架、外部 CDN、建置工具或與本次範圍無關的 CSS 技巧。
14. 每個 README 應說明來源筆記、對應知識點、開啟方式、操作方式、觀察重點與常見誤解。
15. 產出前請檢查：
    - 每個 demo 是否能直接用瀏覽器開啟。
    - 每個 demo 是否只聚焦一個主要知識點。
    - 每個 demo 是否能清楚觀察主要 CSS 宣告造成的畫面變化。
    - 是否沒有引用不存在的 CSS、JS、圖片或其他資產。
    - 是否沒有把正式筆記大段複製進 demo 頁面。
    - README 是否讓學習者知道要觀察什麼，而不是只列檔案清單。
16. 最後將 demo 檔案產生到目標輸出路徑下的對應子目錄。

## Notes

本模板只負責單次任務入口，不應承擔角色定義、完整工作流程或固定 demo 檔案格式。

若後續建立 `prompts/formats/css-demo-format.md`，應再將它加入 `Apply`，並把檔案格式細節移到 format 規則中。
