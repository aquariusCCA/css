# 產生 CSS 查表型資料

## Goal

根據指定 `notes/` 正式筆記，產生一組 CSS 查表型 appendix 資料。

產出內容應優先根據來源筆記，整理成短、準、可掃描的速查資料；不得改寫成教學筆記，也不得加入來源筆記未支持的新知識。

---

## Use When

當你要把 `notes/<章節>/*.md` 中的 CSS 正式教學筆記整理成 `appendix/<章節>/` 下的查表型資料時，使用這份請求模板。

適合產生：

- CSS 屬性速查表。
- 屬性值對照表。
- 名詞表。
- 問題對應屬性表。
- 常見誤解速查。
- 關鍵字索引。

---

## Apply

請依序套用以下提示詞：

- `prompts/system/base-role.md`
- `prompts/system/css-teacher-role.md`
- `prompts/workflows/notes-to-css-appendix.md`

## Prompt Template

請根據以下設定產生 CSS 查表型資料。

### 必要設定

- 筆記路徑：`{{notes_path}}`
- 章節名稱：`{{chapter_name}}`
- 預期輸出目錄：`{{output_dir}}`
- 查表類型：`{{reference_types}}`

### 任務要求

1. 先閱讀並理解指定筆記。
2. 在產生 appendix 前，先判斷筆記中哪些內容可抽成：
   - CSS 屬性速查表。
   - 屬性值對照表。
   - 名詞表。
   - 問題對應屬性表。
   - 常見誤解速查。
   - 關鍵字索引。
3. 只整理筆記已支持的內容，不新增筆記沒有提到的新 CSS 知識。
4. 若某個欄位缺少來源依據，填入「待補來源」；若整個條目缺少來源依據，請刪除該條目。
5. 查表內容應短、準、可掃描，優先使用表格。
6. 不要把正式筆記重寫成摘要，也不要複製正式筆記的大段文字。
7. 每份 appendix 檔案開頭都要標明來源筆記路徑與用途。
8. 若筆記路徑是章節目錄，請整理該目錄下與章節主題相關的 `.md` 檔案。
9. 若本次只指定部分查表類型，只產生指定類型。
10. 預設輸出到 `appendix/<章節>/`，章節名稱應與 `notes/<章節>/` 保持一致。
11. 若預期輸出目錄被填成 `.md` 檔案路徑，請先指出欄位不一致；除非任務明確要求單檔輸出，否則改用該路徑的父層資料夾作為輸出目錄。
12. 每列表格都要包含 `來源` 欄，指向來源知識所在的最近標題，標題層級可為 H2-H6。
13. 產出前請檢查：
    - 每個條目是否能回到來源筆記。
    - 每列 `來源` 是否能定位到來源筆記中最直接支持該列的標題。
    - 是否避免加入來源筆記未支持的新知識。
    - 是否符合指定查表格式。
    - 是否適合快速查找，而不是教學閱讀。
    - 是否清楚區分屬性、屬性值、名詞、問題、誤解與關鍵字。
14. 最後將查表型資料產生到預期輸出目錄。

---

## Notes

如果某些查表類型在指定筆記中沒有足夠內容，應說明未產生或只產生有限條目的原因。

本模板只負責單次任務入口，不應承擔角色定義、完整工作流程或固定格式細節。
