# prompts/

`prompts/` 用來存放可複用的提示詞，不保存單次對話紀錄。

提示詞依照「作用範圍」與「生命週期」拆成四類：長期規則、處理流程、輸出格式與單次請求模板。

## 目錄結構

```text
prompts/
  system/      # 長期角色與全域規則
  workflows/   # 可重複執行的處理流程
  formats/     # 固定輸出格式與品質規則
  requests/    # 單次任務模板與具體輸入欄位
```

## 目錄定位

| 目錄 | 負責內容 | 適合放什麼 |
|---|---|---|
| `system/` | 長期角色與全域規則 | 角色設定、語言規則、教學風格、全域限制 |
| `workflows/` | 可重複執行的處理流程 | 筆記生成流程、Debug 流程、Code Review 流程、重構流程 |
| `formats/` | 固定輸出格式與品質規則 | 筆記格式、報告格式、表格格式、練習題格式 |
| `requests/` | 單次任務模板與臨時要求 | 來源路徑、目標主題、輸出路徑、補充限制 |

## 使用原則

* `prompts/` 放可複用提示詞，不放完整對話紀錄。
* `system/` 管長期規則，避免放入單次資料。
* `workflows/` 管可重複流程，避免放入某一次任務的完整資料。
* `formats/` 管輸出樣式與品質規則，避免放入實際輸入資料。
* `requests/` 管本次任務模板與可替換欄位，避免沉澱長期規則。
* 如果某套流程、格式規則或檢查步驟已經穩定，應整理到 `skills/`。
* `requests/` 範本應保留可替換欄位，避免綁死單一章節或單次任務。
