# CSS 教科書型筆記格式

## 使用情境

當任務要求輸出 CSS 正式教學筆記時，套用這份格式規則。

這份格式不是僵硬的摘要模板，而是教科書型章節的品質標準。章節可以依 CSS 主題調整中間段落，但必須完整支撐學習者理解。

---

## 必要章節功能

每篇筆記必須包含下列教學功能。章節標題可以依主題調整，但功能不可缺漏。

1. **清楚標題**：標明本章 CSS 主題。
2. **學習目標**：說明讀完後應該能理解或做到什麼。
3. **前置知識**：列出理解本章需要先知道的 HTML 或 CSS 觀念。
4. **本章範圍**：說明本章會深入處理什麼，以及哪些相關主題只簡短提及或另開章節。
5. **核心概念**：用循序漸進的段落說明主題本身，不只列重點。
6. **語法、規則或心智模型**：依主題需要說明語法、視覺模型、運作方式或判斷流程。
7. **範例與拆解**：提供小型 CSS 範例，並逐段解釋範例如何對應核心觀念。
8. **常見誤解與修正方式**：指出初學者容易誤解或寫錯的地方，並說明正確理解方式。
9. **實務使用情境**：說明這個主題通常在哪些 CSS 任務中會用到。
10. **自我檢查**：提供可以驗證理解的小問題或練習提示。
11. **小結**：用短段落整理本章最重要的觀念。
12. **來源與補充說明**：區分來源支持內容、補充說明、無法讀取或待確認的資料。

---

## 建議章節順序

可使用下列順序作為預設結構：

```text
# 主題標題

## 學習目標
## 前置知識
## 本章範圍
## 核心概念
## 語法、規則或心智模型
## 範例與拆解
## 常見誤解與修正方式
## 實務使用情境
## 自我檢查
## 小結
## 來源與補充說明
```

若主題需要更自然的教學節奏，可以調整中間章節，但仍必須完整覆蓋必要章節功能。

---

## 各章節品質要求

### 學習目標

學習目標應具體描述學習成果，例如：

- 能判斷某個 selector 會選到哪些元素。
- 能說明某個 CSS 屬性在畫面上造成的效果。
- 能辨識樣式失效是因為 selector、cascade、specificity、display 或父子關係問題。

避免使用空泛描述，例如「了解 CSS」或「熟悉排版」。

### 前置知識

前置知識應只列理解本章必要的概念，不要變成延伸閱讀清單。

若前置知識對初學者可能困難，應簡短補一句背景，不要直接跳到進階用語。

### 本章範圍

本章範圍應說明：

- 本章會深入說明的內容。
- 只會簡短提及的背景內容。
- 不會在本章展開、建議另開章節的內容。

這個章節用來避免筆記因 CSS 關聯概念過多而發散。

### 核心概念

核心概念應用完整段落說明，不要只列名詞定義。

說明時應盡量包含：

- 這個概念解決什麼問題。
- 它如何影響 HTML 元素或畫面結果。
- 它和其他 CSS 概念的基本關係。
- 初學者應先抓住哪個判斷方式。

### 語法、規則或心智模型

依主題選擇最適合的說明方式：

- selector 主題：說明匹配邏輯與選取範圍。
- property 主題：說明屬性、值、效果與適用條件。
- layout 主題：說明視覺模型、父子關係與空間分配。
- cascade 主題：說明規則競爭、繼承、specificity 與 source order。
- responsive 主題：說明條件、斷點、流式設計與內容優先順序。
- animation 主題：說明狀態變化、時間、節奏與觸發條件。

### 範例與拆解

範例必須小而完整，優先包含最小必要 HTML 與 CSS。

範例後應拆解：

1. 哪些 HTML 元素是 CSS 的作用對象。
2. 哪條 CSS 規則造成主要效果。
3. 為什麼會出現這個視覺結果。
4. 如果修改某個值，結果會如何改變。
5. 初學者可能在哪裡誤解。

若主題適合對照教學，可以使用「設定前 / 設定後」或「錯誤範例 / 正確範例」。

### 常見誤解與修正方式

每個常見誤解應包含：

- 錯誤理解或錯誤寫法。
- 為什麼這樣會出問題。
- 正確理解或修正方式。

不要只列「注意事項」，要說明錯誤背後的原因。

### 實務使用情境

實務使用情境應說明這個主題在真實 CSS 工作中通常用在哪裡。

避免過度延伸到工具推薦、框架差異或與本章無關的設計方法。

### 自我檢查

自我檢查題應對應學習目標。題目應具體，能讓學習者判斷自己是否真的理解。

可以使用：

- 判斷題。
- 修改範例題。
- 觀察結果題。
- 找錯題。
- 簡短實作題。

避免只問：「你是否理解本章內容？」

### 來源與補充說明

本章最後必須說明來源使用狀態，可包含：

- 本章主要根據哪些來源內容整理。
- 哪些重點是來源直接支持。
- 哪些內容屬於「補充說明」。
- 哪些資訊因來源不足而標示為「待確認」。
- 哪些相關主題適合另開章節。

若沒有使用來源以外的補充內容，也應明確寫出「本章未加入來源以外的主要補充說明」。

## 彈性章節規則

不要把必要章節功能機械地寫成固定表格。中間段落應依 CSS 主題調整。

- selector 主題應強調選取邏輯、套用範圍、specificity、source order 與錯誤匹配。
- property 主題應強調屬性用途、可用值、效果差異、適用情境與常見失效原因。
- box model 主題應強調 content、padding、border、margin、尺寸計算與 `box-sizing`。
- typography 主題應強調字體、字級、行高、繼承、可讀性與排版節奏。
- color 與 background 主題應強調前景與背景差異、可讀性、背景行為與狀態表達。
- layout 主題應強調盒模型、normal flow、排列方向、空間分配、父子元素關係與除錯方式。
- positioning 主題應強調定位模式、containing block、偏移、堆疊與版面影響。
- flexbox 主題應強調 container/item、main axis/cross axis、空間分配、對齊與父子屬性位置。
- grid 主題應強調欄列、軌道、格線、區域、項目放置與 grid/flex 使用差異。
- cascade 主題應強調繼承、層疊、specificity、source order 與覆蓋策略。
- responsive 主題應強調流式版面、媒體條件、斷點設計與內容優先順序。
- transition 與 animation 主題應強調狀態變化、時間、easing、觸發條件、效能與可讀性。
- debugging 主題應強調 selector、computed style、box model、父子條件與覆蓋來源的排查順序。

---

## 禁止輸出樣式

- 不要只列條列重點。
- 不要只重排來源文字。
- 不要把每個章節寫成簡短定義。
- 不要用過度固定的模板壓縮主題差異。
- 不要加入與目標主題無關的大量延伸內容。
- 不要讓補充說明比來源支持內容更像主線。
- 不要省略範例拆解。
- 不要產生無法驗證學習目標的自我檢查題。
