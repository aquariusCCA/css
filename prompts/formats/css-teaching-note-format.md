# CSS 教科書型筆記格式

## 使用情境

當任務要求輸出 CSS 正式教學筆記時，套用這份格式規則。

這份格式不是僵硬的摘要模板，而是教科書型章節的品質標準。章節可以依 CSS 主題調整中間段落，但必須完整支撐學習者理解。

---

## 正文組織格式

正式筆記的主體應呈現教科書章節，而不是欄位展開或資料清單。

筆記主體應讀起來像一連串課程：先讓讀者看見具體問題或畫面狀況，再建立概念，接著進入語法、規則或心智模型。不要直接從屬性名稱、規格列表或最終答案開始。

若主題適合拆成多段，建議使用「先看問題」、「第一課」、「第二課」等問題導向標題。每一課應能自然完成概念、語法、範例與拆解，不要把中間主體寫成固定欄位表。

每個課程段落可以包含下列小節，但不必在所有主題中硬性逐項出現：

- 問題情境。
- 畫面觀察。
- 最小修改。
- 結果解釋。
- 誤判修正。
- 小整理。

每一課至少應交代修改前觀察、最小 CSS 變更與結果解釋，讓讀者看出新規則解決了哪個原本存在的畫面問題，以及為什麼會產生新的 CSS 結果。

條列可以輔助整理，但主體說明應使用完整段落，不應只用摘要清單取代教學推進。

屬性表、值表與速查整理可以保留，但只能放在概念建立之後，作為回查工具，不應取代教學段落。

---

## 必要章節功能

每篇筆記必須包含下列教學功能。這些是功能要求，不是固定章節模板；章節標題可以依主題調整，功能也可以融入課程段落中完成，但不可缺漏。

1. **清楚標題**：標明本章 CSS 主題。
2. **學習目標**：說明讀完後應該能理解或做到什麼。
3. **前置知識**：列出理解本章需要先知道的 HTML 或 CSS 觀念。
4. **本章範圍**：說明本章會深入處理什麼，以及哪些相關主題只簡短提及或另開章節。
5. **核心概念**：用循序漸進的段落說明主題本身，不只列重點。
6. **語法、規則或心智模型**：依主題需要說明語法、視覺模型、運作方式或判斷流程。
7. **範例與拆解**：提供小型 CSS 範例，並逐段解釋範例如何對應核心觀念。
8. **常見誤解與修正方式**：指出初學者容易誤解或寫錯的地方，並說明正確理解方式。
9. **實務使用情境**：說明這個主題通常在哪些 CSS 任務中會用到。
10. **自我檢查**：提供可以驗證理解的小問題或練習提示。
11. **小結**：用短段落整理本章最重要的觀念。
12. **來源與補充說明**：區分來源支持內容、補充說明、無法讀取或待確認的資料。

## 建議章節順序

可使用下列課程式順序作為預設結構：

```text
# 主題標題

## 學習目標
## 前置知識
## 本章範圍
## 先看問題：讀者會遇到的具體畫面狀況
## 第一課：用讀者問題或學習任務命名
### 問題情境
### 畫面觀察
### 最小修改
### 結果解釋
### 小整理
本課內應自然完成：核心概念、語法 / 規則 / 心智模型、範例與拆解。
## 第二課：用讀者問題或學習任務命名
...
## 常見誤解與修正方式
## 實務使用情境
## 自我檢查
## 小結
## 來源與補充說明
```

中間主體章節應優先使用讀者問題或學習任務作為標題，不要機械使用「核心概念」、「語法、規則或心智模型」、「範例與拆解」作為固定上層章節。這些功能應在每一課內自然完成。

## 章節與教學功能品質要求

下列要求有些對應固定章節，有些對應課內教學功能。課內教學功能應融入每一課，不必作為實際章節標題。

### 學習目標

學習目標應具體描述學習成果，例如：

- 能判斷某個 selector 會選到哪些元素。
- 能說明某個 CSS 屬性在畫面上造成的效果。
- 能辨識樣式失效是因為 selector、cascade、specificity、display 或父子關係問題。

避免使用空泛描述，例如「了解 CSS」或「熟悉排版」。

### 前置知識

前置知識應只列理解本章必要的概念，不要變成延伸閱讀清單。

若前置知識對初學者可能困難，應簡短補一句背景，不要直接跳到進階用語。

### 本章範圍

本章範圍應說明：

- 本章會深入說明的內容。
- 只會簡短提及的背景內容。
- 不會在本章展開、建議另開章節的內容。

本章範圍應讓主線聚焦目標主題；背景概念只能支撐理解，不應搶走主題焦點，延伸內容應標示為可另開章節。

若筆記由來源資料生成，目標主題的邊界應依來源分析收斂；來源中的背景概念只保留理解本章必要的部分。

這個章節用來避免筆記因 CSS 關聯概念過多而發散。

### 核心概念

核心概念應用完整段落說明，不要只列名詞定義。

說明時應盡量包含：

- 這個概念解決什麼問題。
- 它如何影響 HTML 元素或畫面結果。
- 它和其他 CSS 概念的基本關係。
- 初學者應先抓住哪個判斷方式。

核心概念段落應先讓讀者看見「為什麼需要這個概念」，再出現術語、屬性名稱或值表。

若主題適合拆成多課，核心概念應跟著每一課的問題逐步出現，不必集中在獨立的「核心概念」大章。

### 語法、規則或心智模型

語法、規則或心智模型應以支撐讀者判斷為目的，不要只是列規格。

內容應放在讀者已看過問題與畫面觀察之後。若拆成多課，每一課只引入解決當前問題所需的語法與判斷方式。

若來源是清單、表格或速查內容，應先轉成問題導向段落；表格只能放在概念建立之後作為回查整理。

可依主題使用匹配邏輯、屬性效果、視覺模型、規則競爭、條件判斷或狀態變化等形式；具體 CSS 主題的教學重點由 `prompts/system/css-teacher-role.md` 與對應 workflow 判斷。

### 範例與拆解

範例必須小而完整，優先包含最小必要 HTML 與 CSS。

每個重要範例前必須先說明要解決的具體問題。範例不應只是展示某個屬性，而應讓讀者看見「修改前的問題」與「修改後的結果」。

範例若用來介紹新的 CSS 屬性、規則或排版模型，應保留「修改前狀態」與「修改後設定」的對照。修改前狀態可以用段落說明，不一定要提供完整錯誤範例，但讀者必須看得出新規則解決了哪個原本存在的畫面問題。

範例後應拆解：

1. 哪些 HTML 元素是 CSS 的作用對象。
2. 哪條 CSS 規則造成主要效果。
3. 修改前畫面依什麼條件或規則運作。
4. 修改後為什麼會出現新的視覺結果。
5. 如果修改某個值，結果會如何改變。
6. 初學者可能在哪裡誤解。
7. 這個範例能解決哪一類真實排版問題。

若主題適合對照教學，可以使用「設定前 / 設定後」或「錯誤範例 / 正確範例」。

範例與拆解應盡量貼近各課的問題情境，不要等到所有概念講完後才集中放一組總範例。集中式範例只能作為補充，不應取代每一課中的最小修改與結果解釋。

若範例來自來源資料，應依初學者理解需要評估保留、簡化、修正或重寫。改寫後的範例必須支撐本章目標主題，不應只是展示來源中的孤立語法；若改寫來源範例，應保留原本教學目的。若來源範例有明顯錯誤，修正後應說明修正原因。

### 常見誤解與修正方式

每個常見誤解應包含：

- 錯誤理解或錯誤寫法。
- 為什麼這樣會出問題。
- 正確理解或修正方式。

不要只列「注意事項」，要說明錯誤背後的原因。

### 實務使用情境

實務使用情境應說明這個主題在真實 CSS 工作中通常用在哪裡。

避免過度延伸到工具推薦、框架差異或與本章無關的設計方法。

### 自我檢查

自我檢查題應對應學習目標。題目應具體，能讓學習者判斷自己是否真的理解。

可以使用：

- 判斷題。
- 修改範例題。
- 觀察結果題。
- 找錯題。
- 簡短實作題。

自我檢查題應優先要求讀者預測畫面、判斷作用對象、找出失效條件或修改一行 CSS。避免只要求背誦屬性名稱。

若有練習提示，應能讓學習者實際操作或觀察 CSS 結果。

避免只問：「你是否理解本章內容？」

### 來源與補充說明

本章最後必須說明來源使用狀態，可包含：

- 本章主要根據哪些來源內容整理。
- 哪些重點是來源直接支持。
- 哪些內容屬於「補充說明」。
- 哪些資訊因來源不足而標示為「待確認」。
- 哪些相關主題適合另開章節。

主要教學內容應由來源支持；若不是來源直接支持，必須明確標示為補充說明。不要把來源沒有支持的規格、瀏覽器支援、歷史脈絡或版本資訊寫成肯定結論；來源不足時應標示為待確認。

若筆記由來源資料生成，應能區分直接支持內容、背景內容、延伸內容、不相關內容、來源不足內容與來源範例。不相關內容不應納入正式筆記；延伸內容只在必要時標示為可另開章節。

若沒有使用來源以外的補充內容，也應明確寫出「本章未加入來源以外的主要補充說明」。

---

## 主題差異呈現規則

不要把必要章節功能機械地寫成固定表格。中間段落應依 CSS 主題調整，但本檔只規定輸出型態：

- 章節標題應反映讀者問題或學習任務。
- 語法表、值表與名詞表應出現在解釋之後。
- 若主題需要多種觀念，應分課呈現，不要把所有概念塞進同一段。
- 每一課只引入解決當前問題所需的觀念，避免一次引入太多無關概念。
- 具體 CSS 主題的教學重點由 `prompts/system/css-teacher-role.md` 判斷。

---

## 禁止輸出樣式

- 不要只列條列重點。
- 不要只重排來源文字。
- 不要把每個章節寫成簡短定義。
- 不要把「核心概念」、「語法、規則或心智模型」、「範例與拆解」機械拆成固定上層章節，導致課程節奏被切斷。
- 不要在建立問題情境之前直接列完整屬性表。
- 不要讓「語法表」取代「問題、觀察、修改、解釋」的教學過程。
- 不要用過度固定的模板壓縮主題差異。
- 不要加入與目標主題無關的大量延伸內容。
- 不要讓補充說明比來源支持內容更像主線。
- 不要省略範例拆解。
- 不要產生無法驗證學習目標的自我檢查題。
