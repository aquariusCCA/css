# Markdown 資產路徑整理報告格式

## 使用情境

當任務要求整理 Markdown 本地資產路徑，並完成掃描、比對、改寫或檢查後，套用這份報告格式。

這份格式只規定整理報告的輸出長相與填寫品質，不負責定義資產掃描、路徑比對、檔名判斷或 Markdown 改寫流程。

---

## 必要章節

整理報告必須包含下列章節，章節名稱不可省略：

1. **處理範圍**：列出章節目錄、Markdown 數量與掃描資產數量。
2. **改寫摘要**：用表格統計改寫、略過、人工確認與建議重新命名數量。
3. **已改寫項目**：列出實際被改寫的 Markdown 資產路徑。
4. **保持不變項目**：列出已符合規則而未改寫的路徑。
5. **已略過項目**：列出外部網址、anchor、非資產連結或 code block 內文字等略過項目。
6. **需要人工確認**：列出無法安全自動處理的項目。
7. **建議重新命名資產**：列出檔名不符合規則或不安全的資產。
8. **最終檢查結果**：列出路徑、語法、檔案存在性與檔名安全檢查。

---

## 填寫規則

* 所有欄位必須依實際掃描與檢查結果填寫。
* 沒有項目時，不刪除章節；表格內容填 `無`，統計數量填 `0`。
* 不得為了填滿報告而補造不存在的資產、hash、路徑或問題。
* `需要人工確認` 與 `建議重新命名資產` 必須清楚說明原因。
* `最終檢查結果` 應依實際結果勾選；未通過或未確認的項目不可勾選。

---

## 報告格式

請使用以下格式輸出：

```md
# Markdown 資產路徑整理報告

## 處理範圍

- 章節目錄：
- Markdown 數量：
- 掃描資產數量：
  - images:
  - pdfs:
  - files:

## 改寫摘要

| 類型 | 數量 |
|---|---:|
| 已改寫路徑 |  |
| 保持不變 |  |
| 外部連結略過 |  |
| 非資產連結略過 |  |
| 需要人工確認 |  |
| 建議重新命名 |  |

## 已改寫項目

| Markdown | 行號 | 原始路徑 | 改寫後路徑 | 類型 |
|---|---:|---|---|---|
|  |  |  |  |  |

## 保持不變項目

| Markdown | 行號 | 路徑 | 原因 |
|---|---:|---|---|
|  |  |  | 已符合規則 |

## 已略過項目

| Markdown | 行號 | 連結 | 略過原因 |
|---|---:|---|---|
|  |  |  | 外部網址 |
|  |  |  | anchor |
|  |  |  | 非資產 Markdown 連結 |
|  |  |  | code block 內文字 |

## 需要人工確認

| Markdown | 行號 | 原始路徑 | 問題 | 建議處理 |
|---|---:|---|---|---|
|  |  |  | 找不到實體檔案 | 確認資產是否已放入對應 assets 目錄 |
|  |  |  | 多個候選檔案 | 指定正確資產 |
|  |  |  | 未知副檔名 | 判斷應歸類為圖片、PDF 或附件 |
|  |  |  | 檔名不安全 | 重新命名資產後再改寫 |
|  |  |  | 無法產生 hash | 確認檔案可讀取 |

## 建議重新命名資產

| 目前檔案 | 建議檔名 | 原因 |
|---|---|---|
|  |  | 檔名未 normalized |
|  |  | 檔名含空白、中文或特殊符號 |
|  |  | 檔名無語意 |

## 最終檢查結果

- [ ] 所有本地資產路徑皆以 `./assets/` 開頭
- [ ] 圖片皆位於 `./assets/images/`
- [ ] PDF 皆位於 `./assets/pdfs/`
- [ ] 一般附件皆位於 `./assets/files/`
- [ ] 圖片皆使用圖片語法
- [ ] PDF 與附件皆使用一般連結語法
- [ ] 沒有絕對路徑
- [ ] 沒有上一層相對路徑
- [ ] 沒有 URL encoded path
- [ ] 沒有 `<hash6>`、`hash6`、`<hash>` placeholder
- [ ] 所有改寫後路徑皆能找到實體檔案
```
