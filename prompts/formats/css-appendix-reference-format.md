# CSS 查表型資料格式

## 使用情境

當任務要求根據 `notes/` 中的 CSS 正式筆記，產生 `appendix/` 下的查表型資料時，套用這份格式規則。

這份格式的目標是讓內容可以快速查找、對照與回憶，不是重新撰寫教學筆記。

---

## 通用格式規則

每份 appendix 檔案開頭應包含：

```markdown
# 標題

來源筆記：`notes/<章節>/<筆記檔名>.md`

用途：一句話說明本檔案適合用來查什麼。
```

通用要求：

- 使用繁體中文撰寫。
- 優先使用表格；不適合表格時使用短條列。
- 每格內容應短而明確，避免長段教學說明。
- CSS 屬性、屬性值、selector、檔案路徑與程式碼片段應使用反引號標示。
- 不輸出來源筆記未支持的新知識。
- 不把正式筆記整段搬到 appendix。
- 不把 appendix 寫成課堂式教學正文。
- 若某欄沒有足夠來源，填入「待補來源」；若整列都缺乏來源，應刪除該列。
- 每個表格都應包含 `來源` 欄，放在最後一欄，指向來源知識所在的最近標題。
- `來源` 欄應使用相對 Markdown 連結，格式為 `[標題短名](../../notes/<章節>/<筆記檔名>.md#標題錨點)`。
- 來源標題層級可使用 H2-H6。原因：在 CSS 教學筆記格式中，H1 是整篇筆記的主題標題，用來代表整份文件；appendix 的 `來源` 欄是為了定位單一知識點在正文中的位置，因此不指向 H1，而是指向正文內最直接支持該知識點的標題。正文標題從 H2 開始，並可依內容細分到 H6，所以來源標題範圍為 H2-H6。
- 每列只放一個主要來源；若同一列可對應多個標題，選最直接支持該知識點的標題，避免表格過寬。
- 若最近標題是重複或過度通用的標題，例如多個「小整理」或「誤判修正」，且錨點不易穩定判斷，應改用最近的唯一父標題。
- 若無法穩定定位來源標題，填入「待補來源」。

## CSS 屬性速查表

檔名：

```text
properties-reference.md
```

用途：查某個 CSS 屬性用來做什麼、寫在哪裡、常見值有哪些。

建議格式：

```markdown
# CSS 屬性速查表

來源筆記：`notes/<章節>/<筆記檔名>.md`

用途：快速查找本章出現的 CSS 屬性、作用對象與使用注意事項。

| 屬性 | 作用對象 | 用途 | 常見值 | 預設值 | 注意事項 | 來源 |
|---|---|---|---|---|---|---|
| `property-name` | 父元素 / 子元素 / 文字 / 盒子 | 短句說明 | `value-a`、`value-b` | `default-value` | 短句說明 | [標題短名](../../notes/<章節>/<筆記檔名>.md#標題錨點) |
```

欄位規則：

- `屬性`：只放 CSS 屬性名稱。
- `作用對象`：說明屬性應寫在哪類元素或規則上。
- `用途`：用一句話描述它解決的畫面或樣式問題。
- `常見值`：只列筆記中出現或直接支持的值。
- `預設值`：筆記有明確提到才填；否則填「待補來源」。
- `注意事項`：放適用條件、常見失效原因或容易誤用之處。
- `來源`：指向最直接說明該屬性的來源標題。

## 屬性值對照表

檔名：

```text
values-reference.md
```

用途：查某個屬性值會造成什麼畫面結果，以及和相近值差在哪裡。

建議格式：

```markdown
# 屬性值對照表

來源筆記：`notes/<章節>/<筆記檔名>.md`

用途：快速對照本章 CSS 屬性值的效果、條件與混淆點。

| 值 | 所屬屬性 | 畫面效果 | 適用條件 | 容易混淆 | 來源 |
|---|---|---|---|---|---|
| `value-name` | `property-name` | 短句說明 | 短句說明 | `other-value`：差異說明 | [標題短名](../../notes/<章節>/<筆記檔名>.md#標題錨點) |
```

欄位規則：

- `值`：只放屬性值。
- `所屬屬性`：可列一個或多個屬性。
- `畫面效果`：描述使用後最直接的視覺或排版結果。
- `適用條件`：說明值生效所需條件，例如父子關係、單行、多行或剩餘空間。
- `容易混淆`：只列筆記有說明的相近值；沒有則填「無」。
- `來源`：指向最直接說明該值效果或條件的來源標題。

## 名詞表

檔名：

```text
concept-glossary.md
```

用途：查本章重要 CSS 名詞、版面模型詞彙與判斷概念。

建議格式：

```markdown
# 名詞表

來源筆記：`notes/<章節>/<筆記檔名>.md`

用途：快速查找本章重要名詞與初學者應掌握的理解方式。

| 名詞 | 簡短定義 | 初學者理解方式 | 相關屬性或概念 | 來源 |
|---|---|---|---|---|
| 名詞 | 一句話定義 | 用畫面或判斷方式說明 | `property-name`、相關名詞 | [標題短名](../../notes/<章節>/<筆記檔名>.md#標題錨點) |
```

欄位規則：

- `名詞`：可包含中文名詞、英文術語與常見縮寫。
- `簡短定義`：避免百科式長定義。
- `初學者理解方式`：用本章語境說明如何判斷或觀察。
- `相關屬性或概念`：列可交叉查找的屬性、值或名詞。
- `來源`：指向最直接建立該名詞或判斷方式的來源標題。

## 問題對應屬性表

檔名：

```text
decision-table.md
```

用途：從畫面問題反查應優先檢查或使用哪個 CSS 屬性。

建議格式：

```markdown
# 問題對應屬性表

來源筆記：`notes/<章節>/<筆記檔名>.md`

用途：遇到畫面問題時，快速判斷應優先檢查哪個 CSS 屬性或條件。

| 畫面問題 | 優先檢查 | 建議屬性 | 判斷理由 | 來源 |
|---|---|---|---|---|
| 問題描述 | 條件或線索 | `property-name` | 短句說明 | [標題短名](../../notes/<章節>/<筆記檔名>.md#標題錨點) |
```

欄位規則：

- `畫面問題`：使用學習者會搜尋或遇到的說法。
- `優先檢查`：列出最先確認的前提，例如是否為父元素、是否有換行、是否有剩餘空間。
- `建議屬性`：列最直接相關的 CSS 屬性；必要時可列多個。
- `判斷理由`：用一句話說明為什麼是這個屬性。
- `來源`：指向最直接提出該畫面問題或判斷流程的來源標題。

## 常見誤解速查

檔名：

```text
common-mistakes.md
```

用途：快速查找本章常見錯誤理解、失效原因與修正方向。

建議格式：

```markdown
# 常見誤解速查

來源筆記：`notes/<章節>/<筆記檔名>.md`

用途：快速排查本章常見誤解與錯誤用法。

| 誤解或錯誤寫法 | 錯誤原因 | 修正方式 | 相關概念 | 來源 |
|---|---|---|---|---|
| 短句描述 | 短句說明 | 短句說明 | `property-name`、名詞 | [標題短名](../../notes/<章節>/<筆記檔名>.md#標題錨點) |
```

欄位規則：

- `誤解或錯誤寫法`：使用初學者可能真的會說或會寫的形式。
- `錯誤原因`：指出失效或誤判的根本原因。
- `修正方式`：給出正確判斷或修改方向，不必展開完整教學。
- `相關概念`：列可回查的屬性、值或名詞。
- `來源`：指向最直接說明該誤解或修正方式的來源標題。

## 關鍵字索引

檔名：

```text
keyword-index.md
```

用途：從關鍵字、同義詞或畫面任務反查本章 appendix 與筆記內容。

建議格式：

```markdown
# 關鍵字索引

來源筆記：`notes/<章節>/<筆記檔名>.md`

用途：使用關鍵字快速定位本章相關屬性、名詞、問題與查表檔案。

| 關鍵字 | 類型 | 對應內容 | 用途說明 | 來源 |
|---|---|---|---|---|
| 關鍵字 | 屬性 / 值 / 名詞 / 問題 / 同義詞 | `property-name` 或檔案名稱 | 短句說明 | [標題短名](../../notes/<章節>/<筆記檔名>.md#標題錨點) |
```

欄位規則：

- `關鍵字`：包含正式名稱、常見搜尋詞、中文說法與必要英文術語。
- `類型`：使用固定分類：屬性、值、名詞、問題、同義詞。
- `對應內容`：指向屬性、值、概念或 appendix 檔案。
- `用途說明`：說明這個關鍵字適合用來找什麼。
- `來源`：指向最直接支持該關鍵字對應內容的來源標題。

---

## 禁止輸出樣式

- 不要寫成教學筆記。
- 不要複製正式筆記的大段文字。
- 不要用長段落取代表格。
- 不要加入沒有來源支持的新條目。
- 不要讓同一份表同時承擔所有用途。
- 不要把自我檢查題、練習題或範例完整搬入 appendix。
- 不要為了填滿欄位而捏造預設值、規格細節或瀏覽器行為。
