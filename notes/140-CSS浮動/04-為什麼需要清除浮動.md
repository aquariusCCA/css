# 為什麼需要清除浮動

## 學習目標

讀完這篇筆記後，你應該能夠：

- 說明浮動子元素為什麼可能造成父級高度塌陷。
- 判斷什麼情況下需要清除浮動。
- 理解清除浮動的本質是閉合浮動。
- 分辨「父級有高度」與「父級靠子元素撐開」兩種布局方式。

## 問題情境

前面學浮動布局時，很多範例都給父盒子設定了固定高度：

```css
.box {
  width: 800px;
  height: 200px;
}
```

但實際頁面中，父盒子的高度不一定適合寫死。

例如產品列表、新聞列表、卡片清單，內容數量可能變多或變少。理想狀態是：父盒子裡有多少內容，就自然被內容撐多高。

![產品列表和新聞欄內容高度不固定的頁面示例](../../origin/140-CSS浮動/assets/images/clear-float-img-001-2d7043.png)

問題是，如果父盒子裡的子元素全部浮動，父盒子可能無法被這些浮動子元素撐開。

## 父級高度塌陷

浮動元素會脫離標準流，不再以普通流方式佔據原本位置。

當父級沒有設定高度，而內部子元素都浮動時，父級在標準流中可能會認為自己沒有內容可以撐開高度，最後高度變成 `0`。

這種情況常稱為父級高度塌陷。

![子盒子浮動導致父盒子高度塌陷且下方盒子上移的示意圖](../../origin/140-CSS浮動/assets/images/clear-float-img-002-2d7402.png)

範例：

```css
.box {
  width: 800px;
  border: 1px solid blue;
  margin: 0 auto;
}

.damao {
  float: left;
  width: 300px;
  height: 200px;
  background-color: purple;
}

.ermao {
  float: left;
  width: 200px;
  height: 200px;
  background-color: pink;
}

.footer {
  height: 200px;
  background-color: black;
}
```

```html
<div class="box">
  <div class="damao">大毛</div>
  <div class="ermao">二毛</div>
</div>

<div class="footer"></div>
```

如果 `.box` 沒有高度，而 `.damao`、`.ermao` 都浮動，`.box` 可能無法被子元素撐開。下方 `.footer` 仍然在標準流中，於是可能向上移動，造成布局重疊或錯位。

## 為什麼不能一律給父級高度

給父級固定高度可以避免高度塌陷，但不是所有場景都適合。

如果內容數量固定，父級高度也固定，直接寫高度可以很簡單：

```css
.box {
  height: 200px;
}
```

但如果內容會變動，固定高度就可能出問題：

- 內容變多時，可能溢出父盒子。
- 內容變少時，可能留下過多空白。
- 響應式頁面中，不同螢幕寬度可能導致內容換行，高度難以預估。

因此，很多時候更合理的方式，是讓父級被內容自然撐開。這時如果子元素用了浮動，就需要清除浮動。

## 清除浮動的本質

清除浮動的本質，是處理浮動元素造成的父級高度塌陷與後續布局影響。

可以把它理解成「閉合浮動」：

```text
讓浮動只在父盒子內部產生影響，不影響父盒子外面的其他盒子。
```

父級被正確撐開後，下方標準流元素就會接在父級後面，而不會向上跑到浮動元素附近。

## 清除浮動在處理什麼

清除浮動通常要解決兩件事：

1. 父級沒有被浮動子元素撐開。
2. 後面的標準流元素受到浮動影響而上移或錯位。

常見策略包含：

- 在浮動元素後面加入帶有 `clear: both` 的清除元素。
- 使用父級的 `::after` 偽元素產生清除效果。
- 讓父級建立 BFC，使父級能包住內部浮動。

這些方法的表面寫法不同，但核心目的都是讓父級能處理內部浮動造成的高度影響。

## 什麼時候需要清除浮動

通常同時符合以下條件時，就要考慮清除浮動：

1. 父級沒有固定高度。
2. 子元素使用了浮動。
3. 浮動影響了父級高度或下面的布局。

如果父盒子本身已經有明確高度，且後續布局沒有受到影響，就不一定需要清除浮動。

## 常見誤解

- **誤解：只要使用浮動，就一定要清除浮動。**  
  不一定。如果父級有高度，或浮動沒有造成後續布局問題，就可能不需要。

- **誤解：清除浮動是把浮動取消。**  
  清除浮動不是取消子元素的 `float`，而是處理浮動造成的父級高度與後續布局影響。

- **誤解：父級高度塌陷是 border 或 margin 的問題。**  
  核心原因是浮動子元素脫離標準流，父級無法靠它們自然撐高。

## 重點整理

- 浮動子元素脫離標準流後，可能無法撐開沒有固定高度的父級。
- 父級高度變成 `0` 後，下方標準流元素可能上移，造成布局錯位。
- 清除浮動的本質是閉合浮動，讓浮動影響留在父盒子內部。
- 當父級沒有高度、子級浮動、並影響後續布局時，通常需要清除浮動。
- 如果父級本身有高度，未必需要清除浮動。
