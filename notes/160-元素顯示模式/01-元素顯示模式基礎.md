# 元素顯示模式基礎

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明什麼是元素顯示模式。
- 分辨塊元素與行內元素的基本差異。
- 理解為什麼有些元素會獨占一行，有些元素可以排在同一行。
- 知道塊元素與行內元素在寬高、間距和嵌套上的常見限制。

## 問題情境

寫 HTML 時，你會很快遇到一個現象：

```html
<div>這是一個 div</div>
<div>這是另一個 div</div>

<span>這是一個 span</span>
<span>這是另一個 span</span>
```

`div` 通常會一個接一個換行顯示；`span` 則可以在同一行並排。這不是瀏覽器隨機決定的，而是元素本身有不同的顯示模式。

## 一句話理解

元素顯示模式就是元素在頁面中「如何佔據空間、是否換行、寬高是否可控」的基本規則。

## 常見顯示模式

HTML 元素常見的顯示模式可先分成兩大類：

- **塊元素**：例如 `<div>`、`<p>`、`<h1>` 到 `<h6>`、`<ul>`、`<ol>`、`<li>`。
- **行內元素**：例如 `<span>`、`<a>`、`<strong>`、`<em>`、`<b>`、`<i>`、`<del>`、`<ins>`。

之後會再遇到 `inline-block`，它像行內元素一樣可以並排，又像塊元素一樣可以設定寬高。

## 塊元素的特點

塊元素最典型的代表是 `<div>`。可以把它想成頁面中的一個盒子或區塊。

塊元素常見特點：

- 通常會獨占一行。
- 可以設定 `width`、`height`、`margin`、`padding`。
- 在正常流中，寬度預設為 `auto`，通常會撐滿父容器的可用寬度。
- 高度預設由內容撐開。
- 常被拿來當作容器，可以放文字、行內元素或其他合適的塊級結構。

範例：

```css
div {
  width: 200px;
  background-color: pink;
}
```

```html
<div>比較霸道，自己獨占一行</div>

旁邊的文字會出現在下一行
```

這裡 `div` 即使只有一小段文字，也會以塊元素的方式佔據一整行，因此後面的文字不會直接接在同一行。

## 行內元素的特點

行內元素最典型的代表是 `<span>`。它比較像文字流中的一段內容。

行內元素常見特點：

- 相鄰行內元素會排在同一行，一行可以顯示多個。
- 對一般非置換行內元素，直接設定 `width`、`height` 不會生效。
- 寬度與高度預設由內容撐開。
- 通常只能容納文字或其他行內元素。

範例：

```css
span {
  background-color: hotpink;
}
```

```html
<span>第一段文字</span>
<strong>重點文字</strong>
<span>第二段文字</span>
```

這些元素會像文字一樣接在同一行。若內容太長，才會依照文字換行規則換到下一行。

## 塊元素與行內元素對照

| 比較項目 | 塊元素 | 行內元素 |
| --- | --- | --- |
| 是否換行 | 通常獨占一行 | 通常可與其他行內內容同排 |
| 預設寬度 | 通常撐滿父容器可用寬度 | 由內容撐開 |
| 是否能設定寬高 | 可以 | 一般非置換行內元素不行 |
| 常見用途 | 頁面區塊、容器、布局結構 | 文字中的一小段內容或語意標記 |
| 典型元素 | `<div>`、`<p>`、`<ul>` | `<span>`、`<a>`、`<strong>` |

這張表不是用來死背，而是用來判斷：你現在要控制的是「一整塊版面」，還是「一段文字流中的內容」。

## 嵌套注意事項

塊元素可以作為容器，但不是所有塊元素都適合放任意內容。文字類元素尤其要小心。

常見規則：

- `<p>` 主要用來放段落文字，不要在裡面放 `<div>`、`<p>`、`<h1>` 到 `<h6>` 這類塊級元素。
- `<h1>` 到 `<h6>` 是標題文字，不適合再嵌套其他塊級元素。
- 行內元素通常只放文字或其他合適的行內內容。

這些規則屬於 HTML 內容模型，不只是 CSS 排版問題。就算你用 CSS 改變顯示效果，也不代表 HTML 嵌套規則跟著改變。

## 常見錯誤

### 以為所有元素都能直接設定寬高

很多初學者會對 `<span>` 寫：

```css
span {
  width: 200px;
  height: 40px;
}
```

但一般行內元素不會因為這樣就變成固定寬高的盒子。如果需要控制尺寸，可以改用 `display: inline-block` 或 `display: block`。

### 把塊元素放進文字類元素

例如：

```html
<p>
  這是一段文字
  <div>這是一個區塊</div>
</p>
```

這種寫法不符合 `<p>` 的語意與內容模型，瀏覽器可能會用自己的方式修正 DOM，結果和你想像不同。

## 實務判斷

- 要做頁面區塊、卡片、欄位：通常從塊元素開始。
- 要標記一段文字中的局部內容：通常使用行內元素。
- 想讓行內元素能控制寬高：考慮 `inline-block` 或 `block`。
- 想改變排版表現時，用 CSS；想表達內容結構時，仍要先選對 HTML 標籤。

## 重點整理

- 元素顯示模式決定元素如何佔據頁面空間。
- 塊元素通常獨占一行，寬高和內外距較容易控制。
- 行內元素通常在同一行排列，寬高由內容撐開。
- CSS 可以改變顯示模式，但不會改變 HTML 標籤本身的語意與嵌套規則。
