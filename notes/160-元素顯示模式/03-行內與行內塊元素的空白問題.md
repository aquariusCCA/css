# 行內與行內塊元素的空白問題

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明行內元素與行內塊元素之間為什麼會出現空白。
- 理解 HTML 原始碼中的換行可能被解析成空白字元。
- 使用 `font-size: 0` 處理 inline / inline-block 之間的空隙。
- 知道使用這個技巧時需要恢復子元素文字大小。

## 問題情境

當你把幾個 `<span>` 或 `inline-block` 小盒子寫成多行時，畫面上可能會出現一段看起來不該存在的空白：

```html
<span class="s1">xxx</span>
<span class="s2">aaa</span>
<span class="s3">zzz</span>
```

你明明沒有設定 `margin`，元素中間卻仍有間距。這通常不是 CSS 外距造成的，而是 HTML 原始碼中的換行和空格被瀏覽器當成文字空白處理。

## 一句話理解

行內與行內塊元素會像文字一樣排版，因此原始碼中的換行或空格也可能變成畫面上的空白。

## 為什麼會產生空白

行內元素與行內塊元素會參與文字流排版。既然它們被當成文字流中的內容，元素之間的空白字元也會被保留下來。

例如：

```html
<span>第一個</span>
<span>第二個</span>
```

兩個 `<span>` 中間有換行，瀏覽器會把它視為一個空白字元，因此畫面上會出現空隙。

## 解法：父元素 font-size 設為 0

常見解法是把父元素的字體大小設為 `0`，讓空白字元沒有可見寬度，再替真正要顯示文字的子元素恢復字體大小。

```css
div {
  width: 400px;
  height: 400px;
  background-color: gray;
  font-size: 0;
}

.s1 {
  background-color: red;
  font-size: 16px;
}

.s2 {
  background-color: green;
  font-size: 16px;
}

.s3 {
  background-color: blue;
  font-size: 16px;
}
```

```html
<div>
  <span class="s1">xxx</span>
  <span class="s2">aaa</span>
  <span class="s3">zzz</span>
</div>
```

父元素中的空白字元也會繼承 `font-size: 0`，所以不再佔據可見寬度；每個 `span` 再各自恢復 `font-size: 16px`，文字就能正常顯示。

## 範例拆解

- `div { font-size: 0; }`：消除父元素內文字空白的可見寬度。
- `.s1 { font-size: 16px; }`：恢復子元素自己的文字大小。
- `.s2`、`.s3` 同理，避免文字也跟著變成 0。

這個技巧處理的是「空白字元的寬度」，不是元素的 `margin` 或 `padding`。

## 常見錯誤

### 忘記恢復子元素字體大小

```css
.list {
  font-size: 0;
}
```

如果只在父元素設定 `font-size: 0`，但沒有在子元素恢復字體大小，子元素中的文字也可能消失或變得不可讀。

### 把空白誤判為 margin

看到元素中間有空隙時，很多人第一反應是找 `margin`。但 inline / inline-block 之間的空隙常常來自 HTML 原始碼中的換行。排查時可以先看元素是否是行內排版，再檢查原始碼中的空白。

### 在複雜布局中過度依賴這個技巧

如果你正在做比較完整的橫向排列，例如導覽列、卡片列或工具列，後續通常更適合使用 Flexbox。`font-size: 0` 是特定場景下的技巧，不是所有水平布局的首選。

## 實務判斷

- 少量 inline / inline-block 元素之間出現空白，可以考慮 `font-size: 0`。
- 若元素內有文字，務必在子元素恢復字體大小。
- 若需求包含等距排列、對齊、換行控制，應優先評估 Flexbox 或 Grid。

## 重點整理

- 行內與行內塊元素會像文字一樣處理空白。
- HTML 原始碼中的換行可能變成畫面上的空白。
- `font-size: 0` 可以讓父元素中的空白字元不佔可見寬度。
- 子元素要另外恢復字體大小，否則文字會跟著受影響。
