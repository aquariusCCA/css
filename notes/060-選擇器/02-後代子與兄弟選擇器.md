# 後代、子與兄弟選擇器

## 學習目標

讀完這篇筆記後，你應該能夠：

- 理解複合選擇器如何透過元素關係更精準地選取目標。
- 分辨後代選擇器與子選擇器的差異。
- 分辨相鄰兄弟選擇器 `+` 與一般兄弟選擇器 `~`。
- 避免因為空格、`>`、`+`、`~` 混用造成選取結果錯誤。

## 使用情境

基礎選擇器只能根據標籤、class、id 等條件選元素。但實務中常常需要根據 HTML 結構來選取，例如：

- 只選導覽列裡面的連結。
- 只選某個容器的直接子元素。
- 只選標題後面的第一個段落。
- 只選某個元素後方的同層元素。

這些需求都屬於「元素之間的關係」，需要使用複合選擇器。

## 複合選擇器是什麼

複合選擇器是建立在基礎選擇器之上的組合寫法，通常由兩個或更多基礎選擇器組成，用來更精準地選中元素。

這篇聚焦四種和元素關係有關的寫法：

| 選擇器 | 符號 | 關係 |
| --- | --- | --- |
| 後代選擇器 | 空格 | 某元素內部所有符合條件的後代 |
| 子選擇器 | `>` | 某元素的直接子元素 |
| 相鄰兄弟選擇器 | `+` | 某元素後方緊接的第一個兄弟 |
| 一般兄弟選擇器 | `~` | 某元素後方所有符合條件的兄弟 |

## 後代選擇器

後代選擇器使用空格表示祖先與後代關係。

```css
元素1 元素2 {
    樣式宣告
}
```

它會選中 `元素1` 裡面所有符合 `元素2` 的後代元素，不限於直接子元素。

```css
ol li {
    color: pink;
}

ol li a {
    color: red;
}

.nav li a {
    color: yellow;
}
```

```html
<ul class="nav">
    <li><a href="#">連結一</a></li>
    <li><a href="#">連結二</a></li>
</ul>
```

`.nav li a` 會選中 `.nav` 裡面、位於 `li` 後代中的所有 `<a>`。中間的空格很重要，它表示「在某個祖先元素裡面的所有指定後代」。

## 子選擇器

子選擇器使用大於符號 `>` 表示，只會選中某個元素的直接子元素。

```css
元素1 > 元素2 {
    樣式宣告
}
```

例如：

```css
.nav > a {
    color: red;
}
```

```html
<div class="nav">
    <a href="#">直接子元素連結</a>
    <p>
        <a href="#">不是直接子元素的連結</a>
    </p>
</div>
```

在這個例子中，`.nav > a` 只會選中 `.nav` 的直接子元素 `<a>`，不會選中包在 `<p>` 裡面的 `<a>`。

子選擇器比後代選擇器更嚴格，適合用在元素層級關係需要明確限定的情境。

## 後代與子的差異

可以用一句話記：

- 空格：只要在裡面，不管隔幾層都算。
- `>`：只能是下一層，隔一層就不算。

```css
.nav a {
    color: red;
}

.nav > a {
    color: blue;
}
```

第一段 `.nav a` 會選中 `.nav` 裡所有 `<a>`；第二段 `.nav > a` 只會選中直接放在 `.nav` 下一層的 `<a>`。

## 兄弟選擇器

兄弟選擇器用來選中與某個元素位於同一父層底下的後方兄弟元素。

### 相鄰兄弟選擇器

相鄰兄弟選擇器 `+` 只會選中緊接在前一個元素後面的第一個兄弟元素。

```css
h1 + p {
    color: red;
}
```

```html
<h1>標題</h1>
<p>這個段落會被選中</p>
<p>這個段落不會被 h1 + p 選中</p>
```

`h1 + p` 的意思是：選中緊接在 `<h1>` 後面的第一個 `<p>`。

也可以用於連續元素之間的分隔：

```css
li + li {
    border-top: 1px solid #ccc;
}
```

這種寫法可以讓「第二個開始的每個 `li`」都有上邊框，避免第一個也出現不必要的分隔線。

### 一般兄弟選擇器

一般兄弟選擇器 `~` 會選中指定元素之後所有符合條件的兄弟元素。

```css
h1 ~ p {
    color: blue;
}
```

```html
<h1>標題</h1>
<p>第一個後方兄弟段落</p>
<p>第二個後方兄弟段落</p>
```

`h1 ~ p` 會選中 `<h1>` 後方所有同層的 `<p>`。

## 對優先級的影響

複合選擇器的優先級會把各個選擇器的權重加總。符號本身，例如空格、`>`、`+`、`~`，不直接增加權重。

例如：

```css
.nav a {
    color: red;
}
```

這個選擇器的權重來自 `.nav` 和 `a`，而不是中間的空格。

## 常見錯誤

- **把後代選擇器當成直接子元素選擇器**：`.nav a` 會選中所有層級裡的 `<a>`，不只下一層。若只想選直接子元素，要用 `.nav > a`。
- **忘記兄弟選擇器只能選後方元素**：`h1 + p` 不會選中 `<h1>` 前面的 `<p>`。
- **以為 `+` 會選中所有兄弟**：`+` 只選緊接的第一個；要選後方所有符合條件的兄弟，使用 `~`。
- **誤刪空格造成意義改變**：`.nav a` 和 `.nava` 是完全不同的選擇器。

## 實務判斷準則

- 想選容器裡所有符合條件的元素：用後代選擇器。
- 想只限制下一層：用子選擇器。
- 想選緊接在某元素後面的下一個元素：用 `+`。
- 想選某元素後方所有同層元素：用 `~`。
- 結構關係越明確，選擇器越不容易誤傷其他元素。

## 重點整理

- 後代選擇器用空格，會選中所有層級中的指定後代。
- 子選擇器用 `>`，只選直接子元素。
- 相鄰兄弟選擇器 `+` 只選下一個兄弟。
- 一般兄弟選擇器 `~` 選後方所有符合條件的兄弟。
- 這些符號表達的是元素關係，權重仍由實際選擇器組成決定。
