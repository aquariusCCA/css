# 選擇器總覽與基礎選擇器

## 學習目標

讀完這篇筆記後，你應該能夠：

- 說明選擇器在 CSS 規則中的作用。
- 分辨標籤選擇器、類選擇器、id 選擇器與通配符選擇器。
- 判斷什麼時候該用 class，而不是只用標籤或 id。
- 理解多類名如何拆分與組合樣式職責。

## 為什麼要學選擇器

寫 CSS 時，第一個問題通常不是「要設定什麼樣式」，而是「這個樣式要套用到誰身上」。

如果選錯元素，樣式可能完全沒效果；如果選太廣，可能讓不該變化的元素也被影響。選擇器的任務就是根據需求找出指定的 HTML 元素，讓後續樣式只套用在目標元素上。

可以把 CSS 規則拆成兩個步驟：

1. 找到元素：選擇器負責「找對人」。
2. 設定樣式：宣告區塊負責「做對事」。

## 基礎選擇器有哪些

選擇器可以先分成兩大類：

- 基礎選擇器：由單一選擇器組成。
- 複合選擇器：在基礎選擇器之上組合而成，用來表達更精準的元素關係或狀態。

這篇先看四種基礎選擇器：

| 選擇器 | 語法 | 選取對象 | 常見用途 |
| --- | --- | --- | --- |
| 標籤選擇器 | `p` | 所有同名標籤 | 設定同類元素的共同樣式 |
| 類選擇器 | `.red` | class 包含 `red` 的元素 | 可重複套用的樣式 |
| id 選擇器 | `#pink` | id 為 `pink` 的元素 | 唯一元素或 JavaScript 取得元素 |
| 通配符選擇器 | `*` | 所有元素 | 全域初始化 |

## 標籤選擇器

標籤選擇器使用 HTML 標籤名稱作為選擇器，會選中頁面中所有同名標籤。

```css
標籤名 {
    屬性名: 屬性值;
}
```

例如：

```css
p {
    color: green;
}

div {
    color: pink;
}
```

這段 CSS 會讓所有 `<p>` 變成綠色，所有 `<div>` 變成粉色。

標籤選擇器適合設定某一類元素的共同樣式。不過它的影響範圍較大，如果只想選中少數特定元素，通常會改用類選擇器或其他更精準的選擇器。

## 類選擇器

類選擇器會選中具有指定 `class` 屬性的元素。CSS 中以英文句點 `.` 開頭，HTML 中透過 `class` 屬性標記。

```css
.red {
    color: red;
}
```

```html
<div class="red">這個 div 會套用 red 類別樣式</div>
```

類選擇器比標籤選擇器更精準，也更適合重複使用。不同標籤可以套用同一個 class，因此 class 很常用來抽出可共用的樣式。

類名命名時建議：

- 使用有語意的名稱，例如 `.nav`、`.header`、`.footer`。
- 多個單字可以用連字號連接，例如 `.star-sing`。
- 避免使用純數字、中文或特殊符號作為類名。

## 多類名

同一個 HTML 元素可以同時擁有多個類名。多個類名寫在同一個 `class` 屬性中，彼此用空格分隔。

```css
.red {
    color: red;
}

.font35 {
    font-size: 35px;
}
```

```html
<div class="red font35">Kevin</div>
```

這個 `<div>` 同時套用了 `.red` 和 `.font35` 兩組樣式。

多類名常用來拆分樣式職責：

- 公共樣式：多個元素都會使用的樣式。
- 個別樣式：只針對某些元素補上的差異。

這種寫法可以減少重複 CSS，也能讓樣式組合更彈性。

## id 選擇器

id 選擇器會選中具有指定 `id` 屬性的元素。CSS 中以井字號 `#` 開頭，HTML 中透過 `id` 屬性指定。

```css
#pink {
    color: pink;
}
```

```html
<div id="pink">這個 div 會套用 pink id 樣式</div>
```

在同一份 HTML 文件中，`id` 應該保持唯一。也就是說，同一個 `id` 名稱不應重複套用在多個元素上。

類選擇器與 id 選擇器可以這樣理解：

- 類選擇器可以重複使用，像人的名字，可以很多人同名。
- id 選擇器應保持唯一，像身分證字號，通常只對應一個元素。

實務上，`id` 也常用於 JavaScript 精準取得特定元素。不過在 CSS 樣式設計中，若樣式需要重複使用，通常優先考慮 class。

## 通配符選擇器

通配符選擇器使用 `*` 表示，會選中頁面中的所有元素。

```css
* {
    margin: 0;
    padding: 0;
}
```

通配符選擇器不需要在 HTML 中額外呼叫，因為它會自動匹配所有標籤。

常見用途是做全域初始化，例如清除瀏覽器預設的外距與內距：

```css
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}
```

由於它會影響整個頁面，使用時要注意不要意外覆蓋太多元素的預設行為。

## 對優先級的基本影響

不同選擇器的優先級不同。先記住大方向即可：

```text
id 選擇器 > 類選擇器 > 標籤選擇器
```

通配符選擇器的優先級最低，通常不會拿來覆蓋具體選擇器。

後續學到層疊與優先級時，會更完整地計算權重；在這一章先建立選取範圍與使用時機的概念。

## 常見錯誤

- **用標籤選擇器改少數元素**：如果只想改某幾個段落，直接寫 `p { ... }` 會影響全部 `<p>`，應改用 class 或更精準的選擇器。
- **把 id 當成 class 重複使用**：同一份 HTML 中 id 應保持唯一。需要重複套用樣式時，應使用 class。
- **濫用通配符選擇器**：`*` 會選中所有元素，適合初始化，但不適合用來處理局部樣式。
- **類名只描述顏色或外觀**：像 `.red` 可以作為入門示範，但實務上更建議用有語意的名稱，例如 `.error-message`。

## 實務判斷準則

- 要設定一類 HTML 元素的基本樣式：用標籤選擇器。
- 要讓多個元素共用一組樣式：用類選擇器。
- 要標記文件中唯一的元素，或讓 JavaScript 精準取得：可用 id。
- 要做全域初始化：可用通配符選擇器，但要控制影響範圍。

## 重點整理

- 選擇器負責找出要套用樣式的元素。
- 標籤選擇器會選中所有同名標籤，影響範圍較大。
- 類選擇器可重複使用，是實務中最常用的基礎選擇器。
- 同一個元素可以有多個 class，彼此用空格分隔。
- id 選擇器應保持唯一。
- 通配符選擇器會選中所有元素，常用於初始化。
