# 連結與 focus 偽類選擇器

## 學習目標

讀完這篇筆記後，你應該能夠：

- 理解偽類選擇器用來選取元素的狀態或位置。
- 使用連結偽類設定 `<a>` 在不同互動狀態下的樣式。
- 說明 LVHA 書寫順序的原因。
- 使用 `:focus` 提示目前正在操作的表單欄位。

## 使用情境

有些樣式不是根據元素名稱或 class 決定，而是根據元素目前的狀態決定。

例如：

- 連結尚未訪問時是一種顏色。
- 使用者滑鼠移到連結上時變色。
- 連結按下時顯示另一種狀態。
- 輸入框取得焦點時改變背景色。

這些都適合使用偽類選擇器。

偽類選擇器用冒號 `:` 表示，用來選中元素的特定狀態或位置。

## 連結偽類選擇器

連結偽類選擇器常用於 `<a>` 的不同狀態：

```css
a:link {
    color: #333;
}

a:visited {
    color: purple;
}

a:hover {
    color: red;
}

a:active {
    color: orange;
}
```

四個常見狀態如下：

| 偽類 | 狀態 |
| --- | --- |
| `:link` | 尚未訪問的連結 |
| `:visited` | 已訪問的連結 |
| `:hover` | 滑鼠移到連結上時 |
| `:active` | 連結被按下但尚未放開時 |

```html
<a href="#">站內示範連結</a>
<a href="http://www.xxxxxxxx.com">外部示範連結</a>
```

## LVHA 書寫順序

連結偽類通常遵守 LVHA 順序：

```text
:link -> :visited -> :hover -> :active
```

這個順序的目的，是避免相同權重下後寫的狀態覆蓋前面的狀態，導致效果不如預期。

例如 `:hover` 如果寫在 `:link` 前面，當兩條規則都可能符合時，後面的 `:link` 可能覆蓋 hover 效果。依照 LVHA 順序寫，互動狀態通常比較穩定。

## focus 偽類選擇器

`:focus` 偽類選擇器用來選中取得焦點的元素，常見於表單控制項。

例如，當使用者點擊輸入框或透過鍵盤切換到輸入框時，可以改變它的背景色：

```css
input:focus {
    background-color: pink;
}
```

```html
<input type="text">
<input type="text">
<input type="text">
```

`:focus` 適合用來提示目前正在操作的表單欄位，有助於使用者理解目前輸入位置。

## 對優先級的影響

偽類選擇器的權重通常與類選擇器相同。

例如：

```css
a:hover {
    color: red;
}
```

這裡的優先級包含 `a` 與 `:hover`。如果有其他更高優先級的規則套用到同一個連結，就可能覆蓋 hover 樣式。

## 常見錯誤

- **連結偽類順序錯誤**：沒有按照 LVHA 順序時，`:hover` 或 `:active` 可能被其他狀態覆蓋。
- **只設 hover，不考慮 focus**：滑鼠使用者可以看到 hover，但鍵盤使用者需要 focus 提示目前位置。
- **把 `:focus` 當成輸入中狀態**：`:focus` 表示元素取得焦點，不代表使用者已輸入內容。
- **移除 focus 外框卻沒有替代樣式**：若把瀏覽器預設 focus 樣式取消，應提供清楚的替代提示。

## 實務判斷準則

- 需要根據滑鼠互動改變連結樣式：使用 `:hover`、`:active`。
- 需要區分已訪問與未訪問連結：使用 `:link`、`:visited`。
- 需要提示表單目前操作位置：使用 `:focus`。
- 寫連結偽類時，優先採用 LVHA 順序，讓狀態覆蓋關係更穩定。

## 重點整理

- 偽類選擇器用 `:` 表示，用來選取元素的狀態或位置。
- 連結常見偽類包含 `:link`、`:visited`、`:hover`、`:active`。
- 連結偽類建議依 LVHA 順序書寫。
- `:focus` 用來選取取得焦點的元素，常見於表單欄位。
- 偽類選擇器的權重通常與類選擇器相同。
