# 並集與交集選擇器

## 學習目標

讀完這篇筆記後，你應該能夠：

- 使用並集選擇器讓多組元素套用同一組樣式。
- 使用交集選擇器選中同時符合多個條件的元素。
- 分辨逗號、無空格與空格造成的選取差異。
- 避免把交集選擇器誤寫成後代選擇器。

## 使用情境

有時候，多個不同元素需要同一組樣式。例如標題、段落與某個列表項都要變成同一種顏色。如果每個選擇器都重複寫一次宣告，CSS 會變得冗長。

也有時候，你不是要選「所有 `.box`」，而是只要選「同時是 `<p>` 且 class 包含 `box`」的元素。這時就需要交集選擇器。

可以用一句話記：

- 並集：多組條件中任一組符合即可，像「或」。
- 交集：必須同時符合多個條件，像「且」。

## 並集選擇器

並集選擇器使用逗號 `,` 分隔多個選擇器，表示同時選中多組元素並套用同一組樣式。

```css
div,
p {
    color: red;
}
```

這段 CSS 會同時選中所有 `<div>` 與所有 `<p>`。

並集選擇器也可以混合不同種類的選擇器：

```css
div,
p,
.pig li {
    color: pink;
}
```

這代表三組目標都會套用 `color: pink`：

- 所有 `<div>`。
- 所有 `<p>`。
- `.pig` 裡面的所有 `<li>`。

## 並集選擇器的可讀性

實務上常把每個選擇器獨立寫在一行：

```css
h1,
h2,
h3 {
    font-weight: 700;
}
```

這樣比較容易新增或刪除其中一組選擇器。注意最後一個選擇器後面不加逗號。

## 交集選擇器

交集選擇器會選中同時符合多個條件的元素。多個選擇器之間不加空格，也不加其他符號。

```css
p.box {
    color: red;
}
```

```html
<p class="box">這個 p 會被選中</p>
<div class="box">這個 div 不會被 p.box 選中</div>
```

`p.box` 的意思是：選中「同時是 `<p>`，且 class 包含 `box`」的元素。

如果交集選擇器中包含標籤選擇器，標籤選擇器通常寫在最前面：

```css
標籤名.類名 {
    樣式宣告;
}
```

## 並集與交集的差異

| 寫法 | 意思 | 類比 |
| --- | --- | --- |
| `div, .box` | 選中所有 `<div>`，也選中所有 `.box` | 或 |
| `div.box` | 選中同時是 `<div>` 且 class 包含 `box` 的元素 | 且 |
| `div .box` | 選中 `<div>` 裡面的 `.box` 後代元素 | 裡面的 |

這三種寫法只差一點點，但意思完全不同。

## 對優先級的影響

並集選擇器中的每一組選擇器各自計算優先級，不會因為寫在一起就互相加總。

```css
h1,
.title {
    color: red;
}
```

`h1` 仍然是標籤選擇器權重，`.title` 仍然是類選擇器權重。

交集選擇器則會把組成條件的權重加總。例如 `p.box` 的權重來自 `p` 和 `.box`。

## 常見錯誤

- **並集最後多寫逗號**：`h1, h2, { ... }` 是錯誤語法，最後一個選擇器後面不要加逗號。
- **交集選擇器中間加空格**：`p.box` 是交集，`p .box` 是後代選擇器，兩者選取結果不同。
- **把並集當成提高權重的方式**：並集只是共用宣告，不會讓每一組選擇器權重互相累加。
- **交集條件過多**：過度限制會讓選擇器難以重用，也可能導致樣式很難覆蓋。

## 實務判斷準則

- 多組元素需要相同宣告：使用並集選擇器。
- 一個元素必須同時符合多個條件才要套用樣式：使用交集選擇器。
- 看到逗號時，先理解為「這些選擇器共用同一組宣告」。
- 看到選擇器緊貼在一起時，先理解為「同一個元素要同時符合條件」。

## 重點整理

- 並集選擇器用逗號 `,`，表示多組選擇器共用同一組樣式。
- 交集選擇器不加空格，表示同一個元素要同時符合多個條件。
- `p.box`、`p .box`、`p, .box` 是三種不同意思。
- 並集不會加總不同選擇器的權重；交集會加總同一組選擇器的權重。
