# CSS 引入方式比較與選擇

## 學習目標

讀完這篇筆記，你應該能夠：

- 說出 CSS 常見的三種引入方式。
- 比較行內樣式表、內部樣式表與外部樣式表的作用範圍。
- 根據使用場景選擇合適的 CSS 引入方式。
- 理解三種方式在維護性、複用性與結構分離程度上的差異。

## 三種常見引入方式

CSS 常見的三種引入方式是：

- 外部樣式表
- 內部樣式表
- 行內樣式表

它們都可以讓 CSS 套用到 HTML，但寫法、作用範圍和適合情境不同。

![CSS 三種引入方式比較表，列出書寫位置、作用範圍與使用場景](../../origin/030-CSS的引入方式/assets/images/ways-to-include-css-img-001-e7905f.png)

## 快速比較

| 引入方式 | 寫在哪裡 | 作用範圍 | 適合情境 |
| --- | --- | --- | --- |
| 行內樣式表 | HTML 標籤的 `style` 屬性 | 當前標籤 | 少量、臨時、單一元素樣式 |
| 內部樣式表 | HTML 文件的 `<style>` 標籤 | 當前頁面 | 練習、示範、單頁樣式 |
| 外部樣式表 | 獨立的 `.css` 檔案 | 可被多個頁面共用 | 實際開發、多頁網站、需要維護的樣式 |

從維護性來看，通常會優先考慮外部樣式表；從快速測試來看，行內樣式和內部樣式表也有各自的使用空間。

## 外部樣式表

外部樣式表把 CSS 寫在獨立的 `.css` 檔案中，再透過 `<link>` 引入 HTML。

```html
<link rel="stylesheet" href="./style.css">
```

它的作用範圍可以是多個頁面。只要多個 HTML 文件都引入同一份 CSS，就能共用同一套樣式。

優點：

- 樣式可以在多個頁面複用。
- 程式碼結構清楚。
- HTML 和 CSS 分離得比較徹底。
- CSS 檔案可觸發瀏覽器快取機制。

限制：

- 需要正確引入後才能使用。
- `href` 路徑寫錯時，樣式不會套用。

外部樣式表適合實際開發，也是長期維護網站時最常使用的方式。

## 內部樣式表

內部樣式表把 CSS 寫在 HTML 文件中的 `<style>` 標籤內，通常放在 `<head>` 裡。

```html
<style>
  p {
    color: pink;
  }
</style>
```

它的作用範圍通常是目前這一個頁面。

優點：

- 比行內樣式更集中，程式碼結構比較清楚。
- 可以透過選擇器控制目前頁面中的多個元素。
- 適合 CSS 練習與單頁示範。

限制：

- CSS 仍然放在 HTML 檔案中，結構與樣式沒有徹底分離。
- 其他 HTML 頁面無法直接共用這段樣式。

內部樣式表適合學習階段，但當樣式變多或需要跨頁共用時，就應該改用外部樣式表。

## 行內樣式表

行內樣式表把 CSS 寫在 HTML 標籤的 `style` 屬性中。

```html
<p style="color: pink; font-size: 20px;">給我一個粉紅的回憶</p>
```

它的作用範圍是當前標籤。

優點：

- 寫法直接，可以快速針對單一元素設定樣式。
- 在一般情況下，行內樣式比外部樣式表與內部樣式表中的普通宣告優先級更高。

限制：

- 結構與樣式混在同一個標籤裡。
- 程式碼容易變得混亂。
- 樣式不能方便複用。
- 大量使用時非常不利於維護。

行內樣式可以少量用於測試或特殊情境，但不建議作為主要寫法。

## 如何選擇

可以用以下方式判斷：

| 情境 | 建議方式 |
| --- | --- |
| 只想快速測試單一元素樣式 | 行內樣式表 |
| 正在練習 CSS 選擇器或製作單頁範例 | 內部樣式表 |
| 樣式較多、需要長期維護 | 外部樣式表 |
| 多個頁面要共用同一套樣式 | 外部樣式表 |
| 想讓 HTML 和 CSS 分工清楚 | 外部樣式表 |

初學時可以先用內部樣式表練習 CSS 規則；進入實際開發或多頁專案時，應逐步習慣外部樣式表。

## 維護性的差異

三種方式最大的差別之一，是樣式是否容易集中修改。

假設網站中有很多段落都要變成粉紅色：

- 用行內樣式表：每個 `<p>` 都要寫 `style="color: pink;"`。
- 用內部樣式表：目前頁面可以集中寫一條 `p { color: pink; }`。
- 用外部樣式表：多個頁面都可以共用同一份 `style.css`。

所以，越需要複用、越需要長期維護，就越應該把樣式集中到外部 CSS 檔案中。

## 常見誤解

### 誤解：行內樣式優先級較高，所以應該多用

優先級較高不代表適合大量使用。行內樣式會降低可維護性，且後續學到 `!important`、選擇器優先級與層疊規則時，還會看到更完整的 CSS 生效判斷。

### 誤解：內部樣式表和外部樣式表只差在寫法不同

兩者不只寫法不同，複用範圍也不同。內部樣式表主要服務目前頁面；外部樣式表可以被多個頁面共用。

### 誤解：外部樣式表比較麻煩，所以初學不用學

外部樣式表需要多一個引入步驟，但它是實際開發最常用的方式。越早理解 HTML 和 CSS 檔案分工，後續越容易銜接真正的網站專案。

## 重點整理

- CSS 常見引入方式有行內樣式表、內部樣式表與外部樣式表。
- 行內樣式表寫在 `style` 屬性中，只影響當前標籤。
- 內部樣式表寫在 `<style>` 標籤中，通常影響目前頁面。
- 外部樣式表寫在 `.css` 檔案中，可被多個頁面共用。
- 實際開發通常優先使用外部樣式表，因為它更利於複用、維護與結構分離。
