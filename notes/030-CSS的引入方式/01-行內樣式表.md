# 行內樣式表：直接寫在標籤上的 CSS

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明行內樣式表的寫法與作用範圍。
- 理解 `style` 屬性中應該撰寫 CSS 聲明。
- 判斷什麼情況可以少量使用行內樣式。
- 說明為什麼不建議大量使用行內樣式。

## 一句話理解

行內樣式表是把 CSS 直接寫在 HTML 標籤的 `style` 屬性中，只影響目前這一個元素。

例如：

```html
<p style="color: pink; font-size: 20px;">給我一個粉紅的回憶</p>
```

這段程式碼會讓這個 `<p>` 元素的文字變成粉紅色，並把字體大小設定為 `20px`。

## `style` 屬性的角色

`style` 是 HTML 標籤上的屬性。它的屬性值不是普通文字，而是一段 CSS 聲明。

```html
<p style="color: pink; font-size: 20px;">文字內容</p>
```

其中：

| 部分 | 說明 |
| --- | --- |
| `<p>` | 要設定樣式的 HTML 元素 |
| `style` | 用來放 CSS 的 HTML 屬性 |
| `color: pink;` | 設定文字顏色 |
| `font-size: 20px;` | 設定文字大小 |

因為 `style` 寫在 `<p>` 標籤裡，所以這些樣式只會作用在這個 `<p>` 元素本身。

## 行內樣式的作用範圍

行內樣式表的作用範圍是「目前這個標籤」。

例如：

```html
<p style="color: pink;">第一段文字</p>
<p>第二段文字</p>
```

只有第一段文字會變成粉紅色，第二段文字不會受到影響。這是行內樣式最直觀的特點：它很精準，但也很難複用。

## 什麼時候可以使用

行內樣式適合用在非常少量、臨時、只針對單一元素的樣式。

例如：

- 測試某個 CSS 屬性是否會產生預期效果。
- 在練習時快速觀察單一元素的樣式變化。
- 對某一個元素做極少量的特殊處理。

如果同樣的樣式會出現在多個元素上，就不適合繼續使用行內樣式，應該改用內部樣式表或外部樣式表。

## 為什麼不建議大量使用

行內樣式雖然好理解，但大量使用會讓 HTML 變得難以維護。

主要問題包含：

- 樣式和結構混在同一個標籤裡，閱讀時不清楚。
- 同樣的樣式無法集中管理，常常需要重複書寫。
- 修改樣式時必須逐一尋找每個標籤。
- 不符合「結構與樣式分離」的思路。

例如：

```html
<p style="color: pink; font-size: 20px;">第一段</p>
<p style="color: pink; font-size: 20px;">第二段</p>
<p style="color: pink; font-size: 20px;">第三段</p>
```

如果之後想把文字改成藍色，就要改三次。若用 CSS 選擇器集中管理，只需要改一處。

## 常見誤解

### 誤解：行內樣式最直接，所以正式開發應該優先使用

行內樣式確實直接，但直接不等於好維護。正式開發通常更重視可讀性、複用性與結構分離，因此不會大量使用行內樣式。

### 誤解：`style` 裡可以隨便寫描述文字

`style` 屬性的內容必須符合 CSS 聲明格式，例如 `color: pink;`。如果語法錯誤，瀏覽器可能會忽略錯誤的樣式。

### 誤解：行內樣式會影響整個頁面

行內樣式只影響目前這個標籤。若要控制整個頁面的多個元素，應改用內部樣式表或外部樣式表。

## 重點整理

- 行內樣式表寫在 HTML 標籤的 `style` 屬性中。
- `style` 屬性的內容要符合 CSS 聲明格式。
- 行內樣式只控制目前這個標籤。
- 行內樣式適合少量、臨時、單一元素的樣式設定。
- 不建議大量使用，因為它會讓結構與樣式混在一起，降低維護性。
