# vw、vh 與流式百分比布局

## 學習目標

讀完這篇筆記後，你應該能夠：

- 說明 `vw`、`vh` 分別以什麼作為換算基準。
- 將設計稿中的 px 尺寸換算為 `vw` 或 `vh`。
- 理解百分比流式布局如何讓寬度自適應。
- 判斷何時使用視口單位，何時使用百分比布局。

## 問題情境

移動端設備寬度不同，如果所有元素都寫死 `px`，在小螢幕上可能太大，在大螢幕上又可能太小。這時可以使用與視口或容器相關的單位，讓元素跟著可用空間調整。

常見做法有兩類：

- 使用 `vw` / `vh`，讓尺寸跟著視口寬度或高度變化。
- 使用百分比，讓元素跟著父容器寬度伸縮。

## 一句話理解

`vw`、`vh` 依視口尺寸換算；百分比布局依父容器或包含區塊伸縮。

## vw 與 vh 的基準

`vw` 是 viewport width，代表視口寬度的百分之一。

```text
1vw = 視口寬度 / 100
```

`vh` 是 viewport height，代表視口高度的百分之一。

```text
1vh = 視口高度 / 100
```

例如設備視口寬度是 `357px`：

```text
1vw = 357px / 100 = 3.57px
```

如果你要把設計稿上的 `68px` 換算成 `vw`，可以用：

```text
vw 值 = px 數值 / (視口寬度 / 100)
```

以 `375px` 設計稿為例：

```text
1vw = 375px / 100 = 3.75px
68px = 68 / 3.75 = 18.13vw
```

所以可以寫成：

```css
.box {
  width: 18.13vw;
}
```

## vw 範例

```html
<!DOCTYPE html>
<html lang="en">

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
  <style>
    * {
      margin: 0;
      padding: 0;
    }

    .box {
      width: 50vw;
      height: 30vw;
      background-color: pink;
    }
  </style>
</head>

<body>
  <div class="box"></div>
</body>

</html>
```

這裡 `.box` 的寬度是視口寬度的 `50%`，高度是視口寬度的 `30%`。因為寬高都使用 `vw`，元素比例會隨視口寬度變化。

## vh 範例

```css
.box {
  width: 50vh;
  height: 30vh;
  background-color: pink;
}
```

這裡寬度與高度都依視口高度換算。`vh` 常用於與螢幕高度相關的場景，例如滿屏區塊、視窗高度分段等。

實作時要注意：不同設備的寬高比例不同。若同一個元件同時混用 `vw` 和 `vh` 控制主要寬高，可能造成比例在不同設備上不一致。通常會選擇一個主要基準，再依需求輔助調整。

## 流式百分比布局

流式布局就是百分比布局，也可以理解為非固定像素布局。它常見效果是：寬度自適應，高度固定。

![百分比布局寬度自適應效果示意圖](../../origin/310-移動端網頁適配/assets/images/percentage-layout-img-001-44d23f.png)

它的核心做法是：將盒子的寬度設定為百分比，使元素根據父容器或屏幕可用寬度伸縮，不受固定像素限制。

```css
section {
  width: 100%;
  max-width: 980px;
  min-width: 320px;
  margin: 0 auto;
}

section div {
  float: left;
  width: 50%;
  height: 400px;
}
```

這段程式碼的意義是：

- `section { width: 100%; }`：容器寬度跟隨可用空間。
- `max-width: 980px`：最大不超過 `980px`。
- `min-width: 320px`：最小不低於 `320px`。
- `margin: 0 auto`：當容器小於最大寬度時可水平置中。
- `section div { width: 50%; }`：兩個子元素各占一半寬度。

## 視口單位與百分比怎麼選

| 情境 | 建議方向 |
| --- | --- |
| 元素尺寸要直接跟整個視口寬度變化 | `vw` |
| 元素尺寸要直接跟整個視口高度變化 | `vh` |
| 子元素寬度要跟父容器變化 | `%` |
| 需要設定最大或最小內容寬度 | `%` 搭配 `max-width` / `min-width` |
| 需要從設計稿按視口比例換算尺寸 | `vw` 或 rem 方案 |

## 常見錯誤

### 把 vw 當成父容器百分比

`vw` 是相對視口寬度，不是相對父容器。若你希望子元素只依父容器寬度伸縮，通常應使用 `%`。

### 混用 vw 與 vh 造成比例失真

`vw` 依視口寬度，`vh` 依視口高度。不同手機寬高比不同，混用時可能導致元件比例在某些設備上變形。

### 百分比布局缺少最大或最小限制

純 `width: 100%` 很彈性，但在大螢幕或極小螢幕上可能不適合。常搭配 `max-width`、`min-width` 控制合理範圍。

## 重點整理

- `1vw` 是視口寬度的百分之一。
- `1vh` 是視口高度的百分之一。
- 設計稿 px 可除以 `視口寬度 / 100` 換算為 `vw`。
- 百分比布局讓元素依父容器或包含區塊伸縮。
- 視口單位適合依整個螢幕變化，百分比適合依容器變化。
