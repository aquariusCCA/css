# rem 加媒體查詢開發方案

## 學習目標

讀完這篇筆記後，你應該能夠：

- 說明 `rem + 媒體查詢` 適配方案的核心流程。
- 依設計稿寬度、等分數與元素尺寸換算 rem。
- 看懂不同斷點下根字級如何影響元素大小。
- 避免斷點不足或換算基準不一致造成的尺寸問題。

## 問題情境

實務開發中，設計稿通常會給一個基準寬度，例如 `750px`。但使用者的手機可能是 `320px`、`375px`、`414px` 或其他寬度。

如果所有元素都照設計稿寫成固定 `px`，在不同設備上比例會不一致。`rem + 媒體查詢` 的做法，是根據設備寬度設定不同的 `html font-size`，再把元素尺寸寫成 `rem`，讓元素跟著根字級等比例縮放。

## 一句話理解

`rem + 媒體查詢` 是用媒體查詢改變根字級，再用 rem 讓頁面元素依根字級等比例縮放的方案。

## 方案核心

假設設計稿寬度是 `750px`，並把整個屏幕劃分為 `15` 等份：

```text
750 / 15 = 50
```

也就是在 `750px` 設計稿基準下：

```text
1rem = 50px
```

如果頁面元素在設計稿中是 `100px * 100px`，換算為 rem：

```text
100px / 50px = 2rem
```

所以元素可以寫成：

```css
div {
  width: 2rem;
  height: 2rem;
}
```

當設備寬度變成 `320px`，同樣分成 `15` 等份：

```text
320 / 15 = 21.33
```

此時：

```text
1rem = 21.33px
2rem = 42.66px
```

元素變小了，但寬高比例仍保持一致。

## 換算公式

可以用兩個等價公式理解：

```text
html font-size = 屏幕寬度 / 劃分份數
```

```text
頁面元素 rem 值 = 頁面元素 px 值 / html font-size
```

如果以設計稿 `750px`、`15` 等份為基準：

```text
html font-size = 750 / 15 = 50px
元素 100px = 100 / 50 = 2rem
```

## 範例程式碼

下方範例只列出 `320px` 與 `750px` 兩個斷點，用來示範換算方法；完整方案應補齊目標設備斷點，或使用 `calc(100vw / 15)` 等方式讓根字級連續隨視口變化。

![rem 適配方案的常見設備寬度表](../../origin/310-移動端網頁適配/assets/images/rem-adaptive-layout-img-002-ff144f.png)

```html
<!DOCTYPE html>
<html lang="en">

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
  <style>
    @media screen and (min-width: 320px) {
      html {
        font-size: 21.33px;
      }
    }

    @media screen and (min-width: 750px) {
      html {
        font-size: 50px;
      }
    }

    div {
      width: 2rem;
      height: 2rem;
      background-color: pink;
    }
  </style>
</head>

<body>
  <div></div>
</body>

</html>
```

## 範例拆解

在 `320px` 以上時：

```css
html {
  font-size: 21.33px;
}
```

此時：

```text
2rem = 42.66px
```

在 `750px` 以上時：

```css
html {
  font-size: 50px;
}
```

此時：

```text
2rem = 100px
```

同一個 CSS 元素 `width: 2rem; height: 2rem;`，會因根字級不同而呈現不同像素尺寸。

## 實務使用流程

1. 確認設計稿寬度，例如 `750px`。
2. 決定等分數，例如 `15` 等份。
3. 算出設計稿基準根字級，例如 `750 / 15 = 50px`。
4. 將設計稿中的元素尺寸用 `px / 根字級` 換算成 rem。
5. 使用媒體查詢在不同設備寬度下設定對應根字級。
6. 檢查常見設備寬度下的顯示效果，必要時補斷點或調整策略。

## 常見錯誤

### 斷點太少卻期待連續適配

如果只寫 `320px` 和 `750px` 兩個斷點，中間寬度不一定會有平滑變化。這種寫法適合示範原理，但正式方案應考慮更多斷點，或用能連續隨視口變化的公式。

### 設計稿基準與工具設定不一致

如果設計稿以 `750px`、`15` 等份為基準，則換算工具或人工換算也要使用相同基準。否則 rem 值會不一致。

### 忘記設定 meta viewport

移動端 rem 適配通常也需要正確的 `meta viewport`。如果佈局視口不等於設備寬度，根字級計算的基準也會偏離預期。

## 重點整理

- `rem + 媒體查詢` 透過不同螢幕寬度設定不同根字級。
- 元素尺寸用 rem 表示，就能隨根字級等比例縮放。
- 核心公式是 `元素 rem 值 = 元素 px 值 / html font-size`。
- 設計稿寬度、等分數、根字級與換算工具要保持一致。
- 示範斷點不等於完整適配方案，正式頁面要檢查目標設備。

3. 為什麼只寫兩個媒體查詢斷點不一定能完整覆蓋所有設備？
4. 如果 rem 換算結果不對，應先檢查哪些基準設定？
