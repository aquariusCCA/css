# CSS 層疊性：當樣式衝突時誰會生效

## 學習目標

讀完這篇筆記後，你應該能夠：

- 說明 CSS 層疊性用來解決什麼問題。
- 判斷同一個元素、同一個屬性被多次設定時，哪個宣告會生效。
- 分辨「後宣告覆蓋前宣告」成立的前提。
- 理解層疊性和選擇器優先級之間的關係。

## 什麼是層疊性

CSS 的全名是 Cascading Style Sheets，其中 Cascading 就是「層疊」的意思。

層疊性表示樣式可以一層一層疊加，當多條 CSS 規則同時影響同一個元素時，瀏覽器會依照一套規則決定最後採用哪個樣式值。

換句話說，層疊性主要用來處理「樣式衝突」。

## 什麼情況會發生樣式衝突

樣式衝突通常需要同時符合三個條件：

1. 多條 CSS 規則選中了同一個元素。
2. 這些規則設定了同一個 CSS 屬性。
3. 不同規則給出的屬性值不同。

例如：

```css
div {
    color: red;
    font-size: 12px;
}

div {
    color: pink;
}
```

```html
<div>长江后浪推前浪,前浪死在沙滩上</div>
```

這兩條規則都選中同一個 `div`，而且都設定了 `color`。第一條設定 `color: red;`，第二條設定 `color: pink;`，這時就產生了樣式衝突。

最後文字會顯示為粉紅色，因為在這個例子中，兩條規則的選擇器相同，重要性相同，來源也相同，所以後面宣告的 `color: pink;` 會覆蓋前面的 `color: red;`。

## 後宣告覆蓋前宣告的前提

初學時常會記成「CSS 後寫的會覆蓋前寫的」。這句話大方向沒錯，但有重要前提。

後宣告覆蓋前宣告，通常是在以下條件相同時才成立：

- 作用到同一個元素。
- 設定同一個屬性。
- 宣告的重要性相同，例如都沒有使用 `!important`。
- 選擇器優先級相同。
- 樣式來源層級相同。

如果選擇器優先級不同，就不能只看誰寫在後面，而要先比較選擇器權重。

例如：

```css
.text {
  color: red;
}

div {
  color: pink;
}
```

```html
<div class="text">文字</div>
```

雖然 `div` 規則寫在後面，但 `.text` 類選擇器的優先級高於 `div` 標籤選擇器，所以最後仍會顯示紅色。

## 層疊性和優先級的關係

層疊性不是只看程式碼順序。當瀏覽器遇到多條可能套用的宣告時，會逐步比較：

1. 宣告是否套用到目標元素。
2. 宣告的重要性。
3. 選擇器優先級。
4. 宣告出現的先後順序。

這也是為什麼「後寫覆蓋前寫」只是一個簡化說法。真正判斷時，順序通常是在重要性與選擇器優先級都無法分出勝負之後，才成為關鍵。

## 常見誤解

- **誤解：CSS 永遠是後面的規則生效。**  
  只有在重要性、來源與選擇器優先級等條件相同時，後宣告才會覆蓋前宣告。

- **誤解：只要兩條規則選到同一個元素，就一定衝突。**  
  如果兩條規則設定的是不同屬性，例如一條設定 `color`，另一條設定 `font-size`，它們可以同時生效，不算同一個屬性的衝突。

- **誤解：層疊性只和程式碼位置有關。**  
  程式碼位置只是層疊判斷的一部分。遇到不同選擇器時，還要比較選擇器優先級。

## 重點整理

- 層疊性用來解決多條 CSS 規則同時影響同一元素時的樣式衝突。
- 衝突通常發生在同一元素的同一屬性被設定成不同值時。
- 在選擇器優先級等條件相同時，後宣告的規則會覆蓋前面的規則。
- 如果選擇器優先級不同，要先比較權重，再看宣告順序。
- 學會層疊性後，才比較容易理解 CSS 為什麼最後顯示某個樣式。
