# CSS 優先級與權重：判斷哪個選擇器勝出

## 學習目標

讀完這篇筆記後，你應該能夠：

- 說明什麼情況下需要比較 CSS 優先級。
- 使用四欄 tuple 理解選擇器權重。
- 判斷不同選擇器之間的優先級高低。
- 理解權重疊加不會進位。
- 分辨 `!important`、繼承與選擇器權重之間的關係。
- 解析常見優先級練習題。

## 什麼時候需要比較優先級

當同一個元素被多個選擇器選中，而且這些規則設定了同一個屬性時，就需要比較優先級。

例如：

```css
div {
  color: pink;
}

.text {
  color: red;
}
```

```html
<div class="text">你笑起来真好看</div>
```

這個 `div` 同時被 `div` 選擇器與 `.text` 選擇器選中，而且兩者都設定 `color`。最後文字會顯示紅色，因為類選擇器 `.text` 的優先級高於標籤選擇器 `div`。

## 用四欄 tuple 理解權重

在 CSS 中，選擇器優先級通常稱為 specificity，中文可稱為權重或特異性。

入門教材有時會用 `1000`、`100`、`10`、`1` 這種方式輔助記憶，但正式理解時，最好把權重看成四欄 tuple，而不是普通十進位數字。

![CSS 選擇器權重對照表，列出繼承、元素、類、ID、行內樣式與 important](../../origin/040-CSS三大特性/assets/images/css-specificity-img-001-a45272.png)

四欄可以理解為：

| 欄位 | 來源 | 範例 |
| --- | --- | --- |
| 第一欄 | 行內樣式 | `style="color: red;"` |
| 第二欄 | ID 選擇器 | `#demo` |
| 第三欄 | 類、屬性、偽類選擇器 | `.text`、`[type="text"]`、`:hover` |
| 第四欄 | 元素、偽元素選擇器 | `div`、`p`、`::before` |

例如：

- `#id` 的權重是 `(0, 1, 0, 0)`。
- `.class1 .class2 .class3` 的權重是 `(0, 0, 3, 0)`。
- `div` 的權重是 `(0, 0, 0, 1)`。
- 行內樣式可記為 `(1, 0, 0, 0)`。

比較時要由左到右看。前一欄能分出勝負時，就不需要再看後面欄位。

## 常見選擇器優先級順序

一般選擇器優先級可以由低到高粗略理解為：

```text
繼承值 ← 通配符選擇器 ← 標籤選擇器 ← 類選擇器 / 屬性選擇器 / 偽類 ← ID 選擇器 ← 行內樣式
```

也就是說：

- 類選擇器通常高於標籤選擇器。
- ID 選擇器高於類選擇器。
- 行內樣式高於一般選擇器。

例如：

```css
.test {
  color: red;
}

div {
  color: pink;
}

#demo {
  color: green;
}
```

```html
<div class="test" id="demo">你笑起来真好看</div>
```

這個元素同時被 `.test`、`div`、`#demo` 選中。最後會顯示綠色，因為 `#demo` 的 ID 選擇器權重最高。

## 權重不會進位

權重疊加時，不同欄位之間不會進位。

例如：

- `.class1 .class2 .class3` 是 `(0, 0, 3, 0)`。
- `#id` 是 `(0, 1, 0, 0)`。

即使類選擇器數量增加，也不會因為第三欄累積很多，就進位成第二欄的 ID 選擇器。

所以 `(0, 0, 10, 0)` 仍然小於 `(0, 1, 0, 0)`。

這一點很重要，因為把 specificity 當成普通十進位數字會讓人誤以為「十個 class 可以抵過一個 id」。實際上不會。

## 複合選擇器的權重疊加

如果使用複合選擇器，權重會在各自欄位內疊加。

例如：

```css
/* ul li 權重是 (0,0,0,2) */
ul li {
  color: green;
}

/* li 權重是 (0,0,0,1) */
li {
  color: red;
}

/* .nav li 權重是 (0,0,1,1) */
.nav li {
  color: pink;
}
```

```html
<ul class="nav">
  <li>大猪蹄子</li>
  <li>大肘子</li>
  <li>猪尾巴</li>
</ul>
```

比較結果：

- `li` 是 `(0,0,0,1)`。
- `ul li` 是 `(0,0,0,2)`。
- `.nav li` 是 `(0,0,1,1)`。

最後 `.nav li` 勝出，因為第三欄的類選擇器權重高於只在第四欄累積的元素選擇器。

## important 不是選擇器權重的一欄

`!important` 不是選擇器權重的一級，而是提高某條宣告的重要性。

例如：

```css
#father {
  color: red !important;
}

p {
  color: pink;
}
```

```html
<div id="father">
  <p>你还是很好看</p>
</div>
```

這裡 `#father` 選中的是父元素，`p` 選中的是子元素。即使父元素的 `color` 使用了 `!important`，子元素本身仍被 `p { color: pink; }` 直接選中，所以 `p` 的文字會使用粉紅色。

原因是繼承得到的值權重為 0。父元素權重再高，傳到子元素時仍只是繼承值；而直接選中子元素的宣告會優先於繼承值。

當多條直接作用在同一元素、同一屬性的宣告都使用 `!important` 時，才會再比較來源、選擇器優先級與宣告順序。

## 練習題解析

### 示例 1

```css
/* (0, 0, 0, 2) */
div div {
  color: skyblue;
}

/* (0, 0, 0, 1) */
div {
  color: red;
}
```

```html
<div>
  <div>
    <div>白日依山尽，黄河入海流。</div>
  </div>
</div>
```

最內層 `div` 同時符合 `div div` 和 `div`。`div div` 的權重是 `(0,0,0,2)`，高於 `div` 的 `(0,0,0,1)`，所以文字會顯示為天藍色。

### 示例 2

```css
/* (0, 0, 0, 7) */
div div div div div div div {
  color: red;
}

/* (0, 0, 1, 0) */
.one {
  color: green;
}
```

```html
<div>
  <div>
    <div>
      <div>
        <div>
          <div>
            <div class="one">白日依山尽，黄河入海流。</div>
          </div>
        </div>
      </div>
    </div>
  </div>
</div>
```

`div div div div div div div` 有 7 個元素選擇器，權重是 `(0,0,0,7)`。`.one` 是類選擇器，權重是 `(0,0,1,0)`。

比較時第三欄先勝出，所以 `.one` 較高，文字會顯示為綠色。這也再次說明元素選擇器累積再多，也不會進位成類選擇器。

### 示例 3

```css
/* (0, 0, 2, 1) */
.c1 .c2 div {
  color: red;
}

/* (0, 1, 0, 1) */
div #box3 {
  color: green;
}

/* (0, 1, 0, 1) */
#box1 div {
  color: blue;
}
```

```html
<div id="box1" class="c1">
  <div id="box2" class="c2">
    <div id="box3" class="c3">白日依山尽，黄河入海流。</div>
  </div>
</div>
```

最內層 `div` 同時符合三條規則：

- `.c1 .c2 div` 是 `(0,0,2,1)`。
- `div #box3` 是 `(0,1,0,1)`。
- `#box1 div` 是 `(0,1,0,1)`。

先比較第二欄，後兩條有 ID 權重，都高於第一條。接著比較 `div #box3` 和 `#box1 div`，兩者權重相同。權重相同時回到層疊性，後宣告的 `#box1 div` 生效，所以文字會顯示為藍色。

## 判斷 CSS 勝負的實用流程

遇到多條 CSS 規則衝突時，可以照這個順序判斷：

1. 確認哪些宣告真的作用到目標元素。
2. 確認是否是同一個屬性。
3. 先看是否有直接選中元素；繼承值的權重是 0。
4. 比較是否有 `!important`。
5. 比較選擇器權重，由左到右看四欄 tuple。
6. 如果權重相同，再看後宣告的規則是否覆蓋前面的規則。

## 常見誤解

- **誤解：權重可以用普通數字相加比較。**  
  權重應視為四欄 tuple，不是普通十進位數字。不同欄位之間不會進位。

- **誤解：很多標籤選擇器可以贏過一個類選擇器。**  
  不會。`(0,0,0,7)` 仍然小於 `(0,0,1,0)`。

- **誤解：父元素的 `!important` 一定能壓過子元素樣式。**  
  如果子元素被直接選中並設定同一屬性，直接作用在子元素上的宣告會優先於繼承值。

- **誤解：`!important` 是選擇器權重最高的一欄。**  
  `!important` 改變的是宣告的重要性，不是選擇器 specificity 的欄位。

## 重點整理

- 優先級用來判斷同一元素、同一屬性被多條規則設定時誰生效。
- specificity 建議用四欄 tuple 理解：行內、ID、類/屬性/偽類、元素/偽元素。
- 權重比較要由左到右，前面欄位能分勝負時，就不看後面。
- 權重疊加不會進位，`(0,0,10,0)` 不會變成 `(0,1,0,0)`。
- 繼承值的權重是 0，直接選中元素的宣告通常會高於繼承值。
- 權重相同時，才回到層疊性，由後宣告覆蓋前宣告。
