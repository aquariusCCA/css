# 字體圖標的用途與特性

## 學習目標

讀完這篇筆記後，你應該能夠：

- 說明字體圖標適合解決什麼介面需求。
- 理解字體圖標「看起來是圖標，本質是字體」的特性。
- 判斷什麼情境適合使用字體圖標，什麼情境不適合。
- 知道字體圖標在實作時需要注意字型檔載入、路徑與相容性。

## 問題情境

網頁中常會出現許多小圖標，例如搜尋、購物車、箭頭、關閉、播放、下載等。這些圖標通常尺寸小、樣式簡單，而且會在很多地方重複出現。

如果每個小圖標都各自用一張圖片管理，專案可能會遇到幾個問題：

- 檔案數量變多，不容易管理。
- 想統一調整顏色或大小時，需要重新處理圖片或替換資源。
- 同一類圖標在不同位置重複使用時，維護成本較高。

字體圖標就是為這類「簡單、常用、可重複使用的小圖標」提供的一種做法。

## 一句話理解

字體圖標是用字型方式顯示的圖標：畫面上看到的是圖標，但 CSS 可以像控制文字一樣控制它。

## 字體圖標是什麼

字體圖標主要用於顯示網頁中通用、常用的小圖標。它顯示出來像圖形，但本質上屬於字體。

這代表它通常不是用一張獨立圖片放到頁面上，而是透過字型檔中的某個 glyph 顯示出對應圖標。因為它本質接近文字，所以可以搭配文字相關或常見視覺 CSS 來調整外觀。

例如，你可以像調整文字一樣控制：

- 顏色。
- 大小。
- 透明度。
- 陰影。
- 旋轉或其他轉換效果。

這也是字體圖標常被用在前端介面中的原因：它能讓很多簡單圖標用一致的方式管理。

## 適合使用的情境

字體圖標適合處理結構和樣式都比較簡單的小圖標，例如：

- 搜尋圖標。
- 購物車圖標。
- 關閉按鈕的叉號。
- 左右箭頭。
- 社群或功能入口的小符號。

這些圖標通常有幾個共同點：

- 尺寸不大。
- 外觀簡單。
- 多半是單色或可以接受單色呈現。
- 需要在多個地方重複使用。
- 常需要依版面調整顏色、大小或狀態。

如果目標是做出網頁中簡潔、通用的小圖標效果，字體圖標是很常見的選項。

## 不適合使用的情境

字體圖標不是所有視覺資產的替代品。下面這些情境通常不適合用字體圖標：

- 照片。
- 複雜插圖。
- 多色圖形。
- 需要細緻紋理或漸層細節的圖像。
- 需要保留原始圖片內容的素材。

原因很簡單：字體圖標最擅長的是簡單符號，不是完整圖片。若圖形本身很複雜，使用圖片、SVG 或其他更合適的格式會更合理。

## 優點

### 輕量級與好管理

在只載入需要的 glyph，並用單一字型檔管理多個小圖標時，字體圖標可能比多張獨立圖片更容易減少請求與管理成本。

這不代表字體圖標永遠比圖片小，而是說：當專案有一組常用小圖標時，用同一套字型圖標管理，通常比散落多張小圖片更容易維護。

### 樣式調整彈性高

因為字體圖標本質上是字體，所以可以用 CSS 調整許多視覺效果。

例如：

```css
.icon {
  color: #333;
  font-size: 24px;
  opacity: 0.8;
}
```

也可以搭配陰影或轉換效果：

```css
.icon-cart {
  color: #1677ff;
  text-shadow: 0 2px 6px rgba(0, 0, 0, 0.2);
  transform: rotate(-8deg);
}
```

這些調整方式都很接近一般文字或元素樣式的寫法。

### 相容性通常較好

字體圖標在許多瀏覽器與專案中都能使用。不過實作時仍要注意：

- 字型格式是否被目標瀏覽器支援。
- 字型檔路徑是否正確。
- 字型檔是否成功載入。
- 如果字型檔來自不同來源，是否有跨域設定問題。

因此，「相容性好」不代表完全不用檢查載入環境。

## 範例情境拆解

假設頁面上有一個購物車按鈕，需要在導覽列、商品卡片與結帳區塊重複出現。

如果使用圖片，每個位置可能都要引用圖檔，並另外處理不同大小與顏色版本。

如果使用字體圖標，則可以把購物車當成一個可被 CSS 控制的圖標：

```html
<span class="icon icon-cart" aria-hidden="true"></span>
<span>購物車</span>
```

```css
.icon-cart {
  font-size: 20px;
  color: #222;
}
```

當你想讓導覽列中的購物車變大或變色時，只要改 CSS 即可，不需要重新輸出另一張圖片。

這個範例的重點不是特定 class 名稱，而是理解字體圖標的使用思路：圖標由字型提供，外觀由 CSS 控制。

實際專案中，`.icon-cart` 還需要和字型檔中的某個 glyph 建立對應。這個對應可能由 iconfont 工具產生的 class、偽元素 `content`、或元素中的特定字元完成；如果沒有這層映射，空的 `<span>` 不會自己顯示出購物車圖標。

## 常見錯誤

### 把字體圖標當成所有圖片的替代品

字體圖標適合簡單、單色、通用的小圖標，不適合照片或複雜插圖。

如果圖像需要保留多色細節、人物、商品照片或複雜視覺資訊，應使用圖片、SVG 或其他合適格式，而不是硬轉成字體圖標。

### 忽略字型檔是否成功載入

字體圖標依賴字型檔。如果字型檔路徑錯誤、格式不支援，或跨域設定有問題，頁面可能無法顯示正確圖標。

遇到圖標顯示異常時，應先檢查：

- CSS 是否正確引用字型檔。
- 字型檔路徑是否正確。
- Network 面板是否有字型檔載入失敗。
- 是否有跨域或 MIME type 問題。

### 只注意圖標外觀，忘記語意

字體圖標看起來像視覺裝飾，但它可能代表重要操作，例如搜尋、刪除、關閉。若圖標本身承載功能意義，HTML 結構仍需要有清楚文字、標籤或無障礙名稱。

例如只有圖標的按鈕，應確保使用者能理解它的功能；如果旁邊已有文字，圖標本身可以視為裝飾。

## 實務判斷準則

- 簡單、常用、單色的小圖標，可以考慮字體圖標。
- 照片、複雜插圖、多色圖形，不適合用字體圖標。
- 需要頻繁調整顏色、大小或狀態時，字體圖標的 CSS 控制很方便。
- 實作時要確認字型檔格式、載入路徑與跨域設定。
- 圖標若代表功能，不要只顧視覺，也要保留清楚語意。

## 重點整理

- 字體圖標看起來是圖標，本質上是字體。
- 它適合用來顯示簡單、通用、常用的小圖標。
- 字體圖標可以用 CSS 調整顏色、大小、透明度、陰影與轉換效果。
- 它不適合照片、複雜插圖或多色圖形。
- 實作時要注意字型檔格式、載入路徑、跨域與語意問題。
