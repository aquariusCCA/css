# 為什麼要重置瀏覽器默認樣式

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明瀏覽器默認樣式為什麼會影響版面設計。
- 舉出常見 HTML 元素自帶的默認樣式。
- 理解「重置默認樣式」的目的不是讓頁面好看，而是讓版面更可控。
- 判斷為什麼全局選擇器重置只能作為簡單入門方案。

## 問題情境

當你剛寫好一個 HTML 頁面，明明還沒有寫 CSS，畫面卻不是完全貼齊瀏覽器邊緣：

- `body` 外面好像有一圈空白。
- `<p>` 段落上下有空隙。
- `<h1>` 到 `<h6>` 標題不只字變大，還會加粗並帶有上下間距。
- 清單項目前面會自動出現圓點或編號。

這些效果不是你寫的 CSS，而是瀏覽器替 HTML 元素準備的默認樣式。

默認樣式能讓純 HTML 文件在沒有 CSS 時仍然可讀，但當你要依照設計稿精準控制版面時，它就可能變成干擾。

## 一句話理解

重置默認樣式，是先把瀏覽器自帶、可能干擾版面的樣式清理或統一，讓後續 CSS 從更可控的起點開始。

它不是為了完成視覺設計，而是為了建立設計之前的基準。

## 常見的默認樣式

不同瀏覽器的預設呈現可能不完全相同，但常見元素通常都會帶一些默認樣式。

| 元素 | 常見默認樣式 | 可能造成的問題 |
| --- | --- | --- |
| `body` | 預設 `8px` 外邊距 | 頁面內容無法貼齊視窗邊緣 |
| `p` | 上下 `margin` | 段落間距和設計稿不一致 |
| `h1` 到 `h6` | 上下 `margin`、字體加粗、字號變化 | 標題距離和大小不受控 |
| `ul`、`ol` | 內距、列表符號 | 導覽列或卡片清單多出縮排與符號 |
| `a` | 預設連結顏色與底線 | 連結樣式和整體設計不一致 |

這些默認樣式本身不是錯，它們只是「瀏覽器為一般文件閱讀準備的預設值」。當目標是精準布局時，就需要重新評估它們是否保留。

## 為什麼現代網頁常需要重置

早期網頁比較偏向文件閱讀，瀏覽器默認樣式可以快速讓內容有基本層次。

但現代頁面常有更細緻的需求：

- 導覽列要和頁面邊緣對齊。
- 卡片、表單、按鈕要有一致間距。
- 多個瀏覽器中的呈現要盡量接近。
- 頁面元素要能依照設計稿精準排列。

如果不先處理默認樣式，開發者常會遇到「明明沒寫 margin，為什麼有空白」這類問題。重置樣式能讓這些隱性影響變少，後續寫 CSS 時更容易判斷樣式從哪裡來。

## 入門方案：使用全局選擇器

最直觀的重置方式，是用全局選擇器 `*` 選中所有元素，清除常見的內外距。

```css
* {
  margin: 0;
  padding: 0;
}
```

這段 CSS 的意思是：

- 對所有元素設定 `margin: 0`。
- 對所有元素設定 `padding: 0`。

對初學者來說，這個範例很好理解：先把所有元素自帶的內外距歸零，畫面就比較容易從同一個起點開始。

## 全局選擇器的限制

全局選擇器雖然簡單，但實際開發中通常不會只依賴它。

原因是 `*` 會選中所有元素，而不是「只選中真正需要重置的元素」。

例如：

```css
* {
  margin: 0;
  padding: 0;
  color: blue;
}
```

這樣會讓所有元素文字都變成藍色，但實務上你可能希望：

- 一般文字是深灰色。
- 連結 `a` 是另一種顏色。
- 按鈕文字依元件狀態變化。

如果全部都用 `*` 統一處理，就無法細緻區分不同元素的需求。

更穩定的做法，是針對會造成問題的元素分別重置，或使用整理好的 `reset.css`、`normalize.css` 作為基礎。

## 常見錯誤

### 誤解：所有默認樣式都一定不好

默認樣式不是錯誤，只是瀏覽器提供的起始值。有些預設其實對可讀性有幫助，例如標題加粗、段落間距、表單可操作狀態。是否重置，要看它是否干擾你的版面或專案規範。

### 誤解：只要寫 `* { margin: 0; padding: 0; }` 就完成重置

這只處理了最常見的內外距，沒有處理列表符號、圖片底部縫隙、連結樣式、表單字型、表格邊框等問題。它可以當作最小示範，但不是完整的 reset 策略。

### 誤解：重置樣式就是設計樣式

重置樣式只是把起點整理乾淨。真正的設計樣式，例如按鈕顏色、卡片間距、標題字級，應該在 reset 之後另外撰寫，避免把「初始化」和「視覺設計」混在一起。

## 實務判斷

可以用這幾個問題判斷是否需要重置某個默認樣式：

- 這個默認樣式是否讓版面和設計稿不一致？
- 不同瀏覽器對它的呈現是否有明顯差異？
- 這個預設是否會讓後續 CSS 判斷變得困難？
- 重置後是否會破壞可讀性或可操作性？

如果某個默認樣式有用，就不一定要清掉；如果它妨礙布局或一致性，就應該納入 reset 或 normalize 策略。

## 重點整理

- 瀏覽器會替 HTML 元素提供默認樣式。
- 默認樣式能讓純 HTML 文件可讀，但可能干擾精準布局。
- 常見默認樣式包含 `body` 外距、段落上下間距、標題加粗與列表符號。
- 重置默認樣式的目的是讓後續 CSS 從更可控的起點開始。
- `* { margin: 0; padding: 0; }` 是簡單入門方案，但不適合當成完整 reset 策略。
