# CSS 顏色表示方式概覽

## 學習目標

讀完這篇筆記後，你應該能夠：

- 說明 CSS 為什麼提供多種顏色表示方式。
- 分辨具名顏色、數值色值與透明度色值的用途。
- 認識 HEX、RGB、HSL 三種常見 sRGB 顏色表示方式。
- 理解具名顏色適合用在學習與快速測試，但不一定適合完整設計系統。

## 為什麼 CSS 有多種顏色寫法

CSS 可以控制文字、背景、邊框、陰影等各種視覺顏色。不同場景需要不同的描述方式，所以 CSS 支援多種顏色表示法。

例如：

- 想快速測試樣式時，可以使用 `red`、`blue` 這類具名顏色。
- 想精準還原設計稿時，常用 HEX，例如 `#ff0b00`。
- 想用程式或工具動態調整紅、綠、藍通道時，可以使用 RGB。
- 想用比較直覺的方式調整色相、飽和度與亮度時，可以使用 HSL。
- 想做半透明背景或遮罩時，需要加上 alpha 透明度。

這些寫法不是互相取代，而是各自適合不同情境。

## CSS 顏色體系

CSS 顏色可以粗略分成幾類：

![CSS 顏色體系心智圖，分為顏色名、顏色值與關鍵字](../../origin/090-顏色/assets/images/colors-img-001-0ca638.png)

常見類型包含：

- 具名顏色，例如 `red`、`blue`、`green`。
- HEX 色值，例如 `#ff0000`。
- RGB / RGBA 色值，例如 `rgb(255, 0, 0)`、`rgba(255, 0, 0, 0.5)`。
- HSL / HSLA 色值，例如 `hsl(0, 100%, 50%)`。
- 透明關鍵字，例如 `transparent`。
- 其他現代色彩函式與色彩空間。

本章先聚焦在入門階段最常見、也最容易遇到的寫法：具名顏色、HEX、RGB / RGBA、HSL / HSLA。

## 具名顏色

CSS 支援一組標準化的具名顏色，例如：

```css
.title {
  color: red;
}

.notice {
  background-color: rebeccapurple;
}
```

這些顏色名稱不區分大小寫。也就是說，`red`、`RED`、`Red` 在 CSS 中都可以被視為同一個具名顏色。

![CSS 顏色名稱範例，顯示 beige、coral、cyan 等色塊](../../origin/090-顏色/assets/images/colors-img-002-f192cf.png)

具名顏色的優點是容易閱讀，也很適合初學者快速測試。但它的限制也很明顯：名稱數量固定，無法精準描述設計稿中的任意顏色。

## 常見數值顏色表示方式

實務上更常使用數值色值，因為它們能精準描述顏色。

本章會整理三種常見 sRGB 數值表示方式：

| 類型 | 範例 | 適合情境 |
| --- | --- | --- |
| HEX | `#ff0b00` | 設計稿、色票、常見靜態顏色 |
| RGB / RGBA | `rgb(141, 0, 0)`、`rgba(0, 0, 0, 0.5)` | 依紅綠藍通道理解或調整顏色，需要透明度時也常見 |
| HSL / HSLA | `hsl(240, 86%, 50%)`、`hsla(198, 100%, 50%, 0.7)` | 想依色調、飽和度、亮度調整顏色時較直覺 |

三者本質上都可以描述顏色，只是思考角度不同。

## 透明度與 alpha

CSS 顏色除了描述「是什麼顏色」，也可以描述「有多透明」。

透明度通常稱為 alpha。alpha 值越低越透明，越高越不透明：

- `0` 或 `0%` 表示完全透明。
- `1` 或 `100%` 表示完全不透明。
- `0.5` 或 `50%` 表示半透明。

RGBA 和 HSLA 都可以加入 alpha。現代 CSS 也支援在 `rgb()` 和 `hsl()` 中使用斜線寫透明度，例如：

```css
.overlay {
  background-color: rgb(0 0 0 / 50%);
}
```

## 選用建議

初學時可以用這幾個方向判斷：

- 快速測試：用具名顏色。
- 設計稿提供色碼：多半直接使用 HEX。
- 需要明確看出紅、綠、藍通道：使用 RGB。
- 需要半透明：使用 RGBA、HSLA，或現代 `rgb(... / ...)`、`hsl(... / ...)` 語法。
- 想調整顏色明暗或飽和度：HSL 通常比 RGB 更直覺。

實務專案中，最重要的是一致性。若團隊已經使用 HEX 作為色票格式，就優先沿用；若設計系統用 CSS 變數管理顏色，也應該優先使用既有變數。

## 常見誤解

- **誤解：具名顏色比較不正式，不能用。**  
  具名顏色是 CSS 標準支援的合法寫法，只是它們不夠精準，較少用來建立完整設計系統。

- **誤解：HEX、RGB、HSL 是完全不同的顏色系統，不能表示同一個顏色。**  
  它們常用來表示同一個 sRGB 顏色，只是書寫方式與思考角度不同。

- **誤解：只有 `rgba()` 和 `hsla()` 才能寫透明度。**  
  現代 CSS 也可以用 `rgb(0 0 0 / 50%)` 或 `hsl(0 0% 0% / 50%)` 這類語法指定 alpha。

## 重點整理

- CSS 支援多種顏色表示方式，因為不同場景需要不同的描述方式。
- 具名顏色容易閱讀，適合學習與快速測試。
- HEX、RGB、HSL 是常見的數值顏色表示方式。
- RGBA、HSLA 或現代斜線語法可以加入 alpha 透明度。
- 選擇哪種寫法時，應考慮可讀性、設計稿格式、是否需要透明度，以及團隊慣例。
