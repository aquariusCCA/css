# HEX、RGB 與 RGBA 顏色值

## 學習目標

讀完這篇筆記後，你應該能夠：

- 說明 HEX 色值 `#RRGGBB` 的組成方式。
- 判斷哪些 HEX 色值可以簡寫成 `#RGB`。
- 說明 RGB 三個通道的意義與取值範圍。
- 使用 RGBA 或現代 RGB 語法設定半透明顏色。
- 分辨 HEX 和 RGB 在使用情境上的差異。

## HEX 顏色值

HEX 是 CSS 中很常見的顏色表示方式。它使用 `#` 加上十六進制數字來描述顏色。

最常見的格式是：

```css
.box {
  color: #ff0b00;
}
```

`#RRGGBB` 可以拆成三組：

| 區段 | 代表色彩 | 範圍 |
| --- | --- | --- |
| `RR` | red，紅色通道 | `00` 到 `ff` |
| `GG` | green，綠色通道 | `00` 到 `ff` |
| `BB` | blue，藍色通道 | `00` 到 `ff` |

其中 `00` 表示該通道強度最低，`ff` 表示該通道強度最高。

![HEX 色值範例，紅色背景顯示 #ff0b00](../../origin/090-顏色/assets/images/colors-img-003-8da288.png)

例如：

```css
.red {
  color: #ff0000;
}

.green {
  color: #00ff00;
}

.blue {
  color: #0000ff;
}
```

## HEX 簡寫

當 `#RRGGBB` 中每一組的兩個字元都相同時，可以簡寫成 `#RGB`。

例如：

```css
/* 完整寫法 */
.box {
  color: #aabbcc;
}

/* 簡寫 */
.box {
  color: #abc;
}
```

`#abc` 會被展開成 `#aabbcc`。

不是所有 HEX 色值都能簡寫。只有 `RR`、`GG`、`BB` 三組都能各自重複時才可以。

| 完整寫法 | 是否可簡寫 | 簡寫結果 |
| --- | --- | --- |
| `#aabbcc` | 可以 | `#abc` |
| `#ff0000` | 可以 | `#f00` |
| `#ff0b00` | 不可以 | 無 |

## HEX 透明度

HEX 也可以表示 alpha 透明度，常見格式包含：

- `#RGBA`
- `#RRGGBBAA`

例如：

```css
.overlay {
  background-color: #00000080;
}
```

這裡最後的 `80` 是 alpha 通道。HEX alpha 對初學者來說通常不如 `rgba()` 或 `rgb(... / ...)` 直覺，因此入門階段遇到透明度時，可以先優先使用 RGB 或 HSL 的透明度語法。

## RGB 顏色值

RGB 也是用紅、綠、藍三個通道描述顏色，但它使用十進位數字。

基本格式如下：

```css
.box {
  color: rgb(141, 0, 0);
}
```

三個數值分別代表：

| 位置 | 代表色彩 | 範圍 |
| --- | --- | --- |
| 第一個值 | 紅色 red | `0` 到 `255` |
| 第二個值 | 綠色 green | `0` 到 `255` |
| 第三個值 | 藍色 blue | `0` 到 `255` |

![RGB 色值範例，深紅背景顯示 rgb(141, 0, 0)](../../origin/090-顏色/assets/images/colors-img-004-09f9b5.png)

例如：

```css
.red {
  color: rgb(255, 0, 0);
}

.black {
  color: rgb(0, 0, 0);
}

.white {
  color: rgb(255, 255, 255);
}
```

RGB 的好處是每個通道的強度很直觀：數字越大，該色光越強。

## RGBA 與 alpha

RGBA 是在 RGB 的基礎上加入 alpha 透明度。

舊式語法常寫成：

```css
.overlay {
  background-color: rgba(0, 0, 0, 0.5);
}
```

其中第四個值是 alpha：

- `0` 表示完全透明。
- `1` 表示完全不透明。
- `0.5` 表示半透明。

![RGBA 透明色值範例，顯示 rgba(60, 60, 200, 0.2)](../../origin/090-顏色/assets/images/colors-img-005-8c78cc.png)

現代 CSS 也可以使用斜線語法：

```css
.overlay {
  background-color: rgb(0 0 0 / 50%);
}
```

這種寫法把顏色通道與 alpha 分開，讀起來也很清楚。

## HEX 和 RGB 怎麼選

HEX 和 RGB 都很常見，可以依情境選擇：

| 情境 | 建議 |
| --- | --- |
| 設計稿或色票提供 HEX | 直接使用 HEX |
| 需要快速貼上色碼 | HEX 通常較短 |
| 需要手動理解紅綠藍通道 | RGB 較直覺 |
| 需要透明度 | RGBA 或現代 `rgb(... / ...)` 較直覺 |
| 團隊已有固定格式 | 優先遵守團隊慣例 |

同一個專案中，最好維持一致的色彩格式。格式混雜太多，日後維護時會比較難掃讀。

## 常見誤解

- **誤解：`#abc` 和 `#aabbcc` 是不同顏色。**  
  `#abc` 是 `#aabbcc` 的簡寫，兩者表示同一個顏色。

- **誤解：所有六位 HEX 都可以改成三位。**  
  只有 `#RRGGBB` 中每組兩個字元都相同時才能簡寫。

- **誤解：RGB 的數值可以任意填。**  
  一般 RGB 通道範圍是 `0` 到 `255`。超出範圍不是正常的入門寫法。

- **誤解：透明度只能用 `rgba()`。**  
  現代 CSS 可以用 `rgb(0 0 0 / 50%)` 指定透明度，HEX 也有 alpha 寫法。

## 重點整理

- HEX 常見格式是 `#RRGGBB`，分別表示紅、綠、藍通道。
- 每個 HEX 通道範圍是 `00` 到 `ff`。
- 當 `#RRGGBB` 每組字元都重複時，可以簡寫為 `#RGB`。
- RGB 使用 `0` 到 `255` 的十進位數字表示紅、綠、藍通道。
- RGBA 或現代 `rgb(... / ...)` 語法可以加入 alpha 透明度。
- 選擇 HEX 或 RGB 時，除了語法本身，也要考慮設計稿、透明度需求與團隊慣例。
