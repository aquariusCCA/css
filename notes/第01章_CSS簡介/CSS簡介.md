# CSS 簡介

## 學習目標

讀完本章後，你應該能夠說明 CSS 在網頁中的角色，理解為什麼 HTML 與 CSS 通常要分工，並能看懂一條基本 CSS 規則由哪些部分組成。

完成本章後，你應能做到：

- 說明 HTML 主要負責內容結構，CSS 主要負責外觀呈現。
- 解釋「結構與樣式分離」為什麼能讓網頁更容易維護。
- 辨識 CSS 規則中的 selector、property 與 value。
- 寫出一個小型 CSS 範例，控制文字顏色與字體大小。
- 分辨開發時常見的展開風格與上線時常見的緊湊風格。

## 前置知識

學習本章前，建議先具備基本 HTML 概念。你需要知道 HTML 元素通常由標籤組成，例如 `<h1>` 表示一級標題，`<p>` 表示段落。

本章不要求你已經熟悉 CSS selector、cascade、specificity、盒模型或版面配置。這些概念會在後續章節逐步展開。

## 核心概念

HTML 可以想像成網頁的內容骨架。它告訴瀏覽器「這是一個標題」、「這是一段文字」、「這是一張圖片」或「這是一個清單」。這些標籤讓內容具有結構與語意，但如果只靠 HTML 來控制外觀，網頁通常會變得難以維護。

CSS 則負責描述這些 HTML 元素應該如何呈現。它可以設定文字顏色、字體大小、圖片外觀、元素間距、版面排列與其他視覺樣式。簡單來說，HTML 先決定內容是什麼，CSS 再決定內容看起來如何。

可以用兩層模型理解網頁：

| 層次 | 負責內容 | 例子 |
|---|---|---|
| HTML 結構層 | 描述內容與語意 | `<h1>` 是標題，`<p>` 是段落 |
| CSS 樣式層 | 描述外觀與排版 | 標題是紅色、段落有行距、區塊有背景色 |

這種分工的最大價值是「結構與樣式分離」。當內容結構交給 HTML，外觀設定交給 CSS，同一份 HTML 可以更容易套用不同樣式，CSS 也可以集中管理，不必把樣式分散寫在每一個 HTML 標籤中。

## CSS 語法、規則與心智模型

一條 CSS 規則可以理解成一句「把哪些元素改成什麼樣子」的指令。它通常包含 selector 與 declaration block。

```css
選擇器 {
  屬性名: 屬性值;
}
```

selector 用來指定 CSS 要影響哪些 HTML 元素。大括號 `{}` 裡面是一組或多組宣告，每一組宣告由 property 與 value 組成。

```css
h1 {
  color: red;
  font-size: 40px;
}
```

在這個範例中，`h1` 是 selector，表示這條規則會選到頁面中的 `<h1>` 元素。`color` 與 `font-size` 是 property，分別代表文字顏色與字體大小。`red` 與 `40px` 是 value，表示實際要套用的樣式值。

CSS 宣告有幾個基本書寫規則：

- property 與 value 之間使用英文冒號 `:` 分隔。
- 每一組宣告通常以英文分號 `;` 結束。
- 多組宣告可以寫在同一個 selector 的大括號中。
- 開發時通常使用換行與縮排，讓規則更容易閱讀與維護。

補充說明：CSS 的全名是 Cascading Style Sheets，常見中文譯名是「層疊樣式表」或「級聯樣式表」。來源提到 CSS 也是一種標記語言；在常見 Web 標準用語中，CSS 更常被稱為樣式表語言，因為它描述樣式規則，而不是像 HTML 一樣標記內容結構。

## 程式碼風格

CSS 可以寫成展開風格，也可以寫成緊湊風格。兩者對瀏覽器來說可以表達相同規則，但適合的使用情境不同。

開發時建議使用展開風格。每一條宣告獨立成行，縮排清楚，方便閱讀、修改與除錯。

```css
h1 {
  color: red;
  font-size: 40px;
}
```

緊湊風格會移除大部分空白與換行，檔案體積較小，但不適合人類長期閱讀與維護。

```css
h1{color:red;font-size:40px;}
```

來源指出，專案上線時常會透過工具把展開風格轉成緊湊風格，以減小檔案體積並節省網路流量。補充說明：實務上這個過程通常稱為 minification，但本章不展開建置工具的使用方式。

## 範例與拆解

下面是一個最小 HTML 與 CSS 範例，用來展示 CSS 如何讓 HTML 從「只有結構」變成「具有外觀」。

```html
<h1>CSS 簡介</h1>
<p>CSS 負責控制網頁的外觀。</p>
```

```css
h1 {
  color: red;
  font-size: 40px;
}

p {
  color: #333;
}
```

這個範例中，CSS 的第一條規則以 `h1` 作為 selector，所以作用對象是 HTML 中的 `<h1>` 元素。只要頁面中有 `<h1>`，這條規則就會嘗試套用到它身上。

`color: red;` 會讓標題文字變成紅色，`font-size: 40px;` 會讓標題文字變大。第二條規則以 `p` 作為 selector，因此會影響段落 `<p>`，讓段落文字顏色變成較深的灰色。

這裡的視覺結果是：標題仍然是標題，段落仍然是段落，但它們的外觀被 CSS 改變了。CSS 沒有把 `<p>` 變成 `<h1>`，也沒有改變 HTML 的語意；它只是改變元素在畫面上的呈現方式。

如果把 `font-size: 40px;` 改成 `font-size: 20px;`，標題會變小。如果移除整條 `h1` 規則，標題會回到瀏覽器或其他 CSS 規則決定的樣式。

初學者常見的誤解是以為 CSS 會改變 HTML 結構。實際上，CSS 主要影響視覺呈現；HTML 的語意結構仍由 HTML 標籤本身決定。

## 常見誤解與修正方式

### 誤解一：CSS 是用來取代 HTML 的

CSS 不能取代 HTML。HTML 負責描述內容結構，例如標題、段落、圖片與連結；CSS 負責控制這些內容的外觀。若用 CSS 製造看起來像標題的大字，但 HTML 仍然不是標題標籤，內容語意並不會自動變正確。

正確理解方式是先用 HTML 寫出合理結構，再用 CSS 控制外觀。

### 誤解二：只要寫了 CSS，所有元素都會變樣式

CSS 必須先透過 selector 選到目標元素，規則才可能生效。若 selector 寫成 `h1`，它只會選到 `<h1>` 元素，不會自動選到 `<p>` 或其他元素。

修正時要先檢查 HTML 結構，再確認 selector 是否真的對應到想要改變的元素。

### 誤解三：分號不重要

CSS 中每一組宣告通常應以分號結束。雖然最後一組宣告在某些情況下省略分號仍可能被瀏覽器解析，但初學階段建議每一組宣告都保留分號，避免新增下一條宣告時出錯。

正確寫法如下：

```css
h1 {
  color: red;
  font-size: 40px;
}
```

### 誤解四：緊湊風格比較專業，所以開發時應該直接這樣寫

緊湊風格的主要目的不是讓人閱讀，而是減少檔案體積。開發時直接寫成緊湊風格會增加維護成本，也比較不容易看出錯誤。

較合理的做法是開發時使用展開風格，上線前再透過工具產生緊湊版本。

## 實務使用情境

CSS 在實務開發中通常用來處理網頁的外觀與排版。例如調整標題大小、設定文字顏色、控制圖片尺寸、安排區塊間距、建立頁面版面與設計不同狀態下的視覺效果。

本章只先建立 CSS 的基本角色與語法觀念。當你能看懂「selector 選到元素，property 與 value 決定樣式」之後，後續學習文字樣式、顏色背景、盒模型、版面配置與響應式設計時，才會更容易判斷每條規則為什麼生效或沒有生效。

## 自我檢查

1. 在 `h1 { color: red; }` 中，哪一部分是 selector？哪一部分是 property？哪一部分是 value？
2. 如果 HTML 中只有 `<p>Hello</p>`，但 CSS 寫的是 `h1 { color: red; }`，段落文字會變紅嗎？為什麼？
3. 請寫出一段最小 HTML 與 CSS，讓 `<p>` 段落文字變成藍色。
4. 為什麼開發時通常使用展開風格，而不是直接使用緊湊風格？
5. 請說明「HTML 負責結構，CSS 負責樣式」這句話的意思。

## 小結

CSS 是用來描述 HTML 外觀的樣式表語言。HTML 讓內容具有結構與語意，CSS 則讓這些內容在畫面上呈現出指定的顏色、大小、外觀與排版效果。

一條基本 CSS 規則由 selector 與宣告組成。selector 決定作用對象，property 與 value 決定實際樣式。開發時應優先使用易讀的展開風格，讓 CSS 更容易維護；上線時可以透過工具轉成較小的緊湊版本。
