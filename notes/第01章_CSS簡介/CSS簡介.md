# CSS 簡介

## 學習目標

讀完本章後，你應該能夠說明 CSS 在網頁中的角色，並能判斷 HTML 與 CSS 各自負責什麼。你也應該能讀懂一條最基本的 CSS 規則，指出其中的 selector、property、value 與 declaration block，並能解釋為什麼 CSS 要用 `:`、`;` 與 `{}` 組成規則。

本章也會建立一個入門判斷：當畫面沒有照預期改變時，先檢查 CSS 是否選到正確元素、語法是否完整，以及樣式是否寫在適合維護的格式中。

## 前置知識

學習本章前，建議先具備基本 HTML 概念：知道 `<h1>` 代表主要標題，`<p>` 代表段落，並理解 HTML 標籤主要用來描述內容結構。

你不需要先學會完整的 CSS selector、cascade、inheritance、box model 或 layout。這些主題會在後續章節深入處理，本章只先建立 CSS 的入口觀念。

## 本章範圍

本章深入處理三件事：CSS 為什麼存在、CSS 如何用規則描述畫面樣式，以及 CSS 程式碼為什麼常用展開風格書寫。

HTML 與 CSS 的分工會作為背景說明：HTML 負責內容與語意結構，CSS 負責外觀與版面表現。來源中提到文字、圖片外型與版面布局，本章只用它們說明 CSS 的用途，不會展開 box model、flexbox、grid 或 responsive design。

selector 的種類、規則競爭、繼承、優先權與 CSS 壓縮工具都適合另開章節。本章只先讓你看懂最基本的 CSS 規則長什麼樣子，以及它如何造成畫面變化。

## 先看問題：HTML 有結構，但畫面不一定符合需求

想像你已經寫好一段 HTML：

```html
<h1>CSS 簡介</h1>
<p>CSS 可以控制文字、圖片外型與網頁版面。</p>
```

這段 HTML 的結構是清楚的：`<h1>` 是標題，`<p>` 是段落。問題是，HTML 本身主要描述「這是什麼內容」，不適合承擔大量外觀設定。若把顏色、字級、間距與版面都塞進 HTML，內容結構會變得臃腫，後續維護也會困難。

所以本章先建立一個核心畫面模型：HTML 像是網頁的內容骨架，CSS 則是在不改變內容語意的前提下，告訴瀏覽器這些元素應該如何呈現。

## 第一課：如何讓 HTML 保持結構，讓 CSS 負責外觀

### 問題情境

你希望標題在畫面上更醒目，例如改成紅色並放大。若只看 HTML，`<h1>` 已經表示「這是標題」，但它沒有說明這個標題一定要是紅色，也沒有說明一定要是 40px。

這時候真正要解決的問題不是「如何再加一個 HTML 標籤」，而是「如何針對既有 HTML 元素描述外觀」。

### 畫面觀察

在沒有額外 CSS 的情況下，瀏覽器仍會用預設樣式顯示 `<h1>` 和 `<p>`。標題通常會比段落大，也會換行，但這只是瀏覽器預設呈現，不一定符合你的設計需求。

如果需求是「讓標題變成紅色、字體更大」，作用對象其實是 `<h1>` 這個元素，改變的是它的視覺外觀，而不是它的內容語意。

### 最小修改

只需要加入一條針對 `h1` 的 CSS 規則：

```css
h1 {
    color: red;
    font-size: 40px;
}
```

### 結果解釋

這段 CSS 的意思是：選到頁面中的 `h1` 元素，將文字顏色設定為紅色，並將字體大小設定為 `40px`。HTML 裡的文字內容沒有改變，`<h1>` 的語意也沒有改變，改變的是瀏覽器繪製它時採用的外觀規則。

這就是 CSS 的基本價值：HTML 專注於結構與內容，CSS 專注於外觀與布局。兩者分開後，同一份 HTML 可以套用不同樣式，CSS 也可以集中管理多個元素的視覺呈現。

### 誤判修正

初學者常把「畫面變漂亮」誤解成「HTML 變得更有語意」。事實上，CSS 不會把一段普通文字變成真正的標題語意；語意仍然要靠正確 HTML 標籤建立。CSS 只是控制這個元素看起來如何。

另一個常見誤解是認為 HTML 不能做任何樣式。更精確地說，HTML 可以承載一些呈現相關寫法，但若把大量樣式混進 HTML，會讓結構與外觀糾纏在一起，增加維護成本。本章採用的做法是：讓 HTML 負責內容結構，讓 CSS 負責樣式。

### 小整理

CSS 主要用來設定 HTML 元素的外觀，例如文字顏色、字體大小、圖片外型與版面布局。本章先抓住最重要的分工：HTML 說明內容是什麼，CSS 說明內容看起來如何。

## 第二課：一條 CSS 規則如何套用到畫面上

### 問題情境

現在你已經知道 CSS 可以改變外觀，但還需要看懂 CSS 的基本語法。若你看到下面這段程式碼，必須能判斷它「選到誰」以及「改了什麼」：

```css
h1 {
    color: red;
    font-size: 40px;
}
```

### 畫面觀察

CSS 不是直接對整個頁面喊一句「變漂亮」。它需要先選到目標元素，再把一組樣式宣告套用到那些元素上。

以上面範例來說，`h1` 是 selector，用來指定作用對象。大括號 `{}` 裡面是一組 declaration block，裡面每一行像 `color: red;` 這樣的內容都是 declaration。`color` 是 property，`red` 是 value。

### 最小修改

如果你只想先改文字顏色，可以把規則縮到最小：

```css
h1 {
    color: red;
}
```

這條規則已經能形成完整的 CSS 指令：選到 `h1`，把它的文字顏色改成紅色。

### 結果解釋

CSS 規則可以拆成兩個問題：

第一個問題是「作用對象是誰？」答案由 selector 決定。在這裡，`h1` 會匹配 HTML 中的 `<h1>` 元素。

第二個問題是「對它做什麼修改？」答案由 declaration 決定。在這裡，`color: red;` 表示將文字顏色設定為紅色。

完整的 CSS 心智模型可以先記成：

```css
selector {
    property: value;
}
```

`property` 和 `value` 之間使用英文冒號 `:`，每一個 declaration 後面通常使用英文分號 `;` 結束。多個 declaration 可以放在同一個 declaration block 中，依序描述同一批元素要套用哪些樣式。

### 範例拆解

下面是最小可觀察範例：

```html
<h1>CSS 簡介</h1>
<p>這是一段介紹 CSS 的文字。</p>
```

```css
h1 {
    color: red;
    font-size: 40px;
}
```

這段 CSS 的作用對象是 `<h1>`，不是 `<p>`。作用條件是頁面上存在能被 `h1` selector 匹配的元素。修改前，標題會使用瀏覽器預設樣式；修改後，標題文字會變成紅色，並以 `40px` 的字體大小呈現。

如果你把 selector 寫成 `p`，改變的就會是段落，而不是標題。如果你把 `font-size` 寫在大括號外，瀏覽器就無法把它理解成針對 `h1` 的宣告。

### 小整理

一條 CSS 規則至少要回答兩件事：選誰，以及怎麼改。selector 負責選誰，declaration 負責描述樣式變更。讀 CSS 時，不要只看 property 名稱，也要回頭確認 selector 是否真的選到你以為的元素。

## 第三課：為什麼開發時常用展開風格

### 問題情境

來源中示範了兩種寫法：

```css
h1 {
    color: red;
    font-size: 40px;
}
```

```css
h1{color:red;font-size:40px;}
```

兩段 CSS 對畫面的效果相同，都會讓 `h1` 變成紅色並設定為 `40px`。問題是：既然效果一樣，平常寫 CSS 時應該用哪一種？

### 畫面觀察

展開風格把 selector、每一條 declaration 與大括號分行呈現。這讓你比較容易看出哪個 selector 控制哪些 declaration，也比較容易新增、刪除或調整其中一行。

緊湊風格把空白和換行拿掉，檔案看起來更短，但人類閱讀與維護會比較辛苦。來源提到，上線時可以透過工具把展開風格轉成緊湊風格，以減少檔案體積與網路傳輸量。

### 最小修改

開發時建議先使用展開風格：

```css
h1 {
    color: red;
    font-size: 40px;
}
```

這樣在檢查錯誤時，你可以逐行確認 selector、property、value、冒號與分號是否正確。

### 結果解釋

展開風格不會讓畫面變得更漂亮，它改善的是可讀性與維護性。緊湊風格也不會改變 CSS 的意思，它主要用在上線前後的檔案最佳化流程。

本章的重點不是學會壓縮工具，而是先養成能讀懂、能檢查、能維護的 CSS 寫法。

### 誤判修正

不要把緊湊風格誤解成比較高級的寫法。它只是少了空白與換行，對初學者來說更不容易看出語法邊界。學習與開發階段應優先使用展開風格，等到理解規則後，再把壓縮與部署流程留給後續工具處理。

### 小整理

同一條 CSS 規則可以有不同排版風格。展開風格適合學習、開發與維護；緊湊風格適合上線時降低檔案體積。本章所有範例會優先使用展開風格。

## 基本語法回查

在理解前面三課後，可以把 CSS 入門語法整理成下表，作為回查工具：

| 名稱 | 範例 | 作用 |
|---|---|---|
| selector | `h1` | 選出要套用樣式的 HTML 元素 |
| declaration block | `{ ... }` | 包住一組樣式宣告 |
| declaration | `color: red;` | 描述一個樣式設定 |
| property | `color` | 指定要修改的樣式類型 |
| value | `red` | 指定該樣式要使用的值 |
| colon | `:` | 分隔 property 與 value |
| semicolon | `;` | 結束一條 declaration |

這張表只能幫你回想名稱，真正判斷 CSS 時仍要回到兩個問題：selector 是否選到正確元素，以及 declaration 是否正確描述了想要的視覺變化。

## 常見誤解與修正方式

### 誤解一：CSS 會改變 HTML 的語意

錯誤理解是：只要把文字變大，就等於它是標題。這會造成結構與外觀混淆。正確理解是：HTML 標籤負責語意，CSS 負責外觀。若某段內容是標題，應先用適合的 HTML 標籤，再用 CSS 調整它看起來的樣子。

### 誤解二：寫了 property，畫面就一定會變

CSS 必須先選到元素。若 selector 沒有匹配到任何 HTML 元素，declaration 寫得再正確也不會影響畫面。排查時先確認 selector 和 HTML 結構是否對得上。

### 誤解三：中英文標點可以混用

CSS 語法中的冒號與分號應使用英文符號，例如 `color: red;`。若誤用全形標點，瀏覽器可能無法正確解析該宣告。

### 誤解四：緊湊風格比較適合日常開發

緊湊風格適合降低檔案體積，但不適合初學者閱讀與修改。日常開發應先使用展開風格，讓每條 declaration 清楚分行。

### 誤解五：CSS 只是讓網頁變好看

來源中用「美化 HTML」幫助初學者理解 CSS，這是入門角度的說法。更完整地看，CSS 不只處理美觀，也負責文字呈現、圖片外型、間距、版面配置與不同狀態下的視覺結果。本章先從最簡單的文字樣式開始，後續章節會逐步展開布局與規則競爭。

## 實務使用情境

當你需要統一網站標題的顏色、調整段落字級、控制圖片尺寸、安排區塊版面，或讓同一類元素使用一致外觀時，就會使用 CSS。

在實務開發中，CSS 的價值不只是讓單一元素變漂亮，而是讓樣式能被集中管理。當樣式獨立於 HTML 結構之外，你可以更容易修改整站外觀，也能降低內容與外觀混在一起造成的維護成本。

本章建立的入門語法也會用於後續所有 CSS 主題。無論之後學 selector、box model、flexbox 還是 grid，本質上仍然是在回答：選到哪些元素，以及對它們套用哪些樣式規則。

## 自我檢查

   ```css

       color: red;
   }
   ```

2. 如果 HTML 中只有 `<p>CSS 簡介</p>`，但 CSS 寫成 `h1 { color: red; }`，畫面中的段落文字會變紅嗎？請說明原因。

3. 將下面的緊湊風格改寫成展開風格。

   ```css
   h1{color:red;font-size:40px;}
   ```

   ```css
   h1 {

   }
   ```

5. 用一句話說明 HTML 與 CSS 的分工：HTML 負責什麼，CSS 負責什麼？

## 小結

CSS 是用來描述 HTML 元素外觀與版面呈現的樣式語言。入門時先抓住兩個核心：HTML 負責內容結構，CSS 負責視覺表現；一條 CSS 規則則由 selector 和 declaration block 組成，前者決定選誰，後者決定怎麼改。

寫 CSS 時，建議先使用展開風格，讓 selector、property、value、冒號與分號都清楚可見。等到開發完成後，再把緊湊風格或壓縮流程視為上線最佳化的一部分。
