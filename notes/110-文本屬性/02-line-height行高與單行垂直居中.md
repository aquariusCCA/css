# line-height 行高與單行垂直居中

## 學習目標

讀完這篇筆記後，你應該能夠：

- 說明 `line-height` 控制的是行高，而不只是文字本身高度。
- 理解行高由文字高度與上下空隙共同組成。
- 使用 `line-height` 調整多行文字的行距。
- 使用「行高等於盒子高度」讓單行文字垂直居中。
- 判斷這個垂直居中技巧的適用範圍。

## 使用情境

當一段文字太擠，閱讀起來會壓迫；當行距太大，段落又會顯得鬆散。這時需要使用 `line-height` 調整文字行與行之間的距離。

另一個常見情境是按鈕、標籤、導覽列項目等固定高度盒子中，只有一行文字需要垂直置中。這時也常用 `line-height` 解決。

## line-height 是什麼

`line-height` 屬性用於設定行間的距離，也就是每一行文字所佔的高度。

行高不是單純的文字高度，而是包含：

- 上間距。
- 文字本身高度。
- 下間距。

![行高由文字高度與上下間距組成的示意圖](../../origin/110-文本屬性/assets/images/line-height-img-001-ec9b78.png)

基本寫法：

```css
p {
  line-height: 26px;
}
```

這代表每一行文字佔用 `26px` 的行高。當段落有多行文字時，行高會直接影響閱讀節奏。

## 範例拆解

```css
p {
  font-size: 16px;
  line-height: 26px;
}
```

這段設定代表文字大小是 `16px`，但每一行佔用的高度是 `26px`。多出來的空間會分布在文字上下，讓行與行之間不會太擠。

如果行高太小，文字行之間可能互相靠得太近；如果行高太大，段落會顯得鬆散。

## 單行文字垂直居中

CSS 沒有提供專門讓單行文字在盒子中垂直居中的文字屬性，但可以使用一個常見技巧：讓文字的行高等於盒子的高度。

```css
div {
  width: 200px;
  height: 40px;
  background-color: pink;
  line-height: 40px;
}
```

```html
<div>我要居中</div>
```

當 `height` 和 `line-height` 都是 `40px` 時，這一行文字會在盒子中看起來垂直居中。

![盒子高度與行高相同時單行文字垂直居中的示意圖](../../origin/110-文本屬性/assets/images/line-height-img-002-d09232.png)

可以這樣理解：

1. 行高中的上空隙和下空隙會把文字擠到中間。
2. 如果行高小於盒子高度，文字會偏上。
3. 如果行高大於盒子高度，文字會偏下。

## 生效條件與限制

這個技巧適合「單行文字」。例如按鈕、導覽列、標籤或固定高度提示文字。

如果內容可能換成多行，就不適合只靠 `line-height` 等於 `height`。多行文字會依照每一行的行高排列，可能超出容器或看起來不自然。

對多行垂直置中，通常會考慮其他布局方式，例如 Flexbox。

## 常見錯誤

- **以為 `line-height` 只控制行與行之間的空白**：它控制的是整行佔用的高度，包含文字高度與上下空隙。
- **把單行技巧套到多行文字**：`height` 等於 `line-height` 只適合單行，多行內容會破壞效果。
- **行高小於字體大小**：文字可能擠在一起，甚至出現視覺重疊。
- **固定高度卻忘記內容可能變多**：如果文案變長換行，原本的單行垂直居中技巧就不可靠。

## 實務判斷準則

- 段落閱讀性不佳：調整 `line-height`。
- 固定高度盒子中只有一行文字：可讓 `line-height` 等於 `height`。
- 按鈕或導覽項目文字需要垂直居中：可用單行行高技巧。
- 內容可能多行：不要只靠這個技巧，改用更合適的布局方式。

## 重點整理

- `line-height` 設定每一行文字佔用的高度。
- 行高包含文字高度與上下空隙。
- 多行文字的行距可以透過 `line-height` 改善。
- 單行文字垂直居中常用 `height` 等於 `line-height`。
- 這個垂直居中技巧只適合單行文字。
