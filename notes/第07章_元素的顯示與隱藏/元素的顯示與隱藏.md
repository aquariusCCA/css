# 元素的顯示與隱藏

## 學習目標

讀完本章後，你應該能做到以下幾件事：

- 判斷 `display: none` 和 `visibility: hidden` 對畫面與版面空間造成的差異。
- 說明 `display: block` 在來源範例中為什麼能讓元素重新顯示，以及它同時會讓元素以區塊形式參與排版。
- 在需要「元素不見且不佔位置」與「元素不見但保留位置」之間做出正確選擇。
- 分辨 `overflow` 控制的是盒子內容溢出後的處理方式，不是把整個元素從版面中移除。
- 看出常見的誤判，例如把 `overflow: hidden` 當成元素隱藏、把 `visibility: hidden` 當成不佔位、或用 `display: block` 還原所有類型元素。

## 前置知識

學習本章前，需要先能看懂基本 HTML 元素與 CSS class 選擇器，例如：

```html
<div class="box">內容</div>
```

```css
.box {
  width: 200px;
  height: 120px;
  background-color: pink;
}
```

你也需要知道瀏覽器會把元素排進頁面的正常排版流程中。多數 `<div>` 預設會像區塊一樣由上往下排列；若元素有寬度、高度、背景色或邊框，畫面上就更容易觀察它佔用的區域。

本章會提到「元素是否可見」和「元素是否佔位」。這兩個問題必須分開看：有些元素看不見，但仍然保留原本空間；有些元素則會完全不參與版面。

## 本章範圍

本章深入處理三個與「顯示與隱藏」相關的 CSS 主題：

- `display: none`：讓元素不顯示，並且不再佔有原本位置。
- `display: block`：來源用來示範讓元素顯示，同時也會讓元素以 block 方式排版。
- `visibility: hidden` 與 `visibility: visible`：控制元素可見或隱藏，但隱藏後仍保留原本位置。

來源中也包含 `overflow`。`overflow` 的主線不是控制整個元素是否存在，而是控制內容超出盒子範圍時要顯示、裁切或出現捲動條。因此本章會把它放在「容易混淆的相鄰概念」中說明，幫助你分辨「隱藏元素」和「隱藏溢出內容」的差異。

本章不深入展開所有 `display` 值，例如 `inline`、`inline-block`、`flex`、`grid`。這些會影響排版模型，適合另開章節。本章也不展開 JavaScript 顯示隱藏效果、動畫淡入淡出、完整定位規則或瀏覽器可及性行為。

## 先看問題：元素不見時，版面要不要留下空位

假設畫面上有兩個相鄰的區塊：

```html
<div class="card card-a">第一個區塊</div>
<div class="card card-b">第二個區塊</div>
```

```css
.card {
  width: 200px;
  height: 100px;
  margin-bottom: 12px;
  padding: 16px;
  border: 2px solid #444;
}

.card-a {
  background-color: pink;
}

.card-b {
  background-color: skyblue;
}
```

修改前，兩個區塊會由上往下排列。第一個區塊佔住上方 100px 高度，再加上 padding、border 與 margin；第二個區塊排在它下方。

當你說「把第一個區塊藏起來」時，實際上可能有兩種不同需求。

第一種需求是：第一個區塊完全不要出現在版面中，第二個區塊要往上補位。這時候要用 `display: none`。

第二種需求是：第一個區塊的內容不要被看到，但它原本佔的位置仍然保留，第二個區塊不要往上移。這時候要用 `visibility: hidden`。

所以本章的核心判斷不是只問「看不看得到」，而是同時問：

```text
元素是否被繪製出來？
元素是否仍然佔用版面空間？
```

## 第一課：想讓元素完全從版面中消失

### 問題情境

頁面上有一個提示區塊和一個主要內容區塊。當提示不需要顯示時，你希望主要內容往上補位，不要留下空白。

這種需求常見於關閉廣告、收合提示、隱藏某個狀態區塊或讓某個頁面片段暫時不參與排版。

### 畫面觀察

修改前，`.notice` 和 `.content` 都是正常元素。因為它們都在正常排版流程中，所以 `.notice` 會先佔一塊空間，`.content` 再排到下面。

```html
<div class="notice">系統維護公告</div>
<div class="content">主要內容</div>
```

```css
.notice,
.content {
  width: 220px;
  height: 80px;
  padding: 16px;
  border: 2px solid #333;
}

.notice {
  background-color: pink;
}

.content {
  background-color: skyblue;
}
```

如果只是把 `.notice` 的文字顏色改成透明，或把背景拿掉，它可能看起來比較不明顯，但元素本身仍然可能佔位置。要讓它完全不參與版面，需要改變它的 `display`。

### 最小修改

```css
.notice {
  display: none;
}
```

### 結果解釋

`.notice` 是這條 CSS 的作用對象。加入 `display: none` 後，瀏覽器不會為 `.notice` 產生可見的盒子，也不會讓它佔有原本位置。

修改前，`.content` 會排在 `.notice` 下方。修改後，`.notice` 從畫面與版面流程中消失，`.content` 會往上移到原本 `.notice` 所在的位置。

來源中特別提醒：`display` 隱藏元素後，不再佔有原來的位置。這正是 `display: none` 和 `visibility: hidden` 最重要的差異。

### 誤判修正

常見誤解是以為 `display: none` 只是「看不到」。更準確地說，它是讓元素不再以原本的盒子形式參與畫面排版。因為它不佔位置，所以後面的元素會重新排列。

另一個常見誤解是以為元素被 `display: none` 後，修改寬高或背景色還會在畫面上看到效果。實際上，只要 `display: none` 還在生效，這個元素就不會顯示出自己的盒子；寬度、高度、背景色等視覺效果都沒有畫面可以呈現。

### 小整理

當你的需求是「元素不見，版面也不要留空位」時，使用：

```css
.target {
  display: none;
}
```

它適合用來完整移除某個元素的視覺與排版占位，但也因為會改變後續元素的位置，所以使用前要確認版面補位是你想要的結果。

## 第二課：想把被隱藏的元素重新顯示

### 問題情境

來源中提到 `display: block` 除了可以把元素轉換為塊級元素之外，也有顯示元素的意思。這句話需要放在具體情境中理解：如果元素目前因為 `display: none` 而不顯示，改成其他會產生盒子的 `display` 值，就能讓它重新出現在畫面中。

最常見的入門範例是：原本隱藏一個盒子，之後把它顯示出來。

### 畫面觀察

修改前，`.panel` 有寬度、高度與背景色，但因為 `display: none`，它不會出現在畫面上，也不會佔用空間。

```html
<div class="panel">提示面板</div>
<div class="next">下一個區塊</div>
```

```css
.panel {
  display: none;
  width: 200px;
  height: 100px;
  background-color: pink;
}

.next {
  width: 200px;
  height: 100px;
  background-color: skyblue;
}
```

此時你會看到 `.next` 直接排在上方，因為 `.panel` 沒有佔位置。

### 最小修改

若要讓 `.panel` 重新出現，可以把 `display: none` 改成 `display: block`：

```css
.panel {
  display: block;
  width: 200px;
  height: 100px;
  background-color: pink;
}
```

### 結果解釋

`.panel` 重新以 block 盒子的形式參與排版，因此畫面上會看到粉色提示面板，`.next` 也會被排到它下方。

這裡要抓住兩層意思。第一，`display: block` 不是「單純取消隱藏」的魔法字，而是指定元素用 block 的方式顯示。第二，因為 block 會產生盒子並參與正常排版，所以它能讓原本 `display: none` 的元素重新出現在畫面中。

補充說明：如果原本被隱藏的是 `span`、`a` 或某個需要用 `flex`、`grid` 排版的元素，直接改成 `display: block` 可能會改變原本排版方式。實務上應該還原成符合需求的 display 值，例如 `inline`、`inline-block`、`flex` 或 `grid`。來源以 `display: block` 作為入門示範，本章不展開所有 `display` 值。

### 誤判修正

常見誤解是把 `display: block` 當成所有元素的通用「顯示」寫法。它確實會讓元素顯示，但也會讓元素以 block 方式排列。若你只是想讓原本的 inline 元素恢復顯示，使用 `block` 可能會造成換行或寬度行為改變。

另一個誤判是只刪除某一條 `display: none`，卻忽略後面可能有其他 CSS 規則又把同一個元素設成 `display: none`。排查時要看最後實際生效的 computed style，而不是只看其中一段 CSS。

### 小整理

`display` 同時回答兩個問題：

- 這個元素是否產生盒子並參與排版。
- 如果產生盒子，它用哪一種排版角色顯示。

來源範例中的重點是：`display: none` 讓元素不顯示也不佔位；`display: block` 讓元素重新以區塊形式顯示。

## 第三課：想讓元素看不見，但保留原本位置

### 問題情境

有時候你不想讓某個元素被看到，但又不希望後面的元素往上補位。例如版面中有一個固定大小的位置要暫時空著，或你想比較「隱藏前後」而不讓版面跳動。

這時候 `display: none` 不適合，因為它會讓元素不佔位置。你需要的是 `visibility: hidden`。

### 畫面觀察

修改前，兩個盒子上下排列：

```html
<div class="avatar">頭像</div>
<div class="name">使用者名稱</div>
```

```css
.avatar,
.name {
  width: 200px;
  height: 80px;
  padding: 16px;
  border: 2px solid #333;
}

.avatar {
  background-color: pink;
}

.name {
  background-color: skyblue;
}
```

如果你希望「頭像」不被看到，但「使用者名稱」仍然留在原本的位置，就不能讓 `.avatar` 從排版中消失。

### 最小修改

```css
.avatar {
  visibility: hidden;
}
```

### 結果解釋

`.avatar` 是作用對象。加入 `visibility: hidden` 後，`.avatar` 的內容和背景不會被顯示出來，但它原本的盒子仍然保留在版面中。

修改前，`.name` 排在 `.avatar` 下方。修改後，`.avatar` 看不見，但 `.name` 仍然維持在下方，不會往上補位。畫面上會留下 `.avatar` 原本佔用的空白區域。

來源中特別提醒：`visibility` 隱藏元素後，繼續佔有原來的位置。這是它和 `display: none` 的核心差異。

### 誤判修正

常見誤解是以為 `visibility: hidden` 和 `display: none` 都只是「隱藏」，差別不大。實際排版時差很多：前者保留位置，後者不保留位置。

另一個誤解是以為看不到就代表不存在。使用 `visibility: hidden` 時，元素雖然不可見，但它的盒子仍在正常排版流程中，所以仍然會影響後面元素的位置。

### 小整理

`visibility` 常用值如下：

| 值 | 畫面結果 | 是否保留原本位置 |
|---|---|---|
| `visible` | 元素可見 | 是 |
| `hidden` | 元素不可見 | 是 |

當你的需求是「元素不要被看到，但版面不要重新補位」時，使用：

```css
.target {
  visibility: hidden;
}
```

## 第四課：想處理超出盒子的內容

### 問題情境

來源中也提到 `overflow`。假設一個盒子只有固定寬高，但裡面的文字很多，內容超出盒子範圍。你不是要把整個盒子隱藏，而是要決定超出去的內容要怎麼處理。

這是 `overflow` 的使用情境。

### 畫面觀察

修改前，`.message` 有固定寬度與高度，但文字內容太長：

```html
<div class="message">
  這是一段很長的公告文字。當內容超過盒子的高度時，超出的文字可能會跑出盒子外面，影響周圍版面觀察。
</div>
```

```css
.message {
  width: 220px;
  height: 100px;
  padding: 12px;
  border: 3px solid pink;
}
```

這裡的問題不是 `.message` 本身要不要顯示，而是「內容超出 `.message` 的盒子時，要不要讓超出的部分被看到」。

### 最小修改

如果不想讓超出的內容露出盒子，可以寫：

```css
.message {
  width: 220px;
  height: 100px;
  padding: 12px;
  border: 3px solid pink;
  overflow: hidden;
}
```

如果希望使用者能透過捲動查看超出內容，可以改成：

```css
.message {
  overflow: auto;
}
```

### 結果解釋

`.message` 仍然顯示，也仍然佔有版面空間。`overflow` 只處理內容超出盒子後的顯示方式。

`overflow: hidden` 會把超出盒子範圍的內容裁切掉。`overflow: auto` 則是在內容真的超出時才出現捲動條。來源也提到 `overflow: scroll` 會顯示捲動條，即使內容沒有溢出也可能顯示捲動條；`overflow: visible` 則是預設可見的方向，讓溢出內容仍可能露出來。

因此，`overflow: hidden` 的「hidden」和 `visibility: hidden` 的「hidden」不是同一件事。前者隱藏的是超出盒子範圍的內容；後者隱藏的是元素本身的可見狀態。

### 誤判修正

常見誤解是把 `overflow: hidden` 當成隱藏元素。實際上，元素本身仍然在，背景、邊框、寬高和佔位都還在。只有超出盒子範圍的內容會被裁切。

來源也提醒：如果盒子內有定位元素，使用 `overflow: hidden` 要慎重，因為超出父盒子範圍的部分也可能被裁掉。這不代表 `overflow: hidden` 錯了，而是要先確認你是不是真的想把超出容器的部分裁切掉。

### 小整理

`overflow` 常見值可以這樣理解：

| 值 | 作用重點 |
|---|---|
| `visible` | 內容溢出時仍可能顯示在盒子外 |
| `hidden` | 裁切超出盒子範圍的內容 |
| `scroll` | 顯示捲動條，不溢出時也可能出現 |
| `auto` | 只有內容溢出時才顯示捲動條 |

使用 `overflow` 前，先問的問題應該是「內容超出盒子後怎麼辦」，而不是「元素要不要從版面消失」。

## 顯示與隱藏的判斷流程

學完前面幾課後，可以用下面的流程選擇屬性：

1. 如果元素要完全不顯示，且後面的內容要補上位置，使用 `display: none`。
2. 如果元素要看不見，但原本位置要保留，使用 `visibility: hidden`。
3. 如果元素本身要顯示，只是內容超出盒子時不要露出來，使用 `overflow: hidden`。
4. 如果元素本身要顯示，超出內容需要可以查看，考慮 `overflow: auto` 或 `overflow: scroll`。
5. 如果要把 `display: none` 的元素顯示回來，選擇符合排版需求的 `display` 值；來源範例使用 `display: block`，但實務上不一定所有元素都適合用 `block` 還原。

這個流程的核心是把「元素本身」和「盒子裡的內容」分開判斷，再把「是否可見」和「是否佔位」分開判斷。

## 顯示與隱藏回查表

| 需求 | 建議寫法 | 元素是否可見 | 是否佔原本位置 | 主要用途 |
|---|---|---|---|---|
| 完全隱藏元素並讓版面補位 | `display: none` | 否 | 否 | 關閉區塊、收合內容、暫時不讓元素參與排版 |
| 讓元素以區塊方式顯示 | `display: block` | 是 | 是 | 讓元素產生 block 盒子，常用於還原來源範例中的隱藏盒子 |
| 隱藏元素但保留空位 | `visibility: hidden` | 否 | 是 | 保留版面位置、避免後續元素移動 |
| 顯示元素 | `visibility: visible` | 是 | 是 | 還原 visibility 隱藏狀態 |
| 裁切超出的內容 | `overflow: hidden` | 元素可見 | 是 | 固定盒子範圍，隱藏超出部分 |
| 內容溢出時才捲動 | `overflow: auto` | 元素可見 | 是 | 文字或內容可能超出固定容器時 |

## 常見誤解與修正方式

第一個常見誤解是把 `display: none` 和 `visibility: hidden` 當成同義。它們都會讓元素看不見，但版面結果不同。`display: none` 不保留位置；`visibility: hidden` 保留位置。排版問題通常就差在這一點。

第二個常見誤解是使用 `visibility: hidden` 後，期待後面的元素往上補位。如果你的需求是補位，應改用 `display: none`。`visibility: hidden` 的重點正是保留原本位置。

第三個常見誤解是把 `display: block` 當成所有隱藏元素的通用恢復值。`block` 會讓元素變成區塊排版。若原本元素應該是 inline、inline-block、flex 或 grid，應改用符合原本版面需求的 display 值。

第四個常見誤解是以為 `overflow: hidden` 可以隱藏整個元素。它只會裁切超出盒子範圍的內容。盒子本身仍然存在，仍然可能顯示背景、邊框，也仍然佔位。

第五個常見誤解是看到元素不見，就只檢查 `display`。實際上元素看不見可能來自多種原因：`display: none`、`visibility: hidden`、內容被 `overflow: hidden` 裁切、CSS selector 沒有選到、或樣式被後面的規則覆蓋。本章只深入前三種，selector 與層疊問題適合回到前面章節或另開章節檢查。

## 實務使用情境

`display: none` 常用在完全關閉某個區塊，例如收合選單、隱藏提示訊息、切換頁籤內容或讓某個狀態下不需要的區塊退出排版。它會讓後續內容重新補位，因此適合用在希望版面重新排列的情境。

`visibility: hidden` 常用在需要保留版面位置的情境，例如暫時隱藏某個固定大小的區塊，或讓前後狀態切換時不造成元素跳動。它不適合用來收合空間，因為空位會被保留下來。

`overflow` 常用在固定尺寸容器中，例如公告盒、卡片摘要、側邊欄、對話框內容區或任何可能出現內容過多的區塊。它處理的是內容超出邊界後的視覺結果，不應和元素本身的顯示隱藏混在一起。

補充說明：在實際專案中，顯示與隱藏常會搭配 class 或 JavaScript 切換，例如 `.is-hidden { display: none; }`。本章只說明 CSS 屬性本身造成的畫面差異，不展開 JavaScript 操作流程。

## 自我檢查

1. 如果你希望一個提示區塊消失後，下方內容立刻往上補位，應使用 `display: none` 還是 `visibility: hidden`？為什麼？

2. 下面這段 CSS 會讓 `.box` 消失後仍保留位置嗎？

   ```css
   .box {
     display: none;
   }
   ```

3. 如果把下面的 `.photo` 改成 `visibility: hidden`，`.caption` 會往上移嗎？請先預測畫面結果。

   ```html
   <div class="photo">圖片區</div>
   <div class="caption">圖片說明</div>
   ```

4. `overflow: hidden` 隱藏的是整個元素，還是超出盒子範圍的內容？如果盒子有背景色和邊框，畫面上還會看到它們嗎？

5. 來源中用 `display: block` 讓元素顯示。為什麼不能把它當成所有元素的通用還原值？

6. 下面兩段 CSS 的畫面差異是什麼？

   ```css
   .panel {
     display: none;
   }
   ```

   ```css
   .panel {
     visibility: hidden;
   }
   ```

7. 一個固定高度的公告盒文字太多，超出盒子後影響下方內容閱讀。你會優先考慮 `display`、`visibility` 還是 `overflow`？請說明原因。

## 小結

元素顯示與隱藏的重點，不只是「看不看得到」，而是要同時判斷「是否佔版面空間」。`display: none` 會讓元素不顯示也不佔位；`visibility: hidden` 會讓元素不可見但保留原本位置；`overflow` 則是處理內容超出盒子後的顯示方式，不是把整個元素移出排版。

寫 CSS 時，先把需求說清楚：你是要讓元素退出版面、只是不被看見，還是只想裁切超出的內容。判斷清楚後，屬性選擇就會穩定很多。
