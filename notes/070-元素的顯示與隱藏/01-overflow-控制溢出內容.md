# overflow：控制超出盒子的內容怎麼顯示

## 學習目標

讀完這篇筆記，你應該能夠：

- 理解什麼是內容溢出。
- 說明 `overflow` 屬性控制的是哪一種顯示行為。
- 分辨 `visible`、`hidden`、`scroll`、`auto` 的差異。
- 判斷什麼情境適合裁切內容，什麼情境適合顯示捲動條。

## 問題情境

當元素被設定固定寬度或高度時，裡面的內容可能比盒子本身更大。例如卡片高度固定，但文字太長；圖片放進圓角容器，但圖片邊緣超出容器；或一段很長的文字沒有換行，直接衝出版面。

這些超出元素盒子寬度或高度的部分，就叫做「溢出內容」。如果不處理，溢出的內容可能覆蓋旁邊元素、破壞版面，或讓畫面看起來不受控。

## 一句話理解

`overflow` 用來決定「內容超出元素盒子後」要直接顯示、裁切，還是透過捲動條查看。

它不是用來隱藏整個元素，而是處理元素內部內容超出盒子範圍的那一部分。

## 基本語法

```css
.box {
  overflow: visible;
}
```

常見值如下：

| 值 | 行為 |
| --- | --- |
| `visible` | 預設值，溢出的內容仍會顯示在盒子外面 |
| `hidden` | 溢出的內容會被裁切，不顯示在盒子外 |
| `scroll` | 顯示捲動條，即使內容沒有溢出也可能保留捲動條空間 |
| `auto` | 只有內容真的溢出時才顯示捲動條 |

## 範例：固定盒子中的長內容

先看一個固定寬高的盒子：

```css
.message {
  width: 200px;
  height: 160px;
  border: 3px solid pink;
  margin: 40px auto;
  overflow: auto;
}
```

```html
<div class="message">
  這是一段很長的內容。當文字超出盒子的高度時，如果沒有處理溢出，
  後面的內容可能會跑出盒子外面，影響周圍版面的閱讀。
  這裡故意放入更多文字，讓盒子產生垂直方向的溢出。
</div>
```

在這個例子中，`.message` 的寬度與高度固定。當內容高度超過 `160px` 時，`overflow: auto` 會讓瀏覽器在需要時顯示捲動條，使用者仍然可以查看完整內容。

## 四種常用值怎麼選

### `overflow: visible`

`visible` 會讓溢出的內容照樣顯示。這是預設行為。

```css
.message {
  overflow: visible;
}
```

這種寫法適合不介意內容超出盒子的情境。不過在卡片、側欄、固定高度列表中，直接顯示溢出內容通常會讓版面變亂。

### `overflow: hidden`

`hidden` 會裁切超出盒子範圍的部分。

```css
.message {
  overflow: hidden;
}
```

這常用在需要把內容限制在容器內的場景，例如圖片裁切、圓角卡片、隱藏超出區域的動畫效果。但要注意：被裁切的內容使用者看不到，也不能透過捲動條查看。

如果元素內部有定位的子元素，`overflow: hidden` 也會把超出容器範圍的部分一起裁掉。這在做下拉選單、提示框或浮動裝飾時很常踩雷。

### `overflow: scroll`

`scroll` 會顯示捲動條，即使內容沒有超出盒子，也可能保留捲動條。

```css
.message {
  overflow: scroll;
}
```

這種寫法適合你希望捲動區域一直保持穩定的情境。不過如果內容沒有溢出也出現捲動條，畫面可能顯得多餘。

### `overflow: auto`

`auto` 會讓瀏覽器自行判斷：內容溢出時顯示捲動條，沒有溢出時不顯示。

```css
.message {
  overflow: auto;
}
```

這是實務上很常用的選擇，因為它能兼顧版面整潔與內容可讀性。

## 常見錯誤

### 把 `overflow: hidden` 當成隱藏元素

`overflow: hidden` 不會讓元素本身消失。元素的邊框、背景、寬高仍然存在，它只會裁切「超出盒子範圍的內容」。

如果你要讓整個元素消失，應該考慮 `display: none` 或 `visibility: hidden`，那是另一類「元素顯示與隱藏」問題。

### 忘記固定尺寸或可溢出的限制

如果元素沒有明確的寬度、高度或其他限制，內容通常會把盒子撐開，看起來就不會產生需要處理的溢出。

例如只有文字內容、沒有固定高度時，盒子會隨文字變高，`overflow` 的效果就不明顯。

### 裁掉定位子元素

容器設定 `overflow: hidden` 後，內部子元素只要超出容器邊界，就可能被裁掉。這對圖片裁切很有用，但對選單、提示框、徽章、外露陰影等效果可能造成問題。

遇到這類問題時，要回頭檢查父層或祖先層是否設定了 `overflow: hidden`。

## 實務判斷準則

- 想保留溢出內容的自然顯示：使用 `visible`。
- 想裁切超出區域，例如圖片或圓角卡片：使用 `hidden`。
- 想固定顯示捲動區域：使用 `scroll`。
- 想在需要時才出現捲動條：使用 `auto`。
- 如果內容被莫名裁掉，先檢查父層是否設定了 `overflow: hidden`。

## 重點整理

- 溢出內容是指內容超出元素盒子寬度或高度的部分。
- `overflow` 控制的是溢出內容的顯示方式，不是整個元素的顯示與隱藏。
- `hidden` 會裁切內容，`scroll` 會顯示捲動條，`auto` 只在需要時顯示捲動條。
- 使用 `overflow: hidden` 時，要留意定位子元素與外露視覺效果也可能被裁掉。
