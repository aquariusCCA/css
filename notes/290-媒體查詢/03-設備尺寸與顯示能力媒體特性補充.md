# 設備尺寸與顯示能力媒體特性補充

## 學習目標

讀完這篇筆記，你應該能夠：

- 分辨 `aspect-ratio` 與已不建議使用的 `device-aspect-ratio`。
- 知道 `device-width`、`device-height` 屬於較舊的設備尺寸查詢。
- 認識顏色、解析度、掃描方式與網格顯示等低頻媒體特性。
- 判斷哪些媒體特性適合現代響應式設計，哪些主要用於閱讀舊程式碼或特殊設備。

## 使用情境

除了常見的 `width`、`height`、`orientation`，媒體查詢還提供許多和設備尺寸、比例、顯示能力有關的特性。

這些特性不是每個專案都會用到，但在下列情境可能會出現：

- 要依寬高比切換多媒體或全屏布局。
- 維護舊專案時看到 `device-width` 或 `device-aspect-ratio`。
- 針對高解析度螢幕載入更清晰圖片。
- 針對特殊設備、老舊設備或單色顯示器提供樣式。

這篇是補充型筆記，重點是幫你建立查閱與判斷能力。

## 一句話理解

這組媒體特性多用來判斷設備或顯示能力；現代響應式主線通常優先使用視口相關特性，低頻或 deprecated 特性則保留為理解舊程式碼與特殊設備的背景知識。

## 寬高比：aspect-ratio

`aspect-ratio` 用來檢測目標顯示區域的寬高比例。在一般網頁中，通常可以理解為視口寬高比。

```css
@media (aspect-ratio: 16/9) {
  .container {
    display: grid;
    grid-template-columns: repeat(3, 1fr);
  }
}

@media (max-aspect-ratio: 1/1) {
  .container {
    flex-direction: column;
  }
}
```

常見用途：

- 橫向與縱向版面切換。
- 全屏應用布局調整。
- 多媒體內容依比例選擇不同排列方式。

`aspect-ratio` 比 `device-aspect-ratio` 更適合現代網頁，因為它更貼近目前視口與實際可用空間。

## 不建議新專案使用的 device-* 特性

`device-width`、`device-height`、`device-aspect-ratio` 偵測的是設備顯示區域本身，而不是瀏覽器視口。

這類特性已不建議在新專案中使用。現代響應式設計通常改用：

- `width`、`min-width`、`max-width`
- `height`、`min-height`、`max-height`
- `aspect-ratio`
- `orientation`

舊式寫法範例：

```css
@media (min-device-width: 1024px) {
  body {
    font-size: 18px;
    margin: 0 auto;
    max-width: 1000px;
  }
}

@media (device-aspect-ratio: 16/9) {
  body {
    background-color: lightblue;
  }
}
```

如果你在舊程式碼中看到這些寫法，要先確認它是否真的需要判斷設備物理尺寸。多數情況可以改成視口相關查詢。

## 顏色能力：color、color-index、monochrome

這組特性用來判斷設備的顏色顯示能力。

### color

`color` 檢測設備每個顏色分量可以顯示的位元數。

```css
@media (min-color: 8) {
  body {
    background-image: url('high-quality-background.png');
  }
}

@media (max-color: 4) {
  body {
    background-image: url('low-quality-background.png');
  }
}
```

### color-index

`color-index` 檢查設備支援的顏色索引數量，也就是調色板中可用顏色總數。

```css
@media (min-color-index: 256) {
  body {
    background-image: url('high-quality-background.png');
  }
}

@media (max-color-index: 16) {
  body {
    background-color: #333;
  }
}
```

### monochrome

`monochrome` 檢測設備是否是單色或灰階顯示。

```css
@media (monochrome: 1) {
  body {
    color: black;
    background-color: white;
  }
}

@media (monochrome: 0) {
  body {
    color: #333;
    background-color: #f0f0f0;
  }
}
```

現代一般網站較少直接使用這些特性，但它們能提醒我們：設計時不應只依賴顏色本身傳達資訊，仍要注意對比與可讀性。

## 解析度：resolution

`resolution` 用來檢測設備顯示器的解析度或像素密度。

常見單位：

- `dpi`：每英寸顯示的點數。
- `dppx`：每 CSS 像素對應的設備像素數，`1dppx` 等於 `96dpi`。

```css
@media (min-resolution: 150dpi) {
  body {
    background-image: url('high-res-background.jpg');
  }
}

@media (min-resolution: 2dppx) {
  body {
    background-image: url('high-res-background.jpg');
  }
}

@media (max-resolution: 72dpi) {
  body {
    background-image: url('low-res-background.jpg');
  }
}
```

`resolution` 常用於選擇更適合高解析度顯示器的圖片資源。

## 掃描與網格顯示：scan、grid

`scan` 用來檢測顯示設備的掃描方式。

```css
@media (scan: progressive) {
  body {
    background-color: lightblue;
  }
}

@media (scan: interlace) {
  body {
    background-color: lightgreen;
  }
}
```

`grid` 用來檢測設備是否使用網格顯示內容。這裡的 `grid` 和 CSS Grid Layout 沒有直接關係。

```css
@media (grid: 1) {
  body {
    font-size: 12px;
  }
}

@media (grid: 0) {
  body {
    font-size: 16px;
  }
}
```

`grid` 只接受 `0` 或 `1` 這類媒體查詢布林值，也可以用 `(grid)` 測試是否為網格顯示設備。`true` 和 `false` 不是合法值。

## 常見錯誤

### 錯誤一：新專案仍使用 device-width 作主要斷點

`device-width` 判斷的是設備物理尺寸，不會隨桌面瀏覽器視窗調整而改變。現代響應式布局通常應使用 `width`、`min-width`、`max-width`。

### 錯誤二：把 media feature 的 grid 當成 CSS Grid

媒體特性 `grid` 是設備顯示能力查詢，不是 `display: grid` 的布局語法。

### 錯誤三：把低頻特性當成主要響應式工具

`scan`、`color-index`、`monochrome` 等特性有特定用途，但不適合作為一般網頁響應式布局的主線工具。

## 實務判斷

- 一般版面斷點：優先使用 `width` / `min-width` / `max-width`。
- 橫向與縱向差異：優先使用 `orientation` 或 `aspect-ratio`。
- 舊程式碼理解：遇到 `device-*` 時，確認是否可改為視口相關查詢。
- 高解析度資源：可考慮使用 `resolution`。
- 特殊設備：才考慮 `scan`、`grid`、`monochrome` 等低頻特性。
