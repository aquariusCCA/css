# 使用者偏好與輸入能力媒體特性

## 學習目標

讀完這篇筆記，你應該能夠：

- 使用 `prefers-color-scheme` 回應使用者的深色或淺色偏好。
- 使用 `prefers-reduced-motion` 為偏好減少動畫的使用者提供替代樣式。
- 使用 `hover`、`any-hover` 判斷設備是否支援懸停。
- 使用 `pointer`、`any-pointer` 判斷指針輸入精確度。
- 為觸控、滑鼠與混合輸入設備設計更合適的互動樣式。

## 使用情境

不是所有使用者都用同一種設備，也不是所有人都適合同一種視覺效果。

例如：

- 有人使用系統深色模式，希望網站也自動切換深色主題。
- 有人對動畫敏感，希望減少轉場與動畫。
- 手機使用觸控操作，不適合只依賴 hover 效果。
- 觸控輸入較不精準，按鈕需要更大的點擊區域。

這些情境可以透過使用者偏好與輸入能力媒體特性處理。

## 一句話理解

使用者偏好媒體特性關心「使用者想要什麼體驗」，輸入能力媒體特性關心「目前設備適合怎麼互動」。

## 深色與淺色偏好：prefers-color-scheme

`prefers-color-scheme` 用於檢測使用者的色彩模式偏好。

常見值如下：

| 值 | 說明 |
| --- | --- |
| `light` | 使用者偏好淺色主題 |
| `dark` | 使用者偏好深色主題 |
| `no-preference` | 系統沒有提供明確偏好 |

範例：

```css
@media (prefers-color-scheme: light) {
  body {
    background-color: white;
    color: black;
  }
}

@media (prefers-color-scheme: dark) {
  body {
    background-color: black;
    color: white;
  }
}
```

也可以在 JavaScript 中檢查：

```js
window.matchMedia("(prefers-color-scheme: dark)").matches;
```

## 減少動畫偏好：prefers-reduced-motion

`prefers-reduced-motion` 用於檢測使用者是否偏好減少動態效果。

| 值 | 說明 |
| --- | --- |
| `reduce` | 使用者偏好減少動態效果 |
| `no-preference` | 使用者沒有特別偏好 |

範例：

```css
@media (prefers-reduced-motion: reduce) {
  .animated-element {
    animation: none;
    transition: none;
  }
}

@media (prefers-reduced-motion: no-preference) {
  .animated-element {
    animation: slide-in 1s ease-in-out;
    transition: opacity 0.5s ease-in-out;
  }
}
```

這個特性對無障礙很重要。它讓網站能尊重使用者的系統設定，避免不必要的動畫造成不適。

## 懸停能力：hover 與 any-hover

`hover` 用來檢測主要輸入方式是否支援懸停操作。

| 值 | 說明 |
| --- | --- |
| `hover: hover` | 主要輸入方式支援 hover，例如滑鼠 |
| `hover: none` | 主要輸入方式不支援 hover，例如觸控 |

範例：

```css
@media (hover: hover) {
  .button {
    background-color: blue;
  }

  .button:hover {
    background-color: darkblue;
  }
}

@media (hover: none) {
  .button {
    background-color: gray;
  }
}
```

`any-hover` 則檢測設備上的任何一種輸入方式是否支援 hover。

```css
@media (any-hover: hover) {
  .button:hover {
    background-color: darkblue;
  }
}

@media (any-hover: none) {
  .button {
    background-color: gray;
  }
}
```

如果設備同時有觸控螢幕和滑鼠，`hover` 與 `any-hover` 的結果可能不同。

## 指針精確度：pointer 與 any-pointer

`pointer` 用來檢測主要輸入方式是否支援指針輸入，以及指針精確度。

| 值 | 說明 |
| --- | --- |
| `pointer: none` | 不支援指針輸入 |
| `pointer: coarse` | 指針精確度較低，例如手指觸控 |
| `pointer: fine` | 指針精確度較高，例如滑鼠或觸控板 |

範例：

```css
@media (pointer: fine) {
  .button {
    padding: 10px 20px;
  }
}

@media (pointer: coarse) {
  .button {
    padding: 20px 40px;
  }
}

@media (pointer: none) {
  .button {
    padding: 20px;
  }
}
```

`any-pointer` 檢測任何一種輸入設備的指針能力。

```css
@media (any-pointer: fine) {
  .button {
    padding: 10px 20px;
  }
}

@media (any-pointer: coarse) {
  .button {
    padding: 20px 40px;
  }
}
```

## 實務範例：觸控裝置放大點擊區域

```css
.button {
  border: 0;
  border-radius: 4px;
  padding: 10px 18px;
}

@media (pointer: coarse) {
  .button {
    padding: 16px 28px;
  }
}

@media (hover: hover) {
  .button:hover {
    filter: brightness(0.92);
  }
}
```

這段 CSS 做了兩件事：

- 指針較粗略時，把按鈕點擊區域放大。
- 支援 hover 時，才提供 hover 視覺反饋。

這比假設所有設備都有滑鼠更可靠。

## 常見錯誤

### 錯誤一：只靠 hover 傳達重要資訊

觸控設備可能沒有 hover。如果重要資訊只在 `:hover` 時出現，觸控使用者可能看不到。

重要操作應該有可點擊、可聚焦或常駐的替代方式。

### 錯誤二：忽略 prefers-reduced-motion

大量動畫、快速滑動、視差效果可能讓部分使用者不舒服。對這類效果應提供減少動畫版本。

### 錯誤三：把 pointer: coarse 當成手機偵測

`pointer: coarse` 表示主要指針精確度較低，不等於設備一定是手機。它描述的是互動能力，不是設備分類。

## 實務判斷

- 主題顏色：使用 `prefers-color-scheme` 尊重系統偏好。
- 動畫：使用 `prefers-reduced-motion` 提供減少動畫版本。
- hover 效果：先判斷 `hover` 或 `any-hover`。
- 點擊區域：用 `pointer: coarse` 放大互動目標。
- 混合設備：理解 `hover` / `any-hover`、`pointer` / `any-pointer` 的差異。
