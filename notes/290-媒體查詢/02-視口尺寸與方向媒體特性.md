# 視口尺寸與方向媒體特性

## 學習目標

讀完這篇筆記，你應該能夠：

- 使用 `width`、`min-width`、`max-width` 根據視口寬度調整樣式。
- 使用 `height`、`min-height`、`max-height` 根據視口高度調整樣式。
- 使用 `orientation` 判斷縱向與橫向顯示。
- 分辨視口尺寸與設備物理尺寸。
- 避免只依賴方向判斷而忽略實際可用空間。

## 使用情境

響應式設計最常面對的問題是：同一份內容在不同視口大小中，需要呈現不同版面。

例如：

- 手機寬度較窄時，卡片改成單欄。
- 桌面寬度足夠時，內容改成多欄。
- 高度不足時，減少垂直間距或縮小某些區塊。
- 平板橫向時，導覽或側欄可以使用更寬的布局。

這些情境大多會用到視口寬度、高度與方向媒體特性。

## 一句話理解

`width`、`height` 與 `orientation` 類媒體特性判斷的是目前可用顯示區域，也就是視口，而不是單純判斷設備型號。

## 視口寬度：width、min-width、max-width

寬度媒體特性是響應式設計中最常用的一組條件。

| 特性 | 觸發條件 |
| --- | --- |
| `width` | 視口寬度剛好等於指定值 |
| `min-width` | 視口寬度大於等於指定值 |
| `max-width` | 視口寬度小於等於指定值 |

範例：

```css
@media (min-width: 1200px) {
  body {
    font-size: 18px;
    margin: 0 auto;
    max-width: 1100px;
  }
}

@media (max-width: 768px) {
  body {
    font-size: 14px;
    padding: 10px;
  }
}

@media (width: 1024px) {
  body {
    background-color: lightblue;
  }
}
```

這三段分別表示：

- 大於等於 `1200px` 時，套用較大的桌面版樣式。
- 小於等於 `768px` 時，套用較適合手機的樣式。
- 剛好等於 `1024px` 時，才套用指定背景色。

實務上，精確的 `width: 1024px` 用得較少；更常用的是範圍型的 `min-width` 和 `max-width`。

## 視口高度：height、min-height、max-height

高度媒體特性用來根據視口高度套用樣式。

| 特性 | 觸發條件 |
| --- | --- |
| `height` | 視口高度剛好等於指定值 |
| `min-height` | 視口高度大於等於指定值 |
| `max-height` | 視口高度小於等於指定值 |

範例：

```css
@media (min-height: 800px) {
  body {
    background-color: lightblue;
  }
}

@media (max-height: 500px) {
  body {
    font-size: 12px;
  }
}

@media (height: 600px) {
  body {
    background-color: lightgreen;
  }
}
```

高度條件常用在：

- 全屏應用。
- 需要依垂直空間調整間距的版面。
- 橫向手機中高度變短時的布局修正。

## 方向：orientation

`orientation` 用來判斷視口方向。

| 值 | 說明 |
| --- | --- |
| `portrait` | 縱向模式，通常視口高度大於或等於寬度 |
| `landscape` | 橫向模式，通常視口寬度大於高度 |

範例：

```css
@media (orientation: portrait) {
  body {
    background-color: lightblue;
  }

  .container {
    padding: 20px;
  }
}

@media (orientation: landscape) {
  body {
    background-color: lightgreen;
  }

  .container {
    padding: 40px;
  }
}
```

這段 CSS 會在縱向與橫向時使用不同背景色和間距。

## 範例：寬度與方向一起判斷

只用方向有時不夠精準。手機橫向與桌面橫向都可能是 `landscape`，但可用寬度差很多。

可以把方向和寬度條件一起使用：

```css
@media (orientation: landscape) and (max-width: 900px) {
  .toolbar {
    position: sticky;
    top: 0;
  }
}
```

這段表示：只有在橫向，而且視口寬度不超過 `900px` 時，才套用這個工具列樣式。它比單純使用 `orientation: landscape` 更精準。

## 視口尺寸不是設備物理尺寸

`width` 和 `height` 查詢的是視口，不是設備螢幕的物理大小。

在桌面瀏覽器中，使用者縮放或調整視窗寬度時，媒體查詢會跟著重新匹配。在手機上，瀏覽器 UI、縮放狀態和 viewport 設定也會影響可用視口。

因此實務上應該把 media query 當成「目前版面可用空間」的判斷，而不是「這台設備是不是手機」的判斷。

## 常見錯誤

### 錯誤一：把視口寬度當成設備寬度

`@media (max-width: 768px)` 表示視口最多 `768px`，不等於「一定是手機」。

桌面瀏覽器被縮到很窄時，也可能符合這個條件。

### 錯誤二：只用 orientation 判斷版面

`orientation: landscape` 只代表寬度大於高度，不代表空間一定很寬。手機橫向仍可能需要緊湊版面。

通常應搭配 `min-width` 或 `max-width` 一起判斷。

### 錯誤三：大量使用精確 width

```css
@media (width: 1024px) {
  /* ... */
}
```

精確寬度只有在剛好等於該值時才生效，通常不如範圍查詢實用。

## 實務判斷

- 主要斷點：優先使用 `min-width` 或 `max-width`。
- 高度不足：使用 `max-height` 修正垂直空間問題。
- 橫向/縱向差異：使用 `orientation`，但最好搭配寬度條件。
- 不要把 media query 當成設備型號偵測；它判斷的是目前環境條件。
