# 媒體查詢關鍵字與條件組合

## 學習目標

讀完這篇筆記，你應該能夠：

- 使用 `and` 串接多個媒體條件。
- 使用逗號分隔多個媒體查詢，達成 OR 效果。
- 理解 `not` 可以對整個媒體查詢取反，也可以在現代條件語法中否定括號內條件。
- 知道 `only` 的歷史用途，以及現代開發中通常不必特別加上。
- 避免把關鍵字用錯範圍或誤解相容性。

## 使用情境

單一條件有時不夠描述你想要的環境。

例如：

- 只想在螢幕設備，且寬度介於 `700px` 到 `800px` 時套用樣式。
- 想在小於 `700px` 或大於 `800px` 時套用同一段樣式。
- 想排除某一類媒體類型。

這時就需要用媒體查詢關鍵字把條件組合起來。

## 一句話理解

媒體查詢關鍵字是用來組合、否定或限制媒體條件的語法工具，讓 `@media` 可以表達更複雜的判斷。

## and：且條件

`and` 用來把多個條件串在一起，所有條件都成立時，整個媒體查詢才成立。

```css
@media screen and (min-width: 700px) and (max-width: 800px) {
  h1 {
    background-color: orange;
  }
}
```

這段表示：

- 媒體類型是 `screen`。
- 視口寬度大於等於 `700px`。
- 視口寬度小於等於 `800px`。

三個條件都符合時，`h1` 背景才會變成橘色。

## OR：逗號分隔多個查詢

將多個媒體查詢用逗號分隔，可以表達「任一條件成立即可」。

```css
@media (max-width: 700px), (min-width: 800px) {
  h1 {
    background-color: orange;
  }
}
```

這段表示：

- 視口寬度小於等於 `700px` 時套用。
- 或視口寬度大於等於 `800px` 時套用。

逗號分隔是相容性很好的 OR 寫法。

Media Queries Level 4 也支援 `or` 組合媒體條件：

```css
@media (max-width: 700px) or (min-width: 800px) {
  h1 {
    background-color: orange;
  }
}
```

實務上仍要依專案目標瀏覽器選擇寫法。如果需要更保守的相容性，逗號分隔通常更穩。

## not：否定查詢或條件

`not` 可以用來否定媒體查詢或媒體條件。像 `@media not screen` 這類寫法，會對整個媒體查詢結果取反。

```css
@media not screen {
  h1 {
    background-color: orange;
  }
}
```

這表示：當媒體類型不是 `screen` 時，套用這段樣式。

在 Media Queries Level 4 之後，也可以用 `not (條件)` 否定括號內的媒體條件：

```css
@media not (hover) {
  .button {
    text-decoration: underline;
  }
}
```

寫複雜查詢時，要注意 `not` 的作用範圍，必要時用括號讓條件更清楚。

## only：舊瀏覽器相容性用途

`only` 不會改變媒體查詢在現代瀏覽器中的匹配結果。

```css
@media only screen and (width: 800px) {
  h1 {
    background-color: orange;
  }
}
```

它最初主要是為了讓非常舊、只理解媒體類型但不理解媒體特性的瀏覽器忽略該段樣式。

在現代開發中，通常不需要刻意加上 `only`。

## 範例：三種條件組合比較

```css
/* 且：寬度必須同時大於等於 700px 且小於等於 800px */
@media (min-width: 700px) and (max-width: 800px) {
  h1 {
    background-color: orange;
  }
}

/* 或：小於等於 700px 或大於等於 800px */
@media (max-width: 700px), (min-width: 800px) {
  h1 {
    color: red;
  }
}

/* 非：不是螢幕媒體時 */
@media not screen {
  h1 {
    border: 2px solid black;
  }
}
```

閱讀媒體查詢時，可以先把它翻譯成中文：

- `and`：而且。
- 逗號：或者。
- `not`：不是這個查詢或括號內條件。

## 常見錯誤

### 錯誤一：以為逗號是 and

逗號不是「且」，而是「或」。

```css
@media (max-width: 700px), (min-width: 800px) {
  /* 任一條件成立即套用 */
}
```

如果你需要兩個條件同時成立，應使用 `and`。

### 錯誤二：沒有看清楚 not 的作用範圍

`@media not screen` 會否定整個媒體查詢；`@media not (hover)` 則是否定括號內的條件。寫複雜查詢時要特別小心可讀性。

### 錯誤三：現代專案中機械式加 only

`only` 的主要價值在非常舊的瀏覽器相容性。現代專案通常不必把它當成必寫關鍵字。

## 實務判斷

- 多個條件都要成立：使用 `and`。
- 任一條件成立即可：使用逗號分隔多個查詢。
- 需要否定整個查詢或括號內條件：使用 `not`，並用括號確認作用範圍。
- 沒有舊瀏覽器相容需求：通常不用特別加 `only`。
