# 響應式原理與布局容器

## 學習目標

讀完這篇筆記後，你應該能夠：

- 說明響應式開發要解決的版面適配問題。
- 理解媒體查詢在響應式中的角色。
- 使用布局容器控制不同螢幕寬度下的內容寬度。
- 看懂常見響應式斷點與容器寬度設定。
- 避免只改子元素、忽略父容器與斷點順序等常見錯誤。

## 問題情境

同一個網頁可能會被手機、平板、筆電和大螢幕桌機開啟。如果所有裝置都使用同一個固定寬度，手機上可能會出現橫向捲動，大螢幕上又可能顯得內容太散。

響應式開發的目標，就是讓同一份頁面能根據螢幕寬度調整布局與樣式，使不同設備都能得到合適的閱讀與操作體驗。

## 一句話理解

響應式開發是透過媒體查詢偵測螢幕寬度，並在不同寬度範圍套用不同布局規則。

也就是說，CSS 不是只寫一套固定版面，而是根據條件切換樣式。

## 響應式與媒體查詢

響應式開發通常會使用媒體查詢針對不同寬度的設備設定布局和樣式，達成適配不同設備的目的。

![響應式開發布局的裝置尺寸區間表](../../origin/300-響應式/assets/images/responsive-design-img-001-c44853.png)

常見的尺寸分段可以這樣理解：

| 裝置類型 | 寬度條件 | 常見容器寬度 |
| --- | --- | --- |
| 超小螢幕（手機） | 小於 `768px` | `100%` |
| 小螢幕（平板） | 大於等於 `768px` | `750px` |
| 中等螢幕（桌面顯示器） | 大於等於 `992px` | `970px` |
| 大螢幕（大桌面顯示器） | 大於等於 `1200px` | `1170px` |

這些數值不是 CSS 語法規定，而是一套常見的響應式設計分段。實際專案可依設計稿與產品需求調整。

## 為什麼需要布局容器

響應式頁面通常需要一個父級布局容器，例如 `.container`。它負責控制主要內容區在不同螢幕下的寬度。

子元素的排列與大小，會依附在這個容器之內調整。這樣做有兩個好處：

- 小螢幕時，容器可變成 `100%`，充分使用可視寬度。
- 大螢幕時，容器可固定在合理寬度，避免內容拉得太寬。

換句話說，容器先決定「內容區有多寬」，子元素再根據這個寬度做排列。

## 基本範例

```html
<!DOCTYPE html>
<html lang="en">

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
  <style>
    .container {
      height: 150px;
      background-color: pink;
      margin: 0 auto;
    }

    @media screen and (max-width: 767px) {
      .container {
        width: 100%;
      }
    }

    @media screen and (min-width: 768px) {
      .container {
        width: 750px;
      }
    }

    @media screen and (min-width: 992px) {
      .container {
        width: 970px;
      }
    }

    @media screen and (min-width: 1200px) {
      .container {
        width: 1170px;
      }
    }
  </style>
</head>

<body>
  <div class="container"></div>
</body>

</html>
```

## 範例拆解

### viewport 設定

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

這行讓行動裝置以設備實際寬度作為網頁寬度基準。若缺少它，手機瀏覽器可能用較大的虛擬寬度渲染頁面，媒體查詢的結果會不符合預期。

### 容器基礎樣式

```css
.container {
  height: 150px;
  background-color: pink;
  margin: 0 auto;
}
```

`.container` 是主要布局容器。`margin: 0 auto;` 讓固定寬度容器在父層中水平置中。

### 手機寬度

```css
@media screen and (max-width: 767px) {
  .container {
    width: 100%;
  }
}
```

當螢幕寬度小於或等於 `767px` 時，容器寬度設為 `100%`，避免手機上出現固定寬度造成的橫向捲動。

### 平板與桌面寬度

```css
@media screen and (min-width: 768px) {
  .container {
    width: 750px;
  }
}

@media screen and (min-width: 992px) {
  .container {
    width: 970px;
  }
}

@media screen and (min-width: 1200px) {
  .container {
    width: 1170px;
  }
}
```

這幾段媒體查詢會隨著螢幕變大逐步調整容器寬度。因為後面的條件也會在更大寬度下成立，所以順序要從小到大排列，讓較大的斷點覆蓋前面的寬度設定。

## 常見錯誤

### 只調整子元素，沒有設計父容器

如果只改子元素寬度，卻沒有控制主要內容容器，版面可能在大螢幕上拉得太寬，或在小螢幕上難以維持一致的排列基準。

響應式布局通常先決定 `.container` 的寬度，再安排裡面的欄位、卡片或導覽項目。

### 忘記固定寬度容器需要置中

```css
.container {
  width: 970px;
}
```

如果沒有 `margin: 0 auto;`，固定寬度容器會靠左。通常應搭配：

```css
.container {
  width: 970px;
  margin: 0 auto;
}
```

### 斷點順序混亂

如果同時使用多個 `min-width` 條件，通常會由小到大排列。這樣當螢幕越來越寬時，後面的規則自然覆蓋前面的規則。

若順序混亂，可能導致大螢幕仍套用較小斷點的容器寬度。

### 手機仍使用固定寬度

手機寬度小於 `768px` 時，容器應通常使用 `width: 100%;`。如果仍套用 `750px` 這種固定寬度，會造成橫向捲動。

## 實務判斷準則

- 響應式布局先規劃斷點，再規劃每個斷點下的容器寬度。
- 小螢幕通常讓容器滿版，大螢幕則限制最大內容寬度。
- 使用 `min-width` 寫法時，媒體查詢通常由小到大排列。
- 固定寬度容器通常搭配 `margin: 0 auto;`。
- `viewport` meta 是行動端響應式頁面的基本設定。

## 重點整理

- 響應式開發透過媒體查詢讓不同螢幕寬度套用不同樣式。
- 布局容器是響應式頁面的主要寬度基準。
- 常見容器寬度：手機 `100%`、平板 `750px`、桌面 `970px`、大桌面 `1170px`。
- 先控制父級容器，再調整子元素排列，版面會更穩定。
- 斷點順序與條件設計會直接影響最終樣式。
