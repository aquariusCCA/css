# 響應式導覽列案例

## 學習目標

讀完這篇筆記後，你應該能夠：

- 拆解一個響應式導覽列的桌面與手機需求。
- 使用布局容器控制導覽列的整體寬度。
- 使用固定寬度與百分比寬度切換不同螢幕下的項目排列。
- 看懂 `@media screen and (max-width: 767px)` 如何改變導覽項目寬度。
- 避免固定寬度沒有在小螢幕改寫、百分比計算錯誤等常見問題。

## 目標效果

這個案例要做一個簡單導覽列：

- 平板或桌面寬度下，容器寬度是 `750px`，8 個導覽項目排成一行。
- 手機寬度下，容器寬度改為 `100%`，導覽項目每個寬 `33.33%`，形成一行 3 個的排列。

## 需求拆解

這個案例可以拆成三件事：

1. 外層 `.container` 控制整體內容寬度。
2. `li` 控制每個導覽項目的寬度與排列。
3. 媒體查詢在小螢幕時改寫 `.container` 和 `li` 的寬度。

核心不是「換一份 HTML」，而是同一份結構在不同寬度下套用不同 CSS。

## HTML 結構

```html
<div class="container">
  <ul>
    <li>導航欄</li>
    <li>導航欄</li>
    <li>導航欄</li>
    <li>導航欄</li>
    <li>導航欄</li>
    <li>導航欄</li>
    <li>導航欄</li>
    <li>導航欄</li>
  </ul>
</div>
```

`.container` 是響應式布局容器，`ul` 裡有 8 個導覽項目。

實作成完整頁面時，`head` 裡也要保留行動端響應式常用的 viewport 設定：

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

這樣手機瀏覽器才會以裝置實際寬度作為版面寬度基準，下面的 `max-width: 767px` 媒體查詢才會符合預期。

## CSS 完整範例

```css
* {
  margin: 0;
  padding: 0;
}

.container {
  width: 750px;
  margin: 0 auto;
}

.container ul li {
  float: left;
  width: 93.75px;
  height: 30px;
  background-color: green;
}

@media screen and (max-width: 767px) {
  .container {
    width: 100%;
  }

  .container ul li {
    width: 33.33%;
  }
}
```

## 桌面版拆解

```css
.container {
  width: 750px;
  margin: 0 auto;
}
```

桌面或平板寬度下，容器固定為 `750px`，並透過 `margin: 0 auto;` 水平置中。

```css
.container ul li {
  float: left;
  width: 93.75px;
  height: 30px;
  background-color: green;
}
```

每個導覽項目寬 `93.75px`。因為容器總寬是 `750px`，8 個項目平均分配：

```text
750px / 8 = 93.75px
```

所以 8 個導覽項目剛好可以排成一行。

## 手機版拆解

```css
@media screen and (max-width: 767px) {
  .container {
    width: 100%;
  }

  .container ul li {
    width: 33.33%;
  }
}
```

當螢幕寬度小於或等於 `767px` 時：

- `.container` 從固定 `750px` 改成 `100%`，避免手機橫向捲動。
- `li` 從固定 `93.75px` 改成 `33.33%`，讓每個項目約占一列的三分之一。

因此手機上會形成一行約 3 個導覽項目的排列。

## 為什麼手機用百分比

手機螢幕寬度不固定，可能是 320px、375px、390px 或其他寬度。如果仍使用固定寬度，導覽列很容易超出螢幕。

百分比寬度會跟著父容器變化：

```css
.container ul li {
  width: 33.33%;
}
```

這表示每個項目占父容器寬度的三分之一，因此不同手機寬度下仍能維持三欄排列。

## 常見錯誤

### 小螢幕沒有改掉固定容器寬度

如果手機版仍保留：

```css
.container {
  width: 750px;
}
```

手機螢幕可能放不下，造成橫向捲動。應在小螢幕媒體查詢中改成：

```css
.container {
  width: 100%;
}
```

### 百分比寬度和預期欄數不一致

如果想一行 3 個，寬度應接近 `33.33%`。如果寫成 `25%`，就會變成一行 4 個；如果寫成 `50%`，就會變成一行 2 個。

寫百分比前，先確認一行要放幾個項目。

### 忘記 float 對後續內容的影響

這個案例使用 `float: left;` 排列導覽項目。浮動元素可能影響父層高度與後續內容布局。若後面還有其他內容，實務上要注意清除浮動，或改用 Flexbox 等更現代的排列方式。

### 斷點條件和需求不一致

這裡使用：

```css
@media screen and (max-width: 767px)
```

代表只在 `767px` 以下套用手機版。如果需求是 `768px` 以下也算手機，就要調整條件。斷點應和設計需求一致。

## 實務判斷準則

- 先確認桌面與手機各自的一行項目數。
- 固定容器在小螢幕通常要改成 `width: 100%`。
- 子項目在小螢幕常用百分比寬度，讓欄數跟著父容器自動縮放。
- 使用 float 時要留意清除浮動；新專案也可考慮 Flexbox。
- 媒體查詢條件要和設計稿斷點一致。

## 重點整理

- 響應式導覽列的核心是同一份 HTML 在不同螢幕寬度下切換 CSS。
- `.container` 控制整體寬度，`li` 控制每個導覽項目的排列寬度。
- 桌面版可用固定寬度精準分配項目。
- 手機版應讓容器滿版，並用百分比控制欄數。
- `33.33%` 表示一行約 3 個項目。
