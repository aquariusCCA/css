# CSS3 多背景圖：在同一個元素上疊多層背景

## 學習目標

讀完這篇筆記，你應該能夠：

- 理解 CSS3 多背景圖的用途。
- 使用逗號分隔多組背景設定。
- 看懂每一層背景各自的圖片、平鋪與位置。
- 判斷什麼情境適合使用多背景圖。

## 問題情境

有時一個元素需要多個裝飾背景：例如主背景鋪在底層，右上角有一張裝飾圖，左下角再放另一個小圖示。這些圖都只是裝飾，不一定值得為每張圖新增一個 HTML 元素。

CSS3 允許同一個元素設定多個背景圖片，讓這類裝飾可以集中在 CSS 中處理。

## 一句話理解

多背景圖就是在同一個 `background` 或 `background-image` 中，用逗號分隔多層背景，每一層都可以有自己的位置與平鋪方式。

多背景圖的順序也會影響堆疊：先寫的背景會畫在較上層，越後面寫的背景越接近底層。因此如果主背景要鋪在底層，通常要把主背景寫在最後一層。

## 基本寫法

範例：

```css
.box {
  width: 2500px;
  height: 2500px;
  border: 1px solid black;
  background: url(./images/ldh.jpg) no-repeat right top,
              url(./images/logo.png) no-repeat left bottom,
              url(./images/bg.jpg) no-repeat;
}
```

```html
<div class="box"></div>
```

這段設定了三層背景：

1. `url(./images/ldh.jpg) no-repeat right top`
2. `url(./images/logo.png) no-repeat left bottom`
3. `url(./images/bg.jpg) no-repeat`

每一層用逗號分隔，每一層都可以指定自己的圖片、是否平鋪、位置等背景設定。

## 範例拆解

第一層：

```css
url(./images/ldh.jpg) no-repeat right top
```

這張背景圖不平鋪，放在右上角。因為它寫在第一層，所以會畫在比較上面。

第二層：

```css
url(./images/logo.png) no-repeat left bottom
```

這張背景圖不平鋪，放在左下角。

第三層：

```css
url(./images/bg.jpg) no-repeat
```

這張背景圖不平鋪，沒有額外指定位置，因此會使用背景位置的預設行為。它寫在最後一層，所以會作為比較底層的主背景。

## 使用時機

多背景圖適合處理裝飾性視覺：

- 一個區塊有多個角落裝飾圖。
- 頁面大背景上還要疊小圖案。
- 不想為純裝飾圖片增加多個空的 HTML 元素。
- 多個背景都屬於同一個元素的視覺樣式。

如果圖片本身是重要內容，例如商品照片、文章插圖、使用者頭像，仍應優先使用 `img` 或適合的 HTML 結構。

## 常見錯誤

### 忘記用逗號分隔每一層背景

多背景圖的每一層需要用逗號分隔。如果少了逗號，瀏覽器會把後面的值當成同一層背景的設定，造成語法不符合預期。

### 以為每層背景都共享同一個位置

每一層背景都可以有自己的位置與平鋪方式。不要只設定第一層，卻期待其他層自動跟著同樣位置。

### 用多背景圖承載重要內容

多背景圖仍然是 CSS 背景，偏向裝飾用途。重要圖片不應只放在 CSS 背景中，否則語意、可替代文字和內容維護都會變差。

## 實務判斷準則

- 裝飾圖可以考慮多背景圖。
- 重要內容圖片仍使用 `img`。
- 每一層背景都要確認是否需要 `no-repeat`、位置和大小。
- 背景層數太多時，CSS 可讀性會下降，應評估是否拆成更清楚的元素結構。

## 重點整理

- CSS3 允許同一個元素設定多個背景圖片。
- 多背景圖用逗號分隔每一層背景。
- 每一層可以有自己的圖片、平鋪與位置設定。
- 多背景圖適合裝飾，不適合承載重要內容圖片。
