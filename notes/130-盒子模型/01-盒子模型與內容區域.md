# 盒子模型與內容區域

## 學習目標

讀完這篇筆記，你應該能夠：

- 用「盒子」的角度理解網頁布局。
- 說出 CSS 盒子模型的四個組成部分。
- 理解 `width` 和 `height` 在預設盒模型中控制的是內容區域。
- 分辨「內容區大小」和「盒子最終大小」不是同一件事。

## 問題情境

在網頁中排版時，我們常會想：

- 這個區塊要多寬？
- 這個區塊和旁邊元素要距離多少？
- 內容和邊框之間要不要留空間？
- 加上邊框後，整個元素會不會變大？

這些問題都可以用盒子模型來理解。瀏覽器在渲染頁面時，會把頁面中的元素看成一個個矩形區域；CSS 布局的核心，也就是設定這些矩形盒子的尺寸、間距和位置。

## 一句話理解

盒子模型是 CSS 用來描述元素尺寸與間距的模型：每個元素都可以看成由 `content`、`padding`、`border`、`margin` 組成的矩形盒子。

![CSS 盒子模型由 content padding border margin 組成的示意圖](../../origin/130-盒子模型/assets/images/composition-of-box-model-img-001-23285e.png)

## 網頁布局的盒子視角

網頁布局通常可以拆成三個步驟：

1. 準備好頁面中的元素。
2. 使用 CSS 設定這些元素的大小、樣式和位置。
3. 把文字、圖片或其他內容放進元素裡。

從盒子模型來看，頁面中的標籤不只是文字或圖片，它們也是可以被 CSS 控制的盒子。

例如：

```html
<div>文字</div>
```

這個 `<div>` 可以有寬高、背景、內邊距、邊框和外邊距。當你開始用這些屬性控制它時，就已經在操作盒子模型。

## 盒子模型的四個部分

一個 CSS 盒子由四個主要部分組成：

| 區域 | 說明 |
| --- | --- |
| `content` | 內容區，放文字、圖片等實際內容 |
| `padding` | 內邊距，內容和邊框之間的空間 |
| `border` | 邊框，包住 padding 和 content 的線條 |
| `margin` | 外邊距，盒子和其他盒子之間的距離 |

可以把它想像成一個包裹：

- `content` 是盒子裡真正裝的東西。
- `padding` 是東西和盒壁之間的緩衝。
- `border` 是盒子的外框。
- `margin` 是這個盒子和其他盒子的距離。

## `width` 和 `height` 控制什麼

在預設盒模型中，`width` 和 `height` 設定的是內容區域，也就是 `content` 的大小。

![盒子內容區域由 width 和 height 控制的示意圖](../../origin/130-盒子模型/assets/images/box-content-img-001-b074cc.png)

例如：

```css
div {
  width: 200px;
  height: 200px;
  background-color: pink;
}
```

```html
<div>文字</div>
```

這裡的 `width: 200px` 和 `height: 200px` 指的是內容區域的寬高。如果之後再加上 `padding` 或 `border`，整個盒子的實際大小可能會變得比 `200px * 200px` 更大。

## 內容區大小不等於盒子最終大小

初學盒模型時，最容易混淆的是：

> `width` 看起來像是盒子的寬度，但在預設情況下，它其實只是內容區寬度。

例如：

```css
.box {
  width: 200px;
  padding: 20px;
  border: 10px solid red;
}
```

在預設 `content-box` 計算方式下，這個盒子的實際寬度會是：

```text
內容寬度 200px
+ 左右 padding 40px
+ 左右 border 20px
= 260px
```

所以學盒子模型時，要一直分清楚兩件事：

- CSS 寫的 `width` 是哪個區域的寬度？
- 瀏覽器最後畫出來的盒子總寬是多少？

後續學到 `box-sizing: border-box` 時，這個計算方式會改變。

## 對前端開發的意義

盒子模型是 CSS 布局的底層心智模型。

當你看到畫面不對時，通常可以先問：

- 是內容區太大嗎？
- 是 padding 把盒子撐大了嗎？
- 是 border 增加了實際尺寸嗎？
- 是 margin 讓盒子之間距離變了嗎？

能用盒子模型拆解問題，就比較不會在 CSS 中盲目試值。

## 常見錯誤

### 誤解：每個元素的 `width` 都代表畫面上的總寬度

在預設盒模型中，`width` 只代表 content 寬度。padding 和 border 會另外加到盒子外側，因此總寬可能比 `width` 大。

### 誤解：margin 是盒子裡面的空白

`margin` 是盒子外部的距離，不屬於盒子內部內容區，也不會被背景色填滿。內容和邊框之間的空白才是 `padding`。

### 誤解：盒子模型只影響 div

頁面中的多數元素都可以用盒子模型理解，不只是 `<div>`。標題、段落、圖片、清單、按鈕都會有自己的盒模型行為，只是不同元素的預設顯示模式可能不同。

## 重點整理

- CSS 布局可以理解成擺放一個個矩形盒子。
- 盒子模型由 `content`、`padding`、`border`、`margin` 組成。
- 在預設盒模型中，`width` 和 `height` 控制的是內容區域。
- padding 和 border 可能讓盒子實際大小比 `width` / `height` 更大。
- 學會用盒子模型拆解尺寸和間距，是理解 CSS 布局的基礎。

4. 如果一個元素有 `width: 200px`、左右 `padding` 各 `20px`、左右 `border` 各 `10px`，預設盒模型下實際寬度是多少？
