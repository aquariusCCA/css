# 盒模型實務注意：預設間距與行內元素限制

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明為什麼布局前常會清除預設內外邊距。
- 比較使用全局選擇器和列舉元素清除預設間距的差異。
- 理解行內元素的垂直 margin 和 padding 限制。
- 判斷什麼時候應改用 `line-height`、`inline-block` 或 `block`。

## 問題情境

盒模型學到最後，常會遇到兩類實務問題：

1. 明明沒寫間距，頁面元素卻自帶空白。
2. 對 `span` 這類行內元素設定上下 margin 或 padding，卻沒有像 block 元素那樣把上下內容推開。

這兩件事都和盒模型有關，但它們不是單純背語法就能解決，需要理解瀏覽器預設樣式與元素顯示模式。

## 清除預設內外邊距

很多網頁元素都帶有預設的內外邊距，而且不同瀏覽器的預設值可能不一致。

因此在布局前，常會先清除常見元素的 `margin` 和 `padding`，讓頁面從比較一致的起點開始。

最簡單的寫法是：

```css
* {
  margin: 0;
  padding: 0;
}
```

這種寫法很直接，所有元素都會被歸零。

## 列舉元素的做法

有些網站或團隊會列出需要重置的元素，而不是用 `*` 全部選中。

例如：

```css
blockquote,
body,
button,
dd,
dl,
dt,
fieldset,
form,
h1,
h2,
h3,
h4,
h5,
h6,
hr,
input,
legend,
li,
ol,
p,
pre,
td,
textarea,
th,
ul {
  margin: 0;
  padding: 0;
}
```

這樣寫比較長，但更明確：你可以看出哪些元素被重置，也可以避免無差別影響所有元素。

## 全局選擇器與列舉元素對照

| 做法 | 優點 | 注意事項 |
| --- | --- | --- |
| `* { margin: 0; padding: 0; }` | 簡單、快速、適合練習 | 選中所有元素，較粗略 |
| 列舉元素 | 更明確、可控 | 規則較長，需要維護清單 |

初學階段可以先理解全局寫法；正式專案要依團隊規範決定採用 reset、normalize 或自訂基礎樣式。

## 行內元素的垂直間距限制

行內元素，例如 `span`，和塊級元素的盒模型表現不同。

如果你想透過 `margin` 或 `padding` 改變行內標籤的垂直位置，需要注意：

- `margin-top` 和 `margin-bottom` 通常不影響行盒高度，也不會把上下內容推開。
- `padding-top` 和 `padding-bottom` 會套用到元素本身，但通常不會像 block 元素一樣推開上下行。

例如：

```css
span {
  margin: 100px;
}
```

```html
<span>span</span>
<span>span</span>
```

你可能期待 `span` 往下移動或撐開上下距離，但行內元素的垂直 margin 不會像 block 元素那樣影響布局。

## 解法一：使用 line-height

如果目標是控制單行文字的垂直空間，常用 `line-height`。

```css
.tag {
  line-height: 32px;
}
```

`line-height` 會影響行盒高度，常用於讓文字在固定高度的按鈕或標籤中垂直居中。

## 解法二：改成 inline-block

如果你希望元素仍然像文字一樣同行排列，但又能設定寬高和上下 padding，可以使用 `inline-block`。

```css
.tag {
  display: inline-block;
  padding: 8px 12px;
  margin-top: 8px;
}
```

`inline-block` 保留同行排列能力，但盒模型行為比純 inline 更接近 block。

## 解法三：改成 block

如果元素本來就應該獨占一行，直接改成 block 更合理：

```css
.label {
  display: block;
  margin-top: 16px;
  padding: 8px;
}
```

選擇 `inline-block` 還是 `block`，取決於元素是否需要和其他內容在同一行。

## 常見錯誤

### 誤解：清除預設間距只是可有可無的習慣

如果不處理預設間距，標題、段落、列表、body 等元素的內外距可能會讓版面和設計稿不一致。清除或統一基礎樣式是布局前的重要準備。

### 誤解：對 `span` 設定 `margin-top` 就會把它往下推

純行內元素的垂直 margin 通常不會像 block 元素那樣影響上下布局。若需要可控盒模型行為，應考慮 `inline-block` 或 `block`。

### 誤解：行內元素完全不能有 padding

行內元素可以有 padding，但垂直 padding 不一定會推開上下行。這和 block 元素的尺寸表現不同，實務上要小心視覺重疊。

## 實務判斷

可以用這幾個問題判斷：

- 是瀏覽器預設間距造成布局不準嗎？先檢查 reset 或 normalize。
- 這個元素是否是 inline？若是，不要期待上下 margin 像 block 一樣工作。
- 想控制文字行高？優先看 `line-height`。
- 想讓元素可設定上下 padding 和尺寸，又要同行排列？使用 `inline-block`。
- 想讓元素獨占一行？使用 `block`。

## 重點整理

- 很多元素有預設 `margin` / `padding`，布局前常會先清除或統一。
- 全局選擇器寫法簡單，但較粗略；列舉元素更明確。
- 行內元素的垂直 margin 通常不會推開上下內容。
- 行內元素的垂直 padding 會套用，但不一定像 block 一樣影響行間布局。
- 控制行內元素垂直空間時，可使用 `line-height`、`inline-block` 或 `block`。
