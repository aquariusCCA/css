# relative：相對定位

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明 `position: relative` 是相對自己原本位置移動。
- 理解相對定位不會脫離文檔流，原位置仍會保留。
- 正確解讀 `top`、`right`、`bottom`、`left` 的偏移方向。
- 知道相對定位最常見的實務用途是作為絕對定位子元素的參照父級。

## 使用情境

當你想讓元素在原位置附近微調，又不希望後面的元素補上它原本的位置，可以使用相對定位。

更常見的實務情境是：父盒子本身仍要保留在普通版面中，但裡面的子元素要用絕對定位放到角落或中心，這時父盒子通常會設定 `position: relative`。

## 一句話理解

相對定位會讓元素相對「自己原來的位置」移動，但原本佔住的空間仍然保留。

## 基本語法

```css
.box {
  position: relative;
  top: 100px;
  left: 100px;
}
```

這表示 `.box` 的視覺位置會從原本位置往下、往右移動，但它在標準流中原本佔住的位置仍然存在。

## 邊偏移方向

相對定位的偏移可以這樣理解：

- `left: 100px`：元素左邊往右推 100px，因此元素向右移。
- `right: 100px`：元素右邊往左推 100px，因此元素向左移。
- `top: 100px`：元素上邊往下推 100px，因此元素向下移。
- `bottom: 100px`：元素下邊往上推 100px，因此元素向上移。

初學時建議同一個軸向只選一邊設定：水平軸用 `left` 或 `right`，垂直軸用 `top` 或 `bottom`。若同時設定同軸兩邊，實際結果會受 CSS 定位規則、元素尺寸與 `auto` 值影響，容易讓判斷變複雜。

## 範例拆解

```css
.box1 {
  position: relative;
  top: 100px;
  left: 100px;
  width: 200px;
  height: 200px;
  background-color: pink;
}

.box2 {
  width: 200px;
  height: 200px;
  background-color: deeppink;
}
```

```html
<body>
  <div class="box1"></div>
  <div class="box2"></div>
</body>
```

拆解：

- `.box1` 原本會在標準流中佔一個 `200px * 200px` 的位置。
- 設定 `position: relative` 後，它仍保留這個原始位置。
- `top: 100px` 與 `left: 100px` 只改變它的視覺位置。
- `.box2` 仍然按照 `.box1` 原本佔位後的位置排列，不會補到 `.box1` 原本的位置。

## 相對定位的特點

相對定位要記住幾個重點：

- 參照點是自己原本在標準流中的位置。
- 元素不脫標，原來位置仍然佔有。
- 後面的盒子仍然按照標準流看待它原本的位置。
- 定位元素的顯示層級通常高於普通元素，重疊時可能覆蓋普通流內容。
- 多個定位元素重疊且沒有額外設定疊放順序時，後寫的元素通常會蓋在先寫的元素上。

## 為什麼常用來當絕對定位父級

相對定位本身會保留父盒子的版面位置，又能讓這個父盒子成為子元素絕對定位的參照祖先。

這正是「子絕父相」常用的原因：

- 父級：`position: relative`，保留在普通版面裡。
- 子級：`position: absolute`，脫標後放到父盒子內的指定位置。

如果父級不加定位，子級的絕對定位可能會往更外層的定位祖先，甚至初始包含塊尋找參照點。

## 常見錯誤

### 誤以為相對定位會讓後面元素補位

相對定位不脫標。即使元素視覺上移走，原本位置仍在，後面的元素不會補上。

### 同時設定同軸兩邊

```css
.box {
  position: relative;
  left: 20px;
  right: 20px;
}
```

初學階段不建議這樣寫。若只是要向右偏移，寫 `left: 20px` 即可；若只是要向左偏移，寫 `right: 20px` 即可。

### 把相對定位當成主要排版工具

大量用 `relative` 搭配邊偏移微調，容易讓版面看起來正常但結構難以維護。一般排版應優先使用標準流、Flexbox、Grid、margin、padding 等工具。

## 實務判斷準則

- 需要微調元素位置，且保留原本佔位：可用 `relative`。
- 需要讓子元素以父盒子為絕對定位參照：父盒子常用 `relative`。
- 不建議用大量 `top/left` 取代正常布局。
- 同一軸向先避免同時設定兩邊偏移，除非你清楚需要這種約束效果。

## 重點整理

- `relative` 參照自己原本位置移動。
- 相對定位不脫標，原位置仍保留。
- `top`、`left` 等邊偏移改變的是視覺位置，不改變原本佔位。
- 相對定位最典型用途是作為絕對定位元素的父級參照。

3. 為什麼「子絕父相」中的父級常用相對定位？
