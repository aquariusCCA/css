# fixed：固定定位

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明固定定位會讓元素固定在瀏覽器可視區附近。
- 理解 fixed 元素不隨頁面滾動，且會脫離文檔流。
- 使用 `top`、`left` 等邊偏移設定固定位置。
- 理解固定在版心右側的常見計算技巧。

## 使用情境

固定定位常用在頁面滾動時仍要留在螢幕上的元素，例如：

- 固定客服工具。
- 返回頂部按鈕。
- 固定側邊欄。
- 固定頂部或底部操作列。

這些元素的位置不應該跟著文檔內容一起往上滾走，而是要固定在使用者看得到的位置。

## 一句話理解

`fixed` 會讓元素通常相對瀏覽器視口定位，不隨頁面滾動，並且不再佔原本位置。

## 基本語法

```css
.tool {
  position: fixed;
  top: 100px;
  left: 40px;
}
```

`top: 100px` 表示距離視口上方 100px，`left: 40px` 表示距離視口左側 40px。

## 範例拆解

```css
.dj {
  position: fixed;
  top: 100px;
  left: 40px;
}
```

```html
<body>
  <div class="dj">
    <img src="./images/logo.png" alt="">
  </div>
  <p>请尽情吩咐妲己，主人</p>
  <p>请尽情吩咐妲己，主人</p>
  <p>请尽情吩咐妲己，主人</p>
</body>
```

這個示例中，`.dj` 會固定在視口左上附近。即使頁面往下滾動，它仍停留在視覺上的同一個位置。

注意：上方 `./images/logo.png` 是 fenced code block 內的示意路徑，不是本章需要改寫的實體資產引用。

## fixed 的特點

固定定位有幾個重要特性：

- 一般情況下相對瀏覽器視口定位。
- 不隨頁面滾動。
- 脫離文檔流，不再佔原本位置。
- 行內、行內塊、塊級元素設定 fixed 後，都會變成定位元素。
- 若未設定寬高，定位元素預設大小常由內容撐開。
- 固定定位可以看作一種特殊的絕對定位。

有些祖先屬性可能影響固定定位的參照，例如 `transform` 可能建立新的包含塊。入門時先記住：一般頁面中 fixed 常相對視口；遇到異常時，再檢查祖先元素是否建立了特殊包含塊。

## 邊偏移注意事項

初學時同一軸向建議選一邊設定：

```css
.tool {
  position: fixed;
  right: 24px;
  bottom: 24px;
}
```

這表示固定在視口右下附近。若同時設定 `left` 與 `right`，或 `top` 與 `bottom`，實際結果會涉及尺寸與約束計算，入門階段較不直觀。

## 固定在版心右側

有時頁面內容有固定寬度的「版心」，例如 `800px` 寬並水平置中，而你想把固定工具列放在版心右側。

範例：

```css
.w {
  width: 800px;
  height: 1400px;
  background-color: pink;
  margin: 0 auto;
}

.fixed {
  position: fixed;
  left: 50%;
  margin-left: 400px;
  width: 50px;
  height: 150px;
  background-color: skyblue;
}
```

```html
<body>
  <div class="fixed"></div>
  <div class="w">版心盒子 800像素</div>
</body>
```

拆解：

- `left: 50%`：先走到視口水平中心。
- 版心寬度是 `800px`，一半是 `400px`。
- `margin-left: 400px`：從視口中心再往右走版心一半寬度。
- 固定盒子就會貼近版心右側。

如果版心寬度改變，`margin-left` 也要跟著改。

## 常見錯誤

### 以為 fixed 會保留原位置

fixed 會脫標，不再佔原本位置。若固定元素覆蓋內容，需要用 padding、margin 或其他布局方式預留空間。

### 把 fixed 寫在某個容器裡，就以為一定相對容器

一般 fixed 相對視口，而不是普通父元素。若發現它似乎被某個祖先影響，要檢查祖先是否有 `transform` 等建立包含塊的屬性。

### 版心右側公式忘記同步更新

版心是 `800px` 時用 `margin-left: 400px`。如果版心變成 `960px`，就應重新計算為 `480px`，否則固定元素會偏位。

## 實務判斷準則

- 想讓元素滾動時仍留在視口：用 fixed。
- 固定元素可能覆蓋內容時，要主動預留空間。
- 固定在右下角可用 `right` 與 `bottom`。
- 固定在版心旁邊時，可用視口中心加版心半寬的計算方式。

## 重點整理

- `fixed` 通常相對瀏覽器視口定位。
- fixed 元素不隨頁面滾動。
- fixed 會脫離文檔流，不佔原本位置。
- 固定在版心右側可用 `left: 50%` 加版心寬度一半的 `margin-left`。
- 特殊祖先屬性可能影響 fixed 的參照，遇到異常時要檢查。

## 自我檢查

1. fixed 元素會隨頁面內容一起滾動嗎？
2. fixed 元素還會佔有原本在標準流中的位置嗎？
3. 版心寬度是 `1000px` 時，固定在版心右側的小技巧中 `margin-left` 應該是多少？
4. 如果 fixed 元素擋住正文，應該思考什麼布局處理？
