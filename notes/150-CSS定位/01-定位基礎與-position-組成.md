# 定位基礎與 position 組成

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明為什麼標準流與浮動之外還需要定位。
- 理解定位由「定位模式」與「邊偏移」兩部分組成。
- 知道 `static`、`relative`、`absolute`、`fixed`、`sticky` 的基本差異。
- 用「是否佔位」與「以誰為參照點」判斷定位模式。

## 問題情境

標準流適合讓元素按照 HTML 順序由上到下、由左到右排列；浮動適合讓多個盒子橫向排列，或做文字環繞。但有些版面需求很難只靠這兩種方式完成：

- 讓一個元素自由放到某個盒子的右上角，並壓在其他內容上。
- 讓客服按鈕或返回頂部按鈕固定在瀏覽器視窗某個位置。
- 讓導覽列在頁面滾動到某個位置後黏住。

這些效果通常需要定位。

## 一句話理解

CSS 定位是用 `position` 決定元素用哪種方式擺放，再用 `top`、`right`、`bottom`、`left` 決定它偏移到哪裡。

## 定位由兩部分組成

定位有兩個核心部分：

- **定位模式**：由 `position` 指定，決定元素按照哪種規則定位。
- **邊偏移**：由 `top`、`right`、`bottom`、`left` 指定，決定元素移動到哪個位置。

定位模式表：

![position 定位模式值與語義對照表](../../origin/150-CSS定位/assets/images/positioning-components-img-001-2e8044.png)

邊偏移表：

![top bottom right left 邊偏移屬性說明表](../../origin/150-CSS定位/assets/images/positioning-components-img-002-96c952.png)

如果只寫邊偏移，卻沒有讓元素進入可偏移的定位模式，通常不會得到預期效果。

```css
.box {
  top: 20px;
  left: 20px;
}
```

上面這段沒有設定 `position`，元素仍是預設的 `position: static`，`top` 和 `left` 不會像定位元素那樣移動它。

## static：預設的「沒有定位」

`static` 是元素的預設定位方式，也就是按照標準流擺放。

```css
.box {
  position: static;
}
```

靜態定位有兩個重點：

- 元素照標準流規則排列。
- `top`、`right`、`bottom`、`left` 對它沒有定位偏移效果。

因此 `static` 很少作為主動布局手段使用；它比較像是「回到沒有定位」的狀態。

## 定位模式的全貌

定位模式包含：

![static relative absolute fixed sticky 定位模式總結表](../../origin/150-CSS定位/assets/images/positioning-modes-summary-img-001-f984fa.png)

可以先用兩個問題記住它們：

1. **元素是否仍佔有原本位置？**
2. **元素以誰作為移動參照？**

| 定位模式 | 是否佔原位置 | 常見參照點 | 初步理解 |
| --- | --- | --- | --- |
| `static` | 佔位 | 標準流 | 預設狀態，沒有定位偏移 |
| `relative` | 佔位 | 自己原本位置 | 相對原位置微調 |
| `absolute` | 不佔位 | 最近定位祖先 | 放到某個定位容器內 |
| `fixed` | 不佔位 | 通常是瀏覽器視口 | 固定在螢幕位置 |
| `sticky` | 佔位 | 最近滾動祖先與包含塊 | 滾動到閾值後黏住 |

## 定位與浮動的差異

浮動常用於讓多個塊級盒子橫向排列，或產生文字環繞效果。定位則更適合控制元素在某個參照範圍內的精確位置，或讓元素覆蓋、固定、黏住。

簡單判斷：

- 想讓多個盒子水平排列，可以先考慮浮動、Flexbox 或 Grid。
- 想讓元素壓住其他元素、固定在視口、或放到父盒子的角落，通常會用定位。

## 常見錯誤

### 只寫 `top` 或 `left`，忘記設定 `position`

邊偏移必須搭配合適的定位模式才有意義。若元素仍是 `position: static`，邊偏移不會按定位方式生效。

### 沒有先判斷元素是否脫標

`absolute` 和 `fixed` 會脫離文檔流，不再佔有原位置；`relative` 和 `sticky` 則仍會佔有原位置。這會影響後面的兄弟元素與父元素高度。

### 把所有定位都理解成「相對瀏覽器移動」

不同定位模式的參照點不同。`relative` 參照自己原位置，`absolute` 參照最近定位祖先，`fixed` 一般參照視口，不能混在一起記。

## 實務判斷準則

- 先問元素是否要保留原位置；不保留時才考慮 `absolute` 或 `fixed`。
- 先找出元素要參照誰移動；如果要參照父盒子，父盒子通常要加定位。
- `static` 是預設狀態，不是常用的主動布局工具。
- 定位能精準控制位置，但過度使用會讓版面依賴固定座標，維護成本變高。

## 重點整理

- 定位用來處理自由移動、覆蓋、固定、黏住等布局需求。
- 定位由 `position` 模式與邊偏移組成。
- `static` 是預設值，沒有定位偏移效果。
- 學定位要抓兩件事：是否佔位、以誰為參照。
- 後續最重要的模式是 `relative`、`absolute`、`fixed`、`sticky`。
