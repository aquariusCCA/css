# 定位元素的特殊特性與覆蓋行為

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明絕對定位與固定定位元素為什麼能設定寬高。
- 理解定位元素脫標後對外邊距合併與父元素高度的影響。
- 分辨浮動與定位元素在覆蓋文字時的差異。
- 避免定位元素不小心壓住普通流內容。

## 問題情境

你可能會發現：原本是行內元素的 `<span>`，加上 `position: absolute` 後竟然可以設定寬高；原本應該在普通流中佔位置的盒子，加上 fixed 後卻壓住了下面的文字。

這些行為都來自定位元素脫離普通流後的特殊規則。

## 一句話理解

絕對定位與固定定位會讓元素脫離普通流，元素可設定寬高，也可能直接覆蓋普通流內容。

## 行內元素加定位後可設定寬高

一般行內元素不能像 block 那樣直接控制寬高。但加上絕對定位或固定定位後，它會變成定位元素，可以設定寬高。

```css
span {
  position: absolute;
  top: 300px;
  width: 200px;
  height: 150px;
  background-color: pink;
}
```

```html
<body>
  <span>123</span>
</body>
```

這裡的 `span` 雖然原本是行內元素，但加上 `position: absolute` 後，可以用 `width` 和 `height` 控制尺寸。

## 塊級元素不設定寬高時由內容撐開

絕對定位或固定定位的塊級元素，如果沒有設定寬高，預設尺寸可能會由內容決定，而不是像普通 block 那樣自然撐滿父容器寬度。

```css
div {
  position: absolute;
  background-color: skyblue;
}
```

```html
<body>
  <div>abcd</div>
</body>
```

這個 `div` 的大小會更接近內容本身，而不是自動佔滿整行。

## 與外邊距合併的關係

浮動元素、絕對定位元素、固定定位元素都脫離或改變普通流的排版規則，因此不會像普通 block 那樣觸發外邊距合併問題。

也可以簡單記：脫標的盒子不會觸發普通流中的外邊距塌陷。

## 覆蓋普通流內容

絕對定位或固定定位元素與普通流內容重疊時，通常可能直接覆蓋下面的內容。

```css
.box {
  position: absolute;
  width: 150px;
  height: 150px;
  background-color: pink;
}
```

```html
<body>
  <div class="box"></div>
  <p>阁下何不同风起，扶摇直上九万里</p>
</body>
```

這裡 `.box` 脫離普通流後，可能壓住 `<p>` 的文字。

## 定位與浮動覆蓋行為的差異

浮動最初的設計目的包含文字環繞，因此浮動元素旁邊的文字通常會繞開它。

定位不同。絕對定位與固定定位元素不會要求普通流文字繞開自己，當它們重疊時，常見效果是直接蓋在內容上方。實際前後關係仍會受層疊規則與 `z-index` 影響。

## 常見錯誤

### 以為定位元素仍會撐開父元素

絕對定位與固定定位元素脫標後，不會像普通流子元素那樣撐開父元素。若父元素高度依賴這些子元素，版面可能坍掉。

### 以為定位元素會讓文字自動避開

absolute/fixed 不像 float 一樣主要用於文字環繞。重疊時它可能壓住普通文字，需要用布局預留空間或調整層級。

### 忘記定位會改變元素尺寸行為

行內元素定位後能設定寬高，block 定位後未設定寬高時可能由內容撐開。這會讓元素表現與普通流時不同。

## 實務判斷準則

- 需要元素覆蓋內容時，用定位前要確認不會遮住重要資訊。
- 需要父元素被內容撐開時，不要只依賴 absolute/fixed 子元素。
- 需要文字環繞效果時，定位不一定合適，應考慮 float 或其他布局方式。
- 使用 fixed 頂部欄或底部欄時，要記得為正文預留空間。

## 重點整理

- 絕對定位與固定定位會讓元素脫標。
- 行內元素加 absolute/fixed 後可設定寬高。
- 定位 block 若未設定寬高，大小可能由內容撐開。
- 脫標盒子不觸發普通流中的外邊距合併。
- absolute/fixed 與普通流重疊時，可能直接覆蓋內容。
