# sticky：黏性定位

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明 `sticky` 是相對定位與固定效果的混合。
- 理解 sticky 元素先佔位，滾動到指定閾值後才黏住。
- 知道 sticky 必須在需要黏住的軸向設定邊偏移。
- 避免因滾動祖先或瀏覽器支援造成 sticky 失效。

## 使用情境

黏性定位常用在滾動頁面中的導覽列、表格表頭、章節標題等元素。它一開始像普通元素一樣在頁面中佔位，當頁面滾動到指定位置後，才像固定定位一樣黏住。

## 一句話理解

`sticky` 會先保留在普通流中，滾動到指定邊偏移閾值後，在最近滾動祖先的範圍內黏住。

## 基本語法

```css
.nav {
  position: sticky;
  top: 0;
}
```

`top: 0` 是關鍵。沒有設定黏住軸向的邊偏移，sticky 在該軸向通常會表現得像相對定位，不會產生黏住效果。

## 範例拆解

```css
body {
  height: 3000px;
}

.nav {
  position: sticky;
  top: 0;
  width: 800px;
  height: 50px;
  background-color: pink;
  margin: 100px auto;
}
```

```html
<body>
  <div class="nav">我是导航栏</div>
</body>
```

拆解：

- `.nav` 一開始位於普通流中，並佔有原本位置。
- 當頁面滾動到 `.nav` 應該距離視口頂部 `0` 的位置時，它會黏住。
- 在常見頁面滾動情境中，效果看起來像固定在視口頂部。
- 它的運作範圍仍會受到最近滾動祖先與包含塊影響。

## sticky 與 relative、fixed 的關係

可以把 sticky 看成兩個階段：

1. **未到閾值前**：像 `relative`，元素仍在普通流中佔位。
2. **到達閾值後**：像 `fixed`，元素在滾動過程中黏住指定位置。

這也是 sticky 常被說成相對定位與固定定位的混合。

## 生效條件

使用 sticky 時要檢查：

- 是否設定 `position: sticky`。
- 是否在需要黏住的軸向設定 `top`、`left`、`right` 或 `bottom`。
- 外層是否有滾動容器，sticky 會受最近滾動祖先影響。
- 目標瀏覽器是否支援 sticky；IE 不支援，現代主流瀏覽器多已支援。

## 常見錯誤

### 忘記設定 `top`

```css
.nav {
  position: sticky;
}
```

這樣沒有告訴瀏覽器何時開始黏住。若要在頂部黏住，應設定：

```css
.nav {
  position: sticky;
  top: 0;
}
```

### 以為 sticky 永遠相對瀏覽器視口

在頁面本身滾動且沒有額外滾動容器時，sticky 常看起來像相對視口黏住。但更精確地說，它會相對最近的滾動祖先與包含塊運作。

### 忽略瀏覽器支援

如果專案需要支援 IE，sticky 不是可直接依賴的方案。正式專案仍要依目標瀏覽器版本確認。

## 實務判斷準則

- 導覽列滾到頂部後固定：適合 sticky。
- 元素一開始就要固定在視口：適合 fixed。
- 元素只需要相對原位置微調：適合 relative。
- sticky 失效時，先查邊偏移與滾動祖先。

## 重點整理

- `sticky` 是相對定位與固定效果的混合。
- sticky 元素仍佔有原本位置。
- 必須設定黏住軸向的邊偏移，例如 `top: 0`。
- sticky 會受最近滾動祖先與包含塊影響。
- IE 不支援 sticky，正式專案需確認目標瀏覽器。
