# CSS 基本語法與書寫風格

## 學習目標

讀完這篇筆記後，你應該能夠：

- 說明一條 CSS 規則由哪些部分組成。
- 分辨選擇器、宣告區塊、屬性名稱與屬性值。
- 正確使用冒號與分號撰寫 CSS 聲明。
- 理解展開風格與緊湊風格的差異。

## CSS 規則的基本結構

CSS 和 HTML 一樣，都需要遵守固定的語法規則。想要用 CSS 修飾網頁，第一步就是看懂一條 CSS 規則的組成方式。

一條 CSS 規則主要由兩個部分構成：

- 選擇器
- 宣告區塊（裡面放一條或多條聲明）

![CSS 語法規則示意圖，標示 h1 選擇器、宣告區塊與屬性名和值](../../origin/010-CSS簡介/assets/images/css-introduction-img-001-3ac98e.png)

基本格式如下：

```css
選擇器 {
    屬性名稱: 屬性值;
}
```

例如：

```css
h1 {
    color: red;
    font-size: 40px;
}
```

這段 CSS 的意思是：選取頁面中的 `h1` 元素，並把文字顏色設定為紅色、字體大小設定為 `40px`。

## 選擇器

選擇器用來指定 CSS 要套用到哪些 HTML 元素。

在這個例子中：

```css
h1 {
    color: red;
}
```

`h1` 就是選擇器，表示這條規則會套用到所有 `<h1>` 元素。

後續學到 class、id、後代選擇器等內容時，選擇器會變得更有彈性。不過在 CSS 入門階段，先記住選擇器的工作是「選到要設定樣式的對象」即可。

## 聲明、屬性與屬性值

選擇器後面的 `{}` 稱為宣告區塊，裡面可以放一條或多條聲明。

每一條聲明通常由兩個部分組成：

- 屬性名稱
- 屬性值

例如：

```css
color: red;
```

其中：

| 部分 | 說明 |
| --- | --- |
| `color` | 屬性名稱，表示要設定文字顏色 |
| `red` | 屬性值，表示設定成紅色 |
| `:` | 用來分隔屬性名稱與屬性值 |
| `;` | 用來結束這一條聲明 |

CSS 聲明的重點如下：

- 屬性和屬性值會以「鍵值對」的形式出現。
- 屬性是要設定的樣式項目，例如字體大小、文字顏色等。
- 屬性名稱與屬性值之間使用英文冒號 `:` 分開。
- 多條聲明之間使用英文分號 `;` 區分。

## 展開風格

展開風格會把選擇器、宣告區塊與每一條聲明分行書寫。

```css
h1 {
    color: red;
    font-size: 40px;
}
```

這種寫法在開發時較推薦，因為結構清楚，便於閱讀、維護與測試。對初學者來說，展開風格也比較容易看出每個語法部分的角色。

## 緊湊風格

緊湊風格會把空白與換行盡量壓縮。

```css
h1{color:red;font-size:40px;}
```

這種寫法比較常出現在專案上線或建置後的檔案中。透過工具壓縮 CSS，可以減小檔案體積、節約網路流量，並讓使用者更快載入網頁。

不過在日常開發和學習階段，通常不建議直接手寫緊湊風格，因為可讀性較差。

## 書寫建議

初學 CSS 時，建議使用展開風格：

```css
h1 {
    color: red;
    font-size: 40px;
}
```

這樣寫有幾個好處：

- 每一條聲明都清楚分行，方便檢查。
- 冒號與分號的位置更容易看清楚。
- 新增、刪除或修改樣式時比較不容易出錯。
- 後續多人協作時比較容易閱讀。

等到專案準備上線時，再交給工具把 CSS 壓縮成緊湊格式即可。

## 常見誤解

- **誤解：CSS 一定要寫成緊湊風格才專業。**  
  開發時更重視可讀性與可維護性。緊湊風格通常是工具壓縮後的結果，不是初學階段需要追求的寫法。

- **誤解：最後一條聲明可以不加分號，所以分號不重要。**  
  有些情況下最後一條聲明省略分號仍可運作，但養成每條聲明都加分號的習慣，可以減少後續新增樣式時出錯。

## 重點整理

- CSS 規則通常由「選擇器」與「宣告區塊」組成。
- 選擇器負責指定要套用樣式的 HTML 元素。
- 宣告區塊中可以包含一條或多條聲明。
- 每條聲明由屬性名稱、冒號、屬性值與分號組成。
- 開發與學習時建議使用展開風格。
- 緊湊風格常用於專案上線後的壓縮版本。
