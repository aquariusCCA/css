# CSS 與 HTML 的分工

## 學習目標

讀完這篇筆記後，你應該能夠：

- 說明 HTML 在網頁中主要負責內容結構與語意。
- 說明只靠 HTML 處理樣式會造成什麼問題。
- 說明 CSS 的用途與價值。
- 理解「結構與樣式分離」為什麼是學習 CSS 的起點。

## 為什麼需要 CSS

HTML 主要負責描述網頁內容的結構與語意。

例如：

- `<h1>` 表示這是一個主要標題。
- `<p>` 表示這是一個段落。

這些標籤的重點不是讓內容變漂亮，而是說明內容在文件中的角色。也就是說，HTML 更像是在回答「這段內容是什麼」。

早期網頁可以只靠 HTML 呈現內容，也可以在 HTML 裡做一些簡單樣式。不過這樣做很快會遇到問題：當樣式需求變多，HTML 會變得臃腫、重複，而且難以維護。

因此，更合理的做法是讓 HTML 專注在結構與內容，把外觀設計交給 CSS。

## CSS 是什麼

CSS 是 Cascading Style Sheets 的縮寫，中文常稱為「層疊樣式表」，也有人稱為 CSS 樣式表或級聯樣式表。

CSS 是一種樣式表語言，用來描述 HTML 等標記語言文件的呈現樣式。簡單來說，HTML 決定頁面有哪些內容，CSS 決定這些內容如何顯示。

CSS 可以控制的範圍包含：

- 文字的大小、顏色與字型。
- 圖片、區塊或元件的外觀。
- 元素之間的間距。
- 網頁版面的排列方式。
- 整體視覺風格。

所以，CSS 不只是讓網頁變漂亮，也讓版面布局更有彈性。

## 結構與樣式分離

CSS 的最大價值，是讓 HTML 專注於結構呈現，讓樣式交給 CSS 管理。

這種分工可以稱為「結構與樣式分離」：

| 負責項目 | 主要任務 | 例子 |
| --- | --- | --- |
| HTML | 描述內容結構與語意 | 標題、段落、圖片、清單 |
| CSS | 描述內容的呈現樣式 | 顏色、字體、間距、布局 |

當結構與樣式分開後，同一份 HTML 可以搭配不同 CSS 產生不同外觀；同一份 CSS 也能套用到多個頁面，讓整個網站更容易維護。

## 範例理解

假設有一段標題：

```html
<h1>我的第一個網頁</h1>
```

HTML 的工作是說明這是一個一級標題。至於標題要不要變成紅色、字體要多大、和下面內容要距離多遠，這些都屬於 CSS 的工作。

例如：

```css
h1 {
    color: red;
    font-size: 40px;
}
```

這段 CSS 會讓所有 `h1` 標題變成紅色，並設定字體大小為 `40px`。

## 常見誤解

- **誤解：HTML 也可以調整外觀，所以不一定需要 CSS。**  
  HTML 的核心任務是描述內容結構。把大量樣式混進 HTML，會讓文件難以閱讀與維護。

- **誤解：CSS 只是讓頁面變漂亮。**  
  CSS 除了美化畫面，也負責版面布局、視覺一致性與樣式維護，是前端頁面結構中很重要的一層。

## 重點整理

- HTML 主要負責網頁結構與內容語意。
- CSS 主要負責網頁外觀、樣式與布局。
- 只靠 HTML 處理樣式會讓文件變得臃腫且難以維護。
- CSS 的核心價值是讓結構與樣式分離。
- 學 CSS 時，要先建立「HTML 管內容，CSS 管呈現」的分工概念。
