# CSS 的引入方式

## 學習目標

讀完本章後，你應該能判斷 CSS 可以透過哪三種方式進入 HTML 頁面：行內樣式表、內部樣式表與外部樣式表。你也應該能說明每一種方式的作用範圍、適合情境、維護限制，以及為什麼實際開發通常優先使用外部樣式表。

本章還會建立一個重要判斷：當 CSS 沒有生效時，不只要檢查 selector 與 property，也要先確認這份 CSS 是否真的被 HTML 頁面讀到了。

## 前置知識

學習本章前，建議先知道 HTML 文件由標籤組成，例如 `<p>` 表示段落，`<head>` 放置頁面設定，`<body>` 放置畫面內容。你也需要能看懂最基本的 CSS 宣告，例如 `color: pink;` 和 `font-size: 20px;`。

本章會出現 `<style>`、`style` attribute 與 `<link rel="stylesheet">`。其中 `<style>` 是 HTML 標籤，`style` 是 HTML attribute，`<link>` 則用來讓 HTML 文件連結外部資源。

## 本章範圍

本章深入處理的是「CSS 程式碼放在哪裡，瀏覽器才會把它套用到 HTML 元素上」。來源資料提供了三種方式：行內樣式表、內部樣式表與外部樣式表，並比較它們的作用範圍、優點與缺點。

本章只會簡短提到 CSS 優先級與瀏覽器快取，因為它們能幫助理解三種引入方式的差異，但完整的 cascade、specificity、source order、`!important` 與快取策略都適合另開章節。本章也不深入討論 selector 寫法、CSS 檔案拆分策略或大型專案架構。

## 先看問題：CSS 寫好了，頁面為什麼沒有變

初學 CSS 時，常見問題不是 property 寫錯，而是 CSS 根本沒有正確進入 HTML 頁面。瀏覽器要套用樣式，至少要先知道兩件事：哪一段 CSS 要讀取，以及那段 CSS 要作用在哪些 HTML 元素上。

假設畫面中有一個段落：

```html
<p>給我一個粉紅的回憶</p>
```

你希望這段文字變成粉紅色、字體變大。CSS 可以做到這件事，但你必須先決定這段樣式要放在哪裡。放在標籤上、放在同一份 HTML 的 `<style>` 裡，或放在獨立的 `.css` 檔案中，會造成不同的作用範圍與維護結果。

本章可以把三種引入方式先理解成三種距離：

| 引入方式 | CSS 放置位置 | 主要作用範圍 | 維護特性 |
|---|---|---|---|
| 行內樣式表 | 寫在單一 HTML 標籤的 `style` attribute | 當前標籤 | 最直接，但最不利於大量維護 |
| 內部樣式表 | 寫在同一份 HTML 文件的 `<style>` 標籤中 | 當前頁面 | 適合單頁練習或小範圍頁面樣式 |
| 外部樣式表 | 寫在獨立 `.css` 檔案，再由 HTML 引入 | 多個頁面 | 最符合結構與樣式分離，實務最常用 |

這張表只是回查工具。真正學習時，應先看每一種方式解決什麼問題，以及它留下什麼維護成本。

## 第一課：只想改一個標籤時，行內樣式表如何工作

### 問題情境

你只有一個段落想快速改成粉紅色，並把字體放大。此時最短的寫法是直接把 CSS 寫在這個 HTML 標籤上：

```html
<p style="color: pink; font-size: 20px;">給我一個粉紅的回憶</p>
```

這就是行內樣式表。它把樣式放在元素標籤內部的 `style` attribute 中。

### 畫面觀察

修改前，`<p>` 會使用瀏覽器或其他 CSS 規則提供的樣式。修改後，這個 `<p>` 自己攜帶了兩條 CSS 宣告：`color: pink;` 讓文字變成粉紅色，`font-size: 20px;` 讓文字大小變成 `20px`。

行內樣式的作用對象非常明確：它只作用在寫了 `style` attribute 的那一個元素上。若頁面上有三個 `<p>`，但只有第一個 `<p>` 寫了 `style`，就只有第一個段落會被這段行內樣式影響。

### 最小修改

先看一個最小完整例子：

```html
<p>這是一般段落。</p>
<p style="color: pink; font-size: 20px;">這是被行內樣式修改的段落。</p>
```

第二個 `<p>` 的開始標籤中多了 `style="..."`，雙引號裡面寫的是 CSS 宣告。多條宣告之間用英文分號 `;` 分隔。

### 結果解釋

行內樣式的優點是直接。你不需要另外建立 CSS 檔案，也不需要寫 selector，因為樣式已經貼在目標元素上。瀏覽器讀到這個元素時，就能知道這個元素要使用哪些外觀設定。

但這種直接性也帶來限制。當樣式寫在 HTML 標籤上，內容結構和視覺設定會混在一起。若十個段落都要變粉紅色，你可能要在十個標籤上重複寫相同的 `style`。之後如果想把粉紅色改成藍色，也要逐一修改，維護成本很高。

來源資料也提醒，行內樣式沒有體現結構與樣式分離的思想，因此不推薦大量使用。它比較適合臨時測試，或對單一元素做非常簡單、局部的樣式調整。

### 誤判修正

不要把「行內樣式最接近元素」誤解成「最適合日常開發」。它確實很快，但當頁面變大，重複、難找、難改的問題會立刻出現。

補充說明：來源提到行內樣式的優先級最高。更精確地說，在一般沒有 `!important` 等特殊競爭條件時，行內樣式通常會比一般外部或內部 CSS 規則更容易勝出。完整優先級判斷屬於 cascade 與 specificity 主題，本章只先記住：行內樣式容易覆蓋其他地方寫的樣式，因此也容易增加除錯難度。

### 小整理

行內樣式表把 CSS 寫在元素自己的 `style` attribute 裡，作用範圍是當前標籤。它適合非常局部、簡單的修改，不適合大量撰寫或長期維護。

## 第二課：想控制整個頁面時，內部樣式表如何集中樣式

### 問題情境

如果同一個 HTML 頁面裡有很多 `<div>` 都要變成粉紅色，行內樣式就會變得很笨重。你不想在每個 `<div>` 上重複寫 `style="color: pink;"`，而是希望在頁面中集中寫一次規則。

這時可以使用內部樣式表，也就是把 CSS 寫在 HTML 文件中的 `<style>` 標籤裡。

### 畫面觀察

內部樣式表通常放在 `<head>` 中。這樣瀏覽器讀取 HTML 時，可以在頁面內容顯示前先知道本頁有哪些 CSS 規則。來源指出 `<style>` 標籤理論上可以放在 HTML 文件的多處，但一般會放在 `<head>`，本章也採用這個教學寫法。

### 最小修改

下面是一份最小 HTML 文件：

```html
<!DOCTYPE html>
<html lang="zh-Hant">
<head>
    <meta charset="UTF-8">
    <title>內部樣式表</title>
    <style>
        div {
            color: pink;
        }
    </style>
</head>
<body>
    <div>這個 div 會變成粉紅色。</div>
    <div>這個 div 也會變成粉紅色。</div>
</body>
</html>
```

這裡的 `<style>` 標籤放在 `<head>` 裡，CSS 規則則寫在 `<style>` 和 `</style>` 之間。

### 結果解釋

這段 CSS 的 selector 是 `div`，因此它會選到本頁面中的 `<div>` 元素。只要是同一份 HTML 文件裡符合 selector 的 `<div>`，都會套用 `color: pink;`。

和行內樣式相比，內部樣式表改善了重複問題。你只要在 `<style>` 裡修改一次 `color`，本頁所有符合條件的 `<div>` 都會一起改變。程式碼結構也比在每個標籤上塞 `style` 清楚。

不過，內部樣式表仍然只屬於當前 HTML 頁面。若網站有三個頁面都要使用相同的樣式，你仍然要把 `<style>` 內容複製到三份 HTML。這也是來源說它「沒有實現結構與樣式完全分離」的原因：樣式雖然從元素標籤中抽出來了，但仍然留在 HTML 文件裡。

### 誤判修正

不要把「集中在 `<style>`」誤解成「已經適合整站共用」。內部樣式表只能讓當前頁面的樣式集中，不會自動分享給其他 HTML 文件。

另一個常見誤解是把 `<style>` 和 `style` 混在一起。`style="..."` 是寫在單一元素上的 attribute；`<style>...</style>` 是 HTML 標籤，用來在文件中放一整段 CSS 規則。兩者名字相似，但作用範圍和維護方式不同。

### 小整理

內部樣式表把 CSS 集中放在同一份 HTML 文件的 `<style>` 標籤中，通常位於 `<head>`。它適合單頁練習、簡單範例或只服務當前頁面的樣式，但不適合多頁共用的長期維護。

## 第三課：多個頁面要共用樣式時，外部樣式表如何分離結構與樣式

### 問題情境

實際開發中，網站通常不只有一個頁面。首頁、關於頁、文章頁可能都要使用相同的標題顏色、卡片間距與按鈕樣式。如果每一頁都各自寫一段 `<style>`，之後修改整站樣式時會很麻煩。

這時應該把 CSS 放到獨立的 `.css` 檔案中，再讓 HTML 頁面透過 `<link>` 引入它。這就是外部樣式表。

### 畫面觀察

外部樣式表把 HTML 結構與 CSS 樣式拆成兩個檔案。HTML 文件保留內容與結構，CSS 檔案集中放樣式規則。當多個 HTML 頁面都引入同一份 CSS 檔案時，它們就能共用同一套樣式。

來源資料指出，實際開發通常使用外部樣式表，特別適合樣式比較多的情況。它也能讓瀏覽器快取 CSS 檔案，當多個頁面使用同一份樣式時，不一定每次都要重新下載同樣內容。

### 最小修改

外部樣式表分成兩步。

第一步，建立一個副檔名為 `.css` 的檔案，例如 `style.css`：

```css
div {
    color: pink;
}
```

這個檔案裡只放 CSS，不放 HTML 標籤。

第二步，在 HTML 的 `<head>` 中使用 `<link>` 引入這份 CSS：

```html
<!DOCTYPE html>
<html lang="zh-Hant">
<head>
    <meta charset="UTF-8">
    <title>外部樣式表</title>
    <link rel="stylesheet" href="./style.css">
</head>
<body>
    <div>這個 div 的樣式來自外部 CSS 檔案。</div>
</body>
</html>
```

`rel="stylesheet"` 表示這個外部資源是一份樣式表，`href="./style.css"` 表示 CSS 檔案的位置。

### 結果解釋

瀏覽器讀到 `<link rel="stylesheet" href="./style.css">` 時，會依照 `href` 找到 `style.css`，讀取裡面的 CSS 規則，再把規則套用到目前 HTML 頁面中符合條件的元素。

這種方式的關鍵好處是共用與分離。若三個 HTML 頁面都引入同一份 `style.css`，你只需要修改 `style.css` 一次，三個頁面的樣式就會一起更新。HTML 不需要混入大量樣式內容，CSS 也能成為獨立維護的檔案。

外部樣式表也比較符合後續專案開發習慣。當樣式變多時，獨立 CSS 檔案更容易整理、搜尋、拆分與重用。

### 誤判修正

不要把 `<link>` 寫在 `<body>` 內容中當成一般連結。教學與實務上，樣式表通常放在 `<head>`，讓頁面在呈現內容前先取得樣式規則。

也不要忘記檢查路徑。若 `href` 寫錯，CSS 檔案沒有被讀到，頁面就會看起來像沒有套用樣式。此時問題不是 `div { color: pink; }` 這條規則錯了，而是 HTML 沒有成功載入那份 CSS。

### 小整理

外部樣式表把 CSS 放在獨立 `.css` 檔案中，再用 `<link rel="stylesheet" href="...">` 引入 HTML。它的作用範圍可以跨多個頁面，最能體現結構與樣式分離，也是實際開發中最常用的方式。

## 三種引入方式的判斷流程

理解三種方式後，可以用下面的順序判斷該選哪一種：

1. 只是臨時測試或單一元素的非常簡單修改，可以使用行內樣式，但不要大量使用。
2. 只有一份 HTML 頁面要練習或展示樣式，可以使用內部樣式表，把規則集中在 `<style>` 裡。
3. 樣式會被多個頁面共用，或希望長期維護，就應優先使用外部樣式表。

更完整地看，三種方式不是只差在語法，而是差在樣式和 HTML 的距離。行內樣式最貼近元素，但最容易造成結構與樣式混雜。內部樣式表把樣式集中到頁面層級，但仍依附於單一 HTML。外部樣式表把樣式獨立成檔案，最適合多頁共用與後續維護。

## 常見誤解與修正方式

### 誤解一：三種方式只是寫法不同，效果都一樣

三種方式都能讓 CSS 生效，但維護結果不同。行內樣式只影響當前元素，內部樣式主要影響當前頁面，外部樣式可以被多個頁面共用。選擇方式時不能只看眼前畫面是否變色，也要看未來是否容易修改。

### 誤解二：內部樣式表已經完全分離 HTML 和 CSS

內部樣式表確實把 CSS 從元素標籤中抽出來，但 CSS 仍然寫在 HTML 文件內。若要讓樣式獨立維護並跨頁共用，應使用外部樣式表。

### 誤解三：外部 CSS 沒效果一定是 CSS 規則寫錯

外部樣式表多了一個載入步驟。若畫面沒有變，除了檢查 selector 和 declaration，也要檢查 `<link>` 是否存在、`rel="stylesheet"` 是否正確，以及 `href` 路徑是否指向真正的 CSS 檔案。

### 誤解四：行內樣式方便，所以適合大量使用

行內樣式只在小範圍修改時方便。大量使用後，樣式會分散在 HTML 各處，重複高、難搜尋，也容易因為優先級較高而覆蓋其他規則，增加除錯成本。

### 誤解五：外部樣式表一定會讓畫面比較漂亮

引入方式只決定 CSS 放在哪裡、如何被載入，不會自動改善設計品質。畫面是否好看仍取決於 CSS 規則本身。外部樣式表的主要價值是結構清楚、可共用、可維護。

## 實務使用情境

在課堂練習中，內部樣式表很常見，因為 HTML 和 CSS 放在同一份檔案中，方便初學者觀察範例。當你只想快速測試單一元素，也可能短暫使用行內樣式。

在正式專案中，通常會使用外部樣式表。它讓 HTML 專注於內容結構，CSS 專注於樣式規則，也讓多個頁面能共用同一套視覺規則。當設計需要調整時，修改獨立 CSS 檔案會比逐頁、逐元素修改更可靠。

除錯時，本章的實務判斷也很重要：如果 CSS 沒生效，先確認樣式在哪裡、作用範圍是什麼、HTML 是否有成功讀取 CSS，再進一步檢查 selector、優先級或屬性值。

## 小結

CSS 的引入方式決定樣式規則如何進入 HTML 頁面。行內樣式表最直接，但只適合單一元素的簡單修改；內部樣式表能集中管理當前頁面的樣式，但仍然依附在同一份 HTML 中；外部樣式表把 CSS 獨立成檔案，能跨頁共用，也最符合結構與樣式分離的維護方向。

學習本章後，判斷 CSS 問題時要多問一步：這段 CSS 放在哪裡，頁面有沒有讀到它，它的作用範圍是不是我預期的範圍。
