# 無縫動畫

## 學習目標

讀完這篇筆記後，你應該能夠：

- 說明無縫動畫要解決的銜接問題。
- 使用外層裁切容器與內層清單位移製作水平連續滾動。
- 理解為什麼需要複製部分清單項目。
- 使用 `linear`、`infinite` 與 `animation-play-state` 控制播放。
- 避免移動距離與內容寬度不一致造成跳動。

## 目標效果

無縫動畫常見於跑馬燈、品牌牆、商品列表或卡片輪播。畫面看起來像一排內容持續向左移動，而且循環時不會突然斷掉。

要做出這種效果，關鍵不只是讓清單移動，還要讓動畫結束回到起點時，畫面能自然接上。

## 一句話理解

無縫動畫的核心是：移動一組內容，同時在尾端複製開頭內容，讓循環跳回起點時看起來像連續播放。

## 結構拆解

基本結構分成兩層：

```html
<div class="box">
  <ul>
    <li>1</li>
    <li>2</li>
    <li>3</li>
    <li>4</li>
    <li>5</li>
    <li>6</li>
    <li>7</li>
    <li>8</li>
    <li>9</li>
    <li>10</li>

    <!-- 複製出來，用來實現無縫銜接 -->
    <li>1</li>
    <li>2</li>
    <li>3</li>
  </ul>
</div>
```

- `.box`：外層視窗，只顯示固定範圍。
- `ul`：真正移動的長清單。
- 前 10 個 `li`：原始內容。
- 後面複製的 `li`：用來補足銜接畫面。

## CSS 完整範例

```css
* {
  margin: 0;
  padding: 0;
}

.box {
  margin: 0 auto;
  width: 600px;
  height: 100px;
  background-color: skyblue;
  overflow: hidden;
}

.box ul {
  list-style: none;
  display: flex;
  width: 2600px;
  animation: move 5s infinite linear;
}

.box:hover ul {
  animation-play-state: paused;
}

.box ul li {
  width: 200px;
  height: 100px;
  line-height: 100px;
  text-align: center;
  font-size: 30px;
  color: white;
  font-weight: 500;
}

@keyframes move {
  from {
    transform: translate(0);
  }

  to {
    transform: translate(-2000px);
  }
}
```

## 範例拆解

### 外層容器負責裁切

```css
.box {
  width: 600px;
  height: 100px;
  overflow: hidden;
}
```

`.box` 像一個固定大小的視窗，只讓使用者看到 600px 寬的內容。超出範圍的清單會被 `overflow: hidden` 裁掉。

如果沒有 `overflow: hidden`，整條清單都會露出來，就不會像一個滾動視窗。

### 內層清單負責移動

```css
.box ul {
  display: flex;
  width: 2600px;
  animation: move 5s infinite linear;
}
```

- `display: flex;`：讓所有 `li` 水平排列。
- `width: 2600px;`：讓 `ul` 有足夠寬度容納所有項目。
- `animation: move 5s infinite linear;`：播放 `move` 動畫，5 秒一輪，無限循環，等速移動。

無縫滾動通常要用 `linear`，因為速度如果忽快忽慢，循環時比較容易看出不自然。

### 每個項目的寬度要穩定

```css
.box ul li {
  width: 200px;
  height: 100px;
  line-height: 100px;
  text-align: center;
}
```

每個 `li` 都是 `200px` 寬。這讓移動距離可以被準確計算。

原始內容有 10 個項目，每個 `200px`，所以原始內容總寬度是：

```text
10 * 200px = 2000px
```

因此動畫結束時移動 `-2000px`。

### keyframes 控制位移距離

```css
@keyframes move {
  from {
    transform: translate(0);
  }

  to {
    transform: translate(-2000px);
  }
}
```

這段動畫讓清單從原位向左移動 `2000px`。剛好移過原始 10 個項目的寬度後，畫面會看到後面複製出的項目。

下一輪動畫跳回起點時，因為複製項目和開頭內容相同，所以視覺上會像連續接上。

### hover 暫停

```css
.box:hover ul {
  animation-play-state: paused;
}
```

滑鼠移到 `.box` 上時，暫停 `ul` 的動畫。這常用於讓使用者有時間查看滾動內容。

注意暫停的是正在播放動畫的元素，也就是 `ul`，不是外層 `.box`。

## 為什麼要複製項目

如果只有 1 到 10，當清單移動完 `2000px` 後，下一輪會立刻跳回起點。使用者會看到明顯斷點。

複製前幾個項目放到尾端後，動畫快結束時畫面看到的是「和起點一樣的內容」。當動畫重新回到起點，視覺上就比較自然。

複製多少項目，要看外層視窗寬度與單項寬度。這個例子中：

- 外層寬度：`600px`
- 每個項目寬度：`200px`
- 可同時看到 3 個項目

所以複製 3 個項目即可補滿銜接畫面。

## 常見錯誤

### 移動距離和原始內容寬度不一致

原始內容寬度是 `2000px`，動畫就應該移動 `-2000px`。如果寫成 `-1800px` 或 `-2200px`，循環時就會提前或延後跳回，造成畫面斷裂。

### 沒有複製銜接項目

如果尾端沒有複製項目，動畫結束切回起點時會很明顯。無縫動畫的「無縫」通常就是靠複製內容完成銜接。

### 忘記使用 linear

```css
animation: move 5s infinite;
```

沒有指定速度曲線時，預設是 `ease`，速度會忽快忽慢。跑馬燈類效果通常更適合：

```css
animation: move 5s infinite linear;
```

### 暫停選錯元素

動畫是加在 `ul` 上，所以暫停也應控制 `ul`：

```css
.box:hover ul {
  animation-play-state: paused;
}
```

如果只寫 `.box:hover { animation-play-state: paused; }`，但 `.box` 本身沒有播放動畫，就不會暫停清單。

## 實務判斷準則

- 外層容器負責裁切：設定固定尺寸與 `overflow: hidden`。
- 內層內容負責移動：動畫通常加在 `ul` 或內容軌道上。
- 移動距離應等於「原始內容總寬度」，不要把複製項目也算進去。
- 需要等速循環時，使用 `linear`。
- 要讓使用者查看內容，可用 hover 暫停動畫。
- 複製項目數量至少要能填滿外層可視區域。

## 重點整理

- 無縫動畫靠「內容位移」與「尾端複製開頭內容」實現銜接。
- `.box` 負責視窗與裁切，`ul` 負責水平排列與動畫。
- `translate(-2000px)` 的距離應對應原始清單的總寬度。
- `infinite linear` 適合連續跑馬燈效果。
- 暫停動畫時，要控制真正播放動畫的元素。
