# 二倍圖的製作與顯示方式

## 學習目標

讀完這篇筆記，你應該能夠：

- 說明二倍圖解決的是什麼圖片清晰度問題。
- 根據 CSS 顯示尺寸推算二倍圖的實際像素尺寸。
- 使用 CSS 把二倍圖顯示成正確的版面尺寸。
- 避免直接用大圖原始尺寸造成版面變大的錯誤。

## 問題情境

假設頁面需要顯示一張 `50px × 50px` 的圖片。若圖片檔案本身也是 `50 × 50` 像素，在普通螢幕上可能看起來沒問題。

但在 iPhone 6 / 7 / 8 這類設備像素比為 2 的裝置上，這個 `50px × 50px` 的 CSS 顯示區域，實際需要 `100 × 100` 的物理像素細節。若仍使用 `50 × 50` 的圖片，圖片會被放大，細節不足，就容易模糊。

二倍圖就是為了解決這件事。

## 一句話理解

二倍圖是把圖片檔案做成 CSS 顯示尺寸的 2 倍，再用 CSS 把它縮回原本要顯示的大小。

## 製作思路

以 `50px × 50px` 的顯示尺寸為例：

1. 頁面實際想顯示：`50px × 50px`。
2. 目標設備 DPR 為 2。
3. 圖片檔案應準備：`100 × 100`。
4. CSS 仍設定圖片顯示尺寸為：`50px × 50px`。

這樣在 DPR 2 的螢幕上，圖片有足夠的像素細節，但版面尺寸不會變大。

## 範例程式碼

```css
img:nth-child(2) {
  width: 50px;
  height: 50px;
}
```

```html
<body>
  <!-- 原圖 -->
  <img src="../../origin/210-N倍圖/assets/images/n-times-image-img-002-702660.jpg" alt="綠底白色 Apple 標誌原圖">

  <!-- 二倍圖 -->
  <img src="../../origin/210-N倍圖/assets/images/n-times-image-img-003-e2d385.jpg" alt="綠底白色 Apple 標誌二倍圖">
</body>
```

## 範例拆解

這個範例中有兩張圖：

- 第一張是原圖，用來對照普通圖片在高 DPR 螢幕上可能出現的模糊問題。
- 第二張是二倍圖，圖片檔案本身準備了更高的實際像素。

CSS 對第二張圖片設定：

```css
width: 50px;
height: 50px;
```

這一步很重要。二倍圖雖然檔案實際像素更大，但它在頁面上的 CSS 顯示尺寸仍然應該是設計稿需要的 `50px × 50px`。如此一來，圖片既不會模糊，也不會把版面撐大。

## 換算方式

二倍圖可用這個規則判斷：

```text
二倍圖實際像素 = CSS 顯示尺寸 × 2
```

例如：

| CSS 顯示尺寸 | 二倍圖實際像素 |
| --- | --- |
| `50px × 50px` | `100 × 100` |
| `80px × 40px` | `160 × 80` |
| `120px × 60px` | `240 × 120` |

如果是三倍圖，則把倍率改成 3。

## 常見錯誤

### 只換大圖，沒有設定 CSS 寬高

如果你把圖片換成 `100 × 100` 的二倍圖，卻沒有把 CSS 顯示尺寸設為 `50px × 50px`，圖片可能會按原始尺寸顯示，導致畫面中的圖片變大。

正確思路是：圖片檔案變大，CSS 顯示尺寸不變。

### 把二倍圖理解成畫面要顯示兩倍大

二倍圖不是讓圖片在畫面上變成兩倍大，而是讓同樣的版面尺寸中有兩倍的像素細節。畫面上的 CSS 尺寸仍應符合設計稿。

### 不依目標設備判斷倍率

如果目標設備主要是 DPR 2，二倍圖通常足夠；若是 DPR 3 的高解析度場景，可能需要三倍圖或其他響應式圖片方案。不要只記「圖片都用二倍圖」，要回到顯示尺寸與設備像素比判斷。

## 實務判斷

- 設計稿標示圖片顯示為 `50px × 50px`，DPR 2 場景準備 `100 × 100` 圖片。
- HTML 使用高解析度圖片檔。
- CSS 把圖片寬高設定回設計稿的顯示尺寸。
- 若圖片需要支援多種裝置，可以進一步搭配 `srcset` 或響應式圖片策略；這屬於後續延伸主題。

## 重點整理

- 二倍圖的圖片檔實際像素是 CSS 顯示尺寸的 2 倍。
- 顯示時仍要用 CSS 設定回原本的版面尺寸。
- 二倍圖讓圖片更清晰，不是讓圖片在畫面上變大。
- 判斷圖片要幾倍，核心是 CSS 顯示尺寸與目標設備像素比。
