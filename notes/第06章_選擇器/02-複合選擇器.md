# 複合選擇器

## 學習目標

讀完本章後，你應該能做到以下幾件事：

- 說明複合選擇器為什麼需要建立在基礎選擇器之上。
- 判斷後代選擇器、子選擇器與兄弟選擇器會依照哪一種 HTML 結構關係選取元素。
- 使用並集選擇器把同一組樣式套用到多個選擇器。
- 使用交集選擇器選中同時符合多個條件的元素。
- 看出常見的誤選、漏選與選取範圍過大的問題。

## 前置知識

學習本章前，需要先理解基礎選擇器，例如標籤選擇器、類選擇器、id 選擇器與通配符選擇器。你也需要能看懂基本 HTML 父子結構，例如：

```html
<ul class="nav">
  <li>
    <a href="#">首頁</a>
  </li>
</ul>
```

在這段 HTML 中，`ul` 是 `li` 的父元素，`li` 是 `a` 的父元素；對 `ul` 來說，`li` 是它的子元素，`a` 則是它更深層的後代元素。

## 本章範圍

本章深入處理與「複合選擇器」直接相關的內容：

- 後代選擇器：用空格描述內外層關係。
- 子選擇器：用 `>` 描述直接父子關係。
- 兄弟選擇器：用 `+` 或 `~` 描述同一層元素之間的關係。
- 並集選擇器：用 `,` 讓多個選擇器共用同一組樣式。
- 交集選擇器：讓同一個元素同時符合多個選擇條件。

來源中也包含屬性選擇器、偽類選擇器、結構偽類選擇器與偽元素選擇器。這些內容屬於進階選擇器，適合另開章節；本章只在來源說明中標示，不在正文展開。

## 先看問題：基礎選擇器不一定夠精準

基礎選擇器可以選到某一種標籤、某個 class 或某個 id。但實際寫 CSS 時，畫面問題常常不是「所有 `<a>` 都要變色」，而是「只有導覽列裡的連結要變色」。這時只靠 `a` 會選太多，只靠 `.nav` 又只會選到外層容器，不會直接選到裡面的連結。

複合選擇器的核心，就是把多個選擇條件組合起來，讓 CSS 可以根據 HTML 結構、元素關係或多個條件選到更精準的目標。

可以先把它想成一句比較完整的描述：

```text
選到 .nav 裡面的 a
選到 .menu 的直接子層 li
選到 h2 後面緊接著的 p
選到同時是 p 標籤且有 intro class 的元素
```

這些需求都不是單一基礎選擇器最擅長處理的情境，因此需要複合選擇器。

## 第一課：想選到某個區塊裡的所有後代元素

### 問題情境

頁面上有很多連結，但你只想讓導覽列中的連結變成橘色。若直接寫 `a { color: orange; }`，頁面上所有連結都會被選到，包含正文、頁尾或其他區塊中的連結。

### 畫面觀察

修改前，所有 `<a>` 都是同一種標籤。導覽列的差異不在 `<a>` 本身，而在它位於 `.nav` 這個外層區塊裡。因此 selector 需要同時描述「外層是 `.nav`」以及「目標是裡面的 `a`」。

### 最小修改

```html
<ul class="nav">
  <li><a href="#">首頁</a></li>
  <li><a href="#">課程</a></li>
</ul>

<p>
  <a href="#">閱讀更多文章</a>
</p>
```

```css
.nav a {
  color: orange;
}
```

### 結果解釋

`.nav a` 是後代選擇器，中間的空格表示「在 `.nav` 裡面的所有 `a`」。它會選到 `.nav` 內部任何層級的 `<a>`，不管 `<a>` 是直接放在 `.nav` 裡，還是包在 `<li>` 裡。

修改後，導覽列中的兩個連結會變成橘色；段落中的「閱讀更多文章」不在 `.nav` 裡，所以不會被這條規則影響。

### 誤判修正

常見誤解是以為 `.nav a` 只會選到 `.nav` 的直接子元素。實際上，空格表示後代關係，只要在 `.nav` 裡面，不管隔了幾層都可能被選到。如果你只想選直接下一層，應使用子選擇器。

### 小整理

後代選擇器語法：

```css
外層選擇器 目標選擇器 {
  屬性: 值;
}
```

它適合用來限制樣式作用範圍，例如「只改某個區塊裡的標題、段落或連結」。

## 第二課：想只選直接子元素

### 問題情境

現在你只想讓 `.menu` 第一層的連結變紅，但不想影響子選單裡更深層的連結。若使用 `.menu a`，所有深層連結也會一起變紅，選取範圍太大。

### 畫面觀察

修改前，第一層連結和子選單連結都是 `<a>`。差別在於第一層連結是 `.menu` 的直接子元素，而子選單連結被包在其他元素裡。這時要描述的是「直接父子關係」。

### 最小修改

```html
<div class="menu">
  <a href="#">主選單</a>
  <p>
    <a href="#">子選單</a>
  </p>
</div>
```

```css
.menu > a {
  color: red;
}
```

### 結果解釋

`.menu > a` 是子選擇器。`>` 表示只選 `.menu` 的直接子層 `<a>`。範例中「主選單」是 `.menu` 的直接子元素，所以會變紅；「子選單」的父元素是 `<p>`，不是 `.menu`，因此不會被選到。

### 誤判修正

常見錯誤是把後代選擇器和子選擇器混用。`.menu a` 看的是「是否在 `.menu` 裡面」，`.menu > a` 看的是「父元素是否正好是 `.menu`」。如果 HTML 多包了一層，子選擇器就不會選到。

### 小整理

子選擇器語法：

```css
父選擇器 > 子選擇器 {
  屬性: 值;
}
```

它適合用來控制第一層結構，例如導覽列第一層項目、卡片中的直接區塊、清單中的直接項目。

## 第三課：想選同一層中後面的元素

### 問題情境

有時候你不是根據父子關係選元素，而是想根據同一層的前後關係選元素。例如標題後面緊接著的第一段說明要有上方距離，或某個元素後面的所有同層段落都要改色。

### 畫面觀察

兄弟元素有兩個條件：它們必須有同一個父元素，而且位在同一層。`+` 和 `~` 都只能往後選，不會回頭影響前面的元素。

### 最小修改：緊接在後面的兄弟

```html
<section>
  <p>前言段落</p>
  <h2>章節標題</h2>
  <p>第一段說明</p>
  <p>第二段說明</p>
</section>
```

```css
h2 + p {
  color: red;
}
```

### 結果解釋

`h2 + p` 會選到「緊接在 `h2` 後面的第一個 `p`」。修改後，只有「第一段說明」會變紅；`h2` 前面的段落不會被影響，第二段說明也不會被這條 `+` 規則選到。

如果你想選的是後面所有同層的段落，可以使用 `~`：

```css
h2 ~ p {
  color: red;
}
```

這會選到 `h2` 後方、同一層的所有 `<p>`，因此第一段與第二段說明都會變紅。

### 誤判修正

常見誤解是以為 `+` 和 `~` 會選到前面的兄弟。它們都只會從左邊選擇器所代表的元素往後找。另一個誤解是忽略「同一個父元素」這個條件；如果兩個元素不在同一層，就不是兄弟關係。

### 小整理

相鄰兄弟選擇器：

```css
前一個選擇器 + 緊接的下一個選擇器 {
  屬性: 值;
}
```

通用兄弟選擇器：

```css
前一個選擇器 ~ 後方同層選擇器 {
  屬性: 值;
}
```

`+` 適合選「下一個」，`~` 適合選「後面所有符合條件的同層元素」。

## 第四課：想讓多種元素共用同一組樣式

### 問題情境

你希望頁面中的 `<h2>`、`<p>` 和 `.note li` 都使用同一種文字顏色。如果分開寫三次，後續修改顏色時就要改三個地方。

### 畫面觀察

這些元素不一定有父子關係，也不一定是同一種標籤。它們的共同點不是結構，而是「要套用同一組樣式」。這種情況適合使用並集選擇器。

### 最小修改

```html
<h2>最新消息</h2>
<p>這是一段說明。</p>

<ul class="note">
  <li>重點一</li>
  <li>重點二</li>
</ul>
```

```css
h2,
p,
.note li {
  color: pink;
}
```

### 結果解釋

逗號 `,` 表示「以及」。上方 CSS 會選到所有 `<h2>`、所有 `<p>`，以及 `.note` 裡面的 `<li>`。這些被選到的元素會共用同一組宣告，因此文字都會變成粉色。

### 誤判修正

常見錯誤是忘記逗號造成 selector 意義改變。例如 `h2 p` 是後代選擇器，意思是選到 `h2` 裡面的 `p`；`h2, p` 才是並集選擇器，意思是選到 `h2` 和 `p`。

### 小整理

並集選擇器語法：

```css
選擇器1,
選擇器2,
選擇器3 {
  屬性: 值;
}
```

它適合用於集體聲明，讓多個不同選擇器共用同一份樣式。

## 第五課：想選同時符合多個條件的元素

### 問題情境

頁面上有些 `<p>` 有 `class="box"`，也有些 `<div>` 有 `class="box"`。你只想選到「同時是 `<p>`，又有 `box` class」的元素。

### 畫面觀察

如果寫 `.box`，`<p class="box">` 和 `<div class="box">` 都會被選到。如果寫 `p`，所有段落都會被選到。這時需要把兩個條件緊貼在一起，讓同一個元素同時滿足。

### 最小修改

```html
<p class="box">這是 p 標籤，而且有 box class。</p>
<p>這是普通 p 標籤。</p>
<div class="box">這是 div 標籤，也有 box class。</div>
```

```css
p.box {
  color: red;
}
```

### 結果解釋

`p.box` 是交集選擇器。它會選到同時符合 `p` 和 `.box` 的元素，也就是「標籤名稱是 `p`，並且有 `class="box"`」的元素。

修改後，只有第一個 `<p class="box">` 會變紅。普通 `<p>` 沒有 `box` class，不符合條件；`<div class="box">` 雖然有 `box` class，但標籤不是 `p`，也不符合條件。

### 誤判修正

常見錯誤是在交集選擇器中間加空格。`p.box` 和 `p .box` 意思完全不同：

```css
p.box {
  color: red;
}
```

這表示選到「同一個元素」同時是 `p` 且有 `box` class。

```css
p .box {
  color: red;
}
```

這表示選到「在 `p` 裡面的 `.box` 後代元素」。

來源中的交集選擇器語法範例寫成 `選擇器1選擇器2: { ... }`，冒號不屬於交集選擇器語法；正式寫法應是選擇器緊貼後直接接 `{ ... }`。

### 小整理

交集選擇器語法：

```css
標籤選擇器.類選擇器 {
  屬性: 值;
}
```

如果有標籤選擇器，通常寫在最前面，例如 `p.box`。中間不能加空格，因為空格會變成後代選擇器。

## 複合選擇器回查表

| 類型 | 寫法 | 選取重點 | 適合情境 |
|---|---|---|---|
| 後代選擇器 | `.nav a` | `.nav` 裡所有層級的 `a` | 限制某區塊內的元素樣式 |
| 子選擇器 | `.menu > a` | `.menu` 的直接子層 `a` | 只選第一層，不選更深層 |
| 相鄰兄弟選擇器 | `h2 + p` | 緊接在 `h2` 後面的第一個 `p` | 控制標題後第一段 |
| 通用兄弟選擇器 | `h2 ~ p` | `h2` 後方所有同層 `p` | 控制某元素後方的一組同層元素 |
| 並集選擇器 | `h2, p, .note li` | 多個選擇器共用樣式 | 集體聲明 |
| 交集選擇器 | `p.box` | 同一元素同時符合多個條件 | 精準選到某類元素 |

## 常見誤解與修正方式

第一個常見誤解是把空格看成可有可無。CSS selector 裡的空格有語意，`.nav a` 表示後代關係，`.nav>a` 或 `.nav > a` 才是直接子層關係，而 `p.box` 中間不能加空格。

第二個常見誤解是用後代選擇器處理所有結構問題。後代選擇器選取範圍很大，只要在外層裡面都可能被選到。若只想選第一層，應使用 `>`。

第三個常見誤解是以為兄弟選擇器可以往前選。`+` 與 `~` 都只能選左邊元素後面的兄弟，不能影響前面已出現的元素。

第四個常見誤解是忘記逗號。`h2, p` 是選 `h2` 和 `p`；`h2 p` 是選 `h2` 裡面的 `p`。少一個逗號，選取邏輯就完全不同。

第五個常見誤解是用很長的 selector 追求「看起來很精準」。selector 太長會增加維護成本，也容易被 HTML 結構變動影響。複合選擇器應該用來表達必要條件，而不是把整條 DOM 路徑都寫進 CSS。

## 實務使用情境

複合選擇器常用在元件內部樣式，例如 `.card h2`、`.nav a`、`.menu > li`。它能把樣式限制在某個區塊中，避免影響頁面其他地方。

後代選擇器適合處理區塊內的一般元素樣式；子選擇器適合控制第一層結構；兄弟選擇器適合處理相鄰或後續元素之間的間距與狀態；並集選擇器適合減少重複宣告；交集選擇器適合在同類元素中挑出特定狀態或類型。

補充說明：複合選擇器會影響 selector 的可讀性與維護性。寫 CSS 時，應先問「我真正需要的限制條件是什麼」，再決定要用空格、`>`、`+`、`~`、`,` 或緊貼條件。

## 小結

複合選擇器的重點不是讓 selector 變長，而是讓 CSS 更準確地描述「要選誰」。後代選擇器看內外層關係，子選擇器看直接父子關係，兄弟選擇器看同層前後關係，並集選擇器讓多個選擇器共用樣式，交集選擇器則要求同一個元素同時符合多個條件。寫複合選擇器時，最重要的是先看 HTML 結構，再決定要用哪一種關係或組合方式。
