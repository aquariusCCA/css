# 基礎選擇器

## 學習目標

讀完本章後，你應該能做到以下幾件事：

- 說明 CSS 選擇器的作用：先選到 HTML 元素，再設定樣式。
- 判斷標籤選擇器、類選擇器、id 選擇器與通配符選擇器各自會選到哪些元素。
- 依照畫面需求選擇合適的基礎選擇器，而不是只記語法。
- 看出常見的選錯、漏選、過度選取與 class / id 混用問題。

## 前置知識

學習本章前，需要先知道 HTML 是由標籤組成的，例如 `p`、`div`、`ul`、`li`。也需要知道 HTML 標籤可以有 attribute（屬性），例如 `class` 與 `id`。

CSS 規則通常由 selector（選擇器）與 declaration block（宣告區塊）組成：

```css
selector {
  property: value;
}
```

selector 負責回答「要改誰」，宣告區塊負責回答「要改成什麼樣子」。

## 本章範圍

本章會先說明選擇器在 CSS 中的角色，再深入介紹四種基礎選擇器，並補充多類名的使用方式：

- 標籤選擇器。
- 類選擇器。
- 多類名的使用方式。
- id 選擇器。
- 通配符選擇器。

複合與進階選擇器，例如後代選擇器、子選擇器、兄弟選擇器、屬性選擇器、偽類與偽元素，會在後續章節另外說明。本章只需要先建立「CSS 如何選到元素」的基本判斷。

## 先看問題：CSS 第一件事是選到元素

當一份 HTML 還沒有套用 CSS 時，瀏覽器會用預設樣式顯示內容。此時你可能會看到段落、標題、清單都長得很普通。若你想把某些文字改成綠色，CSS 不能直接說「把畫面上的那一塊改成綠色」，它必須先有辦法指出要改的是哪些 HTML 元素。

這就是選擇器的工作。選擇器可以理解成 CSS 用來「找人」的方式：先找到符合條件的元素，再把宣告區塊中的樣式套上去。

```html
<p>這是一段說明文字。</p>
<p>這是另一段說明文字。</p>
<div>這是一個區塊。</div>
```

如果只想讓所有段落變成綠色，可以寫：

```css
p {
  color: green;
}
```

這段 CSS 做了兩件事。第一，`p` 選擇器找到頁面上所有 `<p>` 元素。第二，`color: green;` 把這些被選到的元素文字改成綠色。沒有被 `p` 選到的 `<div>`，不會受到這條規則影響。

初學者常見的誤判是把 CSS 宣告區塊當成全部重點，只記得 `color: green;`，卻忘記前面的 `p` 決定了樣式會套到誰身上。實際寫 CSS 時，選錯元素往往比屬性寫錯更常見。

## 第一課：想一次改同一種標籤

### 問題情境

假設頁面上有多個段落和一個提示區塊。你希望所有段落文字都變成綠色，但提示區塊保持原樣。這時候最直接的做法是使用標籤選擇器。

### 畫面觀察

修改前，三個元素都使用瀏覽器或既有 CSS 的預設文字顏色。從 HTML 結構來看，兩段文字都使用 `<p>`，提示區塊使用 `<div>`。也就是說，如果 CSS 能用標籤名稱找到 `<p>`，就可以一次選到所有段落。

### 最小修改

```html
<p>第一段說明。</p>
<p>第二段說明。</p>
<div>這是提示區塊。</div>
```

```css
p {
  color: green;
}
```

### 結果解釋

`p` 是標籤選擇器。它會選到頁面上所有 `<p>` 元素，所以兩段 `<p>` 文字都會變成綠色。`<div>` 的標籤名稱不是 `p`，因此不會被這條規則選中。

標籤選擇器的優點是快速、直接，適合替同一類 HTML 標籤設定統一樣式。它的限制也很明顯：只要是同一種標籤都會被選到，因此不適合用來處理「只有某幾個段落要不同」的情境。

### 誤判修正

錯誤理解是以為 `p` 只會選到第一個 `<p>`。實際上，標籤選擇器會選到所有符合該標籤名稱的元素。若你只想改其中一段，就不應該只靠標籤選擇器。

### 小整理

標籤選擇器的語法是直接寫 HTML 標籤名稱：

```css
標籤名 {
  屬性: 值;
}
```

它適合用在整體文字、段落、標題、清單等同類元素的基礎樣式。

## 第二課：想只改某幾個元素

### 問題情境

現在頁面上有一組清單，你只想讓其中幾個項目變成紅色。這些項目同樣都是 `<li>`，如果使用 `li` 標籤選擇器，所有清單項目都會一起被改掉。這時候需要類選擇器。

### 畫面觀察

修改前，所有 `<li>` 都是同一種標籤。若只從標籤名稱判斷，CSS 無法分辨哪幾個項目比較重要。因此需要在 HTML 上加上 `class`，替特定元素取一個可以被 CSS 辨識的名字。

### 最小修改

```html
<ul>
  <li class="important">冰雨</li>
  <li class="important">來生緣</li>
  <li>李香蘭</li>
  <li>江南 style</li>
</ul>
```

```css
.important {
  color: red;
}
```

### 結果解釋

`.important` 是類選擇器。前面的 `.` 表示 CSS 要找的是 `class="important"` 的元素。範例中前兩個 `<li>` 有 `important` 這個 class，所以文字會變紅；後兩個 `<li>` 沒有這個 class，所以不會套用這條規則。

類選擇器最常用在需要重複套用的樣式。只要元素有相同 class，不管它是 `<li>`、`<p>` 還是 `<div>`，都可以被同一條 `.important` 選到。

### 誤判修正

常見錯誤是 HTML 寫了 `class="important"`，CSS 卻寫成 `important { ... }`。少了前面的 `.`，CSS 會把 `important` 當成標籤名稱，而不是 class 名稱。

另一個誤判是以為 class 只能用一次。實際上，同一個 class 可以被很多元素共用，這正是類選擇器適合做 reusable style（可重複使用樣式）的原因。

### 小整理

類選擇器的語法是：

```css
.類名 {
  屬性: 值;
}
```

HTML 中則寫在 `class` 屬性上：

```html
<div class="類名">內容</div>
```

補充說明：來源提到類名可由數字、字母、底線與中橫線組成。實務上為了讓 CSS selector 好讀、好寫，也避免特殊字元造成選取困難，建議使用有語意的英文名稱，並優先採用小寫英文與中橫線，例如 `main-title`、`card-item`。不要使用純數字或中文作為 class 名稱。

## 第三課：想把共用樣式拆開重組

### 問題情境

有時候一個元素不只需要一種樣式。例如你想讓某段文字同時具備「紅色」與「大字」兩個效果。如果把所有效果都寫進同一個 class，之後其他元素只想要紅色、不要大字，就很難重用。

### 畫面觀察

修改前，樣式需求其實可以拆成兩個獨立目的：一個 class 負責文字顏色，一個 class 負責文字大小。HTML 的 `class` 屬性允許同一個元素放多個 class 名稱，並用空格隔開。

### 最小修改

```html
<p class="red large-text">重要提醒</p>
```

```css
.red {
  color: red;
}

.large-text {
  font-size: 32px;
}
```

### 結果解釋

這個 `<p>` 同時有 `red` 與 `large-text` 兩個 class，所以會同時被 `.red` 與 `.large-text` 兩條規則選到。結果是文字變紅，也變成 32px。

多類名的重點不是產生新的選擇器種類，而是讓同一個 HTML 元素可以被多個 class 選中。這樣可以把共用樣式拆小，讓不同元素依需要組合。

### 誤判修正

常見錯誤是把多個 class 寫成逗號，例如 `class="red, large-text"`。在 HTML 的 `class` 屬性中，多個 class 必須用空格隔開，不是逗號。

另一個誤判是以為 class 的順序一定決定最後效果。class 在 HTML 中的順序不是本章主線；當多條 CSS 規則設定到同一個屬性時，還會牽涉到層疊、權重與來源順序。這些內容適合另開章節說明。

### 小整理

多類名寫法：

```html
<div class="class-a class-b">內容</div>
```

它適合用來組合共用樣式，例如文字顏色、字級、間距、狀態提示等。

## 第四課：想選頁面中的唯一元素

### 問題情境

假設頁面上有一個主標題區塊，它在整份 HTML 中只會出現一次。你可以用 id 替這個唯一元素命名，再用 id 選擇器選到它。

### 畫面觀察

修改前，如果這個元素只是普通 `<div>`，CSS 很難知道它是頁面中獨一無二的主標題。`id` 屬性提供了一種「唯一識別」的方式。來源資料也提醒：同一份 HTML 文件中，id 應該只出現一次。

### 最小修改

```html
<div id="main-title">CSS 基礎選擇器</div>
```

```css
#main-title {
  color: purple;
}
```

### 結果解釋

`#main-title` 是 id 選擇器。前面的 `#` 表示 CSS 要找的是 `id="main-title"` 的元素。因為 id 用來表示頁面中的唯一元素，所以這條規則應該只套用到一個元素上。

id 選擇器可以用來設定樣式，但在實務中，若某個樣式需要被多個元素重複使用，通常更適合用 class。來源資料也提到 id 經常會和 JavaScript 搭配，用來定位頁面中唯一的元素。

### 誤判修正

常見錯誤是把 id 當成 class 重複使用。若多個元素都需要同一種樣式，應使用 class，而不是替多個元素寫相同 id。

另一個錯誤是 HTML 寫 `id="main-title"`，CSS 卻寫 `.main-title`。`.` 是 class 選擇器，`#` 才是 id 選擇器。

### 小整理

id 選擇器的語法是：

```css
#id名稱 {
  屬性: 值;
}
```

HTML 中則寫在 `id` 屬性上：

```html
<div id="id名稱">內容</div>
```

## 第五課：想對所有元素套用基礎規則

### 問題情境

有時候你希望頁面上所有元素都先套用同一組基礎設定。例如在練習版面時，瀏覽器預設的 `margin` 與 `padding` 可能讓元素之間出現你沒有預期的空白。這時可以用通配符選擇器選到所有元素。

### 畫面觀察

修改前，不同 HTML 元素可能帶有不同的瀏覽器預設樣式。你看到的空白不一定是自己寫的 CSS 造成的，也可能是瀏覽器預設樣式。若想先把所有元素的外距與內距歸零，可以使用 `*`。

### 最小修改

```css
* {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}
```

### 結果解釋

`*` 是通配符選擇器，表示選取頁面中所有元素。上方範例會讓所有元素的 `margin` 與 `padding` 先變成 `0`，並讓盒模型計算方式改成 `border-box`。

這類寫法常見於基礎 reset 或練習版面時的起手式，但它的影響範圍非常大。只要寫在 `*` 裡，就會套用到所有元素，所以不適合用來處理只屬於某一小塊內容的樣式。

### 誤判修正

常見錯誤是把通配符選擇器當成方便的萬用解法。例如：

```css
* {
  color: red;
}
```

這會讓頁面中所有元素的文字都嘗試變成紅色，通常會造成過度選取。若只想改某一區或某一類元素，應改用更精準的標籤、class 或 id 選擇器。

### 小整理

通配符選擇器語法：

```css
* {
  屬性: 值;
}
```

它適合用於全頁基礎規則，但不適合用來取代精準選擇。

## 基礎選擇器回查表

| 選擇器 | CSS 寫法 | HTML 對應 | 適合情境 |
|---|---|---|---|
| 標籤選擇器 | `p` | `<p>` | 同一種標籤都要套用同樣樣式 |
| 類選擇器 | `.important` | `class="important"` | 多個元素共用同一種樣式 |
| 多類名 | `.red`、`.large-text` | `class="red large-text"` | 同一元素組合多個可重用樣式 |
| id 選擇器 | `#main-title` | `id="main-title"` | 頁面中唯一元素 |
| 通配符選擇器 | `*` | 所有元素 | 全頁基礎規則或 reset |

## 常見誤解與修正方式

第一個常見誤解是只看 CSS 宣告，不看 selector。`color: red;` 本身不會決定誰變紅，前面的 selector 才決定作用對象。排查樣式問題時，應先問：「這條規則有沒有選到我想改的元素？」

第二個常見誤解是把標籤選擇器用在差異化樣式上。若只想改某幾個 `<li>`，寫 `li { ... }` 會選到所有清單項目。這種情況應該使用 class。

第三個常見誤解是混淆 class 與 id。`.box` 對應 `class="box"`，`#box` 對應 `id="box"`。符號不同，HTML 屬性也不同。

第四個常見誤解是把 id 當成可重複樣式名稱。id 應該用於唯一元素；若樣式需要重複使用，應使用 class。

第五個常見誤解是濫用 `*`。通配符選擇器會影響所有元素，適合做全頁基礎設定，不適合用來處理局部畫面。

## 實務使用情境

在一般 CSS 開發中，標籤選擇器常用來設定基礎排版，例如段落、標題或清單的基本樣式。類選擇器最常用，適合命名可重用的元件或狀態，例如 `.card`、`.active`、`.warning`。多類名則適合把樣式拆成多個可組合的小單位。

id 選擇器通常用於頁面中唯一的區塊，或搭配 JavaScript 定位元素。若只是為了重複套用外觀，class 會比 id 更適合維護。通配符選擇器則常見於 reset 或全域盒模型設定，但使用時要先確認自己真的想影響所有元素。

## 小結

基礎選擇器的核心不是背符號，而是判斷 CSS 要選到誰。標籤選擇器適合選同一種 HTML 標籤，類選擇器適合重複使用的樣式，id 選擇器適合唯一元素，通配符選擇器則會選到所有元素。當樣式沒有照預期生效時，先檢查 selector 是否真的匹配到目標 HTML，通常比急著修改屬性更有效。
