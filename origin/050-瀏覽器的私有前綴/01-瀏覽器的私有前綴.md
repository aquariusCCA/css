# **什麼是瀏覽器私有屬性前綴**

> W3C 標準所提出的某個 CSS 特性，在被瀏覽器正式支持之前，瀏覽器廠商會根據瀏覽器的內核，使用私有前綴來測試該 CSS 特性，在瀏覽器正式支持該 CSS 特性之後，就不需要私有前綴了。

瀏覽器的私有前綴（Private Prefix）是指瀏覽器廠商在實現 HTML、CSS 和 JavaScript 等標準規範時，使用自己的前綴字母作為標識的一種方式。

私有前綴 **主要是用來實驗和測試新功能**，這些功能還未被標準化或尚未被所有瀏覽器支援。 **私有前綴可以區分這些實驗性功能和標準功能**，使開發人員能夠避免使用這些未正式納入標準的功能，以免出現兼容性問題。

<aside>
👉

**私有前綴:**

- moz-: 代表 firefox 瀏覽器的私有屬性。
- ms-: 代表 ie 瀏覽器的私有屬性。
- webkit-: 代表 safari、chrome 瀏覽器的私有屬性。
- o-: 代表 Opera 瀏覽器的私有屬性。
</aside>

<aside>
💡

瀏覽器私有前綴是為了兼容老版本的寫法，比較新版本的瀏覽器無須添加。

</aside>

# **書寫順序**

> 💡 先寫私有的 CSS3 屬性，再寫標準的 CSS3 屬性。

提倡寫法:

```css
-moz-border-radius: 10px;

-webkit-border-radius: 10px;

-o-border-radius: 10px;

border-radius: 10px;
```

# **什麼時候去掉私有前綴**

> 💡 當一個屬性成為標準，並且被 Firefox、Chrome 等主流的瀏覽器的最新版普遍兼容的時候，可以去掉該屬性的瀏覽器前綴。
