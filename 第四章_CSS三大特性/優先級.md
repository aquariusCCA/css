# 優先級

> 所屬章節：[第四章 CSS 三大特性](./README.md)  
> 課程入口：[CSS 筆記總覽](../README.md)  
> 關鍵字：優先級、權重、specificity、行內樣式、ID 選擇器、類選擇器、`!important`、複合選擇器  
> 建議回查情境：同一元素被多條規則命中時、想判斷哪條 CSS 會生效時、忘了類別和 ID 哪個優先時、想重新計算複合選擇器權重時

## 本節導讀

這一節在說明 CSS 的優先級，也就是當同一個元素同時符合多條規則時，瀏覽器最後要採用哪一條。

第一次閱讀時，先把兩件事分開理解：一個是「權重」在比較選擇器強弱，另一個是「層疊」在權重相同時決定誰覆蓋誰。把這兩件事拆開後，優先級就不容易混亂。

## 你會在這篇學到什麼

- 什麼是 CSS 優先級
- 權重是怎麼比較的
- 為什麼複合選擇器會有權重疊加
- 權重相同時為什麼後寫的規則會生效
- `!important` 應該怎麼理解

## 先講結論

- 同一個元素被多條規則選中時，會進入優先級判斷
- 權重高的選擇器，通常會覆蓋權重低的選擇器
- 如果權重相同，通常由後面寫的規則生效
- `!important` 會提高聲明的優先層級，但不建議濫用

一句話記法：

> 先比有沒有 `!important`，再比選擇器權重；如果還一樣，再看誰寫在後面。

## 1. 什麼是優先級

當同一個元素同時符合多條 CSS 規則，而且這些規則設定了同一個屬性時，就要判斷最後採用哪一條。

例如：

```css
div {
    color: pink;
}

.text {
    color: red;
}
```

```html
<div class="text">你笑起來真好看</div>
```

這個 `div` 同時符合：

- `div`
- `.text`

兩條規則都設定了 `color`，所以會發生衝突。  
最後通常顯示紅色，因為類選擇器的權重高於標籤選擇器。

## 2. 權重怎麼看

CSS 權重通常用四組數字來理解：

`(行內樣式, ID, 類別/屬性/偽類, 元素/偽元素)`

可以先這樣記：

- 行內樣式：`(1, 0, 0, 0)`
- ID 選擇器：`(0, 1, 0, 0)`
- 類別選擇器、屬性選擇器、偽類：`(0, 0, 1, 0)`
- 元素選擇器、偽元素：`(0, 0, 0, 1)`

![權重.png](./images/1776172854791_uJd5DUiZKK.png)

### 這裡要注意兩件事

- 權重不是單純十進位加總，而是四個位置分開比較
- 比較時從左到右看，前面那一位較大就直接勝出，不需要看後面

所以：

- `#id` 是 `(0,1,0,0)`
- `.class1 .class2 .class3` 是 `(0,0,3,0)`

雖然後者看起來很多，但它不會「進位」成 `(0,1,0,0)`。

## 3. 常見優先順序

在不考慮 `!important` 的前提下，可以先記住這個基礎順序：

- 行內樣式
- ID 選擇器
- 類別選擇器 / 屬性選擇器 / 偽類
- 元素選擇器 / 偽元素
- 通配符選擇器
- 繼承而來的值

### 小提醒

- 繼承不是在和一般選擇器「比權重」，而是當元素自己沒有更直接的設定時，才可能沿用父元素的值
- 所以實務上可以把繼承理解成很弱的來源

## 4. 權重相同時怎麼辦

如果兩條規則的權重完全一樣，通常就回到層疊性：後寫的覆蓋前寫的。

例如：

```css
.title {
    color: red;
}

.title {
    color: green;
}
```

這兩條規則權重一樣，所以後面的 `color: green;` 會生效。

## 5. 複合選擇器的權重疊加

如果是複合選擇器，權重會把各部分分開累加。

例如：

```css
ul li {
    color: green;
}

li {
    color: red;
}

.nav li {
    color: pink;
}
```

對同一個 `li` 來說：

- `li` 是 `(0,0,0,1)`
- `ul li` 是 `(0,0,0,2)`
- `.nav li` 是 `(0,0,1,1)`

所以 `.nav li` 的權重最高，通常會生效。

### 重點

- 複合選擇器的權重會疊加
- 但只是各位置相加，不會跨位進位

## 6. `!important` 怎麼理解

`!important` 不是一般選擇器權重的一部分，而是把某條「聲明」提升到更高的優先層級。

例如：

```css
#father {
    color: red !important;
}

p {
    color: pink;
}
```

```html
<div id="father">
    <p>你還是很好看</p>
</div>
```

這裡如果 `p` 自己被直接選中並設定了 `color: pink;`，那它是作用在 `p` 身上的直接規則；  
而父元素的 `color: red !important;` 是先作用在父元素，再透過繼承傳給子元素。

所以這個例子最重要的結論不是「父元素權重高」，而是：

- 子元素若有自己被直接選中的規則，通常要先看這些直接規則
- 父元素就算權重高，傳下來的仍然只是繼承值，不會自動壓過子元素自己的明確設定

### 小提醒

- `!important` 能少用就少用
- 如果你常想靠 `!important` 解問題，通常代表選擇器設計或樣式結構需要整理

## 7. 練習題怎麼判斷

### 示例 1

```css
div div {
    color: skyblue;
}

div {
    color: red;
}
```

```html
<div>
    <div>
        <div>白日依山盡，黃河入海流。</div>
    </div>
</div>
```

判斷：

- `div div` 是 `(0,0,0,2)`
- `div` 是 `(0,0,0,1)`

結果：通常顯示 `skyblue`。

### 示例 2

```css
div div div div div div div {
    color: red;
}

.one {
    color: green;
}
```

```html
<div>
    <div>
        <div>
            <div>
                <div>
                    <div>
                        <div class="one">白日依山盡，黃河入海流。</div>
                    </div>
                </div>
            </div>
        </div>
    </div>
</div>
```

判斷：

- 七個 `div` 是 `(0,0,0,7)`
- `.one` 是 `(0,0,1,0)`

結果：通常顯示 `green`，因為類別那一位比元素那一位更高。

### 示例 3

```css
.c1 .c2 div {
    color: red;
}

div #box3 {
    color: green;
}

#box1 div {
    color: blue;
}
```

```html
<div id="box1" class="c1">
    <div id="box2" class="c2">
        <div id="box3" class="c3">白日依山盡，黃河入海流。</div>
    </div>
</div>
```

判斷：

- `.c1 .c2 div` 是 `(0,0,2,1)`
- `div #box3` 是 `(0,1,0,1)`
- `#box1 div` 是 `(0,1,0,1)`

先比前兩位可知，後兩條都比第一條高。  
後兩條權重相同，所以再比書寫順序，通常後面的 `#box1 div` 生效。

結果：通常顯示 `blue`。

## 8. 常見混淆點

### 類別選擇器很多個，能不能贏過一個 ID？

不能直接這樣想。

權重比較是分位比較，不是十進位湊整數。  
所以 `(0,0,9,0)` 也不會因為「9 很大」就自動贏過 `(0,1,0,0)`。

### 繼承值能不能和直接選中的規則比輸贏？

通常先把它理解成不能直接這樣比。  
子元素如果有自己被規則直接命中，通常優先看那些直接規則；繼承值只是在沒有更直接設定時才出場。

## 9. 30 秒複習

- 優先級是同一元素、同一屬性衝突時的判斷規則
- 先看有沒有 `!important`
- 再比選擇器權重
- 權重相同時，看誰寫在後面
- 複合選擇器會疊加，但不會進位

如果只記一句話：

> 先比優先層級，再比權重，最後才比書寫順序。
