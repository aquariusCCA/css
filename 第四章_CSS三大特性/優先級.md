# 優先級

> 所屬章節：[第四章 CSS 三大特性](./README.md)  
> 課程入口：[CSS 筆記總覽](../README.md)  
> 關鍵字：優先級、權重、specificity、行內樣式、ID 選擇器、類選擇器、`!important`、複合選擇器、`:where()`、`:is()`  
> 建議回查情境：同一元素被多條規則命中時、想判斷哪條 CSS 會生效時、忘了類別和 ID 哪個優先時、想重新計算複合選擇器權重時

## 本節導讀

這一節說明 CSS 的**優先級**，也就是當同一個元素同時符合多條 CSS 規則，而且這些規則設定了同一個屬性時，瀏覽器最後要採用哪一條。

第一次閱讀時，先把兩件事分開：

- **層疊性**：處理多條規則同時存在時，最後怎麼決定結果。
- **優先級 / 權重**：在同一屬性衝突時，用來比較選擇器強弱。

如果把這兩件事拆開，就比較不會誤以為「後寫一定贏」或「離標籤近一定贏」。

## 你會在這篇學到什麼

- 什麼是 CSS 優先級
- 權重是怎麼比較的
- 為什麼複合選擇器會有權重疊加
- 權重相同時為什麼後寫的規則會生效
- `!important` 應該怎麼理解
- 為什麼很多類別選擇器不會進位贏過一個 ID
- `:where()`、`:is()`、`:not()`、`:has()` 的基本權重觀念
- 實務上如何用 DevTools 排查優先級問題

## 先講結論

- 優先級用來判斷：同一個元素、同一個屬性被多條規則設定時，哪一條最後生效。
- 通常先看有沒有 `!important`。
- 再比較選擇器權重，也就是 specificity。
- 權重相同時，通常看誰寫在後面或誰後載入。
- 權重不是一般十進位加總，而是分欄位比較，不會跨位進位。
- 繼承值通常不能直接拿來和子元素自己的直接規則比權重。

一句話記法：

> 先比優先層級，再比選擇器權重，最後才比書寫順序。

## 1. 什麼是優先級

當同一個元素同時符合多條 CSS 規則，而且這些規則設定了同一個屬性時，就要判斷最後採用哪一條。

例如：

```css
div {
    color: pink;
}

.text {
    color: red;
}
```

```html
<div class="text">你笑起來真好看</div>
```

這個 `div` 同時符合：

- `div`
- `.text`

兩條規則都設定了 `color`，所以發生衝突。

最後通常顯示紅色，因為：

```text
.text 的權重 > div 的權重
```

也就是說，這裡不是因為 `.text` 寫在後面才贏，而是因為類別選擇器本身權重就高於元素選擇器。

## 2. 優先級的簡化判斷流程

日常開發可以先用這個流程：

```text
1. 是不是同一個元素、同一個屬性發生衝突？
   ├─ 不是：不需要比較優先級
   └─ 是：繼續
   ↓
2. 有沒有 !important？
   ├─ 有：先比較 important 聲明
   └─ 沒有：繼續
   ↓
3. 比選擇器權重 specificity
   ↓
4. 權重相同時，比書寫順序或載入順序
```

完整 CSS 層疊規則還包含來源、層級、動畫、`@layer` 等進階因素；本篇先聚焦在最常見的作者樣式與選擇器權重。

## 3. 權重怎麼看

CSS 權重通常用四組數字理解：

```text
(行內樣式, ID, 類別/屬性/偽類, 元素/偽元素)
```

可以先這樣記：

| 類型 | 權重表示 | 範例 |
|---|---:|---|
| 行內樣式 | `(1,0,0,0)` | `<div style="color:red">` |
| ID 選擇器 | `(0,1,0,0)` | `#title` |
| 類別選擇器 | `(0,0,1,0)` | `.text` |
| 屬性選擇器 | `(0,0,1,0)` | `[type="text"]` |
| 偽類 | `(0,0,1,0)` | `:hover`、`:focus` |
| 元素選擇器 | `(0,0,0,1)` | `div`、`p` |
| 偽元素 | `(0,0,0,1)` | `::before`、`::after` |
| 通配符 | `(0,0,0,0)` | `*` |
| 組合符 | 不增加權重 | 空格、`>`、`+`、`~` |

### 權重比較方式

比較時從左到右看：

```text
(行內樣式, ID, 類別/屬性/偽類, 元素/偽元素)
```

哪一欄先比較出大小，就由那一方勝出。

例如：

```text
#id                  → (0,1,0,0)
.class1 .class2      → (0,0,2,0)
```

雖然 `.class1 .class2` 有兩個類別，但 ID 那一欄比較高，所以 `#id` 贏。

## 4. 權重不會跨位進位

這是優先級最容易搞錯的地方。

```text
#box                          → (0,1,0,0)
.a .b .c .d .e .f .g .h .i    → (0,0,9,0)
```

即使類別選擇器很多個，也不會進位成 ID 選擇器。

所以：

```text
(0,1,0,0) > (0,0,9,0)
```

也就是：

> 很多個 class，不會因為數量多就自動贏過一個 ID。

## 5. 常見優先順序

在不考慮 `!important`、`@layer` 等進階因素時，可以先記住：

```text
行內樣式
> ID 選擇器
> 類別選擇器 / 屬性選擇器 / 偽類
> 元素選擇器 / 偽元素
> 通配符選擇器
> 繼承而來的值
```

### 小提醒

繼承不是在和一般選擇器直接「比權重」。

比較好的理解是：

> 繼承值通常是在元素自己沒有更直接設定時才使用。

例如：

```css
#parent {
    color: red;
}

p {
    color: blue;
}
```

```html
<div id="parent">
    <p>文字</p>
</div>
```

雖然 `#parent` 是 ID 選擇器，但它設定的是父元素；`p` 是直接設定在子元素。最後 `p` 通常會是 `blue`。

## 6. 權重相同時怎麼辦

如果兩條規則權重完全相同，就回到層疊性：後寫的規則通常生效。

```css
.title {
    color: red;
}

.title {
    color: green;
}
```

兩條規則都是 `.title`：

```text
.title → (0,0,1,0)
.title → (0,0,1,0)
```

權重相同，所以後面的 `color: green;` 會生效。

多個 CSS 檔案也一樣：如果權重相同，後載入的規則通常會覆蓋前面載入的規則。

```html
<link rel="stylesheet" href="base.css">
<link rel="stylesheet" href="page.css">
```

如果兩個檔案都有：

```css
/* base.css */
.title {
    color: red;
}
```

```css
/* page.css */
.title {
    color: green;
}
```

最後通常是 `green`。

## 7. 複合選擇器的權重疊加

複合選擇器會把各部分的權重分別累加。

```css
ul li {
    color: green;
}

li {
    color: red;
}

.nav li {
    color: pink;
}
```

對同一個 `li` 來說：

| 選擇器 | 權重 |
|---|---:|
| `li` | `(0,0,0,1)` |
| `ul li` | `(0,0,0,2)` |
| `.nav li` | `(0,0,1,1)` |

所以 `.nav li` 的權重最高，通常會生效。

### 重點

- 複合選擇器的權重會疊加。
- 只在各自欄位內累加。
- 不會跨位進位。
- 組合符號本身不增加權重，例如空格、`>`、`+`、`~`。

## 8. `!important` 怎麼理解

`!important` 不是一般選擇器權重的一部分，而是把某一條「聲明」提升到更高的優先層級。

```css
.text {
    color: red !important;
}

#title {
    color: blue;
}
```

```html
<h1 id="title" class="text">標題</h1>
```

判斷：

- `.text` 權重是 `(0,0,1,0)`。
- `#title` 權重是 `(0,1,0,0)`。
- 正常情況下 `#title` 會贏。
- 但 `.text` 的 `color` 使用了 `!important`。

結果通常是：

```text
color: red;
```

### `!important` 是提升聲明，不是提升整個選擇器

```css
.box {
    color: red !important;
    font-size: 16px;
}

#main {
    color: blue;
    font-size: 20px;
}
```

這裡：

- `color: red !important;` 會比較強。
- `font-size: 16px;` 沒有 `!important`，所以還是可能被 `#main` 的 `font-size: 20px;` 覆蓋。

也就是說，`!important` 是作用在單一聲明上，不是讓整個 `.box` 規則全部升級。

### 不建議濫用 `!important`

`!important` 可以解決眼前覆蓋問題，但也會讓後續維護更困難。

如果你常常想靠 `!important` 解問題，通常代表：

- 選擇器寫得太複雜。
- 樣式結構不夠清楚。
- 共用樣式和頁面樣式的責任沒有分好。
- CSS 載入順序需要整理。

## 9. 父元素的 `!important` 能不能壓過子元素？

看情況，但有一個初學階段很重要的觀念：

> 父元素的樣式就算有 `!important`，如果它只是透過繼承傳給子元素，通常仍然不會自動壓過子元素自己的直接設定。

例如：

```css
#father {
    color: red !important;
}

p {
    color: pink;
}
```

```html
<div id="father">
    <p>你還是很好看</p>
</div>
```

這裡：

- `#father` 直接設定的是父元素 `div`。
- `p` 自己被 `p { color: pink; }` 直接命中。
- 父元素的 `red` 對 `p` 來說只是繼承值。

所以 `p` 通常會是：

```text
color: pink;
```

這個例子最重要的結論不是「父元素權重高不高」，而是：

> 子元素若有自己被直接命中的規則，通常先看子元素自己的直接規則；父元素傳下來的是繼承值。

## 10. 現代選擇器的權重提醒

初學時先掌握 ID、class、元素選擇器即可。但讀開源專案時，可能會看到一些現代選擇器。

### `:where()` 權重永遠是 0

```css
:where(.card .title) {
    color: red;
}
```

`:where()` 裡面就算放了 class，整個 `:where(...)` 的權重仍然是 `0`。

常見用途：寫低權重的基礎樣式，方便後面覆蓋。

### `:is()`、`:not()`、`:has()` 取括號內最高權重

```css
:is(.card, #main) .title {
    color: red;
}
```

`:is(.card, #main)` 會取括號中最高的權重，也就是 `#main` 的 ID 權重。

同樣概念也適用於：

- `:not(...)`
- `:has(...)`

初學階段不用一次背太細，但至少要知道：

> `:where()` 是刻意降低權重；`:is()`、`:not()`、`:has()` 會受到括號內選擇器權重影響。

## 11. 練習題怎麼判斷

### 示例 1

```css
div div {
    color: skyblue;
}

div {
    color: red;
}
```

```html
<div>
    <div>
        <div>白日依山盡，黃河入海流。</div>
    </div>
</div>
```

判斷：

| 選擇器 | 權重 |
|---|---:|
| `div div` | `(0,0,0,2)` |
| `div` | `(0,0,0,1)` |

結果：

```text
color: skyblue;
```

因為 `div div` 權重比較高。

### 示例 2

```css
div div div div div div div {
    color: red;
}

.one {
    color: green;
}
```

```html
<div>
    <div>
        <div>
            <div>
                <div>
                    <div>
                        <div class="one">白日依山盡，黃河入海流。</div>
                    </div>
                </div>
            </div>
        </div>
    </div>
</div>
```

判斷：

| 選擇器 | 權重 |
|---|---:|
| `div div div div div div div` | `(0,0,0,7)` |
| `.one` | `(0,0,1,0)` |

結果：

```text
color: green;
```

因為類別那一欄比元素那一欄更高，不會因為七個 `div` 很多就贏過 `.one`。

### 示例 3

```css
.c1 .c2 div {
    color: red;
}

div #box3 {
    color: green;
}

#box1 div {
    color: blue;
}
```

```html
<div id="box1" class="c1">
    <div id="box2" class="c2">
        <div id="box3" class="c3">白日依山盡，黃河入海流。</div>
    </div>
</div>
```

判斷：

| 選擇器 | 權重 |
|---|---:|
| `.c1 .c2 div` | `(0,0,2,1)` |
| `div #box3` | `(0,1,0,1)` |
| `#box1 div` | `(0,1,0,1)` |

先比 ID 欄位：

- `div #box3` 和 `#box1 div` 都有一個 ID。
- 它們都比 `.c1 .c2 div` 高。

再比兩條 ID 規則：

- `div #box3` 是 `(0,1,0,1)`。
- `#box1 div` 是 `(0,1,0,1)`。
- 權重相同，所以看書寫順序。
- `#box1 div` 寫在後面。

結果：

```text
color: blue;
```

## 12. 實務排查流程

如果某條 CSS 沒生效，可以這樣查：

```text
1. 先在 DevTools 選中目標元素
2. 看 Styles 面板中該屬性有哪些規則
3. 找出被劃掉的聲明
4. 比較生效與被劃掉的規則：
   - 有沒有 !important？
   - 權重誰比較高？
   - 權重相同時誰寫在後面？
   - 是否來自後載入的 CSS 檔案？
5. 到 Computed 面板確認最後實際值
```

常見修正方式：

- 不要一開始就加 `!important`。
- 優先調整 CSS 載入順序。
- 優先使用清楚的 class 命名。
- 避免為了覆蓋而堆疊過長選擇器。
- 共用樣式權重低一點，頁面或元件覆蓋樣式權重清楚一點。

## 13. 常見混淆點

### 類別選擇器很多個，能不能贏過一個 ID？

通常不能。

```text
(0,0,10,0) 不會進位成 (0,1,0,0)
```

權重比較是分欄位比較，不是十進位湊整數。

### 繼承值能不能和直接選中的規則比輸贏？

初學階段建議先理解成：不要直接這樣比。

如果子元素自己有直接規則，通常先看子元素自己的規則。繼承值只是在沒有更直接設定時才出場。

### 後寫是不是一定贏？

不是。

後寫只在權重相同、優先層級相同時才會成為決勝條件。

### 行內樣式是不是永遠不能被覆蓋？

不是絕對不能。

一般情況下，行內樣式很強：

```html
<div class="box" style="color: red;">文字</div>
```

```css
.box {
    color: blue;
}
```

結果通常是 `red`。

但如果外部 CSS 使用 `!important`：

```css
.box {
    color: blue !important;
}
```

就可能覆蓋一般行內樣式。

不過如果行內樣式自己也寫了 `!important`，那又會變得更難覆蓋。

## 14. 30 秒複習

- 優先級是同一元素、同一屬性衝突時的判斷規則。
- 先看有沒有 `!important`。
- 再比選擇器權重。
- 權重相同時，看誰寫在後面或誰後載入。
- 權重用四組數字理解：`(行內樣式, ID, 類別/屬性/偽類, 元素/偽元素)`。
- 權重不會跨位進位，很多 class 不會自動贏過 ID。
- 複合選擇器會疊加，但只在各自欄位中疊加。
- `:where()` 權重為 0，`:is()`、`:not()`、`:has()` 取括號內最高權重。
- 繼承值通常是在沒有直接規則時才使用。

如果只記一句話：

> CSS 衝突判斷不是單看順序，而是先看優先層級，再看權重，最後才看誰寫在後面。
