# 繼承性

> 所屬章節：[第四章 CSS 三大特性](./README.md)  
> 課程入口：[CSS 筆記總覽](../README.md)  
> 關鍵字：繼承性、父元素、子元素、可繼承屬性、不可繼承屬性、行高、瀏覽器預設樣式、inherit、initial、unset  
> 建議回查情境：忘了哪些 CSS 屬性會自動往下傳、看到子元素沒寫樣式卻跟著變化時、想確認 `line-height` 為什麼會一起變動時

## 本節導讀

這一節說明 CSS 的另一個重要特性：**繼承性**。

第一次閱讀時，先記住一個核心觀念：

> 子元素不是會繼承所有 CSS 屬性，而是在自己沒有設定某些可繼承屬性時，才可能沿用父元素的值。

理解繼承性後，你會比較容易判斷：

- 哪些樣式適合寫在 `body`、外層容器或父元素上。
- 哪些樣式一定要寫在元素自己身上。
- 為什麼子元素明明沒寫 `color` 或 `font-size`，畫面卻跟著變了。
- 為什麼有些元素看起來像「沒有繼承」。

## 你會在這篇學到什麼

- 什麼是 CSS 的繼承性
- 哪些類型的屬性通常會被繼承
- 哪些類型的屬性通常不會被繼承
- 子元素自己有設定時，為什麼會優先用自己的值
- 為什麼 `a` 和 `h1` 看起來像沒有繼承
- `line-height` 沒寫單位時為什麼特別常用
- `inherit`、`initial`、`unset` 的基本用途

## 先講結論

- 繼承性是指：子元素在自己沒有設定某個屬性時，可能沿用父元素的值。
- 常見可繼承屬性多半和文字、字型、顏色、行高有關。
- 盒模型、背景、邊框、寬高、定位這類影響版面的屬性，通常不會自動繼承。
- 子元素如果自己有明確設定某個屬性，通常會優先使用自己的值。
- 有些元素看起來像「繼承失效」，其實常常是瀏覽器預設樣式或其他 CSS 規則已經替它們指定了值。

一句話記法：

> 父元素的部分文字相關樣式，會在子元素沒自己設定時被沿用。

## 1. 什麼是繼承性

繼承性可以理解成：父元素設定某些樣式後，子元素如果沒有自己指定，就可能直接沿用。

例如：

```css
div {
    color: pink;
    font-size: 50px;
}
```

```html
<div>
    <p>龍生龍，鳳生鳳，老鼠生的孩子會打洞。</p>
</div>
```

如果 `p` 沒有自己設定 `color` 或 `font-size`，它通常會沿用 `div` 的對應值。

所以 `p` 最後看起來會像是：

```text
color: pink;
font-size: 50px;
```

### 重點

- 不是所有屬性都會繼承。
- 只有部分屬性會自然往下傳。
- 如果子元素自己有寫值，通常優先用子元素自己的設定。

## 2. 繼承性解決什麼問題

繼承性讓我們不用每個元素都重複寫相同的文字樣式。

例如：

```css
body {
    font-family: Arial, sans-serif;
    color: #333;
    line-height: 1.6;
}
```

這樣頁面中的多數文字元素都可以沿用這些基礎排版設定，而不用每個 `p`、`li`、`span` 都重複寫一次。

這也是為什麼實務上常會把下面這些樣式寫在 `body` 或外層容器：

```css
body {
    font-family: system-ui, sans-serif;
    color: #222;
    line-height: 1.5;
}
```

因為這些屬性大多和文字顯示有關，適合透過繼承往下傳。

## 3. 哪些屬性通常會繼承

不用一開始就背完整清單，先抓方向即可。

### 常見可繼承屬性

| 類型 | 常見屬性 | 說明 |
|---|---|---|
| 文字顏色 | `color` | 子元素常會沿用父元素文字顏色 |
| 字型 | `font-family`、`font-size`、`font-style`、`font-weight` | 和文字長相有關，常見會繼承 |
| 行高 | `line-height` | 常用在整體文字排版 |
| 文字排版 | `text-align`、`letter-spacing`、`word-spacing` | 多數文字相關設定可能繼承 |
| 清單相關 | `list-style` | 清單項目可能受到父層清單設定影響 |
| 可見性 | `visibility` | 父元素隱藏時，子元素通常也受影響 |

可以先用一句話記：

> 和文字呈現、閱讀排版有關的屬性，通常比較容易繼承。

## 4. 哪些屬性通常不會繼承

常見不會自動繼承的屬性包括：

| 類型 | 常見屬性 | 為什麼通常不繼承 |
|---|---|---|
| 盒模型 | `width`、`height`、`margin`、`padding` | 如果每個子元素都繼承，版面會很難控制 |
| 邊框 | `border`、`border-radius` | 子元素通常不應自動擁有父元素邊框 |
| 背景 | `background`、`background-color` | 背景通常屬於該元素自己的盒子 |
| 定位 | `position`、`top`、`left`、`z-index` | 版面定位不能隨意往下傳 |
| 佈局 | `display`、`flex`、`grid`、`overflow` | 會直接影響元素排版，不適合自動繼承 |

例如：

```css
div {
    width: 300px;
    padding: 20px;
    background-color: pink;
}
```

```html
<div>
    <p>我不會自動繼承 div 的 width、padding、background-color。</p>
</div>
```

`p` 不會因為在 `div` 裡面，就自動變成 `width: 300px`、`padding: 20px`、背景粉紅色。

## 5. 子元素自己有設定時怎麼辦

如果子元素自己明確設定了同一個屬性，就會優先使用自己的設定。

```css
div {
    color: red;
}

p {
    color: blue;
}
```

```html
<div>
    <p>這段文字是什麼顏色？</p>
</div>
```

判斷：

- `div` 設定 `color: red;`，這個值可以被子元素繼承。
- 但 `p` 自己設定了 `color: blue;`。
- 所以 `p` 最後使用自己的 `blue`。

結果：

```text
p 的 color 是 blue
```

重點是：

> 繼承值通常是在子元素沒有自己設定時才出場；子元素自己有直接設定時，優先看直接設定。

## 6. 行高的繼承

`line-height` 是很常見、也很值得特別記的繼承案例。

```css
body {
    font: 12px/1.5 "Microsoft YaHei";
    color: pink;
}

div {
    font-size: 14px;
}

p {
    font-size: 16px;
}
```

```html
<div>粉紅色的回憶</div>
<p>粉紅色的回憶</p>
<ul>
    <li>我沒有指定文字大小</li>
</ul>
```

### 這段程式碼怎麼看

`body` 裡的：

```css
font: 12px/1.5 "Microsoft YaHei";
```

代表：

- `font-size` 是 `12px`
- `line-height` 是 `1.5`
- `font-family` 是 `"Microsoft YaHei"`

因為 `line-height` 寫成沒有單位的數字，所以子元素會用自己的字體大小去計算行高。

| 元素 | 字體大小 | 行高計算 | 結果 |
|---|---:|---:|---:|
| `div` | `14px` | `14 × 1.5` | `21px` |
| `p` | `16px` | `16 × 1.5` | `24px` |
| `li` | `12px` | `12 × 1.5` | `18px` |

### 為什麼建議使用無單位 `line-height`

如果寫成：

```css
body {
    line-height: 1.5;
}
```

子元素可以根據自己的 `font-size` 重新計算，比較彈性。

如果寫成固定像素：

```css
body {
    line-height: 18px;
}
```

子元素可能會繼承固定的 `18px`，當子元素字體變大時，行高可能不夠，文字會顯得擁擠。

所以實務上，整體排版常見寫法是：

```css
body {
    font-size: 16px;
    line-height: 1.5;
}
```

## 7. 為什麼 `a` 和 `h1` 看起來像沒有繼承

有些元素看起來像沒有繼承，其實常常是因為它們有瀏覽器預設樣式。

### `a` 的顏色

```css
body {
    color: red;
}
```

```html
<a href="#">這是一個連結</a>
```

你可能會發現連結不一定是紅色，因為瀏覽器通常會替連結設定預設顏色。

這不是繼承機制壞掉，而是：

- `color` 本來可以繼承。
- 但 `a` 可能已經被瀏覽器預設樣式指定了顏色。
- 直接指定在 `a` 身上的值，比從父元素繼承而來的值更明確。

如果你希望連結沿用父元素顏色，可以寫：

```css
a {
    color: inherit;
}
```

### `h1` 的字體大小

```css
body {
    font-size: 16px;
}
```

```html
<h1>標題</h1>
```

`h1` 看起來通常比 `16px` 大，因為瀏覽器預設樣式會替 `h1` 設定較大的字體大小與 margin。

這也不是繼承失效，而是有更直接的預設規則參與了最後結果。

## 8. `inherit`、`initial`、`unset` 基本認識

這幾個值可以幫你更明確控制繼承行為。

### `inherit`

強制使用父元素的值。

```css
a {
    color: inherit;
}
```

常見用途：讓連結顏色跟外層文字一致。

### `initial`

使用 CSS 屬性的初始值。

```css
.box {
    color: initial;
}
```

意思是不要用父元素傳下來的值，而是回到這個屬性本身的初始狀態。

### `unset`

`unset` 會根據屬性是否可繼承來決定：

- 如果是可繼承屬性，效果類似 `inherit`。
- 如果是不可繼承屬性，效果類似 `initial`。

例如：

```css
.child {
    color: unset;
    margin: unset;
}
```

大致可以理解成：

- `color` 可繼承，所以會回到繼承邏輯。
- `margin` 不可繼承，所以會回到初始值。

初學階段不用急著大量使用，但看到別人的 CSS 寫這些值時，要知道它們和繼承有關。

## 9. 繼承性和層疊性的關係

繼承性和層疊性常常一起出現，但它們不是同一件事。

```css
div {
    color: red;
}

p {
    color: blue;
}
```

```html
<div>
    <p>文字</p>
</div>
```

判斷時可以分開看：

1. `div` 的 `color: red;` 可以被 `p` 繼承。
2. 但 `p` 自己有 `color: blue;`。
3. 所以 `p` 最後使用自己的直接設定。

也就是說：

> 繼承提供「沒有直接設定時的候選值」，但如果元素自己有直接規則，通常會優先看直接規則。

## 10. 實務排查流程

如果你看到子元素樣式怪怪的，可以這樣查：

```text
1. 先確認該屬性是否通常可繼承
   例如 color、font-size、line-height
   ↓
2. 檢查子元素自己有沒有直接設定該屬性
   ↓
3. 如果沒有，往父元素一層一層找來源
   ↓
4. 如果有，優先看子元素自己的規則
   ↓
5. 在 DevTools 中查看 Inherited from 與 Computed 值
```

在 DevTools 裡，常會看到：

```text
Inherited from body
font-family: ...
color: ...
line-height: ...
```

這代表該樣式是從父層繼承而來。

## 11. 常見混淆點

### 子元素是不是會繼承父元素所有 CSS？

不是。

大多數會影響版面的屬性，例如 `width`、`height`、`padding`、`margin`、`border`，通常不會自動繼承。

### 可繼承屬性是不是一定會用父元素的值？

不是。

如果子元素自己有明確設定，就優先用自己的值。

```css
.parent {
    color: red;
}

.child {
    color: blue;
}
```

`.child` 最後會是 `blue`。

### 繼承值可以和直接選中的規則比權重嗎？

初學階段可以先這樣理解：

> 直接作用在元素上的規則，通常比從父元素繼承來的值更優先。

所以不要把父元素的高權重選擇器，直接拿來和子元素自己的低權重選擇器硬比。

例如：

```css
#parent {
    color: red;
}

p {
    color: blue;
}
```

```html
<div id="parent">
    <p>文字</p>
</div>
```

雖然 `#parent` 是 ID 選擇器，但它設定的是父元素；`p` 是直接設定在子元素身上。最後 `p` 通常會是 `blue`。

## 12. 小練習

請判斷 `p` 最後的文字顏色與字體大小。

```css
div {
    color: red;
    font-size: 20px;
}

p {
    color: blue;
}
```

```html
<div>
    <p>CSS 繼承性練習</p>
</div>
```

判斷：

- `color` 可以繼承，但 `p` 自己設定了 `color: blue;`，所以使用 `blue`。
- `font-size` 可以繼承，而 `p` 沒有自己設定，所以沿用 `div` 的 `20px`。

結果：

```text
color: blue;
font-size: 20px;
```

## 13. 30 秒複習

- 繼承性是子元素在沒自己設定時，可能沿用父元素的某些樣式。
- 可繼承屬性多半和文字、字型、顏色、行高有關。
- 盒模型、背景、邊框、寬高、定位通常不會自動繼承。
- 子元素自己有明確設定時，通常優先使用自己的值。
- `line-height` 寫成無單位數字時，通常更適合整體文字排版。
- `a`、`h1` 看起來不像繼承，常常是因為瀏覽器預設樣式已經先指定了值。
- `inherit` 可以強制使用父元素的值。

如果只記一句話：

> 繼承性不是所有樣式都往下傳，而是部分文字相關屬性會在子元素沒自己設定時沿用父元素。
