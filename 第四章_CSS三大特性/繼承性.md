# 繼承性

> 所屬章節：[第四章 CSS 三大特性](./README.md)  
> 課程入口：[CSS 筆記總覽](../README.md)  
> 關鍵字：繼承性、父元素、子元素、可繼承屬性、不可繼承屬性、行高、瀏覽器預設樣式  
> 建議回查情境：忘了哪些 CSS 屬性會自動往下傳、看到子元素沒寫樣式卻跟著變化時、想確認 `line-height` 為什麼會一起變動時

## 本節導讀

這一節在說明 CSS 的另一個重要特性：繼承性。

第一次閱讀時，先記住最核心的觀念就夠了：不是所有 CSS 屬性都會自動從父元素傳給子元素，但有一部分和文字呈現有關的屬性，通常會被子元素繼承。理解這個機制後，會比較容易判斷哪些樣式可以集中寫在外層，哪些一定要寫在元素本身。

## 你會在這篇學到什麼

- 什麼是 CSS 的繼承性
- 哪些類型的屬性通常會被繼承
- 哪些類型的屬性通常不會被繼承
- 為什麼 `a` 和 `h1` 看起來像沒有繼承
- `line-height` 沒寫單位時為什麼特別好用

## 先講結論

- 繼承性是指子元素在自己沒有設定某個屬性時，可能會沿用父元素的值
- 常見可繼承屬性多半和文字、字型、行高、顏色有關
- 盒模型、背景、邊框、寬高這類會直接影響版面的屬性，通常不會自動繼承
- 有些元素看起來像「繼承失效」，其實常常是瀏覽器預設樣式另外指定了值

一句話記法：

> 父元素的某些文字樣式，子元素沒自己寫時，通常會跟著用。

## 1. 什麼是繼承性

繼承性可以先理解成：父元素先設定一部分樣式，子元素如果沒有自己指定，就可能直接沿用。

例如：

```css
div {
    color: pink;
    font-size: 50px;
}
```

```html
<div>
    <p>龍生龍，鳳生鳳，老鼠生的孩子會打洞。</p>
</div>
```

如果 `p` 沒有自己設定 `color` 或 `font-size`，就會沿用 `div` 的對應值。

### 重點

- 子元素不是「一定」繼承所有樣式
- 只有部分屬性會自然往下傳
- 如果子元素自己有寫值，會優先用自己的設定

## 2. 哪些屬性通常會繼承

可以先抓大方向，不需要背完整清單。

### 常見可繼承屬性

- `color`
- `font-size`
- `font-family`
- `font-style`
- `font-weight`
- `line-height`
- 多數 `text-` 開頭的文字相關屬性

這些屬性大多和「文字如何顯示」有關，所以常常適合寫在較外層的元素上。

## 3. 哪些屬性通常不會繼承

常見不會自動繼承的屬性包括：

- `width`
- `height`
- `margin`
- `padding`
- `border`
- `background`
- `overflow`

這些屬性大多直接影響盒子大小、邊界或版面配置，所以通常不會直接往下傳給子元素。

## 4. 行高的繼承

`line-height` 是很常見、也很值得特別記的繼承案例。

```css
body {
    font: 12px/1.5 "Microsoft YaHei";
    color: pink;
}

div {
    font-size: 14px;
}

p {
    font-size: 16px;
}
```

```html
<div>粉紅色的回憶</div>
<p>粉紅色的回憶</p>
<ul>
    <li>我沒有指定文字大小</li>
</ul>
```

### 這段程式碼怎麼看

- `body` 設定 `font: 12px/1.5 "Microsoft YaHei";`
- 其中 `1.5` 代表行高是目前字體大小的 `1.5` 倍
- `div` 自己把字體大小改成 `14px`，所以它的行高通常會跟著變成 `14 × 1.5 = 21px`
- `p` 自己把字體大小改成 `16px`，所以它的行高通常會跟著變成 `16 × 1.5 = 24px`
- `li` 沒有自己設定字體大小，因此會沿用 `body` 的 `12px`，行高通常就是 `12 × 1.5 = 18px`

### 為什麼這樣寫有用

當 `line-height` 寫成沒有單位的數字時，子元素會依照自己的字體大小去計算，通常比直接寫死像素值更靈活。

## 5. 常見混淆點

### `a` 的 `color`、`h1` 的 `font-size` 是不是繼承失效？

更準確的說法不是「繼承失效」，而是：

- 這些元素常常帶有瀏覽器預設樣式
- 預設樣式已經直接幫它們指定了某些值
- 所以畫面上看起來不像單純沿用父元素的結果

例如很多瀏覽器會預設讓連結有自己的顏色，也會替 `h1` 設定較大的字體大小。  
這不代表繼承機制壞掉，而是有其他樣式一起參與了最後結果。

### 子元素有自己的值時，還會繼承嗎？

子元素如果自己明確設定了某個屬性，通常就以自己的值為主，不會再直接使用父元素那個屬性的繼承結果。

## 6. 30 秒複習

- 繼承性是子元素在沒自己設定時，可能沿用父元素的某些樣式
- 可繼承屬性多半和文字、字型、顏色、行高有關
- 盒模型、背景、邊框、寬高通常不會自動繼承
- `line-height` 寫成無單位數字時，通常更適合做整體文字排版
- 有些元素看起來不像繼承，常常是因為瀏覽器預設樣式已經先指定了值

如果只記一句話：

> 繼承性不是所有樣式都往下傳，而是部分文字相關屬性會在子元素沒自己設定時沿用父元素。
