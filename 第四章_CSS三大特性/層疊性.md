# 層疊性

> 所屬章節：[第四章 CSS 三大特性](./README.md)  
> 課程入口：[CSS 筆記總覽](../README.md)  
> 關鍵字：層疊性、樣式衝突、同一元素、同一屬性、後寫覆蓋前寫、CSS 規則、樣式覆蓋  
> 建議回查情境：忘了 CSS 樣式衝突時怎麼判斷、看到同一元素被重複設定樣式時、想確認後面的 CSS 為什麼會覆蓋前面的 CSS 時

## 本節導讀

這一節說明 CSS 的第一個重要特性：**層疊性**。

第一次閱讀時，先抓住一句話：

> 當多條 CSS 規則同時作用在同一個元素，而且設定了同一個屬性時，瀏覽器必須決定最後採用哪一條聲明，這種決定最後結果的機制，就是層疊性要處理的事。

層疊性不是說「所有後面的樣式都會覆蓋前面的樣式」，而是說：

- 多條規則可以同時作用在一個元素上。
- 不同屬性通常可以一起生效。
- 同一屬性發生衝突時，才需要決定最後用哪一個值。

## 你會在這篇學到什麼

- 什麼是 CSS 的層疊性
- 層疊性主要解決什麼問題
- 什麼情況叫做樣式衝突
- 什麼情況下後面的樣式會覆蓋前面的樣式
- 為什麼「後寫一定贏」這句話不夠精確
- 實務上如何用 DevTools 排查層疊問題

## 先講結論

- 層疊性處理的是「樣式最後由誰生效」的問題。
- 只有在**同一個元素、同一個屬性**被多條聲明設定時，才會出現真正的衝突。
- 如果是不同屬性，通常不是互相覆蓋，而是一起生效。
- 在相同選擇器、相同權重、相同優先層級下，通常是**後寫的樣式覆蓋先寫的樣式**。
- 如果選擇器權重不同，就不能只看誰寫在後面，還要進入優先級判斷。

一句話記法：

> 同一元素、同一屬性發生衝突時，瀏覽器會根據層疊規則決定最後採用哪一條聲明。

## 1. 什麼是層疊性

「層疊」可以先想像成：樣式一層一層套到元素身上，最後由瀏覽器算出實際呈現的結果。

例如同一個元素可能同時受到這些規則影響：

```css
div {
    color: red;
}

.box {
    font-size: 20px;
}

#app .box {
    color: blue;
}
```

這些規則不是每一條都互相覆蓋。真正要看的是：

1. 它們有沒有命中同一個元素？
2. 它們設定的是不是同一個 CSS 屬性？
3. 如果是同一屬性，哪一條規則最後比較強？

所以層疊性的核心不是「全部覆蓋」，而是：

> 多條規則同時存在時，衝突的地方要決定誰生效，不衝突的地方可以一起存在。

## 2. 層疊性在解決什麼問題

層疊性主要解決的是**樣式衝突**。

例如，同一個 `div` 同時被設定兩次 `color`：

```css
div {
    color: red;
    font-size: 12px;
}

div {
    color: pink;
}
```

```html
<div>長江後浪推前浪，前浪死在沙灘上。</div>
```

### 這段程式碼會發生什麼事

- `font-size: 12px;` 會保留，因為後面的規則沒有再次設定 `font-size`。
- `color: pink;` 會覆蓋前面的 `color: red;`，因為兩條規則都設定了同一個元素的 `color`。

所以最後通常會是：

- 文字顏色：`pink`
- 字體大小：`12px`

這個例子要記住的不是「第二段 CSS 全部覆蓋第一段 CSS」，而是：

> 只有發生衝突的 `color` 被覆蓋，沒有衝突的 `font-size` 仍然保留。

## 3. 怎麼判斷有沒有衝突

可以用三個問題判斷。

### 問題一：是不是同一個元素？

如果不是同一個元素，就不算同一個元素上的直接衝突。

```css
.box {
    color: red;
}

.title {
    color: blue;
}
```

```html
<div class="box">內容 A</div>
<h1 class="title">內容 B</h1>
```

這裡兩條規則作用在不同元素上，不是在同一個元素上搶同一個 `color`。

### 問題二：是不是同一個屬性？

```css
div {
    color: red;
}

div {
    font-size: 16px;
}
```

這裡雖然都是作用在 `div`，但一個設定 `color`，一個設定 `font-size`。它們不互相覆蓋，而是一起生效。

### 問題三：是不是多條規則都想設定同一個值？

```css
div {
    color: red;
}

div {
    color: pink;
}
```

這裡兩條規則都想決定 `div` 的 `color`，所以發生衝突。

## 4. 後寫覆蓋前寫的前提

「後寫覆蓋前寫」只適用於比較基礎的情境。

比較完整的前提是：

- 作用在同一個元素。
- 設定同一個屬性。
- 選擇器權重相同。
- 沒有 `!important` 等更高優先層級。
- 規則來自相同層級的 CSS 來源。

例如：

```css
.title {
    color: red;
}

.title {
    color: green;
}
```

兩條規則都是 `.title`，權重相同，所以後面的 `color: green;` 會生效。

但是下面這個例子就不能只看誰寫在後面：

```css
#main-title {
    color: red;
}

.title {
    color: green;
}
```

```html
<h1 id="main-title" class="title">標題</h1>
```

雖然 `.title` 寫在後面，但 `#main-title` 的權重更高，所以通常會是 `red`。

這就是為什麼層疊性和優先級要一起理解。

## 5. 樣式衝突與樣式合併

初學者常把「多條規則同時命中」都理解成衝突，但其實不一定。

### 情況一：不同屬性，會合併

```css
.card {
    color: red;
}

.card {
    padding: 20px;
}
```

結果：

```text
color: red;
padding: 20px;
```

兩個屬性都會生效。

### 情況二：同一屬性，會競爭

```css
.card {
    color: red;
}

.card {
    color: blue;
}
```

結果通常是：

```text
color: blue;
```

因為兩條規則都在設定 `color`，所以後面的贏。

### 情況三：同一屬性的不同子屬性，不一定互相覆蓋

```css
.box {
    border-color: red;
}

.box {
    border-width: 2px;
}
```

這兩條不會互相覆蓋，因為一個設定邊框顏色，一個設定邊框寬度。

但如果寫成縮寫屬性，就要小心：

```css
.box {
    border-color: red;
}

.box {
    border: 1px solid blue;
}
```

`border` 是縮寫屬性，可能同時影響 `border-width`、`border-style`、`border-color`，所以它可能覆蓋前面某些邊框相關設定。

## 6. 多個 CSS 檔案也會層疊

層疊性不只發生在同一個 CSS 檔案裡，也會發生在多個 CSS 檔案之間。

```html
<link rel="stylesheet" href="base.css">
<link rel="stylesheet" href="page.css">
```

如果 `base.css` 和 `page.css` 都設定了同一個元素的同一個屬性，而且權重相同，通常後載入的 `page.css` 會覆蓋前面的 `base.css`。

例如：

```css
/* base.css */
.btn {
    color: black;
}
```

```css
/* page.css */
.btn {
    color: red;
}
```

結果通常是：

```text
color: red;
```

這也是為什麼實務上常會有：

- reset.css / normalize.css 放前面
- 基礎樣式放中間
- 頁面樣式或覆蓋樣式放後面

## 7. 實務排查流程

當你看到某個樣式沒有生效時，可以照下面檢查。

```text
1. 先確認目標元素有沒有被你想像中的選擇器命中
2. 再確認是否有其他規則也設定同一個屬性
3. 如果有，檢查被劃掉的是哪一條
4. 看贏的那條規則：
   - 是否寫在後面？
   - 是否權重更高？
   - 是否使用 !important？
   - 是否來自後載入的 CSS 檔案？
```

在瀏覽器 DevTools 中：

- `Styles` 面板可以看到哪些聲明被劃掉。
- `Computed` 面板可以看到最後實際計算出來的值。
- 點開屬性來源，可以知道最後是哪一個 CSS 檔案、哪一行生效。

## 8. 常見混淆點

### 層疊性是不是「誰離標籤最近就用誰」？

這樣講太粗略，容易誤解。

更準確的說法是：

- 如果是行內樣式，確實通常比一般選擇器強。
- 如果都是 CSS 規則，不能只看離 HTML 近不近。
- 需要看是否同一屬性衝突、選擇器權重、是否 `!important`、書寫順序等因素。

所以重點不是「離標籤近」，而是：

> 多條規則競爭同一個屬性時，最後哪一條在層疊規則中勝出。

### 是不是只要後寫就一定贏？

不是。

後寫會贏，通常要建立在「權重相同、優先層級相同」的前提下。

```css
#title {
    color: red;
}

.title {
    color: blue;
}
```

即使 `.title` 寫在後面，`#title` 通常仍然會贏，因為 ID 選擇器權重更高。

### 不同屬性會不會互相覆蓋？

通常不會。

```css
.box {
    color: red;
}

.box {
    margin: 20px;
}
```

這兩個屬性會一起生效。真正要注意的是同一個屬性，或縮寫屬性影響到其他子屬性的情況。

## 9. 小練習

請判斷最後文字顏色與字體大小。

```css
p {
    color: red;
    font-size: 14px;
}

p {
    color: blue;
}
```

```html
<p>CSS 層疊性練習</p>
```

判斷：

- `color` 被設定兩次，發生衝突。
- 兩條規則都是 `p`，權重相同。
- 後面的 `color: blue;` 生效。
- `font-size` 只設定一次，所以保留 `14px`。

結果：

```text
color: blue;
font-size: 14px;
```

## 10. 30 秒複習

- 層疊性處理的是樣式最後由誰生效的問題。
- 同一元素、同一屬性才會發生真正的覆蓋競爭。
- 不同屬性通常會一起生效。
- 權重相同時，後寫或後載入的規則通常會覆蓋前面的規則。
- 後寫不一定永遠贏，還要看優先級與權重。
- 用 DevTools 可以快速確認哪條規則被劃掉、哪條規則最後生效。

如果只記一句話：

> 層疊性不是所有樣式互相覆蓋，而是同一元素的同一屬性發生衝突時，瀏覽器決定最後採用哪一條聲明。
