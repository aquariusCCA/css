# 層疊性

> 所屬章節：[第四章 CSS 三大特性](./README.md)  
> 課程入口：[CSS 筆記總覽](../README.md)  
> 關鍵字：層疊性、樣式衝突、同一元素、同一屬性、後寫覆蓋前寫、CSS 規則  
> 建議回查情境：忘了 CSS 樣式衝突時怎麼判斷、看到同一元素被重複設定樣式時、想確認後面的 CSS 為什麼會覆蓋前面的 CSS 時

## 本節導讀

這一節在說明 CSS 的第一個重要特性：層疊性。

第一次閱讀時，先抓住一句話就夠了：當多條 CSS 規則同時作用在同一個元素、而且設定了同一個屬性時，最終會由其中一條規則生效。這種「發生衝突時要怎麼決定最後結果」的機制，就是層疊性要處理的事。

## 你會在這篇學到什麼

- 什麼是 CSS 的層疊性
- 層疊性主要在解決什麼問題
- 什麼情況下後面的樣式會覆蓋前面的樣式
- 什麼情況下樣式不會互相衝突

## 先講結論

- 層疊性處理的是「樣式衝突」問題
- 只有在同一個元素、同一個屬性發生衝突時，才需要看哪一條規則最後生效
- 在這種基礎情況下，如果選擇器條件相同，通常是後面寫的樣式覆蓋前面寫的樣式

一句話記法：

> 同一個元素、同一個屬性、規則衝突時，後寫的通常會覆蓋先寫的。

## 1. 什麼是層疊性

所謂層疊，可以先理解成樣式一層一層套上去，最後由瀏覽器決定實際生效的結果。

它不是指所有樣式都互相覆蓋，而是指：

- 多條 CSS 規則可能同時作用在同一個元素上
- 如果這些規則設定的是不同屬性，就可以同時存在
- 如果這些規則設定的是同一個屬性，就會發生衝突，必須決定最後用哪一條

所以，層疊性的核心不是「全部覆蓋」，而是「衝突時如何決定最後結果」。

## 2. 層疊性在解決什麼問題

層疊性主要解決的是樣式衝突。

例如，同一個 `div` 同時被設定兩次 `color`：

```css
div {
    color: red;
    font-size: 12px;
}

div {
    color: pink;
}
```

```html
<div>長江後浪推前浪，前浪死在沙灘上。</div>
```

### 這段程式碼會發生什麼事

- `font-size: 12px;` 會保留，因為後面的規則沒有再次設定 `font-size`
- `color: pink;` 會覆蓋前面的 `color: red;`

也就是說，最後畫面通常會是：

- 文字顏色：`pink`
- 字體大小：`12px`

## 3. 怎麼判斷有沒有衝突

可以先用最簡單的方式判斷：

### 有衝突

- 作用在同一個元素
- 設定的是同一個屬性

例如：

```css
div {
    color: red;
}

div {
    color: pink;
}
```

這裡衝突的是 `color`。

### 沒有衝突

- 雖然作用在同一個元素
- 但設定的是不同屬性

例如：

```css
div {
    color: red;
}

div {
    font-size: 16px;
}
```

這裡 `color` 和 `font-size` 不會互相覆蓋，而是會一起生效。

## 4. 常見混淆點

### 層疊性是不是「誰離標籤最近就用誰」？

這樣講太粗略，容易誤解。

在這一篇的基礎情境裡，更準確的說法是：

- 當同一個元素套用到多條規則
- 而且這些規則設定了同一個屬性
- 如果條件相同，通常是後面寫的覆蓋前面寫的

所以重點不是「離標籤近」，而是「規則衝突時，最後由哪一條生效」。

### 是不是只要後寫就一定贏？

這一篇先掌握最基礎的情況即可：相同選擇器、相同條件下，後寫的通常會覆蓋前寫的。  
更進一步還會牽涉到選擇器權重等規則，但那是後續主題。

## 5. 30 秒複習

- 層疊性主要解決樣式衝突
- 同一元素、同一屬性才會發生覆蓋問題
- 相同條件下，後寫的樣式通常會覆蓋先寫的樣式
- 不同屬性不一定衝突，很多時候會一起生效

如果只記一句話：

> 層疊性不是所有樣式互相覆蓋，而是衝突時決定最後用哪一條規則。
