# 第四章 CSS 三大特性

本章整理 CSS 的三個核心觀念：**層疊性、繼承性、優先級**。

它們共同回答一個問題：

> 當瀏覽器看到一個元素時，最後到底要用哪一個 CSS 值來顯示畫面？

如果你只會寫 CSS，很容易覺得樣式偶爾「莫名其妙沒生效」；但只要理解這三個特性，就能開始判斷：

- 是不是父元素的樣式被子元素繼承了？
- 是不是同一個元素的同一個屬性發生衝突？
- 是不是某條選擇器的權重比較高？
- 是不是後面載入的 CSS 覆蓋了前面載入的 CSS？

## 本章和整體學習地圖的關係

這一章承接前面的 CSS 基礎語法，開始進入「CSS 規則如何真正生效」的核心。

前面你可能已經會寫：

```css
.box {
    color: red;
    font-size: 16px;
}
```

但從這一章開始，你要進一步理解：

- 為什麼某些樣式寫在父元素，子元素也會跟著變？
- 為什麼同一個元素被多條規則命中時，有些規則會被劃掉？
- 為什麼有時候後寫的會生效，有時候卻是前面或權重更高的規則生效？

這一章適合作為從「會寫 CSS」走向「看得懂 CSS 為什麼這樣生效」的起點。

## 三大特性的定位

| 特性 | 主要解決的問題 | 判斷重點 | 常見情境 |
|---|---|---|---|
| 繼承性 | 子元素沒寫樣式時，能不能沿用父元素的值 | 這個屬性是否可繼承、子元素是否有自己設定 | `color`、`font-size`、`line-height` 從父元素傳到子元素 |
| 層疊性 | 多條 CSS 聲明同時作用時，最後採用哪一條 | 是否作用在同一元素、同一屬性、是否發生衝突 | 同一個 `div` 被重複設定 `color` |
| 優先級 | 發生衝突時，哪條規則比較強 | `!important`、選擇器權重、書寫順序 | `.text` 比 `div` 更容易覆蓋顏色 |

可以先這樣理解：

> 繼承性決定「沒寫時能不能沿用」，層疊性處理「多條規則同時存在」，優先級處理「衝突時誰比較強」。

## CSS 樣式生效的簡化判斷流程

當你不知道某個樣式為什麼生效或沒生效時，可以照這個流程查：

```text
1. 先確認目標元素是誰
   ↓
2. 確認你要看的屬性是哪一個
   例如 color、font-size、margin、width
   ↓
3. 這個元素有沒有被 CSS 直接設定這個屬性？
   ├─ 有：進入層疊與優先級判斷
   └─ 沒有：看這個屬性是否可能從父元素繼承
   ↓
4. 如果多條規則都設定同一個屬性
   ├─ 先看有沒有 !important
   ├─ 再看選擇器權重
   └─ 權重相同時，看誰寫在後面或誰後載入
   ↓
5. 用 DevTools 的 Styles / Computed 面板驗證
```

這個流程不是 CSS 規範的完整版本，但足以處理大部分初學與日常開發中的樣式問題。

## 哪些內容適合快速回查

### 層疊性

- 忘了樣式衝突時怎麼判斷誰生效
- 想確認為什麼後面的 CSS 會覆蓋前面的 CSS
- 想分辨「樣式衝突」和「不同屬性同時生效」的差異
- 想知道多個 CSS 檔案載入順序為什麼會影響結果

### 繼承性

- 忘了哪些 CSS 屬性會自動往下傳給子元素
- 看到子元素沒寫樣式卻跟著父元素變化
- 想確認 `line-height` 為什麼會一起跟著變
- 想理解 `a`、`h1` 看起來沒有繼承的原因

### 優先級

- 想判斷兩條規則到底哪一條會生效
- 想重新計算複合選擇器的權重
- 忘了類別選擇器和 ID 選擇器哪個優先
- 想知道 `!important` 為什麼不建議濫用
- 想排查 DevTools 裡被劃掉的 CSS 規則

### Demo

- 想直接打開小範例觀察繼承、層疊與優先級差異
- 想對照「多檔案載入順序」與「選擇器權重」的實際效果

## 這一章先不深入的內容

為了先建立穩固基礎，本章主要討論最常見的作者樣式，也就是你自己在 CSS 檔案或 `<style>` 中寫的規則。

更進階的主題，例如：

- 瀏覽器預設樣式、使用者樣式、作者樣式的完整來源順序
- `@layer` 的層疊層級
- CSS 動畫、transition 在層疊中的特殊位置
- Shadow DOM 或 Scoped CSS 的進階規則

之後學到更複雜的 CSS 架構時再補即可。現在先把「繼承、層疊、權重」這三個最常遇到的核心觀念打穩。

## 30 秒總複習

- **繼承性**：子元素沒有自己設定時，可能沿用父元素的某些樣式。
- **層疊性**：多條 CSS 規則同時作用時，要決定最後採用哪一條。
- **優先級**：同一元素、同一屬性發生衝突時，根據重要性、權重與順序判斷勝負。

如果只記一句話：

> CSS 不是單純「後寫就贏」，而是先看有沒有直接設定與繼承，再看衝突時的層疊與優先級。
