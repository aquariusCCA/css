# Demo 07：父元素權重不會搬到子元素身上

來源筆記：`notes/第04章_CSS三大特性/CSS三大特性.md`

對應知識點：父元素用高權重 selector，甚至加上 `!important`，只能決定父元素自己的樣式；子元素若只是繼承該值，仍會被直接作用在子元素上的宣告覆蓋。

開啟方式：直接用瀏覽器開啟 `index.html`。

操作方式：不需要操作，觀察左右兩組段落顏色。

觀察重點：左側 `#father` 只直接選到父元素，`p` 仍使用自己身上的粉紅色；右側改成 `#father-fixed p`，才直接選到子元素並讓段落變紅。

常見誤解：不要把父元素 selector 的權重直接套到子元素身上；判斷前要先確認 CSS 直接選到的是哪個元素。
