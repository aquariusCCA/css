CSS 屬性書寫順序

同樣的畫面，整理後更容易讀懂

屬性排序不會讓一般宣告變得更強；它讓讀者先看版面，再看盒子、文字與裝飾。

排序對照

比較未整理與分組後的 CSS

布局定位 自身屬性 文本屬性 其他效果

未整理

同類屬性分散，掃讀時需要來回找。

.card {
  color: #333;
  padding: 16px;
  display: block;
  box-shadow: 0 8px 24px rgba(0, 0, 0, .12);
  width: 320px;
  overflow: hidden;
  font: 16px/1.6 Arial, sans-serif;
  margin: 24px auto;
  border-radius: 8px;
  background: #fff;
}

分組後

閱讀路徑固定：版面、盒子、文字、效果。

.card {
  display: block;
  overflow: hidden;

  width: 320px;
  margin: 24px auto;
  padding: 16px;
  background: #fff;

  color: #333;
  font: 16px/1.6 Arial, sans-serif;

  border-radius: 8px;
  box-shadow: 0 8px 24px rgba(0, 0, 0, .12);
}

未整理 CSS 的結果

課程卡片

畫面結果相同，但維護時比較難快速定位屬性。

分組後 CSS 的結果

課程卡片

同類屬性集中，讀者更快理解這段規則。

真正會改變結果的情況

同名屬性：後面的宣告覆蓋前面的宣告

.box {
  color: red;
  color: blue;
}

最後生效顏色

blue 因為後面的 color: blue 覆蓋了前面的 color: red。
