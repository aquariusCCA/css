# CSS 程式碼風格

來源筆記：`notes/第01章_CSS簡介/CSS簡介.md`

對應知識點：展開風格與緊湊風格可以產生相同畫面效果，但開發時展開風格更容易閱讀、修改與檢查。

開啟方式：直接用瀏覽器開啟 `index.html`。

觀察重點：

- 兩側標題的顏色與大小相同，因為宣告內容相同。
- 左側把每條 declaration 分行，較容易檢查 selector、property、value、冒號與分號。
- 右側將空白與換行壓縮，適合上線最佳化，不適合作為初學與日常維護的主要書寫方式。

常見誤解：緊湊風格不是更高級的 CSS，只是更少空白與換行。開發時應優先讓程式碼可讀。
