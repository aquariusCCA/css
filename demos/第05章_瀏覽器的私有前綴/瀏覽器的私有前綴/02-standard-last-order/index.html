Demo 02

需要前綴時，標準屬性放在最後

這個範例用不同圓角值放大順序差異。當瀏覽器同時理解前綴版本和標準版本時，後面的宣告會成為最後結果。

建議順序

標準寫法在最後 
.recommended-order {
    -webkit-border-radius: 4px;
    border-radius: 28px;
}

觀察：標準屬性放最後，圓角以 28px 作為主要結果。

反向順序

前綴寫法在最後 
.reversed-order {
    border-radius: 28px;
    -webkit-border-radius: 4px;
}

觀察：若瀏覽器理解 -webkit-border-radius，後面的前綴版本可能蓋過標準值。

如果右側在你的瀏覽器中仍然是大圓角，代表該前綴寫法被忽略；本 demo 仍保留順序對照，用來觀察筆記中的維護規則。
