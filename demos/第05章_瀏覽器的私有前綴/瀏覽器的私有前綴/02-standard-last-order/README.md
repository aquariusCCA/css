# Demo 02：標準屬性放在最後

來源筆記：`notes/第05章_瀏覽器的私有前綴/瀏覽器的私有前綴.md`

對應知識點：來源筆記建議先寫私有前綴屬性，再寫標準屬性，讓標準寫法成為主要維護結果。

開啟方式：直接用瀏覽器開啟 `index.html`。

操作方式：不需要操作，觀察左右兩張卡片的圓角，再對照各自的 CSS 順序。

觀察重點：左側把前綴版本放前面、標準 `border-radius` 放最後。右側刻意反過來，若瀏覽器同時理解兩種寫法，後面的前綴版本可能蓋過標準值。

常見誤解：不要只把前綴當成「多寫幾行就好」。前綴版本是相容補充，標準屬性應該保留在最後，讓 CSS 的長期意圖清楚。

補充說明：不同瀏覽器對舊前綴屬性的支援不同；若右側沒有出現視覺差異，代表目前瀏覽器忽略了該前綴寫法。
