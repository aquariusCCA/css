# Demo 04：日常先寫標準 CSS

來源筆記：`notes/第05章_瀏覽器的私有前綴/瀏覽器的私有前綴.md`

對應知識點：來源筆記提醒，平常不用刻意記憶所有私有前綴；常用 CSS 功能應先使用標準寫法，有舊版瀏覽器或實驗功能需求時再處理前綴。

開啟方式：直接用瀏覽器開啟 `index.html`。

操作方式：將游標移到兩個按鈕上，觀察背景色與位置變化。

觀察重點：左側用標準 `transition` 作為日常主線；右側在標準寫法前面補上 `-webkit-transition`，表示相容補充。兩者的目標視覺結果相同。

常見誤解：不要把「不用日常手寫前綴」理解成「永遠不用檢查相容性」。若專案要求支援舊版瀏覽器，仍要依目標環境判斷是否需要保留前綴。
