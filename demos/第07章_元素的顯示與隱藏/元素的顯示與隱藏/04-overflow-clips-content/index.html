元素的顯示與隱藏

overflow 控制的是溢出內容，不是整個元素

三個盒子有相同寬高與文字內容，只改 overflow 的值，觀察超出盒子範圍的文字如何被處理。

visible：溢出仍可見

這是一段很長的公告文字。當內容超過盒子的高度時，超出的文字可能會跑出盒子外面，讓你看到內容已經超過邊框。

盒子本身仍然存在。

.message {
  overflow: visible;
}

hidden：裁切超出部分

這是一段很長的公告文字。當內容超過盒子的高度時，超出的文字可能會跑出盒子外面，讓你看到內容已經超過邊框。

盒子本身仍然存在。

.message {
  overflow: hidden;
}

auto：需要時才捲動

這是一段很長的公告文字。當內容超過盒子的高度時，超出的文字可以留在盒子裡，使用者透過捲動查看被容器限制住的內容。

盒子本身仍然存在。

.message {
  overflow: auto;
}

主要觀察

三個範例的粉色邊框都還在，表示元素本身沒有被隱藏。改變的是內容超出固定盒子後，要露出、裁切，還是透過捲動查看。
